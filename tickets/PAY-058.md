# [GW] Payroll - Submit Payroll Run for Review

## Description

As a payroll officer, I want to submit a completed draft payroll run for review so that the finance team can verify all figures before finalization. Submission triggers a full recomputation of all payslips, validates the run is complete, and transitions the state from `draft` to `submitted`.

## Technical Implementation

### Action
- **`app/Payroll/Actions/SubmitPayrollRunAction.php`** (lorisleiva/laravel-actions)
  - Accepts: `PayrollRun $payrollRun`, `User $submittedBy`
  - Pre-submit validations:
    - Run must be in `draft` state
    - Run must have at least one employee (payslip count > 0)
    - No unresolved flags exist on any payslip in the run
    - All employees have compensation records
  - Triggers `RecomputePayrollRunAction` to ensure all payslips are up-to-date
  - Transitions state: `draft → submitted` via spatie/laravel-model-states
  - Sets `submitted_by = $submittedBy->id`, `submitted_at = now()`
  - Dispatches `PayrollRunSubmittedNotification` to all users with `finance` role

### Controller
- **`PayrollRunController@submit`** — `POST /payroll-runs/{payrollRun}/submit`
- Middleware: `auth`, `role:payroll_officer|owner`

### Form Request
- **`SubmitPayrollRunRequest`** — authorization check (user is payroll_officer or owner)

### Notification
- **`PayrollRunSubmittedNotification`** — database channel
  - Content: "Payroll run '{run_name}' for {period} has been submitted for review"
  - Sent to all users with `finance` role

### Frontend
- Submit button on payroll run detail page (visible only in `draft` state)
- Confirmation dialog showing:
  - Employee count
  - Total gross pay
  - Total deductions
  - Total net pay
  - "Are you sure you want to submit this run for review?"
- Button disabled during processing (Inertia `processing` state)

### Audit
- State transition logged via laravel-auditing (old: draft, new: submitted, actor)

### Tests
- `tests/Feature/PayrollRun/SubmitPayrollRunTest.php`
  - Submits draft run successfully
  - Rejects submission of non-draft run (422)
  - Rejects empty run (no employees) with validation error
  - Rejects run with unresolved flags
  - Verifies finance users receive notification
  - Verifies submitted_by and submitted_at are set
  - Verifies payslips are recomputed before submission

## Acceptance Criteria

- [ ] Only draft runs can be submitted
- [ ] Confirmation dialog shows employee count, total gross, and total net before confirming
- [ ] State changes to `submitted` after confirmation
- [ ] All payslips are recomputed before state transition
- [ ] Finance users receive in-app notification upon submission
- [ ] Empty run (zero employees) is blocked from submission with error message
- [ ] Run with unresolved flags is blocked from submission
- [ ] `submitted_by` and `submitted_at` fields are populated
- [ ] Audit trail records state transition with actor and timestamp
