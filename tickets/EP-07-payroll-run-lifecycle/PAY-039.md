# [GW] Payroll - Payroll Run List Page

## Description

Build the payroll run list page where the Payroll Officer and Owner can view all payroll runs with their statuses, track payroll history, and quickly navigate to active runs. The list is filterable by state and date range, paginated, and sorted by newest first.

## Technical Implementation

### Queries

- Payroll run list query with filtering by state (multi-select) and date range (created date). Eager loads payroll period data. Default sort by created date descending, with additional sort options for name and state.

### Routes & Controllers

- `GET /payroll-runs` — paginated list of payroll runs with filters.

### Policies & Authorization

- Owner, Payroll Officer, and Finance can view the payroll run list.
- Employee role cannot access the run list.

### Tests

- Assert payroll runs are listed with correct data (period, name, state, date, employee count, total net).
- Assert filters by state and date range work correctly.
- Assert pagination works with configurable page sizes.
- Assert default sort is newest first.
- Assert Employee role cannot access the run list (403).
- Assert finalized runs show the download summary action.

## Acceptance Criteria

### Client-side

#### Layout

- The Payroll Runs page is accessible from the sidebar for Owner, Payroll Officer, and Finance roles

#### Behavior

- The payroll run list displays: Period Name, Run Name, State (color-coded badge), Created Date (relative), Employee Count, and Total Net Pay (formatted as currency)
- State badges use distinct colors: Draft (gray), Submitted (blue), Under Review (yellow), Finalized (green)
- Users can filter the list by state (multi-select dropdown) and date range (date picker)
- The list is paginated with a page size selector (10, 25, 50)
- Default sort order is newest first
- Clicking a run row navigates to the run detail page
- Only finalized runs show a "Download Summary" action button
- A "Create New Run" button is visible in the header for Owner and Payroll Officer roles

---

### Server-side

#### Side-effects

- The run list returns paginated results with payroll period data eager-loaded for performance
- Only authorized roles (Owner, Payroll Officer, Finance) can access the run list
- Employee role is denied access to the run list
