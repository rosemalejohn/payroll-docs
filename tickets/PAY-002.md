# [GW] Payroll - Configure Tailwind CSS v4 and Shadcn/ui Component Library

## Description

Install and configure Tailwind CSS v4 with project-specific design tokens, then initialize the Shadcn/ui component library with a default theme. Install all base UI primitives needed across the application. This ticket ensures the design system is in place before any feature pages are built.

## Technical Implementation

### Environment & Config
- Install Tailwind CSS v4 and its Vite plugin: `npm install tailwindcss @tailwindcss/vite`.
- Configure `vite.config.ts` to include the Tailwind CSS Vite plugin.
- Update `resources/css/app.css` with the Tailwind v4 `@import "tailwindcss"` directive.
- Define project design tokens (colors, border radius, font family) as CSS custom properties within `resources/css/app.css` using Tailwind v4's `@theme` directive.
- Initialize Shadcn/ui: `npx shadcn@latest init`.
- Configure `components.json` with:
  - Style: "default" or "new-york" (team preference).
  - Path alias: `@/components/ui`.
  - Tailwind CSS path: `resources/css/app.css`.
  - TypeScript enabled.
- Create `resources/js/lib/utils.ts` with the `cn()` helper function using `clsx` and `tailwind-merge`:
  ```typescript
  import { type ClassValue, clsx } from "clsx";
  import { twMerge } from "tailwind-merge";
  export function cn(...inputs: ClassValue[]) {
    return twMerge(clsx(inputs));
  }
  ```
- Install dependencies: `npm install clsx tailwind-merge class-variance-authority lucide-react`.
- Install Shadcn/ui base primitives via CLI:
  - `npx shadcn@latest add button input label card dialog sheet select table badge dropdown-menu tabs tooltip separator skeleton`
- Verify all installed components are placed under `resources/js/components/ui/`.
- Ensure `app.css` imports are ordered correctly (Tailwind base, components, utilities, then custom overrides).

### Tests
- No automated tests for this ticket (visual/manual verification).
- Verify by importing and rendering at least one Shadcn component (e.g., Button) on the welcome page.

### Edge Cases & Notes
- Tailwind v4 uses CSS-first configuration instead of `tailwind.config.js`. Do not create a JS config file unless absolutely necessary for plugin compatibility.
- Shadcn/ui components are copied into the project (not a dependency), so they can be freely customized.
- Ensure `lucide-react` icons render correctly alongside Shadcn components.
