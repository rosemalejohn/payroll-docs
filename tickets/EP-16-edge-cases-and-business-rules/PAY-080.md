# [GW] Payroll - New Employee HMO Proration for Mid-Period Join

## Description

When an employee is added to a payroll run and their hire date falls within the run period, prorate their HMO deduction and other mandatory deductions (SSS, PhilHealth, Pag-IBIG) for the first partial period. The system calculates deductions proportionally based on the number of working days from the hire date to the period end relative to the total working days in the period. Payslip notes document the proration basis and the employee's start date. Payroll officers can override prorated amounts when needed.

## Technical Implementation

### What Migrations

- Add proration fields to payslips table: is_mid_period_join (boolean, default false), proration_factor (decimal 8,6, nullable), employee_start_date (nullable date).
- Add override fields to payslip deductions table: original_amount (decimal 14,2, nullable), prorated_amount (decimal 14,2, nullable), is_override (boolean, default false), override_amount (decimal 14,2, nullable).

### What Models

- Payslip model: add mid-period join fields (is_mid_period_join, proration_factor, employee_start_date) to fillable attributes and casts.
- PayslipDeduction model: add override fields (original_amount, prorated_amount, is_override, override_amount) to fillable attributes and casts.

### What Actions

- Detect Mid-Period Join action: checks if an employee's hire date falls between the payroll run period start and end (inclusive but not on the first day). Returns proration data: days worked, total working days, and proration factor.
- Prorate New Employee Deductions action: prorates deductions for mid-period joiners — HMO, SSS, PhilHealth, and Pag-IBIG are each multiplied by the proration factor and rounded to 2 decimal places. Stores both original and prorated amounts on each deduction line.
- Enhanced Compute Payslip action: after gross pay calculation, checks for mid-period join. If detected, prorates gross pay for monthly employees and delegates deduction steps to the proration action. Withholding tax is annualized from the prorated gross. Adds payslip notes documenting proration.
- Override Payslip Deduction action: allows payroll officer to override a prorated deduction amount. Sets the override flag, recalculates net pay, and adds a note recording the override.

### Validation & DTOs

- Deduction Override DTO: payslip ID, deduction type (SSS, PhilHealth, Pag-IBIG, HMO), override amount.
- Override Payslip Deduction request: validates deduction type is one of the allowed types, override amount is non-negative, and payslip belongs to a draft run.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/payslips/{payslip}/override-deduction` — overrides a prorated deduction amount.

### Policies & Authorization

- Only Payroll Officer and Owner roles can override deductions.
- Payroll run must be in draft state.

### Tests

- Assert employee hired mid-period has HMO prorated by days worked / total working days.
- Assert SSS contribution is prorated based on proration factor.
- Assert PhilHealth contribution is prorated based on proration factor.
- Assert Pag-IBIG contribution is prorated based on proration factor.
- Assert gross pay is prorated for monthly employees.
- Assert daily rate employee gross equals daily_rate × days_worked (unchanged behavior).
- Assert withholding tax is calculated on prorated gross (annualized).
- Assert payslip notes include hire date and proration basis.
- Assert payslip has is_mid_period_join flag and correct proration_factor.
- Assert employee hired before period start is not flagged as mid-period join.
- Assert employee hired on first day of period is not flagged (full period).
- Assert employee hired on last day of period is prorated to 1 day.
- Assert override deduction changes the amount and sets is_override flag.
- Assert override recalculates net pay correctly.
- Assert only Payroll Officer and Owner can override deductions.
- Assert override only allowed in draft runs.
- Assert external salary employees skip deduction proration entirely.

### Edge Cases & Notes

- Employee hired on the first day of the period (hire date equals period start) should NOT be flagged as mid-period — they work the full period.
- Employee hired on the last working day should be prorated to 1 working day.
- Working days calculation must account for weekends; consider using a working days helper that can optionally include company holidays.
- Proration factor must use DECIMAL precision to avoid rounding errors in deduction calculations.
- External salary employees skip deduction steps 3-9 entirely, so proration of those deductions does not apply.
- If the payroll officer overrides a prorated amount, subsequent recomputations should preserve the override (not recalculate from proration).
- SSS proration is a simplification — in practice, SSS may not allow partial-month contributions. Current implementation prorates for consistency.

## Acceptance Criteria

### Client-side

#### Behavior

- Employees hired within the payroll run period are automatically detected as mid-period joiners
- Mid-period join payslips display a "New Hire (Prorated)" badge
- Each deduction line shows the original full amount and the prorated amount with the proration factor
- A proration note displays the hire date and working days ratio (e.g., "12/22 working days")
- An "Override" button is available next to each prorated deduction in draft runs
- The override dialog shows the current prorated amount and allows entering a new amount
- After override, the deduction displays the override amount with an "Overridden" indicator
- Only Payroll Officer and Owner roles can override deduction amounts

---

### Server-side

#### Side-effects

- Employees with a hire date within the payroll run period are flagged as mid-period joiners
- HMO, SSS, PhilHealth, and Pag-IBIG deductions are prorated by the ratio of days worked to total working days
- Gross pay is prorated for monthly employees; daily rate employees are unaffected
- Withholding tax is calculated based on the prorated gross amount
- Both original and prorated amounts are stored for each deduction for transparency
- Payroll officers can override any prorated deduction amount in draft runs
- Overrides are preserved during payslip recomputation
- Employees hired on the first day of the period are treated as full-period employees
- External salary employees skip deduction proration entirely
