# [GW] Payroll - Audit Trail Page with Filters

## Description

Build a dedicated audit trail page accessible only to the Owner role. The page displays all audit log entries showing timestamp, acting user, model type, action, and old/new values in a readable format. Filters for model type, date range, user, and payroll run help owners quickly locate specific changes. The audit trail is strictly read-only.

## Technical Implementation

### Queries

- Filter audits query: supports filtering by model type, user, date range, and payroll run. Eager loads the user relationship, orders by created timestamp descending, and returns paginated results (25 per page).

### Routes & Controllers

- GET /audit-trail — paginated audit trail with filters.

### API Resources

- Audit resource: includes ID, event type, readable model type, auditable ID, formatted old/new values, actor name and role, timestamp, URL, and IP address.

### Policies & Authorization

- Only Owner role can view the audit trail page.
- Audits are immutable — no edit or delete actions available.

### Tests

- Assert Owner can access the audit trail page.
- Assert non-Owner roles receive 403.
- Assert filter by model type returns only matching audits.
- Assert filter by user returns only audits by that actor.
- Assert filter by date range returns audits within range.
- Assert filter by payroll run returns related audits.
- Assert results are paginated at 25 per page.
- Assert results are sorted by newest first.

### Edge Cases & Notes

- Map friendly model names to fully qualified class names for the type filter.
- Handle gracefully if old/new values JSON is null or malformed.
- Use server-side pagination for performance with large audit tables.

## Acceptance Criteria

### Client-side

#### Layout

- The Audit Trail page is accessible from the sidebar for the Owner role only

#### Behavior

- The page displays audit entries with: Timestamp, User (actor), Action (created/updated), Model Type, and Changes (old/new values in a readable diff format)
- Filters are available for: Model Type (dropdown), User (dropdown), Date Range (date pickers), and Payroll Run (dropdown)
- Changes are displayed in a diff-style format showing field name, old value, and new value
- Results are paginated (25 per page) and sorted by newest first
- No edit or delete actions are available anywhere on the page

---

### Server-side

#### Side-effects

- Only the Owner role can access the audit trail page — all other roles are denied
- Audit entries are read-only and cannot be modified or deleted through the UI
- Filters correctly narrow results by model type, user, date range, and payroll run
- Results use server-side pagination for performance with large datasets
