# [GW] Payroll - Configure Audit Trail with Laravel Auditing

## Description

Install and configure the owen-it/laravel-auditing package to provide a comprehensive audit trail across the application. The Auditable interface must be implemented on all critical models: PayrollRun, Payslip, EmployeeCompensation, Currency, and Employee (payroll-specific flags only). Each model declares which attributes are audited and which are excluded, ensuring state transitions, salary changes, currency rate changes, and payroll flag changes are all captured with old/new values and the acting user.

## Technical Implementation

### What Migrations

- Publish and run the audits table migration: includes user and auditable morph columns, event type, old/new values (JSON), URL, IP address, user agent, and created timestamp.

### What Models

- Update PayrollRun to audit: state, name, transition actor/timestamp fields, and notes.
- Update Payslip to audit monetary fields: gross pay, net pay, deductions, bonuses, and individual deduction amounts.
- Update EmployeeCompensation to audit: salary fields, currency, and effective date.
- Update Currency to audit: code, name, rate, and active status.
- Update Employee to audit payroll-specific flags only (external salary, MP2, HMO, cash advance), excluding all HRIS-synced fields.

### Environment & Config

- Configure the audit package: use database driver, resolve users via web guard, and disable console auditing.

### Tests

- Assert updating PayrollRun status creates an audit record with old/new values.
- Assert updating Payslip monetary fields creates an audit record.
- Assert updating EmployeeCompensation salary creates an audit record.
- Assert updating Currency rate creates an audit record.
- Assert updating Employee payroll flags creates an audit record.
- Assert HRIS-synced fields on Employee are excluded from audit.
- Assert audit record captures the authenticated user as actor.
- Assert audit record stores the correct event type.

### Edge Cases & Notes

- Queue jobs that modify auditable models must set user context so changes are attributed correctly.
- HRIS-synced employee fields must be excluded to avoid noisy entries from daily syncs.
- Monetary values must be stored accurately in the JSON old/new columns.

## Acceptance Criteria

### Server-side

#### Side-effects

- The owen-it/laravel-auditing package is installed and configured with the audits table created
- PayrollRun, Payslip, EmployeeCompensation, Currency, and Employee models implement the Auditable interface
- Each model audits only relevant attributes — HRIS-synced employee fields are excluded
- Audit records are created automatically on model changes with correct old and new values
- The authenticated user is captured as the actor on every audit record
- State transitions on PayrollRun are captured with before and after status values
- Salary and rate changes are captured with precise decimal values
