# PAY-004: Configure Authentication System

## Description

Set up session-based authentication using Laravel Breeze with the Inertia/React stack. This provides login, registration, password reset, and email verification pages built with React and styled with Tailwind/Shadcn. The authentication system will serve as the entry point for all role-based access in the payroll application.

## Technical Implementation

### What Actions
- No custom domain actions. Laravel Breeze scaffolds its own auth controllers.

### What Migrations
- Laravel Breeze may add or modify the default `users`, `password_reset_tokens`, and `sessions` tables. Verify these align with the schema from `docs/01-database-schema.md` (customization of the users table happens in PAY-007).

### Routes & Controllers
- Install Laravel Breeze with the Inertia React stack: `composer require laravel/breeze --dev`, then `php artisan breeze:install react --typescript`.
- This scaffolds:
  - `routes/auth.php` with login, register, password reset, email verification, and logout routes.
  - Auth controllers under `app/Http/Controllers/Auth/` (LoginController, RegisterController, etc.).
  - Inertia page components under `resources/js/pages/auth/` (login.tsx, register.tsx, forgot-password.tsx, reset-password.tsx, verify-email.tsx, confirm-password.tsx).
- Configure `routes/web.php` to include `auth.php` routes.
- Add `auth` middleware group to all protected routes.
- Configure `authenticated-layout.tsx` to check for auth state and redirect to login if unauthenticated.

### Validation & DTOs
- Breeze provides Form Requests: `LoginRequest`, `RegisterRequest`, etc.
- Frontend forms use Inertia's `useForm` hook with inline validation display from Laravel errors.

### Environment & Config
- Set `SESSION_DRIVER=database` in `.env.example`.
- Ensure `sessions` table migration exists (included with Laravel).
- Configure `config/auth.php` guards and providers for the default web guard.
- Set redirect paths: authenticated users go to `/dashboard`, guests go to `/login`.
- Configure "remember me" functionality (enabled by default in Breeze).

### Tests
- `tests/Feature/Auth/LoginTest.php` (Pest): asserts login with valid credentials redirects to dashboard, invalid credentials return errors, unauthenticated access redirects to login.
- `tests/Feature/Auth/RegistrationTest.php` (Pest): asserts new user creation with valid data, validation errors on duplicate email.
- `tests/Feature/Auth/LogoutTest.php` (Pest): asserts session is destroyed on logout.
- `tests/Feature/Auth/PasswordResetTest.php` (Pest): asserts reset link is sent and password can be reset.

### Edge Cases & Notes
- Breeze scaffolding may overwrite files created in PAY-001 and PAY-002 (e.g., `app.tsx`, `app.css`). After installation, verify and restore any customizations (Tailwind v4 config, Shadcn setup, design tokens).
- The User model will be migrated to `app/Company/Models/User.php` in PAY-007. For now, keep it at `app/Models/User.php`.
- Email verification is optional for the initial setup but the routes should exist for future enablement.

## Acceptance Criteria

- [ ] Laravel Breeze is installed with the Inertia React TypeScript stack.
- [ ] Login page renders at `/login` with email and password fields.
- [ ] Register page renders at `/register` with name, email, password, and password confirmation fields.
- [ ] Forgot password page renders at `/forgot-password`.
- [ ] User can register a new account and is redirected to the dashboard.
- [ ] User can log in with valid credentials and is redirected to the dashboard.
- [ ] User can log out and session is destroyed.
- [ ] Invalid login credentials display error messages on the form.
- [ ] Unauthenticated users accessing protected routes are redirected to `/login`.
- [ ] "Remember me" checkbox persists the session beyond browser close.
- [ ] Session driver is set to `database` in `.env.example`.
- [ ] All Breeze-generated Pest auth tests pass (`php artisan test --filter=Auth`).
- [ ] Tailwind and Shadcn configurations from PAY-002 are intact after Breeze installation.
