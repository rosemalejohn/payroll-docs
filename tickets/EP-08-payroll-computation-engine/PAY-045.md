# [GW] Payroll - ComputePayslipAction - Bonus Application (Step 2)

## Description

Implement Step 2 of the 10-step payroll computation pipeline. This step evaluates all active bonuses attached to the payroll run, applies them per employee (respecting exclusions and eligibility), computes the bonus amount based on its computation type, and records individual line items. The payslip's `total_bonuses` field is updated with the sum of all applied bonuses.

## Technical Implementation

### Action

**`app/Payroll/Actions/ComputePayslipAction.php`** (Step 2 segment)

After Step 1 completes and `gross_pay` is available, iterate over `payroll_run_bonuses` associated with the current run.

**Bonus evaluation loop:**

```php
$totalBonuses = 0;

foreach ($payrollRun->payrollRunBonuses as $bonus) {
    // 1. Check exclusions
    if ($bonus->excludedEmployees->contains($employee->id)) {
        continue;
    }

    // 2. Compute amount based on computation_type
    $amount = match ($bonus->computation_type) {
        'fixed'      => $this->computeFixed($bonus),
        'percentage'  => $this->computePercentage($bonus, $grossPay),
        'formula'     => $this->computeFormula($bonus, $compensation, $grossPay),
        'attendance'  => $this->computeAttendance($bonus, $payslip, $grossPay),
    };

    if ($amount <= 0) continue;

    // 3. Record line item
    PayslipLineItem::create([
        'id'         => Str::ulid(),
        'payslip_id' => $payslip->id,
        'code'       => $bonus->code,
        'label'      => $bonus->name,
        'type'       => 'bonus',
        'amount'     => $amount,
        'sort_order' => $sortOrder++,
    ]);

    $totalBonuses += $amount;
}

$payslip->update(['total_bonuses' => $totalBonuses]);
```

**Computation methods:**

- **Fixed:** `return $bonus->amount;` -- flat amount from the `payroll_run_bonuses` record.

- **Percentage:** `return $basis * ($bonus->rate / 100);` where `$basis` is either `gross_pay` (Step 1) or `basic_salary` depending on `bonus->basis_type`.

- **Formula (13th Month Pay):**
  ```php
  if ($compensation->salary_basis === 'monthly') {
      return ($compensation->monthly_salary * $snapshotRate * 12) / 12;
  }
  if ($compensation->salary_basis === 'daily') {
      return ($compensation->daily_rate * $snapshotRate * 261) / 12;
  }
  ```
  Uses 261 standard working days per year for daily-basis employees.

- **Formula (June Special):** `return $grossPay * 1.0;` -- 100% of gross pay.

- **Attendance:**
  ```php
  $attendedDays = $payrollPeriod->working_days - $payslip->absent_days;
  if ($attendedDays < $bonus->attendance_threshold) {
      return 0; // Does not qualify
  }
  return $bonus->amount; // Fixed amount awarded for meeting threshold
  ```

### Exclusions

The `bonus_exclusions` pivot table links `payroll_run_bonus_id` to `employee_id`. Excluded employees are skipped entirely for that bonus.

### Edge Cases

- Bonus with zero amount or rate: line item not created (skip zero amounts).
- Employee excluded from all bonuses: `total_bonuses` = 0.
- No bonuses on the run: Step 2 completes with `total_bonuses` = 0.

## Acceptance Criteria

- [ ] Fixed bonuses apply the flat amount from the `payroll_run_bonuses` record.
- [ ] Percentage bonuses compute correctly as `basis * (rate / 100)`.
- [ ] 13th Month Pay formula uses `annual_basic / 12` (monthly: `monthly_salary * 12 / 12`; daily: `daily_rate * 261 / 12`).
- [ ] June Special formula applies 100% of gross pay.
- [ ] Attendance bonuses check `attended_days >= threshold`; employees below threshold receive nothing.
- [ ] Excluded employees (via `bonus_exclusions` table) skip the bonus entirely.
- [ ] A `bonus`-type line item is created per applied bonus with the correct code and amount.
- [ ] Zero-amount bonuses do not generate line items.
- [ ] `payslip.total_bonuses` is updated to the sum of all applied bonus amounts.
- [ ] Foreign currency employees have bonuses computed on the PHP-converted gross pay.
