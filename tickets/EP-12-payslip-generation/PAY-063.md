# [GW] Payroll - Payslip PDF Generation

## Description

Implement queued payslip PDF generation triggered upon payroll run finalization. Each payslip gets an individual PDF generated using barryvdh/laravel-dompdf. The PDF includes company information, employee details, payroll period, earnings breakdown, itemized deductions, and a gross-to-net summary. External salary employees receive a simplified PDF. Foreign currency employees see dual amounts with exchange rates.

## Technical Implementation

### What Actions

- Generate Payslip PDF action: determines which template to use based on the external salary flag, prepares view data including currency conversion details, generates the PDF via barryvdh/laravel-dompdf with A4 portrait format, and returns the generated PDF content.

### Jobs

- Generate Payslip PDF job: queued on the payslips queue, dispatched per payslip. Loads the payslip with all related data, renders the appropriate Blade template, generates the PDF, stores the file to the payslips storage directory, updates the payslip's PDF path and generated timestamp. Retries up to 3 times on failure.

### Routes & Controllers

- `GET /payslips/{payslip}/download` — download the generated PDF.

### Policies & Authorization

- Employees can download only their own finalized payslips.
- Owner, Payroll Officer, and Finance can download any finalized payslip.

### Tests

- Assert a PDF is generated for each payslip upon run finalization.
- Assert PDF content includes correct employee name, period, earnings, deductions, and net pay.
- Assert external salary employee PDF uses a simplified template with no deductions section.
- Assert foreign currency employee PDF shows original currency amount and PHP equivalent with exchange rate.
- Assert PDF path is saved correctly on the payslip record.
- Assert generated timestamp is set after PDF generation.
- Assert unauthorized users cannot download payslips they do not own.
- Assert the job retries up to 3 times on failure.

### Edge Cases & Notes

- PDF generation is asynchronous; employee notifications should account for a brief delay before the PDF is downloadable.
- Store PDFs in a non-public disk; serve through a controller that checks authorization.
- Use consistent decimal formatting for all monetary values.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Download PDF" button is available on finalized payslip rows for authorized users
- The button is disabled with a tooltip if the PDF has not yet been generated

---

### Server-side

#### Side-effects

- A PDF is generated for each payslip in the run upon finalization
- PDF content accurately reflects the payslip data: earnings, deductions, bonuses, and net pay
- External salary employees receive a simplified PDF with no deduction section, labeled "External Salary — No Deductions Applied"
- Foreign currency employees see the original currency amount, PHP equivalent, and exchange rate used
- The PDF is stored on disk and the path is saved to the payslip record
- A generated timestamp is set upon successful PDF generation
- PDF generation runs asynchronously on a dedicated queue
- Failed generation retries up to 3 times
- Only authorized users can download payslip PDFs
