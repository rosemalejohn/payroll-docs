# PAY-009: Create Currencies Migration, Model, and Seeder

## Description

Create the `currencies` table, Eloquent model, factory, seeder, and auditing configuration. The currency system supports multi-currency payroll with PHP (Philippine Peso) as the immutable base currency. All rates are stored as "1 [FOREIGN] = X PHP" and use `DECIMAL(18,6)` for precision. Rate changes are tracked via audit logs for compliance and historical integrity.

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_currencies_table.php`:
  ```php
  Schema::create('currencies', function (Blueprint $table) {
      $table->char('id', 26)->primary();                    // ULID
      $table->char('company_id', 26);                        // ref: companies.id
      $table->string('code', 10);                            // ISO 4217 e.g. USD, EUR, PHP
      $table->string('name', 100);                           // e.g. "Philippine Peso"
      $table->string('symbol', 10);                          // e.g. ₱, $, €
      $table->decimal('rate', 18, 6)->default(1.000000);     // 1 [code] = rate PHP
      $table->boolean('is_base')->default(false);            // only PHP has this = true
      $table->boolean('is_active')->default(true);
      $table->timestamp('rate_updated_at')->nullable();      // last time rate was manually changed
      $table->char('updated_by', 26)->nullable();            // ref: users.id
      $table->timestamps();

      // Composite unique constraint
      $table->unique(['company_id', 'code'], 'uq_currencies_company_code');
      $table->index('company_id', 'idx_currencies_company');
  });
  ```

### What Models
- Create `app/Currency/Models/Currency.php`:
  ```php
  namespace App\Currency\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;
  use OwenIt\Auditing\Contracts\Auditable;
  use OwenIt\Auditing\Auditable as AuditableTrait;

  class Currency extends Model implements Auditable
  {
      use HasUlids, HasFactory, AuditableTrait;

      protected $fillable = [
          'company_id', 'code', 'name', 'symbol',
          'rate', 'is_base', 'is_active',
          'rate_updated_at', 'updated_by',
      ];

      protected $casts = [
          'rate' => 'decimal:6',
          'is_base' => 'boolean',
          'is_active' => 'boolean',
          'rate_updated_at' => 'datetime',
      ];

      // Audit rate changes
      protected $auditInclude = ['rate', 'is_active'];

      // Relationships
      public function company() { return $this->belongsTo(\App\Company\Models\Company::class); }
      public function updatedByUser() { return $this->belongsTo(\App\Company\Models\User::class, 'updated_by'); }

      // Scopes
      public function scopeActive($query) { return $query->where('is_active', true); }
      public function scopeBase($query) { return $query->where('is_base', true); }
      public function scopeByCompany($query, string $companyId) { return $query->where('company_id', $companyId); }

      // Helpers
      public function isBase(): bool { return $this->is_base; }

      /**
       * Convert an amount in this currency to PHP.
       * Formula: amount_in_php = amount * rate
       */
      public function toPhp(float|string $amount): string
      {
          return bcmul((string) $amount, (string) $this->rate, 6);
      }

      /**
       * Convert an amount in PHP to this currency.
       * Formula: amount_in_currency = amount / rate
       */
      public function fromPhp(float|string $amount): string
      {
          return bcdiv((string) $amount, (string) $this->rate, 6);
      }
  }
  ```

### Edge Cases & Notes
- PHP (Philippine Peso) is the immutable base currency. Its `rate` must always be `1.000000` and `is_base` must always be `true`. Application logic should prevent any modification of the base currency's rate or base flag.
- Rates are stored as `DECIMAL(18,6)` to avoid floating-point precision issues. All arithmetic should use `bcmath` functions (`bcmul`, `bcdiv`) instead of native PHP float operations.
- `rate_updated_at` is set to `NOW()` whenever `rate` changes, independently of `updated_at`. This can be handled in a model observer or the update action.
- `updated_by` tracks which user last changed the rate (only `owner` role should be allowed to update rates).
- The unique constraint `uq_currencies_company_code` allows the same currency code (e.g., USD) to exist for different companies but not duplicated within one company.
