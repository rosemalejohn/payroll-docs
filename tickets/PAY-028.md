# PAY-028: Employee Compensation Management

## Description

Build the employee compensation management page, allowing authorized users to view and update an employee's salary details. The page is nested under the employee context and displays different fields depending on the employee's `pay_basis` (monthly vs. daily). It includes a currency selector for multi-currency salary entry, optional voluntary deduction fields (MP2, SSS WISP), and a full audit trail of all compensation changes via `owen-it/laravel-auditing`.

## Technical Implementation

### What Actions

#### `app/Compensation/Actions/UpdateEmployeeCompensationAction.php`
- Single-responsibility action class using `lorisleiva/laravel-actions`.
- Accepts the `Employee` model and validated compensation data.
- Uses `updateOrCreate` on the `EmployeeCompensation` model keyed by `employee_id`.
- Sets `company_id` from the employee's company.
- Sets `updated_by` to the authenticated user's ID.
- If creating (first time), also sets `created_by`.
- Conditionally sets fields based on `employee.pay_basis`:
  - If `monthly`: sets `monthly_salary`, nulls `daily_rate`.
  - If `daily`: sets `daily_rate`, nulls `monthly_salary`.
- Sets `currency_id`, `mp2_amount`, `sss_wisp_amount`, `effective_date`, `notes`.
- Returns the updated `EmployeeCompensation` model.

### What Models

#### `app/Compensation/Models/EmployeeCompensation.php`
- Ensure the model implements `OwenIt\Auditing\Contracts\Auditable` and uses the `Auditable` trait.
- `$fillable`: `company_id`, `employee_id`, `currency_id`, `monthly_salary`, `daily_rate`, `mp2_amount`, `sss_wisp_amount`, `effective_date`, `notes`, `created_by`, `updated_by`.
- `$casts`: `monthly_salary` => `decimal:2`, `daily_rate` => `decimal:2`, `mp2_amount` => `decimal:2`, `sss_wisp_amount` => `decimal:2`, `effective_date` => `date`.
- Relationships:
  - `employee()`: `belongsTo(Employee::class)`
  - `currency()`: `belongsTo(Currency::class)`
- Audit configuration: track changes to `monthly_salary`, `daily_rate`, `mp2_amount`, `sss_wisp_amount`, `currency_id`, `effective_date`.

### Validation & DTOs

#### `app/Http/Requests/Compensation/UpdateCompensationRequest.php`
- Authorization: user must have `owner` or `payroll_officer` role.
- Rules (conditional based on the employee's `pay_basis`):
  - `currency_id`: `required|string|exists:currencies,id`
  - `monthly_salary`: `required_if:pay_basis,monthly|nullable|numeric|min:0|max:99999999999999.99` (when pay_basis is monthly)
  - `daily_rate`: `required_if:pay_basis,daily|nullable|numeric|min:0|max:99999999999999.99` (when pay_basis is daily)
  - `mp2_amount`: `nullable|numeric|min:0|max:999999999999.99`
  - `sss_wisp_amount`: `nullable|numeric|min:0|max:999999999999.99`
  - `effective_date`: `required|date`
  - `notes`: `nullable|string|max:1000`
- The `pay_basis` value is read from the employee model (route-bound), not from the request body.
- Custom validation messages for negative salary: "Salary amount cannot be negative."

### Routes & Controllers

#### `app/Http/Controllers/Web/EmployeeCompensationController.php`
- `edit(Employee $employee)`:
  - Authorize: `owner` or `payroll_officer`.
  - Load the employee's current compensation with currency, and the employee's payroll flags.
  - Fetch all active currencies for the dropdown.
  - Return Inertia response rendering `employees/compensation/edit` with props:
    - `employee`: employee data including `pay_basis`, `has_mp2`, `has_sss_wisp`.
    - `compensation`: current compensation data (or null if not yet set).
    - `currencies`: list of active currencies `[{ id, code, name, symbol }]`.
- `update(UpdateCompensationRequest $request, Employee $employee)`:
  - Execute `UpdateEmployeeCompensationAction` with employee and validated data.
  - Redirect back to the same page with flash success message: "Compensation updated successfully."

#### Route definitions (`routes/web.php`)
```php
Route::middleware(['auth', 'role:owner|payroll_officer'])->group(function () {
    Route::get('/employees/{employee}/compensation', [EmployeeCompensationController::class, 'edit'])
        ->name('employees.compensation.edit');
    Route::put('/employees/{employee}/compensation', [EmployeeCompensationController::class, 'update'])
        ->name('employees.compensation.update');
});
```

### Frontend Components

#### `resources/js/pages/employees/compensation/edit.tsx`
- Inertia page component using `authenticated-layout`.
- Page header: "Compensation - {employee.full_name}" with breadcrumb to employee list.
- Uses Inertia `useForm` initialized with current compensation data (or empty defaults).
- Form sections (using `FormSection` from PAY-024):
  - **Salary Details**:
    - Currency selector (`SelectField`) populated from `currencies` prop.
    - Conditional field based on `employee.pay_basis`:
      - If `monthly`: `NumberInput` for `monthly_salary` with currency symbol prefix.
      - If `daily`: `NumberInput` for `daily_rate` with currency symbol prefix.
    - Display a read-only label showing the employee's `pay_basis` for context.
  - **Voluntary Deductions** (conditionally visible):
    - `NumberInput` for `mp2_amount` - only visible if `employee.has_mp2` is true.
    - `NumberInput` for `sss_wisp_amount` - only visible if `employee.has_sss_wisp` is true.
    - If neither flag is true, show a muted message: "No voluntary deductions enabled for this employee."
  - **Effective Date & Notes**:
    - `DatePicker` for `effective_date`.
    - `TextareaField` for `notes`.
- Form actions: "Save" button, "Cancel" button (links to employee list).
- On submit: `form.put(route('employees.compensation.update', employee.id))`.

### Tests

#### `tests/Feature/Compensation/UpdateCompensationTest.php` (Pest)
- Assert `owner` can access the compensation edit page.
- Assert `payroll_officer` can access the compensation edit page.
- Assert `employee` role receives 403.
- Assert successful update for monthly employee sets `monthly_salary` and nulls `daily_rate`.
- Assert successful update for daily employee sets `daily_rate` and nulls `monthly_salary`.
- Assert validation fails when `monthly_salary` is missing for a monthly employee.
- Assert validation fails when `daily_rate` is missing for a daily employee.
- Assert validation fails for negative salary value.
- Assert validation fails for invalid `currency_id`.
- Assert `effective_date` is required.
- Assert compensation change creates an audit record.
- Assert compensation update redirects back with flash success message.
- Assert first-time compensation creation works (no prior record for employee).

### Edge Cases & Notes
- The compensation page must correctly handle the case where no compensation record exists yet (first-time setup).
- When `has_mp2` or `has_sss_wisp` is false, the corresponding amount fields should not be submitted (or should be ignored server-side).
- The currency symbol displayed next to the salary input should update dynamically when the user changes the currency selector.
- Audit trail captures the old and new values for all changed fields, including the user who made the change.
- Salary fields use `DECIMAL(14,2)` precision; the UI should allow up to 2 decimal places.

## Acceptance Criteria

- [ ] `/employees/{id}/compensation` page loads with current compensation data for `owner` and `payroll_officer`.
- [ ] `employee` and `finance` roles receive 403 on the compensation page.
- [ ] Currency selector displays all active currencies and pre-selects the current compensation currency.
- [ ] Monthly employees see only the `monthly_salary` field; daily employees see only the `daily_rate` field.
- [ ] Voluntary deduction fields (`mp2_amount`, `sss_wisp_amount`) appear only when the corresponding employee flags are enabled.
- [ ] Submitting valid data updates the compensation record and redirects with a success flash message.
- [ ] Validation prevents negative salary values with a clear error message.
- [ ] Validation requires the salary field matching the employee's `pay_basis`.
- [ ] First-time compensation entry creates a new record successfully.
- [ ] All compensation changes are recorded in the audit log with old and new values.
- [ ] Currency symbol next to salary input updates when the currency selector changes.
- [ ] All Pest feature tests pass.
