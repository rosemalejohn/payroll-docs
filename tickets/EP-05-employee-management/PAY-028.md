# [GW] Payroll - Employee Compensation Management

## Description

Build the employee compensation management page, allowing authorized users to view and update an employee's salary details. This ticket also creates the employee compensation table and model. The page displays different fields depending on the pay basis (monthly vs. daily), includes a currency selector for multi-currency salary entry, optional voluntary deduction fields, and a full audit trail of all compensation changes.

## Technical Implementation

### What Migrations

- Create the employee compensations table with ULID primary key, company and employee references (unique constraint on employee — one record per employee), currency reference, monthly salary (nullable for daily-rate employees), daily rate (nullable for monthly employees), MP2 and SSS WISP voluntary deduction amounts, effective date, notes, and creator/updater tracking.

### What Models

- Create EmployeeCompensation model in the Compensation domain with ULID support, factory, and audit trail integration.
- Relationships: belongs to Employee, Company, Currency, creator, and updater.
- Factory includes state methods for daily rate, with MP2, and with SSS WISP.

### What Actions

- Update Employee Compensation action: accepts the employee and validated data, uses update-or-create keyed by employee ID, conditionally sets salary fields based on pay basis (monthly sets monthly salary and nulls daily rate; daily does the reverse), and tracks who made the change.

### Validation & DTOs

- Update Compensation request: validates currency (required, must exist), monthly salary (required when pay basis is monthly), daily rate (required when pay basis is daily), MP2 amount (optional, non-negative), SSS WISP amount (optional, non-negative), effective date (required), and notes (optional).
- Pay basis is read from the employee record, not from the request body.

### Routes & Controllers

- `GET /employees/{employee}/compensation` — edit compensation form
- `PUT /employees/{employee}/compensation` — update compensation
- Controller loads current compensation, employee flags, and active currencies for the dropdown.

### Tests

- Assert Owner and Payroll Officer can access the compensation page.
- Assert Employee and Finance roles receive 403.
- Assert successful update for monthly employee sets monthly salary and nulls daily rate.
- Assert successful update for daily employee sets daily rate and nulls monthly salary.
- Assert validation fails for missing salary, negative values, and invalid currency.
- Assert compensation changes create an audit record with old and new values.
- Assert first-time compensation creation works when no prior record exists.
- Assert unique constraint prevents duplicate compensation records for the same employee.
- Assert factory and state methods produce valid data.

### Edge Cases & Notes

- Handle the case where no compensation record exists yet (first-time setup for a new employee).
- When MP2 or SSS WISP flags are disabled on the employee, the corresponding amount fields should be ignored server-side.
- The currency symbol next to the salary input should update dynamically when the user changes the currency selector.
- Salary fields use decimal precision; the UI should allow up to 2 decimal places.

## Acceptance Criteria

### Client-side

#### Layout

- The compensation page is nested under the employee context, accessible from the employee profile

#### Behavior

- The currency selector displays all active currencies and pre-selects the current compensation currency
- Monthly employees see only the monthly salary field; daily employees see only the daily rate field
- The employee's pay basis is displayed as a read-only label for context
- Voluntary deduction fields (MP2, SSS WISP) appear only when the corresponding employee flags are enabled
- When neither voluntary deduction flag is enabled, a message indicates "No voluntary deductions enabled for this employee"
- After saving, the page refreshes with a success message
- The currency symbol next to the salary input updates when the currency selector changes

#### Validation

- An error is shown if the salary field matching the employee's pay basis is missing
- An error is shown for negative salary or deduction amounts
- An error is shown for an invalid or missing currency selection

---

### Server-side

#### Side-effects

- Each employee has exactly one compensation record — saving creates it the first time and updates it thereafter
- Compensation changes are recorded in the audit trail with old and new values
- Only Owner and Payroll Officer roles can manage compensation
- Either monthly salary or daily rate is set based on the employee's pay basis — never both simultaneously
