# [GW] Payroll - New Employee HMO Proration for Mid-Period Join

## Description

When an employee is added to a payroll run and their hire date falls within the run period, prorate their HMO deduction and other mandatory deductions (SSS, PhilHealth, Pag-IBIG) for the first partial period. The system calculates deductions proportionally based on the number of working days from the hire date to the period end relative to the total working days in the period. Payslip notes document the proration basis and the employee's start date. Payroll officers can override prorated amounts when needed.

## Technical Implementation

### What Migrations

- Add payslip fields for join status, factor, start date.
- Add deduction fields for original, prorated, overridden amounts.

### What Models

- Add mid-period join fields to Payslip fillable and casts.
- Add override fields to PayslipDeduction fillable and casts.

### What Actions

- Detect mid-period joins and calculate proration data.
- Prorate new-hire deductions and store both amounts.
- Prorate monthly gross and annualize withholding tax.
- Allow deduction overrides and recalculate net pay.

### Validation & DTOs

- DTO includes payslip, deduction type, and override amount.
- Validate allowed deduction types and non-negative override amounts.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/payslips/{payslip}/override-deduction` overrides deductions.

### Policies & Authorization

- Only Payroll Officer and Owner can override deductions.
- Payroll run must be draft.

### Tests

- Assert mid-period hires get prorated HMO deductions.
- Assert SSS contribution uses the proration factor.
- Assert PhilHealth contribution uses the proration factor.
- Assert Pag-IBIG contribution uses the proration factor.
- Assert monthly gross pay is prorated.
- Assert daily-rate gross stays unchanged.
- Assert withholding tax uses annualized prorated gross.
- Assert payslip notes include hire date and proration.
- Assert payslip stores join flag and proration factor.
- Assert pre-period hires are not flagged.
- Assert first-day hires are not flagged.
- Assert last-day hires prorate to one day.
- Assert deduction overrides update amount and flag.
- Assert overrides recalculate net pay correctly.
- Assert only authorized roles can override deductions.
- Assert overrides only work in draft runs.
- Assert external-salary employees skip deduction proration.

### Edge Cases & Notes

- First-day hires should not be flagged as mid-period.
- Last-day hires should prorate to one working day.
- Working days must exclude weekends and optional holidays.
- Use DECIMAL precision for proration factors.
- External-salary employees skip deduction proration.
- Preserve overrides during later recomputations.
- SSS proration remains a simplified rule.

## Acceptance Criteria

### Client-side

#### Behavior

- Detect employees hired within the payroll run period.
- Show a "New Hire (Prorated)" badge.
- Show original and prorated deduction amounts.
- Show hire date and working-days ratio.
- Show "Override" beside prorated deductions in draft runs.
- Override dialog shows current and new amounts.
- Show an "Overridden" indicator after overrides.
- Restrict overrides to Payroll Officer and Owner.

---

### Server-side

#### Side-effects

- Flag employees hired within the payroll run period.
- Prorate HMO, SSS, PhilHealth, and Pag-IBIG deductions.
- Prorate monthly gross; keep daily-rate gross unchanged.
- Calculate withholding tax from prorated gross.
- Store original and prorated deduction amounts.
- Allow deduction overrides in draft runs.
- Preserve overrides during payslip recomputation.
- Treat first-day hires as full-period employees.
- Skip deduction proration for external-salary employees.
