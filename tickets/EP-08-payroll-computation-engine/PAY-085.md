# [GW] Payroll - BIR Tax Override Per Employee

## Description

Allow payroll officers to manually override the computed BIR withholding tax for a specific employee within a draft payroll run. Overrides require a reason/note and are tracked in the audit trail with both the original computed value and the new override value. Overridden BIR amounts are marked with an asterisk on the payslip to indicate manual adjustment. Recomputation of the payroll run preserves manual overrides and does not reset them to the computed value.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/OverrideBirTaxAction.php`** — Single-responsibility action that:
  - Accepts payslip ID, override amount (decimal), and reason (string)
  - Validates the payslip's payroll run is in `draft` state
  - Stores the original computed BIR value in `bir_computed` (if not already stored)
  - Sets the new BIR amount in `bir_tax` field
  - Sets `bir_override = true` flag
  - Sets `bir_override_reason` with the provided note
  - Triggers audit log entry via `owen-it/laravel-auditing`
- **`app/Payroll/Actions/ComputePayslipAction.php`** — Update BIR tax computation step (Step 9):
  - Before computing BIR, check if `bir_override = true` on the payslip
  - If overridden, skip BIR computation and retain the manually set value
  - Recompute net pay using the overridden BIR amount
- **`app/Payroll/Actions/RecomputePayrollRunAction.php`** — Ensure recomputation respects override flags:
  - When iterating payslips, pass the override flag to `ComputePayslipAction`
  - Overridden values are not recalculated

### Models
- **`app/Payroll/Models/Payslip.php`** — Add to `$fillable`:
  - `bir_computed` (decimal:2, nullable) — stores the original system-computed BIR value
  - `bir_override` (boolean, default false) — flag indicating manual override
  - `bir_override_reason` (text, nullable) — required reason for the override
  - Cast `bir_override` to boolean, `bir_computed` to `decimal:2`

### Migrations
- **`database/migrations/xxxx_add_bir_override_fields_to_payslips_table.php`**
  - Add `bir_computed` column (decimal 12,2, nullable)
  - Add `bir_override` column (boolean, default false)
  - Add `bir_override_reason` column (text, nullable)
  - Index on `bir_override` for filtering overridden payslips

### Validation & DTOs
- **`app/Payroll/DTOs/OverrideBirTaxData.php`** — spatie/laravel-data DTO:
  - `payslip_id`: string (ULID)
  - `override_amount`: string (decimal, must be >= 0)
  - `reason`: string (required, min 10 characters)
- **`app/Http/Requests/Payroll/OverrideBirTaxRequest.php`** — Validates:
  - `override_amount` is numeric and >= 0
  - `reason` is required and at least 10 characters
  - Payslip exists and its payroll run is in `draft` state

### Routes & Controllers
- **`PATCH /payroll-runs/{payrollRun}/payslips/{payslip}/bir-override`** — Route name: `payslips.bir-override`
- **`app/Http/Controllers/Web/PayslipBirOverrideController@update`**
  - Middleware: `auth`, `role:owner|payroll_officer`
  - Authorizes via `PayslipPolicy@overrideBir`
  - Delegates to `OverrideBirTaxAction`
  - Redirects back with flash message

### Policies & Authorization
- **`app/Payroll/Policies/PayslipPolicy.php`** — Add `overrideBir()` method:
  - `owner` and `payroll_officer` roles can override
  - Payslip's payroll run must be in `draft` state

### Tests
- **`tests/Feature/Payroll/BirTaxOverrideTest.php`**
  - Payroll officer can override BIR tax for an employee in a draft run
  - Override requires a reason (422 if reason is missing or too short)
  - Override amount must be >= 0 (422 for negative values)
  - Override is not allowed on non-draft runs (422)
  - Audit trail records old (computed) and new (override) BIR values
  - Recomputation of payroll run preserves the overridden BIR value
  - Overridden BIR is reflected in net pay calculation
  - Payslip shows asterisk indicator for overridden BIR
  - `bir_computed` stores the original system-calculated value
  - Non-authorized roles receive 403
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- The `bir_computed` field preserves the original system-calculated value so it can be displayed alongside the override for transparency
- If an override is set and then the payroll officer wants to revert to the computed value, they can set the override amount equal to `bir_computed` — or a "Reset to Computed" action could be added as a follow-up ticket
- Multiple overrides on the same payslip should update the existing override, not create duplicates — each override replaces the previous one
- Audit trail via `owen-it/laravel-auditing` captures each override as a separate audit entry with full before/after values

## Acceptance Criteria

- [ ] Override field is available per employee in the payroll run detail page (draft state only)
- [ ] Reason/note is required when overriding BIR tax (minimum 10 characters)
- [ ] Audit trail logs the override with old (computed) and new (override) values
- [ ] Override persists through payroll run recomputation (not auto-reset)
- [ ] Net pay is recalculated using the overridden BIR amount
- [ ] Asterisk (*) is shown on the payslip for overridden BIR with footnote explaining the adjustment
- [ ] Original computed BIR value is preserved and accessible for reference
