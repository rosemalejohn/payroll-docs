# PAY-003: Set Up Domain-Driven Folder Structure

## Description

Establish the domain-driven directory structure for both the backend (Laravel) and frontend (React/Inertia). This ticket creates all domain folders and their standard subdirectories so that subsequent tickets can place files in the correct locations without creating directories ad hoc. The autoloader must recognize the new namespaces.

## Technical Implementation

### Backend Domain Folders
Create the following domain directories under `app/`, each with the standard set of subdirectories:

| Domain | Path |
|---|---|
| Company | `app/Company/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Employee | `app/Employee/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Payroll | `app/Payroll/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Compensation | `app/Compensation/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Currency | `app/Currency/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Bonus | `app/Bonus/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| CashAdvance | `app/CashAdvance/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |
| Audit | `app/Audit/{Actions,Models,Policies,DTOs,Events,Listeners,Jobs,Queries}` |

### HTTP Layer Directories
- `app/Http/Controllers/Web/` (thin Inertia controllers)
- `app/Http/Requests/` (Form Request classes)
- `app/Http/Resources/` (API Resource classes for JSON shaping)

### Frontend Directories
Create under `resources/js/`:
- `pages/` (Inertia page components)
- `components/ui/` (Shadcn/ui base primitives, may already exist from PAY-002)
- `components/layout/` (shell components: sidebar, header, breadcrumbs)
- `components/forms/` (reusable form controls and field wrappers)
- `components/tables/` (data grids, filters, pagination)
- `components/feedback/` (alerts, empty states, loaders, confirmations)
- `layouts/` (authenticated-layout.tsx, guest-layout.tsx)
- `hooks/` (custom React hooks)
- `types/` (TypeScript type definitions and interfaces)
- `lib/` (utility functions, constants)

### Autoloading Configuration
- Update `composer.json` PSR-4 autoload map to include domain namespaces:
  ```json
  "autoload": {
    "psr-4": {
      "App\\": "app/",
      "App\\Company\\": "app/Company/",
      "App\\Employee\\": "app/Employee/",
      "App\\Payroll\\": "app/Payroll/",
      "App\\Compensation\\": "app/Compensation/",
      "App\\Currency\\": "app/Currency/",
      "App\\Bonus\\": "app/Bonus/",
      "App\\CashAdvance\\": "app/CashAdvance/",
      "App\\Audit\\": "app/Audit/"
    }
  }
  ```
- Run `composer dump-autoload` to regenerate the autoloader.
- Place a `.gitkeep` file in each empty directory so Git tracks the structure.

### Tests
- `tests/Feature/DomainStructureTest.php` (Pest): asserts that all expected domain directories exist and that classes in domain namespaces can be autoloaded (create a dummy class, instantiate it, then clean up).

### Edge Cases & Notes
- Laravel's default `app/Models/User.php` will be moved to `app/Company/Models/User.php` in a later ticket (PAY-007). Do not move it here to avoid breaking default auth.
- Ensure no namespace collisions with Laravel's default `App\Http` namespace.
- The `App\\` base mapping already covers all subdirectories, so the explicit domain mappings are optional but improve IDE support.

## Acceptance Criteria

- [ ] All 8 backend domain directories exist under `app/` with their 8 subdirectories each (Actions, Models, Policies, DTOs, Events, Listeners, Jobs, Queries).
- [ ] `app/Http/Controllers/Web/`, `app/Http/Requests/`, and `app/Http/Resources/` directories exist.
- [ ] All frontend directories exist under `resources/js/` (pages, components/ui, components/layout, components/forms, components/tables, components/feedback, layouts, hooks, types, lib).
- [ ] `composer dump-autoload` runs without errors.
- [ ] A PHP class placed in `app/Company/Models/` with namespace `App\Company\Models` is autoloaded correctly.
- [ ] A PHP class placed in `app/Payroll/Actions/` with namespace `App\Payroll\Actions` is autoloaded correctly.
- [ ] `.gitkeep` files are present in all empty directories.
- [ ] No existing Laravel functionality is broken (default routes, artisan commands still work).
- [ ] `php artisan test` passes with no namespace-related errors.
