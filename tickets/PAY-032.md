# PAY-032: User Management Page (Owner Only)

## Description

As an owner, I want to manage user accounts and assign roles so that access to the payroll system is controlled. Only the owner can create users, assign roles, and toggle active status.

## Technical Implementation

### Controller
- **`UserController`** — `app/Http/Controllers/Web/UserController.php`
  - `index`: paginated user list with filters
  - `create`: create user form
  - `store`: create user with role
  - `edit`: edit user form
  - `update`: update user info, role, active status

### Routes
- `GET /users` — index
- `GET /users/create` — create form
- `POST /users` — store
- `GET /users/{user}/edit` — edit form
- `PUT /users/{user}` — update

### Policy
- **`UserPolicy`** — all actions restricted to `owner` role only

### Actions
- **`app/Company/Actions/CreateUserAction.php`**: creates user, assigns role, hashes password
- **`app/Company/Actions/UpdateUserAction.php`**: updates user, changes role if needed
  - Guard: cannot remove last owner account

### Form Requests
- **`StoreUserRequest`**: name (required), email (required, unique), password (required, min:8), role (required, in:owner,payroll_officer,finance,employee)
- **`UpdateUserRequest`**: name, email (unique except current), role, is_active

### Frontend
- **`resources/js/pages/users/index.tsx`**: user list table with name, email, role badge, active status, last login
- **`resources/js/pages/users/create.tsx`**: create form
- **`resources/js/pages/users/edit.tsx`**: edit form with role dropdown, active toggle
- Role displayed as colored badge (owner=purple, payroll_officer=blue, finance=green, employee=gray)

### Tests
- Owner can access user management pages
- Non-owner roles get 403
- User created with correct role
- Cannot remove last owner
- Role changes logged in audit trail

## Acceptance Criteria

- [ ] Only owner role can access user management
- [ ] Users can be created with name, email, password, and role
- [ ] Role can be changed and takes effect immediately
- [ ] Active status can be toggled (inactive users cannot login)
- [ ] Cannot remove the last owner account (validation error)
- [ ] Role changes are logged in audit trail
- [ ] Non-owner roles receive 403 on all user management routes
