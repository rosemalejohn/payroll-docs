# [GW] Payroll - Create Companies Migration, Model, and Seeder

## Description

Create the `companies` table, Eloquent model, factory, and seeder. The companies table stores the company/tenant record. While the system is designed for single-company deployment, this table supports future multi-tenancy. This is the first domain model in the application and establishes the pattern for all subsequent models (ULID primary keys, no database-level FK constraints, app-layer integrity).

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_companies_table.php`:
  ```php
  Schema::create('companies', function (Blueprint $table) {
      $table->char('id', 26)->primary();          // ULID
      $table->string('name', 255);
      $table->string('code', 50)->unique();
      $table->text('address')->nullable();
      $table->string('tin', 50)->nullable();       // Tax Identification Number
      $table->string('sss_number', 50)->nullable();
      $table->string('philhealth_no', 50)->nullable();
      $table->string('pagibig_no', 50)->nullable();
      $table->string('logo_path', 500)->nullable();
      $table->boolean('is_active')->default(true);
      $table->timestamps();
  });
  ```

### What Models
- Create `app/Company/Models/Company.php`:
  ```php
  namespace App\Company\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;

  class Company extends Model
  {
      use HasUlids, HasFactory;

      protected $fillable = [
          'name', 'code', 'address', 'tin',
          'sss_number', 'philhealth_no', 'pagibig_no',
          'logo_path', 'is_active',
      ];

      protected $casts = [
          'is_active' => 'boolean',
      ];

      // Relationships
      public function users() { return $this->hasMany(User::class); }
      public function employees() { return $this->hasMany(\App\Employee\Models\Employee::class); }
      public function currencies() { return $this->hasMany(\App\Currency\Models\Currency::class); }
      public function salaryComponents() { return $this->hasMany(\App\Compensation\Models\SalaryComponent::class); }

      // Scopes
      public function scopeActive($query) { return $query->where('is_active', true); }
  }
  ```

### Edge Cases & Notes
- The `id` column uses `CHAR(26)` to store ULIDs, not the default auto-incrementing integer. Ensure `HasUlids` trait is properly generating ULID values.
- No database-level FK constraints are created. Relationships are defined only at the Eloquent model level.
- The factory must be discoverable by Laravel. If the model is in a non-standard namespace (`App\Company\Models`), the factory's `$model` property must be explicitly set.
- Logo upload functionality is out of scope for this ticket; `logo_path` is nullable.
