# PAY-017: Create Payroll Run Bonuses Migration and Model

## Description

Create the `payroll_run_bonuses` table and model to link bonus types to specific payroll runs. Each record represents a bonus configured for a particular run with optional overrides of the default amount/rate.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_payroll_run_bonuses_table.php`**
  ```sql
  id                      CHAR(26) PRIMARY KEY  -- ULID
  payroll_run_id          CHAR(26) NOT NULL
  bonus_type_id           CHAR(26) NOT NULL
  bonus_code              VARCHAR(50) NOT NULL   -- denormalized from bonus_types
  bonus_name              VARCHAR(150) NOT NULL  -- denormalized from bonus_types
  amount                  DECIMAL(12,2) NULL     -- override for this run
  rate                    DECIMAL(7,4) NULL      -- override for this run
  percentage_basis        ENUM('gross','basic') NULL
  attendance_threshold_days INT NULL
  apply_to_all            TINYINT(1) NOT NULL DEFAULT 1
  is_active               TINYINT(1) NOT NULL DEFAULT 1
  added_by                CHAR(26) NULL          -- ref: users.id
  timestamps
  UNIQUE KEY uq_payroll_run_bonuses_run_type (payroll_run_id, bonus_type_id)
  ```
- Index: `idx_run_bonuses_run` on `payroll_run_id`

### Model
- **`app/Bonus/Models/PayrollRunBonus.php`**
  - Uses `HasUlids` trait
  - Casts: `amount` → `decimal:2`, `rate` → `decimal:4`, `apply_to_all` → `boolean`, `is_active` → `boolean`
  - Relationships: `belongsTo(PayrollRun::class)`, `belongsTo(BonusType::class)`, `hasMany(PayrollRunBonusExclusion::class)`, `belongsTo(User::class, 'added_by')`
  - Scope: `scopeActive()`

### Tests
- One bonus type per run enforced by unique constraint
- Denormalized code/name populated correctly
- Override amounts don't affect bonus_type defaults

## Acceptance Criteria

- [ ] Migration runs without errors
- [ ] One bonus type per run enforced (unique constraint on `payroll_run_id + bonus_type_id`)
- [ ] Denormalized `bonus_code` and `bonus_name` populated from `bonus_types` on creation
- [ ] Override `amount`/`rate` is per-run and does not modify `bonus_types` defaults
- [ ] `apply_to_all` flag controls whether exclusions are checked
- [ ] `is_active` toggle allows soft-disabling a bonus for a run
