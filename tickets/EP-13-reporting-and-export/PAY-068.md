# [GW] Payroll - CSV and PDF Export for Finalized Runs

## Description

Implement CSV and PDF export functionality for finalized payroll runs. The CSV export uses maatwebsite/laravel-excel and includes per-employee payroll data with all earnings, deductions, bonuses, and currency details. The PDF export uses barryvdh/laravel-dompdf and generates a company-level summary report.

## Technical Implementation

### What Actions

- Generate Payroll Summary PDF action: reuses the payroll summary query for data aggregation, renders a summary Blade template, and returns the PDF.

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}/export/csv` — download CSV export.
- `GET /payroll-runs/{payrollRun}/export/pdf` — download PDF summary export.

### Policies & Authorization

- Only Owner and Finance roles can export.
- Payroll run must be in finalized state.

### Tests

- Assert CSV downloads with correct filename format and column headers.
- Assert CSV data matches finalized payslip data for each employee.
- Assert CSV includes all deduction types, bonus data, and currency details.
- Assert foreign currency employees show salary currency, exchange rate, and PHP equivalent.
- Assert PDF downloads with correct filename and summary totals matching the report page.
- Assert only Owner and Finance roles can export (others receive 403).
- Assert non-finalized run returns 403.
- Assert empty run produces valid CSV with headers only.

### Edge Cases & Notes

- CSV uses chunked query reading for large runs to avoid memory issues.
- Period slug in filenames should be URL-safe.
- CSV decimal values should not have thousands separators (raw numbers for spreadsheet compatibility).
- PDF summary should use formatted numbers with thousands separators.

## Acceptance Criteria

### Client-side

#### Behavior

- Export CSV and Export PDF buttons are available on the payroll run detail page and summary report for finalized runs
- CSV filename follows the pattern "payroll-{period-slug}.csv"
- PDF filename follows the pattern "payroll-summary-{period-slug}.pdf"

---

### Server-side

#### Side-effects

- CSV includes per-employee payroll data with all earnings, deductions, bonuses, and currency details
- Foreign currency employees show salary currency, exchange rate, and PHP equivalent in the CSV
- PDF summary report totals match the payroll summary report page
- Only finalized payroll runs can be exported
- Only Owner and Finance roles can access export endpoints
- CSV handles special characters in names correctly
- Large payroll runs export without memory issues
