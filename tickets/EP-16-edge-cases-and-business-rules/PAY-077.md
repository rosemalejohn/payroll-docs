# [GW] Payroll - Mid-Month Employee Termination Handling

## Description

Handle the scenario where an employee is marked as terminated with an effective date falling within an active payroll run period. When this occurs, the payroll officer is prompted to specify the employee's last working day. The system prorates the employee's gross pay based on actual days worked, recomputes all deductions on the prorated amount, and annotates the payslip with termination details. This ensures terminated employees are compensated accurately for their partial period.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/HandleTerminatedEmployeePayslipAction.php`** — Orchestrates termination proration:
  - Accepts: payroll run, employee, last working day (date)
  - Validates last working day falls within the run period
  - Calculates `days_worked` based on working days from `period_start` to `last_working_day`
  - Calculates `total_working_days` for the full period
  - Prorates gross pay:
    - Monthly employee: `(basic_salary / total_working_days) × days_worked`
    - Daily rate employee: `daily_rate × days_worked`
  - Calls `ComputePayslipAction` with overridden prorated gross
  - Updates payslip notes with termination info

- **`app/Payroll/Actions/ProrateDeductionsAction.php`** — Prorates mandatory deductions based on prorated gross:
  - SSS: look up bracket based on prorated gross (not full salary)
  - PhilHealth: 2.5% of prorated gross, respecting floor/ceiling
  - Pag-IBIG: fixed ₱100 (not prorated, as per standard rules, unless partial month policy applies)
  - HMO: `(hmo_monthly / total_working_days) × days_worked`
  - Withholding tax: annualize prorated gross → apply TRAIN brackets → de-annualize

- **`app/Payroll/Actions/SetEmployeeTerminationAction.php`** — Marks the employee payslip as terminated:
  - Sets `is_terminated` flag on the payslip
  - Sets `termination_date` and `last_working_day` on the payslip
  - Adds payslip notes: "Terminated effective {date}. Last working day: {date}. Prorated for {days_worked}/{total_working_days} working days."

### Routes & Controllers
- **`POST /payroll-runs/{payrollRun}/payslips/{payslip}/termination`** — Route name: `payroll-runs.payslips.termination.store`
- **`app/Http/Controllers/Web/PayslipTerminationController@store`**
  - Middleware: `auth`, `role:payroll_officer|owner`

  - Delegates to `HandleTerminatedEmployeePayslipAction`

### Tests
- **`tests/Feature/Payroll/MidMonthTerminationTest.php`**
  - Monthly employee prorated correctly: salary / working_days × days_worked
  - Daily rate employee prorated correctly: daily_rate × days_worked
  - Deductions (SSS, PhilHealth, HMO) recalculated based on prorated gross
  - Withholding tax recalculated based on prorated gross
  - Payslip notes include termination date and proration basis
  - Last working day must be within the run period
  - Last working day after period end returns validation error
  - Terminated flag is set on the payslip
  - Only draft runs allow termination marking
  - Non-payroll_officer/owner roles receive 403
  - External salary employees skip deduction proration (steps 3-9)
  - Net pay is computed correctly after proration

### Edge Cases & Notes
- External salary employees (`has_external_salary=true`) still have their gross prorated but skip deduction steps 3-9 per the computation engine rules
- Pag-IBIG proration rules may vary by company policy; current implementation keeps the fixed ₱100 but this should be configurable
- If the termination date is the first day of the period, the employee effectively has 0 working days — handle gracefully (payslip with zero gross)
- Working days calculation must account for weekends and potentially company holidays
- If the employee is un-terminated (correction), the payslip should be recomputable to the full period amount

## Acceptance Criteria

- [ ] All deductions (SSS, PhilHealth, HMO, withholding tax) are recalculated based on prorated gross
- [ ] Payslip notes display termination date, last working day, and proration basis
- [ ] Terminated payslip shows a visual "Terminated" indicator
- [ ] Last working day validation ensures it falls within the run period
- [ ] External salary employees have gross prorated but skip deduction steps 3-9
- [ ] Only payroll_officer and owner roles can set termination
- [ ] Only draft runs allow termination marking
- [ ] All termination proration tests pass
