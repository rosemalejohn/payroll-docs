# PAY-083: Employer Contributions in Payroll Run Summary

## Description

Add employer-side contribution calculations and display to the payroll run detail and summary report. For each employee in a payroll run, the system computes the employer share of SSS, PhilHealth, and Pag-IBIG using the same bracket tables and rates as the employee deductions. These amounts are displayed alongside employee deductions in the run summary, included in the total employer cost calculation, and exported in the CSV report.

Employer contributions are computed and stored per payslip for historical accuracy. The total employer cost for a run is the sum of all employee net pay plus all employer contributions across the run.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/ComputeEmployerContributionsAction.php`** — Single-responsibility action that:
  - Accepts a payslip and its associated employee gross_pay
  - Computes employer SSS: looks up the employer share column from the SSS bracket table based on the employee's gross_pay (Step 1 basic salary)
  - Computes employer PhilHealth: 2.5% of basic salary (same as employee share), subject to the same salary floor and ceiling
  - Computes employer Pag-IBIG: fixed ₱100 matching contribution
  - Returns an `EmployerContributionsData` DTO with all three values and their sum
  - Skips computation for employees with `has_external_salary = true` (returns zeroes)
- **`app/Payroll/Actions/ComputePayslipAction.php`** — Update to invoke `ComputeEmployerContributionsAction` after employee deduction computation and store results on the payslip

### Edge Cases & Notes
- Employer contributions use the same gross_pay (Step 1 basic salary) as the basis, not adjusted gross after bonuses/deductions
- SSS employer share is a separate column in the bracket lookup table — do not confuse with the employee share
- Employer contributions are informational and do not affect the employee's net pay calculation
- For external salary employees (`has_external_salary = true`), all employer contributions are zero
