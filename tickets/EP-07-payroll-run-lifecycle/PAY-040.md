# [GW] Payroll - Payroll Run Detail Page

## Description

Build the payroll run detail page with a tabbed layout showing employee payslips, bonuses, currency rates, and audit history. The page adapts based on the run's current state — draft runs are editable, submitted/under review runs are read-only with review actions, and finalized runs are fully read-only with download options. Action buttons are role-dependent.

## Technical Implementation

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}` — detail view with relationships loaded based on active tab. Eager loads relationships to avoid N+1 queries.

### Policies & Authorization

- Owner, Payroll Officer, and Finance can view payroll run details.
- Draft state: Payroll Officer and Owner can edit.
- Submitted state: Finance and Owner can place under review.
- Under Review state: Finance can approve, Owner can finalize.
- Non-finalized runs: Payroll Officer and Owner can return to draft.

### Tests

- Assert the detail page loads with correct run data and financial totals.
- Assert tabs switch correctly between Employees, Bonuses, Currency Rates, and Audit History.
- Assert draft mode allows inline editing of absent days and bonus management.
- Assert finalized mode is fully read-only with download options.
- Assert role-based action buttons are visible only to appropriate roles.
- Assert Employee role cannot access the detail page (403).

## Acceptance Criteria

### Client-side

#### Layout

- The detail page header shows: Run Name, Period, State Badge, and Financial Totals (Gross, Deductions, Net) with Employee Count

#### Behavior

- The page has four tabs: Employees, Bonuses, Currency Rates, and Audit History
- The Employees tab shows a table with: Employee Name, Gross Pay, Total Bonuses, Total Deductions, Net Pay, and Absent Days (editable in draft state)
- The Bonuses tab shows attached bonuses with amount/rate and apply-to-all status, with add/remove/edit controls in draft state only
- The Currency Rates tab shows snapshotted exchange rates with code, rate, and snapshot date
- The Audit History tab shows a timeline of state transitions and changes
- In draft state, an action bar shows a "Submit for Review" button for Payroll Officer and Owner
- In submitted state, a "Place Under Review" button is visible for Finance and Owner
- In under review state, an "Approve" button is visible for Finance and a "Finalize" button for Owner only
- On any non-finalized state, a "Return to Draft" button with reason input is available
- Finalized runs show download actions (PDF, CSV) and all fields are read-only

---

### Server-side

#### Side-effects

- The detail page eager-loads relationships based on the active tab to optimize performance
- Only authorized roles can view the detail page
- Action buttons are determined by both the run state and the user's role
