# [GW] Payroll - Build Reusable Form Components

## Description

Create a library of reusable form field wrapper components that handle labels, validation error display, and description text. These components integrate with Inertia's form handling for simple forms and display server-side validation errors inline. This library will be used across all create/edit pages in the application.

## Technical Implementation

### Edge Cases & Notes

- Field components must handle missing error values gracefully (no error displayed when undefined).
- All inputs must have proper label associations for accessibility.
- Number input must handle empty values on clear and reject non-numeric paste.
- Date picker values must always be stored as ISO date strings for server compatibility.
- Form section component should support nesting without visual conflicts.

## Acceptance Criteria

### Client-side

#### Layout

- Each form field component renders a label above the input, with optional description text below

#### Behavior

- **Text Input:** Renders a standard text field supporting text, email, and password types
- **Number Input:** Renders a numeric field with min/max/step support that blocks non-numeric entry
- **Select Field:** Renders a dropdown with a list of options and displays the selected value
- **Date Picker:** Renders a date selector and stores the value in a standard date format
- **Toggle Field:** Renders a switch control with a label for boolean on/off settings
- **Textarea:** Renders a multi-line text area with configurable height
- **Form Section:** Groups related fields with a title and optional description for visual separation
- **Form Actions:** Renders submit and cancel buttons at the bottom of the form

#### Validation

- Server-side validation errors display inline below the relevant field in red text
- The submit button shows a loading indicator and is disabled while the form is being submitted
- Each field component works seamlessly with Inertia's form handling hook
