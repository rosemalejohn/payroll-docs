# [GW] Payroll - Cash Advance Management

## Description

Build the cash advance management interface including the cash advances database table and model. Cash advances track money issued to employees that is later recovered through payroll deductions. The Owner and Payroll Officer can create, view, and manage cash advances. Each advance tracks the issued amount, recovery progress, and status (active, fully recovered, or cancelled).

## Technical Implementation

### What Migrations

- Create the cash advances table with ULID primary key, company and employee references, amount issued, amount recovered, remaining balance, issued date, status (active/fully_recovered/cancelled), notes, creator reference, and standard timestamps.

### What Models

- Create CashAdvance model in the CashAdvance domain with ULID support, factory, and audit trail integration.
- Relationships: belongs to Company, Employee, and creator (User).
- Scopes: active advances, filter by employee.
- Balance is auto-calculated as amount minus amount recovered.

### What Actions

- Create Cash Advance action: creates a new advance with initial balance equal to the amount, zero recovery, and active status.
- Update Cash Advance action: allows updating notes and cancelling active advances. Amount can only be changed if no recovery has started. Cancelled advances have their balance set to zero.

### Validation & DTOs

- Store Cash Advance request: validates employee (required, must exist), amount (required, minimum 0.01), issued date (required), and notes (optional).
- Update Cash Advance request: validates notes (optional), status (only cancellation allowed), and amount (only if no recovery has started).
- DTO for cash advance data using spatie/laravel-data.

### Routes & Controllers

- `GET /cash-advances` — paginated list with filters for employee, status, and date range.
- `POST /cash-advances` — create new advance.
- `GET /cash-advances/{cashAdvance}` — detail view with recovery history.
- `PUT /cash-advances/{cashAdvance}` — update advance.
- All routes restricted to Owner and Payroll Officer.

### Queries

- Cash advance list query with filtering by employee, status, and date range. Eager loads employee data, ordered by issued date descending.

### Policies & Authorization

- Owner and Payroll Officer can view and manage cash advances.
- Employees can view their own cash advances only.
- Finance and other roles cannot create or manage advances.
- Only active advances can be updated or cancelled.

### Tests

- Assert Owner and Payroll Officer can create cash advances with correct initial state.
- Assert Finance and Employee roles cannot create advances (403).
- Assert balance is calculated correctly as amount minus amount recovered.
- Assert active advances can be cancelled.
- Assert fully recovered and cancelled advances cannot be edited.
- Assert amount can only be changed if no recovery has started.
- Assert filters by employee, status, and date range work correctly.
- Assert the list is paginated and ordered by issued date descending.
- Assert all changes are captured in the audit trail.

## Acceptance Criteria

### Client-side

#### Layout

- Accessible from sidebar for Owner and Payroll Officer

#### Behavior

- List shows employee, amount, recovered, balance, status, date
- Status badges: Active (blue), Recovered (green), Cancelled (gray)
- Filter by employee, status, and date range
- Create form: Employee, Amount, Issued Date, Notes
- Active advances: update notes or cancel
- Detail view shows advance info and recovery timeline

#### Validation

- Error if amount is zero or negative
- Error if employee selection is missing
- Error if editing cancelled or fully recovered advance
- Error if changing amount after recovery started

---

### Server-side

#### Side-effects

- New advances: active status, zero recovery, full balance
- Cancelling sets balance to zero, prevents recovery
- Only active advances can be modified
- Amount only changeable before recovery starts
- All changes recorded in audit trail
- Only Owner and Payroll Officer can manage
