# PAY-067: Payroll Summary Report Page

## Description

Implement a payroll summary report page accessible to `owner` and `finance` roles for finalized payroll runs. The report aggregates all payslip data into a company-level summary: total gross pay, total deductions broken down by type (SSS, PhilHealth, Pag-IBIG, HMO, MP2, WISP, Cash Advance, BIR), total net pay, and employer contributions (SSS employer share, PhilHealth employer share, Pag-IBIG employer share). The total employer cost is calculated as the sum of net pay plus all employer contributions.

The report supports filtering by department and pay basis, and can be exported as CSV or PDF.

## Technical Implementation

### Routes & Controllers
- **`GET /reports/payroll-summary/{payrollRun}`** — Route name: `reports.payroll-summary`
- **`app/Http/Controllers/Web/ReportController@payrollSummary`**
  - Middleware: `auth`, `role:owner,finance`
  - Authorizes via `PayrollRunPolicy@viewReport`
  - Accepts query parameters: `department` (filter), `pay_basis` (filter: monthly/daily)
  - Returns Inertia response with aggregated report data

### Queries
- **`app/Payroll/Queries/PayrollSummaryQuery.php`** — Aggregation query class:
  - Accepts payroll run ID, optional department filter, optional pay_basis filter
  - Aggregates across all payslips in the run:
    - `total_gross_pay`: SUM of gross_pay
    - `total_sss`: SUM of SSS deductions
    - `total_philhealth`: SUM of PhilHealth deductions
    - `total_pagibig`: SUM of Pag-IBIG deductions
    - `total_hmo`: SUM of HMO deductions
    - `total_mp2`: SUM of MP2 deductions
    - `total_wisp`: SUM of WISP deductions
    - `total_cash_advance`: SUM of cash advance deductions
    - `total_bir_tax`: SUM of BIR withholding tax
    - `total_deductions`: SUM of all deductions
    - `total_net_pay`: SUM of net_pay
  - Employer contributions:
    - `total_sss_employer`: SUM of SSS employer share
    - `total_philhealth_employer`: SUM of PhilHealth employer share
    - `total_pagibig_employer`: SUM of Pag-IBIG employer share
  - `total_employer_cost`: total_net_pay + total_sss_employer + total_philhealth_employer + total_pagibig_employer
  - Applies department and pay_basis filters via WHERE clauses on joined employee records

### API Resources
- **`app/Http/Resources/Payroll/PayrollSummaryResource.php`** — Shapes the response:
  - `payroll_run`: run details (ID, period, status, finalized_at)
  - `employee_count`: number of employees in the run (after filters)
  - `totals`: all aggregated amounts as formatted decimals
  - `employer_contributions`: SSS, PhilHealth, Pag-IBIG employer shares
  - `total_employer_cost`: computed total
  - `filters`: current active filters
  - `filter_options`: available departments and pay basis values

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Add `viewReport()` method:
  - Only `owner` and `finance` roles can view reports
  - Payroll run must be in `finalized` state

### Frontend Components
- **`resources/js/pages/reports/payroll-summary.tsx`** — Inertia page component:
  - Page title: "Payroll Summary — {period}"
  - Uses authenticated layout
  - Filter bar: department dropdown, pay basis dropdown, apply button
  - Summary cards: total gross pay, total deductions, total net pay, total employer cost
  - Deductions breakdown table: type, employee share total, employer share total (where applicable)
  - Employer contributions section: SSS, PhilHealth, Pag-IBIG employer shares
  - Export buttons: "Export CSV" and "Export PDF" (link to PAY-068 export endpoints)
- **`resources/js/components/reports/summary-cards.tsx`** — Summary metric cards
- **`resources/js/components/reports/deductions-breakdown-table.tsx`** — Deductions breakdown table

### Tests

  - Total deductions match sum of all individual deduction types
  - Total net pay matches sum of all payslip net pay values
  - Employer contributions are calculated correctly
  - Total employer cost equals net pay + employer contributions

### Edge Cases & Notes
- Employer contribution amounts must be stored or computable per payslip (SSS employer share, PhilHealth employer share, Pag-IBIG employer share) — verify these fields exist in the schema
- External salary employees are excluded from deduction totals since they have no deductions
- All aggregated amounts must use `DECIMAL` math to avoid floating point errors
- The department list should be dynamically pulled from employees in the current run, not a static list
- Large runs may have many employees; ensure the aggregation query is efficient (use database-level SUM, not PHP-level iteration)
