# [GW] Payroll - Create Bonus Types Migration, Model, and Seeder

## Description

Create the `bonus_types` table, Eloquent model, and seeder for the 4 system bonus types. Bonus types define the catalog of bonuses available in the system, including built-in types (Meal Allowance, Attendance Bonus, 13th Month, June Special) and custom types created by the owner.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_bonus_types_table.php`**
  ```sql
  id                       CHAR(26) PRIMARY KEY  -- ULID
  company_id               CHAR(26) NOT NULL
  code                     VARCHAR(50) NOT NULL
  name                     VARCHAR(150) NOT NULL
  is_system                TINYINT(1) NOT NULL DEFAULT 0
  computation_type         ENUM('fixed','percentage','formula') NOT NULL DEFAULT 'fixed'
  default_amount           DECIMAL(12,2) NULL      -- for fixed type
  default_rate             DECIMAL(7,4) NULL       -- for percentage (e.g. 0.1000 = 10%)
  percentage_basis         ENUM('gross','basic') NULL DEFAULT 'gross'
  attendance_threshold_days INT NULL
  attendance_threshold_type ENUM('days','percentage') NULL
  auto_trigger             TINYINT(1) NOT NULL DEFAULT 0
  auto_trigger_month       INT NULL                -- 6=June, 12=December, NULL=all months
  is_active                TINYINT(1) NOT NULL DEFAULT 1
  description              TEXT NULL
  sort_order               INT NOT NULL DEFAULT 0
  timestamps
  UNIQUE KEY uq_bonus_types_company_code (company_id, code)
  ```

### Model
- **`app/Bonus/Models/BonusType.php`**
  - Uses `HasUlids` trait
  - Casts: `default_amount` → `decimal:2`, `default_rate` → `decimal:4`, `is_system` → `boolean`, `auto_trigger` → `boolean`
  - Relationships: `belongsTo(Company::class)`, `hasMany(PayrollRunBonus::class)`
  - Scope: `scopeActive()`, `scopeSystem()`, `scopeAutoTriggerable(int $month)`
  - Guard: `isDeletable()` returns false if `is_system = 1`

### Seeder
- **`BonusTypeSeeder`** — seeds 4 system bonus types:

| code | name | computation_type | auto_trigger | auto_trigger_month |
|---|---|---|---|---|
| MEAL_ALLOWANCE | Meal Allowance | fixed | 1 | NULL (all months) |
| ATTENDANCE_BONUS | Attendance Bonus | percentage | 0 | NULL |
| 13TH_MONTH | 13th Month Pay | formula | 1 | 12 |
| JUNE_SPECIAL | June Special Bonus | formula | 1 | 6 |

### Tests
- System bonuses seeded correctly
- System bonuses cannot be deleted
- `computation_type` determines which fields are relevant
- Unique constraint prevents duplicate codes per company

## Acceptance Criteria

- [ ] Migration runs without errors
- [ ] 4 system bonus types seeded with correct configurations
- [ ] System bonuses (`is_system=1`) cannot be deleted
- [ ] `computation_type` determines which default fields are used (amount for fixed, rate for percentage)
- [ ] `auto_trigger` and `auto_trigger_month` configure auto-attachment behavior
- [ ] Unique constraint on `company_id + code` prevents duplicates
