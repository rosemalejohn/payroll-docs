# [GW] Payroll - Build Reusable Data Table Component

## Description

Create a reusable, server-side-driven data table component that supports pagination, sorting, and filtering through URL query parameters. This component will be used across all list pages in the application (employees, payroll runs, payslips, cash advances, etc.) and must handle empty states gracefully.

## Technical Implementation

### Edge Cases & Notes

- All data fetching is server-side — the table receives data from page props, not client-side API calls.
- Search input should be debounced (300ms) to prevent excessive page requests while typing.
- When filters change, pagination must reset to page 1 to avoid showing an empty page.
- The component must handle tables with varying column counts (3 to 12+) without layout breaking.
- All query parameters (sort, filters, page, per_page) should be URL-encoded and bookmarkable.

## Acceptance Criteria

### Client-side

#### Layout

- The data table consists of a filter bar at the top, a sortable column header row, data rows, and pagination controls at the bottom

#### Behavior

- Clicking a sortable column header sorts the table by that column, cycling through ascending, descending, and default order
- A sort direction indicator (arrow icon) appears on the active sort column
- Pagination controls show page numbers, previous/next buttons, and a "Showing X to Y of Z results" summary
- A page size selector allows switching between 10, 25, 50, and 100 items per page
- The search input filters results as the user types (with a slight delay to avoid excessive requests)
- Dropdown filters narrow results by category (e.g., status, department)
- A "Clear filters" button appears when any filter is active and resets all filters when clicked
- Changing a filter resets the page back to page 1
- Changing pages or page size preserves the current filters and sort order
- All filter, sort, and page settings are reflected in the URL and can be bookmarked or shared
- When no data matches the current filters, an empty state message is displayed
- When no data exists at all, a different empty state message is shown
