# [GW] Payroll - Payslip Preview with Draft Watermark

## Description

Implement an in-memory payslip PDF preview for payroll runs in draft or submitted states. The preview uses the same layout as the final payslip PDF but overlays a prominent "DRAFT" watermark across the page. The preview is generated on-the-fly and is never saved to disk. Accessible to Payroll Officer and Owner roles from the payroll run detail page.

## Technical Implementation

### What Actions

- Generate Payslip Preview action: reuses the PDF generation view data logic, passes a preview flag to the template, generates the PDF with a DRAFT watermark, and returns the raw content without saving to disk.

### Routes & Controllers

- GET /payroll-runs/{payrollRun}/payslips/{payslip}/preview — preview the payslip as an inline PDF in the browser.

### Policies & Authorization

- Only Owner and Payroll Officer roles can preview.
- Payroll run must be in draft or submitted state.
- Under review and finalized runs return 403 (use download for finalized).

### Tests

- Assert Payroll Officer can preview a payslip for a draft run.
- Assert Payroll Officer can preview a payslip for a submitted run.
- Assert Owner can preview a payslip for a draft or submitted run.
- Assert Finance and Employee roles receive 403.
- Assert preview of a payslip for an under review or finalized run returns 403.
- Assert preview response has correct PDF content type and inline content disposition.
- Assert preview content includes "DRAFT" watermark.
- Assert preview reflects current computation data.
- Assert no file is written to disk during preview.

### Edge Cases & Notes

- Preview generates synchronously (not queued) to return immediately.
- If computation data changes between previews, the latest data is reflected.
- The watermark must be clearly visible but not obscure the underlying data.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Preview" button is available on each payslip row in the payroll run detail page for Owner and Payroll Officer roles
- The button is visible only when the run is in draft or submitted state
- Clicking opens a modal or side panel displaying the PDF preview without navigating away
- A loading state is shown while the preview PDF is being generated

---

### Server-side

#### Side-effects

- Preview is available for payslips in draft and submitted payroll runs only
- A "DRAFT" watermark is clearly visible on the preview PDF
- Preview content matches the current payslip computation data
- The preview PDF is not saved to disk
- Only Owner and Payroll Officer roles can access the preview
