# [GW] Payroll - Absent Days Entry Per Employee Per Run

## Description

Provide an inline-editable absent days field on each employee row within a draft payroll run's detail view. Updating absent days triggers automatic payslip recomputation so that absent deductions and net pay reflect the change immediately. An optional note field captures the reason for the absence. Half-day increments are supported.

## Technical Implementation

### What Actions

- Update Payslip Absent Days action: validates the parent payroll run is in draft state, validates absent days is not negative, validates absent days does not exceed the payroll period's working days, updates the payslip absent days and note fields, triggers individual payslip recomputation, and updates the parent run totals.

### Validation & DTOs

- Update Absent Days request: validates absent days (required, numeric, minimum 0, maximum equal to the payroll period's working days) and absent note (optional, maximum 500 characters).
- DTO for absent days data using spatie/laravel-data.

### Routes & Controllers

- `PATCH /payroll-runs/{payrollRun}/payslips/{payslip}/absent-days` — update absent days for a specific payslip.

### Policies & Authorization

- Owner and Payroll Officer can update absent days on draft runs only.

### Tests

- Assert absent days can be updated on a draft run payslip.
- Assert absent days field is read-only on non-draft runs.
- Assert validation prevents negative values.
- Assert validation prevents values exceeding the payroll period's working days.
- Assert half-day increments (e.g., 0.5, 1.5) are supported.
- Assert changing absent days triggers automatic payslip recomputation.
- Assert recomputation updates the absent deduction and net pay on the payslip.
- Assert the optional absent note is saved on the payslip record.
- Assert parent run totals are recalculated after the update.

## Acceptance Criteria

### Client-side

#### Behavior

- Inline-editable absent days field per employee row
- Supports half-day increments (0.5, 1.5)
- Auto-submits on blur or Enter
- Optional expandable note for absence reason
- Row values update after recomputation
- Read-only when run is not draft

#### Validation

- Error if value is negative
- Error if value exceeds period working days

---

### Server-side

#### Side-effects

- Update triggers automatic payslip recomputation
- Absent note saved on payslip record
- Parent run totals recalculated after update
- Blocked on non-draft runs
- Changes persisted and reflected on reload
