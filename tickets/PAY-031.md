# [GW] Payroll - HRIS Employee Sync Job

## Description

Implement a daily background job that syncs employee data from the external HRIS application via API. The sync creates new employees and updates existing ones using `hris_employee_id` as the key, while preserving payroll-specific local fields that are never overwritten.

## Technical Implementation

### Job
- **`app/Employee/Jobs/SyncEmployeesFromHrisJob.php`** (queued on Redis)
  - Makes HTTP request: `GET {HRIS_API_URL}/api/payroll/employees`
  - Uses Laravel HTTP client with timeout, retry (3 attempts with exponential backoff)
  - Environment variable: `HRIS_API_URL` in `.env`

### Sync Logic
- For each employee in API response:
  - `updateOrCreate` keyed on `hris_employee_id`
  - **Synced fields (overwritten):** employee_number, first_name, last_name, middle_name, email, department, position, employment_type, employment_status, date_hired
  - **Local-only fields (NEVER overwritten):** pay_basis, has_external_salary, has_mp2, has_sss_wisp, hmo_amount, sss_number, philhealth_number, pagibig_number, bir_tin
  - Updates `last_synced_at = now()` on each synced record

### Action
- **`app/Employee/Actions/SyncEmployeeFromHrisAction.php`** — handles individual employee upsert logic

### Scheduling
- Register in `app/Console/Kernel.php` (or `routes/console.php` for Laravel 12):
  ```php
  Schedule::job(new SyncEmployeesFromHrisJob)->dailyAt('02:00');
  ```

### Error Handling
- API connection failure: log error, retry job (max 3 attempts)
- Individual employee parse error: log and skip, continue with remaining
- Dispatch `HrisSyncCompletedEvent` with summary (created, updated, failed counts)

### Config
- Add to `.env.example`: `HRIS_API_URL=`, `HRIS_API_TOKEN=`

### Tests
- New employees created from HRIS data
- Existing employees updated (synced fields only)
- Local-only fields preserved on sync
- API failure triggers retry
- `last_synced_at` updated after sync

## Acceptance Criteria

- [ ] Job calls HRIS API and syncs employee data
- [ ] New HRIS employees are created locally with correct data
- [ ] Existing employees have synced fields updated without overwriting local-only fields
- [ ] `last_synced_at` updated on each synced employee
- [ ] API failures are logged and job retries up to 3 times
- [ ] Job runs daily at 02:00 via Laravel scheduler
- [ ] Individual employee errors don't halt the entire sync
