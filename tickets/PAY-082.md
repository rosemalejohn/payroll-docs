# PAY-082: Employee Draft Payslip Access Restriction

## Description

Restrict employees to viewing only payslips belonging to finalized payroll runs. Employees must not be able to access payslips in draft, submitted, or under_review states through any means — the payslip list query, direct URL navigation, or API endpoints. No partial or preview payslip data should be exposed to the employee role.

When a payroll run is finalized and the employee's payslip becomes available, they receive an in-app notification linking to the payslip detail page.

## Technical Implementation

### Policies & Authorization
- **`app/Payroll/Policies/PayslipPolicy.php`** — Add/update `view()` method:
  - If the user has the `employee` role, return `true` only when:
    - The payslip belongs to the authenticated employee (`payslip.employee_id === user.employee_id`)
    - The payslip's payroll run state is `finalized`
  - For `owner`, `payroll_officer`, `finance`: allow viewing payslips in any state (existing behavior)
- **`app/Payroll/Policies/PayslipPolicy.php`** — Add `viewAny()` method:
  - Employees can list payslips (filtered to finalized only at query level)
  - Other roles can list all payslips

### Queries
- **`app/Payroll/Queries/EmployeePayslipQuery.php`** — Dedicated query class for employee-facing payslip listing:
  - Joins `payroll_runs` table
  - Filters: `payroll_runs.state = 'finalized'` AND `payslips.employee_id = {authenticated_employee_id}`
  - Orders by payroll period descending (most recent first)
  - Paginates results
  - Never exposes payslips from non-finalized runs regardless of URL parameters

### Routes & Controllers
- **`GET /my-payslips`** — Route name: `my-payslips.index`
- **`GET /my-payslips/{payslip}`** — Route name: `my-payslips.show`
- **`app/Http/Controllers/Web/EmployeePayslipController@index`**
  - Middleware: `auth`, `role:employee`
  - Uses `EmployeePayslipQuery` to fetch only finalized payslips for the authenticated employee
  - Returns Inertia page with paginated payslip list
- **`app/Http/Controllers/Web/EmployeePayslipController@show`**
  - Middleware: `auth`, `role:employee`
  - Authorizes via `PayslipPolicy@view`
  - Returns 403 if payslip's payroll run is not finalized
  - Returns Inertia page with payslip detail

### API Resources
- **`app/Http/Resources/Payroll/EmployeePayslipResource.php`** — Shapes payslip data for employee view:
  - Includes: period, gross pay, deductions breakdown, net pay, bonuses
  - Excludes: internal flags, draft notes, override markers not relevant to employee
  - Never includes data from non-finalized runs

### Events & Listeners
- **`app/Payroll/Listeners/NotifyEmployeesOfAvailablePayslipsListener.php`** — Listens to `PayrollRunFinalizedEvent`:
  - For each payslip in the finalized run, sends notification to the corresponding employee
  - Uses queue to avoid blocking finalization

### Notifications
- **`app/Payroll/Notifications/PayslipAvailableNotification.php`** — Database notification to employee:
  - Payroll period name
  - Net pay amount
  - Link to view the payslip (`/my-payslips/{payslip}`)

### Frontend (Inertia/React)
- **`resources/js/pages/my-payslips/index.tsx`** — Employee payslip list page:
  - Displays only finalized payslips (server-enforced)
  - Columns: period, gross pay, total deductions, net pay, date finalized
  - Link to detail page
- **`resources/js/pages/my-payslips/show.tsx`** — Employee payslip detail page:
  - Full breakdown of earnings, deductions, and net pay
  - Download PDF button (if PDF has been generated)

### Tests
- **`tests/Feature/Payroll/EmployeePayslipAccessTest.php`**
  - Employee can list their own finalized payslips
  - Employee cannot see payslips from draft/submitted/under_review runs in the list
  - Employee accessing a non-finalized payslip via direct URL receives 403
  - Employee cannot access another employee's payslip (403)
  - No draft payslip data leaks through any API response or Inertia props
  - Employee receives notification when a new finalized payslip is available
  - Owner/payroll_officer/finance can view payslips in any state (control test)
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- The query-level filter is the primary defense; the policy is the secondary check — both must enforce the finalized-only rule for employees
- If a payroll run is somehow reverted from finalized (which should not be possible per state machine rules), previously visible payslips should become inaccessible to employees
- Ensure no Inertia shared props or global queries leak draft payslip counts or summaries to the employee role

## Acceptance Criteria

- [ ] Employee payslip list shows only payslips from finalized payroll runs
- [ ] Direct URL navigation to a non-finalized payslip by an employee returns 403
- [ ] No draft, submitted, or under_review payslip data is exposed through any endpoint to the employee role
- [ ] Employee receives an in-app notification when a new finalized payslip is available
- [ ] Employee can only view their own payslips, not other employees' payslips
- [ ] `PayslipPolicy@view` enforces finalized-only access for the employee role
- [ ] Other roles (owner, payroll_officer, finance) can still view payslips in any state
