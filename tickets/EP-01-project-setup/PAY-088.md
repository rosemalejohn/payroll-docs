# [GW] Payroll - Comprehensive Pest Test Suite Setup

## Description

Set up the foundational Pest testing framework with helpers, factories, and test suite structure covering authentication, authorization, payroll computation, and the full payroll run lifecycle. This ensures the application has reliable automated testing from the start and establishes patterns for all subsequent test development.

## Technical Implementation

### Tests

- Configure Pest with database refresh support and parallel testing.
- Create test helper functions for common operations: acting as a specific role, seeding system data, creating a complete payroll run with employees and payslips, creating an employee with compensation, and custom payslip amount assertions.
- Define test suite structure:
  - Authentication and authorization: login, role-based route access for all 4 roles.
  - Employee management: CRUD, compensation updates, payroll flag toggling.
  - Payroll run lifecycle: creation, state transitions, submission, finalization.
  - Payroll computation: gross pay, SSS brackets, PhilHealth, Pag-IBIG, BIR tax brackets, net pay, external salary path.
  - Currency: management, snapshot creation and usage in computation.
  - Bonus: auto-trigger logic, exclusions, computation types.
  - Cash advance: CRUD, recovery after finalization.
  - Access control: payroll officer cannot finalize (403), employee cannot see draft payslips (403).
- Ensure all model factories are created with realistic data for every domain model.

### Edge Cases & Notes

- Tests must be CI-ready with no external dependencies and deterministic results.
- Test helpers should significantly reduce boilerplate across test files.
- Factories must handle non-standard model namespaces by explicitly setting the model property.

## Acceptance Criteria

### Server-side

#### Side-effects

- The full test suite runs without errors using the test command
- Tests can be run in parallel for faster execution
- Test helpers reduce repetitive setup code across all test files
- All model factories generate valid, persistable data
- Authentication tests verify login/logout and role-based access
- Computation tests verify correct monetary amounts for all 10 steps of the payroll pipeline
- Access control tests verify that restricted actions return proper denial responses
- Tests are deterministic and do not depend on external services
