# PAY-048: ComputePayslipAction - BIR Withholding Tax (Step 9)

## Description

Implement Step 9 of the 10-step payroll computation pipeline: BIR withholding tax calculation using the TRAIN Law (Tax Reform for Acceleration and Inclusion) annual tax brackets. The taxable base is gross pay minus statutory deductions only (SSS, PhilHealth, Pag-IBIG). The monthly income is annualized, the tax bracket is applied, and the annual tax is divided by 12 to yield the monthly withholding tax. This step is skipped for employees with `has_external_salary = true`.

## Technical Implementation

### Action

**`app/Payroll/Actions/ComputePayslipAction.php`** (Step 9 segment)

Skipped if `$employee->has_external_salary === true`.

**Compute taxable income:**

```php
$taxableMonthly = $grossPay - $sssEmployeeShare - $philhealthDeduction - $pagibigDeduction;
$taxableAnnual = $taxableMonthly * 12;
```

The taxable base excludes bonuses, absent deductions, HMO, voluntary deductions, and cash advances. Only the three mandatory statutory deductions (SSS, PhilHealth, Pag-IBIG) are subtracted from `gross_pay` (Step 1).

**BIR TRAIN Law bracket lookup:**

```php
$brackets = config('payroll.bir_tax_brackets');
// Returns: annual_tax = base_tax + (taxable_annual - excess_over) * marginal_rate
$annualTax = BirTaxLookup::computeAnnualTax($taxableAnnual);
$monthlyBirTax = round($annualTax / 12, 2);
```

**Tax brackets (stored in `config/payroll.php`):**

```php
'bir_tax_brackets' => [
    ['min' => 0,         'max' => 250000,    'base_tax' => 0,         'rate' => 0.00, 'excess_over' => 0],
    ['min' => 250001,    'max' => 400000,    'base_tax' => 0,         'rate' => 0.15, 'excess_over' => 250000],
    ['min' => 400001,    'max' => 800000,    'base_tax' => 22500,     'rate' => 0.20, 'excess_over' => 400000],
    ['min' => 800001,    'max' => 2000000,   'base_tax' => 102500,    'rate' => 0.25, 'excess_over' => 800000],
    ['min' => 2000001,   'max' => 8000000,   'base_tax' => 402500,    'rate' => 0.30, 'excess_over' => 2000000],
    ['min' => 8000001,   'max' => PHP_INT_MAX, 'base_tax' => 2202500, 'rate' => 0.35, 'excess_over' => 8000000],
],
```

**Record line item:**

```php
if ($monthlyBirTax > 0) {
    PayslipLineItem::create([
        'id'         => Str::ulid(),
        'payslip_id' => $payslip->id,
        'code'       => 'BIR_TAX',
        'label'      => 'BIR Withholding Tax',
        'type'       => 'deduction',
        'amount'     => $monthlyBirTax,
        'sort_order' => $sortOrder++,
    ]);
}

$payslip->update(['bir_tax' => $monthlyBirTax]);
```

### Helper Class

**`app/Payroll/Support/BirTaxLookup.php`**

```php
class BirTaxLookup
{
    public static function computeAnnualTax(float $taxableAnnual): float
    {
        if ($taxableAnnual <= 0) {
            return 0;
        }

        $brackets = config('payroll.bir_tax_brackets');

        foreach ($brackets as $bracket) {
            if ($taxableAnnual >= $bracket['min'] && $taxableAnnual <= $bracket['max']) {
                return $bracket['base_tax']
                    + ($taxableAnnual - $bracket['excess_over']) * $bracket['rate'];
            }
        }

        // Fallback to highest bracket
        $last = end($brackets);
        return $last['base_tax']
            + ($taxableAnnual - $last['excess_over']) * $last['rate'];
    }
}
```

### Edge Cases

- Taxable annual <= 250,000: zero tax (first bracket).
- Negative taxable monthly (unlikely but guarded): treated as 0 taxable, zero tax.
- Very high salaries (> 8M annual): falls into the 35% top bracket.
- Monthly BIR tax is rounded to 2 decimal places.

## Acceptance Criteria

- [ ] BIR withholding tax is computed from the correct taxable base: `gross_pay - SSS - PhilHealth - Pag-IBIG`.
- [ ] Bonuses, absent deductions, HMO, and voluntary deductions are NOT subtracted from the taxable base.
- [ ] Annualization is correct: `taxable_monthly * 12`.
- [ ] Bracket lookup returns the proper `base_tax + (annual - excess_over) * rate` for each tier.
- [ ] Employees with taxable annual income <= 250,000 have zero BIR tax.
- [ ] Monthly BIR tax = `annual_tax / 12`, rounded to 2 decimal places.
- [ ] A `BIR_TAX` line item of type `deduction` is created for non-zero tax amounts.
- [ ] No line item is created when BIR tax is zero.
- [ ] BIR tax amount is stored on `payslip.bir_tax`.
- [ ] Tax brackets are stored in `config/payroll.php` and are easily updatable.
- [ ] Step 9 is skipped entirely for employees with `has_external_salary = true`.
