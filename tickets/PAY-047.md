# [GW] Payroll - ComputePayslipAction - HMO, Voluntary Deductions, Cash Advance (Steps 7-8)

## Description

Implement Steps 7 and 8 of the 10-step payroll computation pipeline. Step 7 handles HMO deductions. Step 8 handles voluntary deductions (MP2, SSS WISP) and cash advance recovery. These steps are skipped for employees with `has_external_salary = true`. Line items are only generated for non-zero deductions.

## Technical Implementation

### Action

**`app/Payroll/Actions/ComputePayslipAction.php`** (Steps 7-8 segment)

All steps are skipped if `$employee->has_external_salary === true`.

---

**Step 7 -- HMO Deduction:**

```php
$hmoAmount = $employee->hmo_amount ?? 0;

if ($hmoAmount > 0) {
    PayslipLineItem::create([
        'id'         => Str::ulid(),
        'payslip_id' => $payslip->id,
        'code'       => 'HMO',
        'label'      => 'HMO Premium',
        'type'       => 'deduction',
        'amount'     => $hmoAmount,
        'sort_order' => $sortOrder++,
    ]);
}
```

The HMO amount is a fixed value stored on the employee record (`employees.hmo_amount`). If the value is 0 or null, no line item is created.

---

**Step 8 -- Voluntary Deductions:**

**MP2 (Modified Pag-IBIG 2):**

```php
if ($employee->has_mp2) {
    $mp2Amount = $compensation->mp2_amount ?? 0;

    if ($mp2Amount > 0) {
        PayslipLineItem::create([
            'id'         => Str::ulid(),
            'payslip_id' => $payslip->id,
            'code'       => 'MP2',
            'label'      => 'Pag-IBIG MP2',
            'type'       => 'deduction',
            'amount'     => $mp2Amount,
            'sort_order' => $sortOrder++,
        ]);
    }
}
```

**SSS WISP (Workers' Investment and Savings Program):**

```php
if ($employee->has_sss_wisp) {
    $wispAmount = $compensation->sss_wisp_amount ?? 0;

    if ($wispAmount > 0) {
        PayslipLineItem::create([
            'id'         => Str::ulid(),
            'payslip_id' => $payslip->id,
            'code'       => 'SSS_WISP',
            'label'      => 'SSS WISP',
            'type'       => 'deduction',
            'amount'     => $wispAmount,
            'sort_order' => $sortOrder++,
        ]);
    }
}
```

**Cash Advance Recovery:**

```php
$cashAdvanceTotal = CashAdvance::where('employee_id', $employee->id)
    ->where('status', 'active')
    ->sum('balance');

if ($cashAdvanceTotal > 0) {
    PayslipLineItem::create([
        'id'         => Str::ulid(),
        'payslip_id' => $payslip->id,
        'code'       => 'CASH_ADVANCE',
        'label'      => 'Cash Advance Recovery',
        'type'       => 'deduction',
        'amount'     => $cashAdvanceTotal,
        'sort_order' => $sortOrder++,
    ]);
}
```

Cash advance recovery sums the remaining `balance` from all `cash_advances` records for the employee where `status = 'active'`. The actual balance deduction from the cash advance records happens at finalization (PAY-041), not at computation time.

---

**Update payslip with deduction amounts:**

```php
$payslip->update([
    'hmo_deduction'          => $hmoAmount,
    'mp2_deduction'          => $employee->has_mp2 ? $mp2Amount : 0,
    'sss_wisp_deduction'     => $employee->has_sss_wisp ? $wispAmount : 0,
    'cash_advance_deduction' => $cashAdvanceTotal,
]);
```

### Edge Cases

- Employee with `hmo_amount = 0`: Step 7 is skipped, no line item.
- Employee with `has_mp2 = false`: MP2 is skipped regardless of `mp2_amount`.
- Employee with `has_sss_wisp = false`: SSS WISP is skipped regardless of `sss_wisp_amount`.
- Employee with no active cash advances: cash advance total = 0, no line item.
- Employee with multiple active cash advances: all balances are summed into a single deduction.

## Acceptance Criteria

- [ ] HMO deduction uses the `hmo_amount` from the employee record.
- [ ] HMO line item is skipped when `hmo_amount` is 0 or null.
- [ ] MP2 deduction is applied only when `has_mp2 = true` on the employee.
- [ ] MP2 uses `mp2_amount` from `employee_compensations`.
- [ ] SSS WISP deduction is applied only when `has_sss_wisp = true` on the employee.
- [ ] SSS WISP uses `sss_wisp_amount` from `employee_compensations`.
- [ ] Cash advance recovery sums the `balance` of all active cash advances for the employee.
- [ ] Cash advance line item is skipped when total balance is 0.
- [ ] Line items are generated only for non-zero deduction amounts.
- [ ] Individual deduction amounts are stored on the payslip record.
- [ ] Steps 7-8 are skipped entirely for employees with `has_external_salary = true`.
