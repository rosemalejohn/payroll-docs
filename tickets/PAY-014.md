# PAY-014: Create Payslips Migration and Model

## Description

Create the `payslips` table, Eloquent model, and factory. A payslip is the core output of the payroll computation engine, representing one employee's pay record within a payroll run. It captures the complete breakdown of gross pay, all deductions (statutory, voluntary, and absences), bonuses, tax, and final net pay. Each payslip also snapshots the employee's salary currency and exchange rate at computation time for historical integrity. A unique constraint ensures one payslip per employee per payroll run.

  use Illuminate\Database\Eloquent\Concerns\HasUlids;

      use HasUlids, HasFactory, AuditableTrait;

          'pay_basis', 'has_external_salary', 'working_days', 'absent_days',
          'monthly_salary', 'daily_rate',
          'salary_currency_id', 'salary_currency_code', 'salary_exchange_rate',
          'gross_pay', 'total_bonuses', 'total_deductions', 'net_pay',
          'sss_deduction', 'philhealth_deduction', 'pagibig_deduction',
          'hmo_deduction', 'mp2_deduction', 'sss_wisp_deduction',
          'cash_advance_deduction', 'absent_deduction', 'bir_tax',
          'pdf_path', 'generated_at',

      protected $casts = [
          'has_external_salary' => 'boolean',
          'working_days' => 'integer',
          'absent_days' => 'integer',
          'monthly_salary' => 'decimal:2',
          'daily_rate' => 'decimal:2',
          'salary_exchange_rate' => 'decimal:6',
          'gross_pay' => 'decimal:2',
          'total_bonuses' => 'decimal:2',
          'total_deductions' => 'decimal:2',
          'net_pay' => 'decimal:2',
          'sss_deduction' => 'decimal:2',
          'philhealth_deduction' => 'decimal:2',
          'pagibig_deduction' => 'decimal:2',
          'hmo_deduction' => 'decimal:2',
          'mp2_deduction' => 'decimal:2',
          'sss_wisp_deduction' => 'decimal:2',
          'cash_advance_deduction' => 'decimal:2',
          'absent_deduction' => 'decimal:2',
          'bir_tax' => 'decimal:2',
          'generated_at' => 'datetime',
      ];

      public function company()
      {
          return $this->belongsTo(\App\Company\Models\Company::class);
      }

      public function payrollRun()
      {

      }

      {
          return $this->belongsTo(\App\Employee\Models\Employee::class);

      {

      {
          return $this->has_external_salary;
      }

      public function isMonthly(): bool
      {
          return $this->pay_basis === 'monthly';
      }

      public function isDaily(): bool
      {
          return $this->pay_basis === 'daily';
      }
  }

### Edge Cases & Notes
- The unique constraint `uq_payslips_run_employee` is critical to prevent duplicate payslips. The computation engine should check for existing payslips before creating new ones.
- External salary employees (`has_external_salary = true`) skip statutory deductions (SSS, PhilHealth, Pag-IBIG) and BIR tax in the computation pipeline. Their deduction fields should remain 0.
- `salary_currency_code` and `salary_exchange_rate` are denormalized from the currency snapshot at computation time. This ensures the payslip is self-contained for historical reporting.
- `pdf_path` and `generated_at` are populated when the payslip PDF is generated, which may happen asynchronously after computation.
