# [GW] Payroll - Payroll Officer Finalization Access Restriction

## Description

Enforce that Payroll Officers cannot finalize payroll runs. The restriction must be enforced at the server level so it cannot be bypassed by direct URL access. The finalize button must be hidden for Payroll Officers, and any direct attempt to finalize returns an access denied response. Payroll Officers can still view run status and receive a notification when their submitted run is finalized.

## Technical Implementation

### Policies & Authorization

- Update the payroll run policy to explicitly deny finalization for the payroll_officer role. Only the owner role is allowed to finalize.
- The policy is the authoritative enforcement layer — even if middleware is misconfigured, the policy blocks unauthorized finalization.

### Routes & Controllers

- The existing finalize endpoint must call the policy check before any state mutation occurs.
- Pass a finalization permission flag via page props derived from the policy (not hardcoded role checks).

### Events & Listeners

- When a run is finalized, send a notification to the payroll officer who originally submitted the run.

### Mail & Notifications

- Create an in-app notification for the submitting payroll officer containing the run name, period, who finalized it, and a link to view the finalized run.

### Tests

- Assert payroll officer receives 403 when attempting to finalize.
- Assert the finalize button is not present in page data for payroll officers.
- Assert payroll officer can view the run detail page and see its current status.
- Assert payroll officer receives notification when their submitted run is finalized.
- Assert owner can still finalize successfully.
- Assert finance role also cannot finalize.

### Edge Cases & Notes

- The permission flag sent to the frontend must be derived from the policy, not hardcoded role checks.
- If a payroll officer submits a direct POST request to the finalize URL, the 403 must be returned before any data changes occur.

## Acceptance Criteria

### Client-side

#### Behavior

- Payroll Officers do not see a Finalize button on the payroll run detail page
- Payroll Officers can view the payroll run and its current status (e.g., "Under Review — Pending Finalization")
- Payroll Officers receive an in-app notification when their submitted run has been finalized

---

### Server-side

#### Side-effects

- Any attempt by a Payroll Officer to finalize a payroll run is denied with a 403 response
- The restriction is enforced at the server level and cannot be bypassed by direct URL access
- Only the Owner role can finalize payroll runs
- The Finance role also cannot finalize payroll runs
