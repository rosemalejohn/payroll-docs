# [GW] Payroll - Configure Authentication System and Users Database

## Description

Set up session-based authentication using Laravel Breeze with the Inertia/React stack, and create the users table with ULID primary keys, company association, optional employee link, and active status tracking. This provides login, registration, password reset, and email verification pages. The User model is relocated to the Company domain and integrates with Spatie roles.

## Technical Implementation

### What Migrations

- Customize the users table to use ULID primary key, add company reference, optional employee reference, active flag, and last login timestamp.
- Include password_reset_tokens and sessions table migrations.

### What Models

- Create User model in the Company domain extending Authenticatable with ULID support, Spatie roles trait, and Notifiable trait.
- Model defines relationships to Company and Employee.
- Factory includes role helper methods (owner, payroll_officer, finance, employee) for test convenience.
- Update authentication configuration to point to the new model location.

### Validation & DTOs

- Breeze provides standard Form Requests for login and registration.
- Frontend forms use Inertia's form hook with inline validation display.

### Routes & Controllers

- Install Laravel Breeze with Inertia React TypeScript stack, scaffolding auth routes and controllers.
- Auth routes: login, register, password reset, email verification, logout.
- Authenticated users redirect to dashboard; guests redirect to login.

### Tests

- Assert login with valid credentials redirects to dashboard.
- Assert invalid credentials return errors.
- Assert unauthenticated access redirects to login.
- Assert new user creation works with valid data.
- Assert logout destroys the session.
- Assert a user can be created with ULID primary key.
- Assert factory role helpers produce users with correct roles.
- Assert authentication works with the relocated User model.

### Environment & Config

- Set session driver to database.
- Configure auth guards and providers for the web guard.
- Configure "remember me" functionality.
- Update all authentication references to the new User model location.

### Edge Cases & Notes

- Breeze scaffolding may overwrite files from previous tickets — verify and restore any customizations after installation.
- The employee reference on users is nullable because admin-only users may not have a corresponding employee record.
- Last login timestamp should be updated on successful login via a listener in a future ticket.

## Acceptance Criteria

### Client-side

#### Behavior

- The login page is accessible and accepts email and password
- The registration page allows creating a new account
- The forgot password page allows requesting a password reset link
- After logging in, the user is redirected to the dashboard
- After logging out, the session is cleared and the user is redirected to the login page
- Invalid login credentials display an error message on the form
- The "remember me" option keeps the user logged in across browser sessions

#### Validation

- Registration shows inline errors for invalid email, missing fields, or duplicate email addresses
- Login shows an error when credentials are incorrect

---

### Server-side

#### Side-effects

- A new user account is created during registration and persisted in the system
- Each user is associated with a company and optionally linked to an employee record
- Unauthenticated requests to protected pages are redirected to the login page
- Inactive users cannot log in
