# PAY-011: Create Employee Compensation Migration and Model

## Description

Create the `employee_compensations` table, Eloquent model, factory, and audit trail integration. This table stores each employee's current compensation details (monthly salary, daily rate, voluntary deduction amounts) with a unique constraint ensuring one compensation record per employee. Salary changes are tracked via the `owen-it/laravel-auditing` package to maintain a full history of compensation adjustments.

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_employee_compensations_table.php`:
  ```php
  Schema::create('employee_compensations', function (Blueprint $table) {
      $table->char('id', 26)->primary();                          // ULID
      $table->char('company_id', 26);                              // ref: companies.id
      $table->char('employee_id', 26)->unique();                   // ref: employees.id (one record per employee)
      $table->char('currency_id', 26);                             // ref: currencies.id
      $table->decimal('monthly_salary', 14, 2)->nullable();        // nullable for daily-rate-only employees
      $table->decimal('daily_rate', 14, 2)->nullable();            // nullable for monthly-salary employees
      $table->decimal('mp2_amount', 12, 2)->default(0);
      $table->decimal('sss_wisp_amount', 12, 2)->default(0);
      $table->date('effective_date');
      $table->text('notes')->nullable();
      $table->char('created_by', 26)->nullable();                  // ref: users.id
      $table->char('updated_by', 26)->nullable();                  // ref: users.id
      $table->timestamps();

      $table->index('employee_id', 'idx_emp_comp_employee');
      $table->index('company_id', 'idx_emp_comp_company');
  });
  ```

### What Models
- Create `app/Compensation/Models/EmployeeCompensation.php`:
  ```php
  namespace App\Compensation\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;
  use OwenIt\Auditing\Contracts\Auditable;
  use OwenIt\Auditing\Auditable as AuditableTrait;

  class EmployeeCompensation extends Model implements Auditable
  {
      use HasUlids, HasFactory, AuditableTrait;

      protected $fillable = [
          'company_id', 'employee_id', 'currency_id',
          'monthly_salary', 'daily_rate',
          'mp2_amount', 'sss_wisp_amount',
          'effective_date', 'notes',
          'created_by', 'updated_by',
      ];

      protected $casts = [
          'monthly_salary' => 'decimal:2',
          'daily_rate' => 'decimal:2',
          'mp2_amount' => 'decimal:2',
          'sss_wisp_amount' => 'decimal:2',
          'effective_date' => 'date',
      ];

      // Relationships
      public function company()
      {
          return $this->belongsTo(\App\Company\Models\Company::class);
      }

      public function employee()
      {
          return $this->belongsTo(\App\Employee\Models\Employee::class);
      }

      public function currency()
      {
          return $this->belongsTo(\App\Currency\Models\Currency::class);
      }

      public function creator()
      {
          return $this->belongsTo(\App\Models\User::class, 'created_by');
      }

      public function updater()
      {
          return $this->belongsTo(\App\Models\User::class, 'updated_by');
      }
  }
  ```

### Edge Cases & Notes
- The unique constraint on `employee_id` ensures one-to-one compensation record per employee. Updates replace the existing record rather than creating new ones.
- Either `monthly_salary` or `daily_rate` should be set (not both), but this is enforced at the application/validation layer, not the database.
- The `Auditable` interface from `owen-it/laravel-auditing` automatically tracks all field changes, which is critical for salary change history and compliance.
- `mp2_amount` and `sss_wisp_amount` default to 0 to indicate no voluntary deductions; they are only used during payroll computation if the employee has opted in.
