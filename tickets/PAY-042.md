# [GW] Payroll - Add/Remove Employees from Draft Payroll Run

## Description

Allow payroll officers and owners to add or remove employees from a payroll run while it is in `draft` state. Adding an employee creates an initial payslip record. Removing an employee deletes the payslip record and requires a reason, which is logged in the audit trail. These operations are blocked on any non-draft run.

## Technical Implementation

### Actions

**`app/Payroll/Actions/AddEmployeeToRunAction.php`**

- Accepts `PayrollRun` and `Employee` models.
- Validates the run is in `draft` state; throws `InvalidRunStateException` otherwise.
- Validates the employee does not already have a payslip in this run (unique constraint on `payroll_run_id` + `employee_id`); throws `DuplicateEmployeeException` if found.
- Creates a `Payslip` record with:
  - `id`: new ULID
  - `payroll_run_id`, `employee_id`
  - Snapshots employee data: `employee_name`, `department`, `position`, `monthly_salary`, `daily_rate`, `salary_currency_id`, `salary_currency_code`
  - All computed fields initialized to `0.00` (gross_pay, total_bonuses, total_deductions, net_pay)
  - `is_locked = false`
- Increments the run's `employee_count`.
- Returns the created `Payslip`.

**`app/Payroll/Actions/RemoveEmployeeFromRunAction.php`**

- Accepts `PayrollRun`, `Employee`, and `reason` (string, required).
- Validates the run is in `draft` state.
- Deletes the `Payslip` record and all associated `payslip_line_items`.
- Decrements the run's `employee_count`.
- Logs removal in audit trail with: employee ID, employee name, reason, actor, timestamp.

### Controller

Add methods on `PayrollRunController`:

- `POST /payroll-runs/{payrollRun}/employees` -> `AddEmployeeToRunAction`
  - Request body: `{ employee_id: string }`
  - Form Request: validates `employee_id` exists in employees table.
- `DELETE /payroll-runs/{payrollRun}/employees/{employee}` -> `RemoveEmployeeFromRunAction`
  - Request body: `{ reason: string }`
  - Form Request: validates `reason` is required, min 5 characters.

### Frontend (Inertia/React)

**Add Employee Modal:**

- Searchable employee list (filtered to exclude employees already in the run).
- Search by name, employee ID, or department.
- Confirmation button to add.
- On success, employee row appears in the payroll run detail table.

**Remove Employee:**

- Remove button on each employee row in the draft run detail view.
- Clicking opens a dialog requiring a reason text input.
- On confirmation, employee row is removed from the table.
- Toast notification confirms removal.

**State guard:** Add/remove UI elements are only visible when the run is in `draft` state.

## Acceptance Criteria

- [ ] An employee can be added to a draft payroll run, creating a payslip record with initial zero values.
- [ ] Adding a duplicate employee to the same run returns a validation error.
- [ ] The employee search modal filters out employees already in the run.
- [ ] Removing an employee requires a non-empty reason (minimum 5 characters).
- [ ] Removal deletes the payslip and all associated line items.
- [ ] Removal is logged in the audit trail with employee details, reason, and actor.
- [ ] The run's `employee_count` is updated on both add and remove.
- [ ] Add and remove operations are blocked on non-draft runs (returns 422 with appropriate message).
- [ ] UI elements for add/remove are hidden when the run is not in `draft` state.
