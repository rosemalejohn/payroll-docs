# [GW] Payroll - Deactivate Currency with Safety Checks

## Description

Allow the Owner to deactivate a currency that is no longer needed. The system prevents deactivation when active employees still have compensation in that currency, and the base currency (PHP) can never be deactivated.

## Technical Implementation

### What Actions

- Deactivate Currency action: prevents deactivating the base currency, checks for active employees with compensation in this currency, returns an error with count if employees are found, and sets the currency to inactive if safe.

### Routes & Controllers

- `PATCH /settings/currencies/{currency}/toggle-active` — toggle active status (Owner only).

### Tests

- Assert deactivation is blocked when active employees use the currency (with employee count in error message).
- Assert PHP base currency cannot be deactivated.
- Assert deactivated currency is hidden from compensation currency selector.
- Assert existing payslips are unaffected by deactivation (they use snapshot data).
- Assert currency can be reactivated (toggled back to active).

## Acceptance Criteria

### Client-side

#### Behavior

- A toggle button on the currency row allows the Owner to activate or deactivate a currency
- If deactivation is blocked due to active employees, an error message states how many employees still use the currency
- If deactivation is safe, a confirmation dialog asks "Deactivate [CODE]?"
- The PHP base currency row has no toggle button

---

### Server-side

#### Side-effects

- Deactivation is blocked if active employees have compensation set in that currency — the Owner must reassign employees first
- The PHP base currency can never be deactivated
- Deactivated currencies no longer appear in the employee compensation currency selector
- Deactivated currencies are not included in future payroll run snapshots
- Existing payslips and historical data are unaffected by deactivation (they reference snapshot data)
- A deactivated currency can be reactivated at any time
