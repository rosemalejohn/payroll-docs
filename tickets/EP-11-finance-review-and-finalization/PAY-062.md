# [GW] Payroll - Finalize Payroll Run Action (Owner Only)

## Description

Implement the finalization step for payroll runs, restricted exclusively to the Owner role. Finalization is the terminal, irreversible state transition (under review to finalized) that permanently locks the payroll run, all associated payslips, and the payroll period. Upon finalization, the system dispatches jobs for payslip PDF generation, cash advance recovery processing, and notifications to payroll officers and employees. Pre-finalization checks ensure all flags are resolved.

## Technical Implementation

### What Actions

- Finalize Payroll Run action: validates the run is in under review state, ensures all flags are resolved, transitions to finalized, sets finalized_by and finalized_at, locks all payslips and the payroll period, and dispatches PayrollRunFinalized event. Wrapped in a database transaction with row-level locking to prevent concurrent finalization.

### Routes & Controllers

- POST /payroll-runs/{payrollRun}/finalize — finalize the run.

### Policies & Authorization

- Only Owner role can finalize.
- Run must be in under review state.
- All flags must be resolved.

### Events & Listeners

- PayrollRunFinalized event fired after successful finalization.
- Listener dispatches PDF generation jobs for each payslip.
- Listener dispatches cash advance recovery job to update balances.
- Listener notifies payroll officers of finalization and employees that payslips are available.

### Jobs

- Generate Payslip PDF job: queued per payslip for parallel processing (detailed in EP-12).
- Process Cash Advance Recovery job: queued to update cash advance balances (detailed in EP-10 PAY-056).

### Mail & Notifications

- PayrollRunFinalized notification to payroll officers via database and mail.
- PayslipAvailable notification to employees via database channel.

### Tests

- Assert Owner can finalize an under review run.
- Assert non-Owner roles (Payroll Officer, Finance, Employee) receive 403.
- Assert finalizing a run not in under review state returns 422.
- Assert unresolved flags prevent finalization (422).
- Assert finalized_by and finalized_at are correctly set.
- Assert all payslips are locked after finalization.
- Assert the payroll period is locked after finalization.
- Assert the PayrollRunFinalized event is dispatched.
- Assert PDF generation jobs are dispatched for each payslip.
- Assert cash advance recovery job is dispatched.
- Assert payroll officers receive finalization notification.
- Assert employees receive payslip-available notification.
- Assert finalization is recorded in the audit trail.

### Edge Cases & Notes

- Finalization is irreversible — there is no state transition out of finalized.
- All operations are wrapped in a database transaction; any failure rolls back entirely.
- Row-level locking prevents concurrent finalization attempts.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Finalize" button is visible only to Owner users on under review runs
- The button is hidden for all other roles including Finance and Payroll Officer
- After finalization, the run detail page shows a fully read-only view with download options

#### Validation

- An error is shown if there are unresolved flags on the run

---

### Server-side

#### Side-effects

- Only the Owner role can finalize a payroll run
- Finalization is irreversible — once finalized, no further edits are possible
- All payslips in the run are locked permanently
- The associated payroll period is locked, preventing new runs from being created against it
- The finalized by and finalized at fields are recorded
- PDF generation is triggered for each payslip upon finalization
- Cash advance balances are updated based on deductions in the finalized run
- Payroll officers are notified of finalization
- Employees are notified that their payslips are available
- Runs with unresolved flags cannot be finalized
- Finalization is logged in the audit trail
