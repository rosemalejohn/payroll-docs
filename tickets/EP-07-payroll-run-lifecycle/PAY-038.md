# [GW] Payroll - Create Payroll Run Action

## Description

Build the payroll run creation feature including the payroll runs, payslips, and payslip line items database tables and models. A payroll run represents a single payroll processing cycle tied to a payroll period. When created, the run auto-populates with all active employees (creating initial payslip records), snapshots currency rates, and attaches auto-trigger bonuses. The payroll run uses a state machine (draft -> submitted -> under_review -> finalized) managed by spatie/laravel-model-states, with all transitions audited. Each payslip captures the complete breakdown of gross pay, deductions, bonuses, tax, and net pay. Payslip line items store individual earnings, bonuses, and deductions as separate rows for itemized display.

## Technical Implementation

### What Migrations

- Create the payroll runs table with ULID primary key, company and payroll period references, name, state (default draft, managed by spatie/laravel-model-states), base currency reference, rates snapshotted timestamp, aggregate totals (total gross, total deductions, total net as high-precision decimals), employee count, transition actor and timestamp fields (submitted by/at, reviewed by/at, finalized by/at), notes, creator reference, and standard timestamps. Indexes on company, payroll period, and state.
- Create the payslips table with ULID primary key, company/payroll run/employee/payroll period references, pay basis (monthly/daily), external salary flag, working days, absent days, monthly salary, daily rate, salary currency reference and denormalized code, salary exchange rate (6 decimal places), gross pay, total bonuses, total deductions, net pay, individual deduction fields (SSS, PhilHealth, Pag-IBIG, HMO, MP2, SSS WISP, cash advance, absent deduction, BIR tax), PDF path, generated timestamp, and standard timestamps. Unique constraint on payroll run and employee. Indexes on payroll run and employee.
- Create the payslip line items table with ULID primary key, payslip reference, component code, component name, type (earning/deduction/bonus), amount (high-precision decimal), sort order, notes, and standard timestamps. Index on payslip reference.

### What Models

- Create PayrollRun model in the Payroll domain with ULID support, factory, state machine integration (spatie/laravel-model-states), and audit trail integration (owen-it/laravel-auditing).
  - Relationships: belongs to Company, PayrollPeriod, base Currency, submitter/reviewer/finalizer/creator (User); has many Payslips, PayrollRunBonuses, PayrollRunCurrencySnapshots.
  - Scopes: filter by company, draft runs, finalized runs.
  - Helpers: check if draft, check if finalized.
  - Factory with states for draft, submitted, under review, and finalized.
- Create state classes for the payroll run state machine: Draft, Submitted, UnderReview, Finalized. Configure allowed transitions: Draft to Submitted, Submitted to UnderReview, UnderReview to Finalized.
- Create Payslip model in the Payroll domain with ULID support, factory, and audit trail integration.
  - Relationships: belongs to Company, PayrollRun, Employee, PayrollPeriod, salary Currency; has many PayslipLineItems.
  - Helpers: check if external salary, check if monthly or daily pay basis.
  - Factory with states for external salary and daily pay basis.
- Create PayslipLineItem model in the Payroll domain with ULID support and factory.
  - Relationships: belongs to Payslip.
  - Default ordering by sort order.
  - Factory with states for earning, deduction, and bonus types.

### What Actions

- Create Payroll Run action: validates no existing non-cancelled run for the same period, creates the payroll run record in draft state, snapshots all active currency rates, attaches applicable auto-trigger bonuses, creates a payslip record for each active employee with initial zero values, and sets employee count and base currency.

### Validation & DTOs

- Store Payroll Run request: validates payroll period (required, must exist), name (required), and notes (optional).
- Duplicate check: rejects creation if a non-cancelled run already exists for the same period.
- DTO for payroll run data using spatie/laravel-data.

### Routes & Controllers

- `POST /payroll-runs` — create new payroll run.
- On success, redirect to the payroll run detail page.

### Policies & Authorization

- Owner and Payroll Officer can create payroll runs.

### Tests

- Assert the payroll run is created in draft state with correct data.
- Assert all active employees are auto-populated with initial payslip records.
- Assert currency rates are snapshotted at creation time.
- Assert auto-trigger bonuses are attached based on period month.
- Assert duplicate run for the same period (non-cancelled) is blocked.
- Assert inactive employees are excluded from auto-population.
- Assert the state machine enforces valid transitions only (draft to submitted to under review to finalized).
- Assert invalid transitions are rejected (skipping states, going backwards).
- Assert finalized state is terminal — no further transitions allowed.
- Assert the unique constraint on payslip prevents duplicate payslips per employee per run.
- Assert all monetary fields use correct decimal precision.
- Assert payslip line items are ordered by sort order.
- Assert audit trail captures state transitions with old/new values and actor.
- Assert all model relationships work correctly.
- Assert factories produce valid records with all state methods.

### Edge Cases & Notes

- The state machine is irreversible by design. Once a run reaches finalized, it cannot be modified. This is critical for payroll compliance.
- Transition classes should record the actor and timestamp during each transition.
- Aggregate fields (total gross, total deductions, total net, employee count) are updated after payslip computation, not during creation.
- The unique constraint on payslips prevents duplicate payslips per employee per run. The computation engine should check for existing payslips before creating new ones.
- External salary employees skip statutory deductions and BIR tax in the computation pipeline.
- Salary currency code and exchange rate are denormalized from the currency snapshot at computation time for historical integrity.
- Payslip line items follow a standard sort order convention: earnings first (sort order 10), then bonuses (20-60+), then deductions (100-180).

## Acceptance Criteria

### Client-side

#### Behavior

- A "Create Payroll Run" button is available on the payroll runs list page for Owner and Payroll Officer roles
- The creation form includes: Payroll Period (dropdown of unlocked periods), Run Name (auto-suggested from period name), and Notes
- The form shows recent runs for the selected period to help avoid duplicates
- On successful creation, the user is redirected to the payroll run detail page

#### Validation

- An error is shown if no payroll period is selected
- An error is shown if the run name is empty
- An error is shown if a non-cancelled run already exists for the selected period

---

### Server-side

#### Side-effects

- New payroll runs are created in draft state with all active employees auto-populated as payslip records
- Each payslip is initialized with zero values for all computed fields
- Currency rates are snapshotted from all active currencies at the time of run creation
- Auto-trigger bonuses are attached based on the payroll period month
- The employee count is set correctly based on the number of auto-populated employees
- Inactive employees are excluded from auto-population
- Duplicate payroll runs for the same period (excluding cancelled runs) are prevented
- A unique constraint ensures only one payslip per employee per payroll run
- All changes are recorded in the audit trail
