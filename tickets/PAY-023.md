# [GW] Payroll - Build Reusable Data Table Component

## Description

Create a reusable, server-side-driven data table component built on Shadcn/ui's Table primitive. The table must support server-side pagination, sorting, and filtering through Inertia visits with query parameters. This component will be used across all list pages in the application (employees, payroll runs, payslips, cash advances, etc.) and must handle empty states gracefully.

## Technical Implementation

### What Actions
- No domain actions in this ticket (frontend component only).

### Frontend Components

#### `resources/js/components/tables/data-table.tsx`
- Generic, typed React component: `DataTable<TData>`.
- Props interface:
  ```typescript
  interface DataTableProps<TData> {
    columns: ColumnDef<TData>[];
    data: TData[];
    pagination: PaginationMeta;
    sorting?: { column: string; direction: 'asc' | 'desc' } | null;
    filters?: Record<string, string | string[] | null>;
    searchPlaceholder?: string;
    emptyMessage?: string;
    emptyDescription?: string;
  }
  ```
- Render a Shadcn `Table` with `TableHeader`, `TableBody`, `TableRow`, `TableCell`.
- Column definitions follow a simple pattern: `{ key: string; label: string; sortable?: boolean; render?: (row: TData) => ReactNode }`.
- Clicking a sortable column header triggers an Inertia `router.get()` visit with `sort` and `direction` query params (toggles asc/desc/none).
- Display sort indicator icons (chevron up/down) on active sort column.
- Compose `FilterBar`, `Pagination`, and `EmptyState` as children within the table layout.

#### `resources/js/components/tables/pagination.tsx`
- Props: `PaginationMeta` matching Laravel's paginator shape: `{ current_page, last_page, per_page, total, from, to, links }`.
- Render previous/next buttons and page number buttons using Shadcn `Button`.
- Page size selector dropdown (options: 10, 25, 50, 100) using Shadcn `Select`.
- Display "Showing X to Y of Z results" text.
- All navigation triggers Inertia `router.get()` with `page` and `per_page` query params, preserving existing filters and sort.

#### `resources/js/components/tables/filter-bar.tsx`
- Props: `filters` config array defining available filters.
- Render a search text input (debounced, 300ms) for text search.
- Render dropdown filters using Shadcn `Select` for categorical filters (e.g., department, status).
- "Clear filters" button appears when any filter is active.
- Filter changes trigger Inertia `router.get()` with updated query params, resetting to page 1.
- Preserve other active query params (sort, per_page) when filtering.

#### `resources/js/components/tables/empty-state.tsx`
- Props: `message` (string), `description` (optional string), `action` (optional ReactNode for a CTA button).
- Render a centered message with an icon when the table has no data.
- Differentiate between "no data exists" and "no results match filters" based on whether filters are active.

#### `resources/js/hooks/use-table-params.ts`
- Custom hook that reads current URL query params and provides helpers:
  - `getParam(key)`: read a query param value.
  - `setParams(params)`: trigger Inertia visit with merged query params.
  - `resetFilters()`: clear all filter params, keeping sort and per_page.
- Uses `usePage().url` and `router.get()` from Inertia.

### TypeScript Types

#### `resources/js/types/pagination.ts`
```typescript
interface PaginationMeta {
  current_page: number;
  last_page: number;
  per_page: number;
  total: number;
  from: number | null;
  to: number | null;
}

interface PaginatedResponse<T> {
  data: T[];
  meta: PaginationMeta;
}
```

### Tests
- No automated backend tests for this ticket.
- Manual verification: table renders, pagination navigates, sort toggles, filter bar triggers visits, empty state displays.

### Edge Cases & Notes
- All data fetching is server-side. The table does not manage its own data state; it receives data from Inertia page props.
- Debounce on search input prevents excessive Inertia visits while typing.
- When filters change, pagination must reset to page 1 to avoid showing an empty page.
- The component must be flexible enough for tables with 3 columns or 12 columns without layout breaking.
- Sorting and filtering query params should be URL-encoded and shareable (bookmarkable URLs).

## Acceptance Criteria

- [ ] `DataTable` component renders a Shadcn Table with dynamic columns and row data from props.
- [ ] Clicking a sortable column header triggers an Inertia visit with `sort` and `direction` query params.
- [ ] Sort direction cycles through ascending, descending, and none on repeated clicks.
- [ ] Pagination component renders page numbers, previous/next buttons, and "Showing X to Y of Z" text.
- [ ] Page size selector allows switching between 10, 25, 50, and 100 items per page.
- [ ] Pagination and page size changes trigger Inertia visits preserving existing filters and sort order.
- [ ] Filter bar renders a debounced search input and dropdown filters.
- [ ] Applying a filter resets pagination to page 1.
- [ ] "Clear filters" button appears when filters are active and resets all filters on click.
- [ ] Empty state component displays when table data is empty, with appropriate messaging.
- [ ] All query params (sort, filters, page, per_page) are reflected in the URL and shareable.
- [ ] Components are fully typed with TypeScript generics.
