# PAY-001: Initialize Laravel 12 Project with Inertia.js and React

## Description

Set up the foundational Laravel 12 application with Inertia.js v2 as the server-side adapter and React 19 with TypeScript as the frontend framework. This ticket establishes the project skeleton that all subsequent tickets build upon. The goal is a working dev environment where Laravel serves an Inertia-powered React page through Vite, with all tooling configured and verified.

## Technical Implementation

### What Actions
- No domain actions in this ticket (project initialization only).

### What Migrations
- Default Laravel migrations (users, password_reset_tokens, sessions, cache, jobs) are included but not customized yet (customization happens in later tickets).

### Routes & Controllers
- `routes/web.php` with a default welcome route returning an Inertia response.

### Environment & Config
- Install Laravel 12 via `composer create-project` or `laravel new`.
- Install Inertia.js v2 server-side adapter: `composer require inertiajs/inertia-laravel`.
- Install Inertia.js React adapter: `npm install @inertiajs/react`.
- Install React 19 and ReactDOM 19: `npm install react react-dom`.
- Install TypeScript and type definitions: `npm install -D typescript @types/react @types/react-dom`.
- Configure Vite (`vite.config.ts`) with the Laravel plugin and React plugin (`@vitejs/plugin-react`).
- Create `tsconfig.json` with strict mode, path aliases (`@/` mapping to `resources/js/`).
- Create `resources/views/app.blade.php` as the Inertia root view with `@inertiaHead` and `@inertia` directives, plus `@vite` asset includes.
- Create `resources/js/app.tsx` entry point with `createInertiaApp()` setup, dynamic page resolution from `resources/js/pages/`.
- Create `resources/js/pages/welcome.tsx` as a basic landing page component.
- Create `resources/js/ssr.tsx` placeholder (optional, for future SSR support).
- Configure `.env.example` with all required variables:
  - `APP_NAME`, `APP_ENV`, `APP_KEY`, `APP_DEBUG`, `APP_URL`
  - `DB_CONNECTION=mysql`, `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD`
  - `REDIS_HOST`, `REDIS_PASSWORD`, `REDIS_PORT`
  - `CACHE_STORE=redis`, `QUEUE_CONNECTION=redis`, `SESSION_DRIVER=database`
  - `HRIS_API_URL`, `HRIS_API_KEY`
  - `MAIL_MAILER`, `MAIL_HOST`, `MAIL_PORT`, `MAIL_USERNAME`, `MAIL_PASSWORD`
- Set `QUEUE_CONNECTION=redis` and `CACHE_STORE=redis` as defaults.
- Add `HandleInertiaRequests` middleware to the web middleware group.
- Configure `HandleInertiaRequests` to share base props (app name, auth user placeholder).

### Tests
- `tests/Feature/WelcomePageTest.php` (Pest): asserts GET `/` returns 200 and renders the correct Inertia component.

### Edge Cases & Notes
- Ensure `php artisan key:generate` is documented in setup steps.
- Confirm that TypeScript compiles without errors via `npx tsc --noEmit`.
- Vite dev server must proxy correctly to Laravel's `php artisan serve`.

## Acceptance Criteria

- [ ] `composer install` completes without errors.
- [ ] `npm install` completes without errors.
- [ ] `npm run dev` starts the Vite development server and watches for file changes.
- [ ] `npm run build` produces production assets in `public/build/` without errors.
- [ ] `php artisan serve` starts the Laravel dev server and serves the welcome page at `http://localhost:8000`.
- [ ] The welcome page is rendered by React via Inertia (not a Blade view).
- [ ] TypeScript compiles without errors (`npx tsc --noEmit` exits 0).
- [ ] `tsconfig.json` has path alias `@/*` mapped to `resources/js/*`.
- [ ] `.env.example` contains all required environment variables (DB, Redis, HRIS API, Queue, Cache).
- [ ] `HandleInertiaRequests` middleware is registered and shares base props.
- [ ] Pest test for the welcome page passes (`php artisan test`).
