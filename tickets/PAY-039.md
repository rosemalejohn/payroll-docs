# PAY-039: Payroll Run List Page

## Description

As a payroll officer, I want to view a list of all payroll runs with their statuses so that I can track payroll history and quickly navigate to active runs.

## Technical Implementation

### Controller
- **`PayrollRunController@index`** — `GET /payroll-runs`
- Returns paginated runs with filters, sorted by `created_at` desc

### Query
- **`app/Payroll/Queries/ListPayrollRunsQuery.php`**
  - Filters: state (multi-select), date range (created_at), period search
  - Eager loads: payrollPeriod (for period name)
  - Sorting: created_at desc (default), name, state

### Frontend
- **`resources/js/pages/payroll-runs/index.tsx`**
- Table columns:
  - Period name
  - Run name
  - State (colored badge: draft=gray, submitted=blue, under_review=yellow, finalized=green)
  - Created date (relative)
  - Employee count
  - Total net pay (formatted as ₱X,XXX.XX)
- Filters: state dropdown (multi-select), date range picker
- Pagination with page size selector (10, 25, 50)
- Row click navigates to run detail page
- Finalized runs show "Download Summary" action button
- "Create New Run" button in header

### Tests
- Runs listed with correct data
- Filters work (by state, date range)
- Pagination works
- Sorted by newest first by default
- Role-based access: employee cannot see run list

## Acceptance Criteria

- [ ] Payroll runs listed with period, name, state badge, date, employee count, total net
- [ ] State badges use distinct colors for each state
- [ ] Filters by state and date range work correctly
- [ ] Default sort is newest first
- [ ] Pagination with configurable page size
- [ ] Clicking a run navigates to its detail page
- [ ] Only finalized runs show "Download Summary" button
