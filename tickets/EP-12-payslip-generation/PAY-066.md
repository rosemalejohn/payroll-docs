# [GW] Payroll - Payslip Detail View with Deduction and Bonus Breakdown

## Description

Implement the employee-facing payslip detail page that displays the full breakdown of a finalized payslip. The view includes an earnings section (basic salary + itemized bonuses), a deductions section (itemized with computation basis descriptions), and a summary section (gross pay, total bonuses, total deductions, net pay).

Each deduction line shows a brief description of the computation basis (e.g., "SSS: bracket-based", "PhilHealth: 2.5% of basic"). Voluntary deductions are labeled as "Voluntary" and overridden deductions are marked with an asterisk and explanatory note. Foreign currency employees see dual amounts with the exchange rate.

## Technical Implementation

### Routes & Controllers
- **`GET /my-payslips/{payslip}`** — Route name: `employee.payslips.show`
- **`app/Http/Controllers/Web/EmployeePayslipController@show`**
  - Middleware: `auth`, `role:employee`
  - Route model binding with `Payslip`
  - Authorizes via `PayslipPolicy@view`
  - Eager loads: employee, payroll run, payroll period, payslip bonuses, payslip deductions, currency snapshot
  - Returns Inertia response with full payslip detail

### API Resources
- **`app/Http/Resources/Payroll/PayslipDetailResource.php`** — Shapes the response:
  - `id`: payslip ULID
  - `employee_name`: string
  - `employee_number`: string
  - `department`: string
  - `position`: string
  - `period`: formatted payroll period string
  - `finalized_at`: formatted date
  - `pay_basis`: "monthly" or "daily"
  - `earnings`:
    - `basic_salary`: decimal
    - `bonuses`: array of `{ label, type, amount }`
    - `total_earnings`: decimal (basic + bonuses)
  - `deductions`: array of `{ label, type, basis_description, amount, is_voluntary, is_overridden, override_note }`
  - `summary`:
    - `gross_pay`: decimal
    - `total_bonuses`: decimal
    - `total_deductions`: decimal
    - `net_pay`: decimal
  - `currency` (if foreign):
    - `code`: string (e.g., "USD")
    - `original_amount`: decimal
    - `exchange_rate`: decimal
    - `php_equivalent`: decimal

### Deduction Basis Descriptions
Map each deduction type to a human-readable description:
- SSS: "Bracket-based (employee share)"
- PhilHealth: "2.5% of basic salary (employee share)"
- Pag-IBIG: "Fixed contribution"
- HMO: "Fixed premium"
- BIR Withholding Tax: "TRAIN Law annualized computation"
- MP2: "Voluntary — Modified Pag-IBIG II"
- SSS WISP: "Voluntary — Workers' Investment and Savings Program"
- Cash Advance: "Voluntary — Cash advance recovery"

### Policies & Authorization
- **`app/Payroll/Policies/PayslipPolicy.php`** — `view()` method:
  - Employee can only view their own payslip
  - Payslip must belong to a finalized payroll run
  - Returns 403 for non-finalized payslips

### Frontend Components
- **`resources/js/pages/employee/payslips/show.tsx`** — Inertia page component:
  - Page title: "Payslip — {period}"
  - Uses authenticated layout
  - Back link to payslip list
  - Sections rendered in order:
    1. **Header**: employee name, number, department, position, period, finalized date
    2. **Earnings**: basic salary row + bonus rows (label, type badge, amount)
    3. **Deductions**: itemized rows with label, basis description, amount; voluntary deductions show "Voluntary" badge; overridden deductions show asterisk (*) with tooltip/note
    4. **Summary**: gross pay, total bonuses, total deductions, net pay (highlighted)
    5. **Currency info** (if foreign): original amount, exchange rate, PHP equivalent
  - Download PDF button at the top
- **`resources/js/components/payslip/earnings-section.tsx`** — Earnings breakdown component
- **`resources/js/components/payslip/deductions-section.tsx`** — Deductions breakdown component
- **`resources/js/components/payslip/summary-section.tsx`** — Summary component
- **`resources/js/components/payslip/currency-info.tsx`** — Foreign currency display component

### Tests
- **`tests/Feature/Payroll/PayslipDetailViewTest.php`**
  - Employee can view their own finalized payslip with full detail
  - All earnings line items (basic + bonuses) are displayed in correct order
  - All deductions are displayed with correct labels and amounts
  - Deduction basis descriptions are shown (e.g., "Bracket-based" for SSS)
  - Voluntary deductions are labeled as "Voluntary"
  - Overridden deductions are marked with an asterisk and note
  - Foreign currency payslip shows original amount, PHP equivalent, and exchange rate
  - Summary totals (gross, bonuses, deductions, net) are mathematically correct
  - Employee cannot view another employee's payslip (403)
  - Employee cannot view a non-finalized payslip (403)

### Edge Cases & Notes
- Ensure deduction order is consistent: SSS, PhilHealth, Pag-IBIG, HMO, BIR Tax, then voluntary deductions
- Overridden deductions should show both the original computed amount (struck through) and the override amount, plus the override reason
- If an employee has no bonuses, the bonuses subsection should still display with an empty state or "None"
- Foreign currency display only applies if the employee's salary currency differs from PHP
- All monetary values formatted with thousands separators and 2 decimal places
