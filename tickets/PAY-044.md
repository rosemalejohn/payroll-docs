# PAY-044: ComputePayslipAction - Gross Pay Calculation (Step 1)

## Description

Implement Step 1 of the 10-step gross-to-net payroll computation pipeline within `ComputePayslipAction`. This step calculates the employee's gross pay from their salary basis (monthly or daily), handles foreign currency conversion using snapshot rates, records a BASIC earning line item, and snapshots salary fields onto the payslip for historical integrity.

## Technical Implementation

### Action

**`app/Payroll/Actions/ComputePayslipAction.php`** (Step 1 segment)

Located at `app/Payroll/Actions/ComputePayslipAction.php`. This action receives a `Payslip` model (which belongs to a `PayrollRun` and an `Employee`).

**Pre-computation checks:**

1. Load employee compensation data (`employee_compensations` record).
2. Check `employee.has_external_salary` flag -- if true, the external salary path is handled in Step 10 (PAY-049) but gross pay is still computed here.
3. Load the currency snapshot rate from `payroll_run_currency_snapshots` for the employee's `salary_currency_id`.

**Gross pay calculation:**

```php
// Monthly basis
if ($compensation->salary_basis === 'monthly') {
    $grossPayInForeign = $compensation->monthly_salary;
    $grossPay = $grossPayInForeign * $snapshotRate; // snapshotRate = 1.0 for PHP
}

// Daily basis
if ($compensation->salary_basis === 'daily') {
    $grossPayInForeign = $compensation->daily_rate * $payrollPeriod->working_days;
    $grossPay = $grossPayInForeign * $snapshotRate;
}
```

All values stored as `DECIMAL` with `decimal:6` casts. No float arithmetic.

**Snapshot salary fields on payslip:**

```php
$payslip->update([
    'monthly_salary'       => $compensation->monthly_salary,
    'daily_rate'           => $compensation->daily_rate,
    'salary_currency_id'   => $compensation->salary_currency_id,
    'salary_currency_code' => $currency->code,
    'salary_exchange_rate' => $snapshotRate,
    'gross_pay'            => $grossPay,
]);
```

**Record BASIC line item:**

Create a `payslip_line_items` record:

```php
PayslipLineItem::create([
    'id'         => Str::ulid(),
    'payslip_id' => $payslip->id,
    'code'       => 'BASIC',
    'label'      => 'Basic Salary',
    'type'       => 'earning',
    'amount'     => $grossPay,
    'sort_order' => 1,
]);
```

### Currency Conversion

- If the employee's salary currency is PHP (Philippine Peso), the snapshot rate is always `1.0`.
- For foreign currencies, the rate is fetched from `payroll_run_currency_snapshots` where `payroll_run_id` matches and `currency_id` matches the employee's salary currency.
- Rate format: `1 [FOREIGN] = X PHP`. So `salary_in_php = salary_in_foreign * rate`.

### Edge Cases

- Employee with no compensation record: throw `MissingCompensationException`.
- Currency snapshot not found for foreign currency employee: throw `MissingCurrencySnapshotException`.
- Zero monthly salary or daily rate: gross_pay = 0, BASIC line item still created with amount 0.

## Acceptance Criteria

- [ ] Monthly-basis employee receives their full `monthly_salary` (converted to PHP) as `gross_pay`.
- [ ] Daily-basis employee receives `daily_rate * working_days` (converted to PHP) as `gross_pay`.
- [ ] PHP-currency employees use a snapshot rate of `1.0` (no conversion effect).
- [ ] Foreign-currency employees have their salary converted using the snapshot rate from `payroll_run_currency_snapshots`.
- [ ] A `BASIC` line item of type `earning` is created with the computed gross pay amount.
- [ ] Salary fields are snapshotted on the payslip: `monthly_salary`, `daily_rate`, `salary_currency_id`, `salary_currency_code`, `salary_exchange_rate`.
- [ ] Missing compensation record throws `MissingCompensationException`.
- [ ] Missing currency snapshot for a foreign-currency employee throws `MissingCurrencySnapshotException`.
- [ ] All monetary values use `DECIMAL` types -- no floating point arithmetic.
