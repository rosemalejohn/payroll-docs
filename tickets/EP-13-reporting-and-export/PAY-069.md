# [GW] Payroll - Deduction Remittance Report

## Description

Build a deduction remittance report that groups all employee deductions by remittance agency (SSS, PhilHealth, Pag-IBIG, BIR) for finalized payroll runs. For each agency, the report shows total employee share, total employer share, and a per-employee breakdown. The report is downloadable as a PDF. Report totals must match finalized payslip figures exactly to ensure accurate remittance.

## Technical Implementation

### Queries

- Remittance report query: accepts a payroll run ID, groups deductions by agency (SSS, PhilHealth, Pag-IBIG, BIR). Returns total employee and employer share per agency with per-employee breakdown. Excludes external salary employees, ordered alphabetically.

### What Actions

- Generate Remittance Report PDF action: uses the report query, renders per-agency Blade sections, and returns the PDF.

### Routes & Controllers

- GET /reports/remittance/{payrollRun} — remittance report page.
- GET /reports/remittance/{payrollRun}/download — download PDF.

### Policies & Authorization

- Only Owner and Finance roles can access.
- Payroll run must be in finalized state.

### Tests

- Assert Owner and Finance can view the report for a finalized run.
- Assert Payroll Officer and Employee roles receive 403.
- Assert non-finalized run returns 403.
- Assert SSS, PhilHealth, Pag-IBIG deductions are grouped correctly with employee and employer shares.
- Assert BIR deductions are grouped correctly (employee share only, no employer share).
- Assert per-employee breakdown lists all employees with correct amounts.
- Assert agency totals match the sum of per-employee amounts.
- Assert report totals match finalized payslip figures exactly.
- Assert external salary employees are excluded.
- Assert PDF download returns valid content.

### Edge Cases & Notes

- BIR withholding has no employer share; display zero for the employer column.
- HMO, MP2, WISP, and Cash Advance are excluded — not government remittances.
- Totals must use DECIMAL math and match payslip values exactly.

## Acceptance Criteria

### Client-side

#### Layout

- The Remittance Report is accessible from the Reports section for Owner and Finance roles

#### Behavior

- The report groups deductions by agency: SSS, PhilHealth, Pag-IBIG, BIR
- Each agency section shows: total employee share, total employer share, and total remittance
- A per-employee breakdown is expandable for each agency, showing individual employee and employer shares
- The report header includes company name, payroll period, and finalization date
- A Download PDF button is available at the top

---

### Server-side

#### Side-effects

- Report totals match finalized payslip figures exactly — no rounding discrepancies
- External salary employees are excluded from the report
- BIR section correctly shows no employer share
- Only Owner and Finance roles can access the report
- Only finalized payroll runs can generate remittance reports
- PDF download produces a well-formatted document with per-agency sections
