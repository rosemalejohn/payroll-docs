# PAY-080: New Employee HMO Proration for Mid-Period Join

## Description

When an employee is added to a payroll run and their `date_hired` falls within the run period, prorate their HMO deduction and other mandatory deductions (SSS, PhilHealth, Pag-IBIG) for the first partial period. The system calculates deductions proportionally based on the number of working days from the hire date to the period end relative to the total working days in the period. Payslip notes document the proration basis and the employee's start date. Payroll officers can override prorated amounts when needed.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/DetectMidPeriodJoinAction.php`** — Checks if an employee is a mid-period joiner:
  - Accepts: employee, payroll run period
  - Returns `true` if `employee.date_hired` falls between `period_start` and `period_end` (inclusive)
  - Returns proration data: `days_worked`, `total_working_days`, `proration_factor` (days_worked / total_working_days)

- **`app/Payroll/Actions/ProrateNewEmployeeDeductionsAction.php`** — Prorates deductions for mid-period joiners:
  - Accepts: payslip, proration factor, total_working_days, days_worked
  - HMO: `hmo_monthly × proration_factor`
  - SSS: look up full bracket amount, then `sss_contribution × proration_factor`
  - PhilHealth: `philhealth_contribution × proration_factor`
  - Pag-IBIG: `pagibig_contribution × proration_factor` (prorated from ₱100)
  - Round all prorated amounts to 2 decimal places
  - Stores both `original_amount` and `prorated_amount` on each deduction line for transparency

- **`app/Payroll/Actions/ComputePayslipAction.php`** — Enhanced for mid-period join:
  - After step 1 (gross pay): check if employee is mid-period joiner via `DetectMidPeriodJoinAction`
  - If mid-period joiner: also prorate gross pay (monthly = salary / working_days × days_worked)
  - At deduction steps 4-7: if mid-period joiner, delegate to `ProrateNewEmployeeDeductionsAction`
  - Step 9 (withholding tax): annualize the prorated gross, not the full salary
  - Add payslip notes documenting proration

- **`app/Payroll/Actions/OverridePayslipDeductionAction.php`** — Allows payroll officer to override prorated amounts:
  - Accepts: payslip ID, deduction type (sss, philhealth, pagibig, hmo), override amount
  - Validates override amount is non-negative
  - Sets `is_override` flag on the deduction
  - Recalculates net pay based on override
  - Adds payslip note: "{deduction_type} overridden from {prorated_amount} to {override_amount} by {user}"

### Edge Cases & Notes
- Employee hired on the first day of the period (`date_hired == period_start`) should NOT be flagged as mid-period — they work the full period
- Employee hired on the last working day of the period should be prorated to 1 working day
- Working days calculation must account for weekends; consider using a working days helper that can optionally include company holidays
- Proration factor must use `DECIMAL` precision to avoid rounding errors in deduction calculations
- External salary employees (`has_external_salary=true`) skip deduction steps 3-9 entirely, so proration of those deductions does not apply to them
- If the payroll officer overrides a prorated amount, subsequent recomputations of the payslip should preserve the override (not recalculate from proration)
- SSS proration is a simplification; in practice, SSS may not allow partial-month contributions — clarify with business rules. Current implementation prorates for consistency.
