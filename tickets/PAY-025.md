# PAY-025: Build Shared Props Middleware

## Description

Create and configure the `HandleInertiaRequests` middleware to share cross-cutting data with every Inertia page response. Shared props include the authenticated user (with roles and permissions), flash messages, application name, and unread notification count. Corresponding TypeScript types must be created so the frontend can consume shared props in a type-safe manner.

## Technical Implementation

### What Actions
- No domain actions in this ticket (middleware and types only).

### Routes & Controllers
- No new routes or controllers. The middleware is applied globally to the `web` middleware group.

### Middleware

#### `app/Http/Middleware/HandleInertiaRequests.php`
- Extend `Inertia\Middleware`.
- Override the `share()` method to return shared props merged with parent.
- Shared props structure:

```php
return array_merge(parent::share($request), [
    'auth' => fn () => $request->user() ? [
        'user' => [
            'id' => $request->user()->id,
            'name' => $request->user()->name,
            'email' => $request->user()->email,
            'roles' => $request->user()->getRoleNames()->toArray(),
            'permissions' => $request->user()->getAllPermissions()->pluck('name')->toArray(),
        ],
    ] : null,

    'flash' => [
        'success' => fn () => $request->session()->get('success'),
        'error' => fn () => $request->session()->get('error'),
    ],

    'appName' => config('app.name'),

    'notifications' => fn () => $request->user()
        ? ['unreadCount' => $request->user()->unreadNotifications()->count()]
        : null,
]);
```

- Use lazy evaluation (`fn ()`) for auth, flash, and notifications to avoid unnecessary queries on non-authenticated requests.
- Keep shared data minimal: only `id`, `name`, `email`, `roles`, `permissions` from the user model. No full model dump.
- `getRoleNames()` and `getAllPermissions()` come from `spatie/laravel-permission` traits on the User model.

### Configuration
- Ensure `HandleInertiaRequests` is registered in `bootstrap/app.php` (Laravel 12) within the `web` middleware group.
- Verify the middleware runs after `auth` middleware so `$request->user()` is available.

### TypeScript Types

#### `resources/js/types/index.d.ts`
```typescript
import { PageProps as InertiaPageProps } from '@inertiajs/core';

export interface AuthUser {
  id: string;
  name: string;
  email: string;
  roles: string[];
  permissions: string[];
}

export interface FlashMessages {
  success: string | null;
  error: string | null;
}

export interface NotificationData {
  unreadCount: number;
}

export interface SharedProps extends InertiaPageProps {
  auth: {
    user: AuthUser;
  } | null;
  flash: FlashMessages;
  appName: string;
  notifications: NotificationData | null;
}
```

#### `resources/js/types/helpers.ts`
- Create a typed `useSharedProps()` hook or helper:
```typescript
import { usePage } from '@inertiajs/react';
import { SharedProps } from '@/types';

export function useAuth() {
  const { auth } = usePage<SharedProps>().props;
  return auth?.user ?? null;
}

export function useFlash() {
  const { flash } = usePage<SharedProps>().props;
  return flash;
}

export function hasRole(user: AuthUser | null, role: string): boolean {
  return user?.roles.includes(role) ?? false;
}

export function hasAnyRole(user: AuthUser | null, roles: string[]): boolean {
  return roles.some((role) => user?.roles.includes(role));
}

export function can(user: AuthUser | null, permission: string): boolean {
  return user?.permissions.includes(permission) ?? false;
}
```

### Tests

#### `tests/Feature/Middleware/HandleInertiaRequestsTest.php` (Pest)
- Assert unauthenticated requests have `auth` as `null`.
- Assert authenticated requests include `auth.user` with `id`, `name`, `email`, `roles`, and `permissions` keys.
- Assert `flash.success` is populated when `session()->flash('success', 'Test')` is set.
- Assert `flash.error` is populated when `session()->flash('error', 'Test error')` is set.
- Assert `appName` matches `config('app.name')`.
- Assert `notifications.unreadCount` is a number for authenticated users.
- Assert no sensitive data (password, remember_token) is included in shared auth props.

### Edge Cases & Notes
- Shared props are serialized on every Inertia response. Keep them lightweight to avoid performance impact.
- `getAllPermissions()` can be expensive with many permissions. Consider caching if performance becomes an issue.
- Flash messages are consumed from the session, so they only appear once (standard Laravel flash behavior).
- Roles and permissions arrays must be plain string arrays, not objects, for frontend consumption.
- The `notifications.unreadCount` query should be indexed (Laravel notifications table default indexes cover this).
