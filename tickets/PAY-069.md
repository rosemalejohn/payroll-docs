# PAY-069: Deduction Remittance Report

## Description

Implement a deduction remittance report that groups all employee deductions by remittance agency (SSS, PhilHealth, Pag-IBIG, BIR) for finalized payroll runs. For each agency, the report shows total employee share, total employer share, and a per-employee breakdown. The report includes a header with company name, payroll period, and finalization date. It is downloadable as a PDF.

The totals in the remittance report must match the finalized payslip figures exactly to ensure accurate remittance to government agencies.

## Technical Implementation

### Routes & Controllers
- **`GET /reports/remittance/{payrollRun}`** — Route name: `reports.remittance`
- **`GET /reports/remittance/{payrollRun}/download`** — Route name: `reports.remittance.download`
- **`app/Http/Controllers/Web/ReportController@remittanceReport`**
  - Middleware: `auth`, `role:owner,finance`
  - Authorizes via `PayrollRunPolicy@viewReport`
  - Returns Inertia response with grouped deduction data
- **`app/Http/Controllers/Web/ReportController@downloadRemittanceReport`**
  - Middleware: `auth`, `role:owner,finance`
  - Generates and returns PDF download

### Queries
- **`app/Payroll/Queries/RemittanceReportQuery.php`** — Query class that:
  - Accepts payroll run ID
  - Groups deductions by agency type: SSS, PhilHealth, Pag-IBIG, BIR
  - For each agency, retrieves:
    - Total employee share (sum across all payslips)
    - Total employer share (sum across all payslips)
    - Per-employee breakdown: employee name, employee number, employee share amount, employer share amount
  - Excludes external salary employees (they have no deductions)
  - Orders per-employee list by employee name alphabetically

### API Resources
- **`app/Http/Resources/Payroll/RemittanceReportResource.php`** — Shapes the response:
  - `payroll_run`: run details (period, finalized_at)
  - `company_name`: string
  - `agencies`: array of agency objects:
    - `name`: agency name (e.g., "Social Security System (SSS)")
    - `code`: agency code (e.g., "SSS")
    - `total_employee_share`: decimal
    - `total_employer_share`: decimal
    - `total_remittance`: employee + employer shares combined
    - `employees`: array of employee objects:
      - `name`: string
      - `employee_number`: string
      - `employee_share`: decimal
      - `employer_share`: decimal

### Actions
- **`app/Payroll/Actions/GenerateRemittanceReportPdfAction.php`** — Generates the PDF:
  - Uses `RemittanceReportQuery` for data
  - Renders the Blade template
  - Returns PDF via `barryvdh/laravel-dompdf`

### Views (Blade Templates)
- **`resources/views/pdf/remittance-report.blade.php`** — PDF template:
  - Header: company name, "Deduction Remittance Report", payroll period, finalization date
  - For each agency (SSS, PhilHealth, Pag-IBIG, BIR):
    - Agency section header with agency name
    - Per-employee table: Employee Name, Employee Number, Employee Share, Employer Share
    - Agency totals row: Total Employee Share, Total Employer Share, Total Remittance
  - Grand totals section: sum of all agency remittances
  - Footer: generation timestamp

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Uses existing `viewReport()` method:
  - Only `owner` and `finance` roles
  - Payroll run must be in `finalized` state

### Frontend Components

  - Page title: "Remittance Report — {period}"
  - Uses authenticated layout
  - Accordion or tab-based layout with one section per agency
  - Each agency section shows:
    - Summary row: total employee share, total employer share, total remittance
    - Expandable per-employee breakdown table
  - Download PDF button at top
- **`resources/js/components/reports/agency-remittance-section.tsx`** — Reusable agency section component

### Tests
- **`tests/Feature/Payroll/RemittanceReportTest.php`**
  - Owner can view the remittance report for a finalized run
  - Finance can view the remittance report for a finalized run
  - Payroll officer and employee roles receive 403
  - Report for a non-finalized run returns 403
  - SSS deductions are grouped correctly with employee and employer shares
  - PhilHealth deductions are grouped correctly with employee and employer shares
  - Pag-IBIG deductions are grouped correctly with employee and employer shares
  - BIR deductions are grouped correctly (employee share only, no employer share)
  - Per-employee breakdown lists all employees with correct amounts
  - Agency totals match the sum of per-employee amounts
  - Grand totals match the sum of all agency totals
  - Report totals match the finalized payslip figures exactly
  - External salary employees are excluded from the report
  - PDF download returns a valid PDF with correct content type
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- BIR withholding tax has no employer share; display "N/A" or "0.00" for employer share column
- HMO, MP2, WISP, and Cash Advance are not remitted to government agencies and are excluded from this report
- Ensure totals use `DECIMAL` math and match payslip values exactly (no rounding discrepancies)
- For agencies with bracket-based contributions (SSS), the employer share per employee is stored on the payslip record, not recomputed
- If no employees have deductions for a given agency (unlikely but possible), that agency section should still appear with zero totals
- Alphabetical ordering of employees within each agency section helps payroll officers verify the report
