# [GW] Payroll - Project Framework Initialization

## Description

- Setup Laravel 12, Inertia.js v2, React.
- Use TypeScript for strict type checking.
- Create foundational companies tenant record table.
- Goal is working dev environment setup.
- Verify basic Inertia page and database.

## Technical Implementation

### What Migrations

- Create companies table with ULID pk.
- Add name, code, address, registration numbers.
- Add logo path and active flag.
- Keep default Laravel migrations as-is.

### What Models

- Create Company model in Company domain.
- Add ULID support and basic factory.
- Add relationships to users and employees.
- Factory must generate realistic company data.

### Routes & Controllers

- Add welcome route returning Inertia response.

### Tests

- Assert GET / returns 200 status.
- Assert route renders correct Inertia component.
- Assert company creation with valid ULID.
- Assert company code is strictly unique.
- Assert factory generates valid company records.

### Environment & Config

- Configure Vite with Laravel, React plugins.
- Setup TypeScript configuration with strict mode.
- Create Inertia root Blade view template.
- Create Inertia app entry point logic.
- Create a basic welcome page component.
- Configure .env.example with required variables.
- Register Inertia middleware in web group.

### Edge Cases & Notes

- Ensure TypeScript compiles without any errors.
- Proxy Vite dev server to Laravel.
- Use application layer for referential integrity.
- Use ULIDs for all primary keys.
