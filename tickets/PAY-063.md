# [GW] Payroll - Payslip PDF Generation

## Description

Implement queued payslip PDF generation triggered upon payroll run finalization. Each payslip in a finalized run gets an individual PDF document generated using `barryvdh/laravel-dompdf`. The PDF includes company information, employee details, payroll period, earnings breakdown (basic salary + bonuses), itemized deductions, and a gross-to-net summary.

External salary employees receive a simplified PDF with no deduction section, clearly labeled "External Salary -- No Deductions Applied." Foreign currency employees see dual amounts: the original currency amount alongside the PHP equivalent with the exchange rate used.

Generated PDFs are stored on disk and the file path is saved to the `payslips.pdf_path` column with a `generated_at` timestamp.

## Technical Implementation

### Jobs
- **`app/Payroll/Jobs/GeneratePayslipPdfJob.php`** — Queued job dispatched per payslip:
  - Queue: `payslips`
  - Accepts payslip ULID as parameter
  - Loads payslip with all related data (employee, payroll run, bonuses, deductions, currency snapshot)
  - Renders the appropriate Blade view based on employee type
  - Generates PDF via `barryvdh/laravel-dompdf`
  - Stores the file to `storage/app/payslips/{payroll_run_id}/{payslip_id}.pdf`
  - Updates `payslips.pdf_path` and `payslips.generated_at`
  - Retries up to 3 times on failure

### Views (Blade Templates)
- **`resources/views/pdf/payslip.blade.php`** — Standard payslip layout:
  - Header: company name, payroll period (start date - end date), date of finalization
  - Employee section: name, employee number, department, position, pay basis
  - Earnings section: basic salary, itemized bonuses with labels and amounts
  - Deductions section: itemized mandatory deductions (SSS, PhilHealth, Pag-IBIG, BIR tax), HMO, voluntary deductions (MP2, WISP, Cash Advance)
  - Summary section: gross pay, total bonuses, total deductions, net pay
  - Footer: generation timestamp, document reference ID
- **`resources/views/pdf/payslip-external.blade.php`** — Simplified layout for external salary employees:
  - Same header and employee section
  - Earnings section: basic salary only (or basic + bonuses if applicable)
  - No deductions section — replaced with label: "External Salary -- No Deductions Applied"
  - Summary section: gross pay, net pay (equal to gross)
- **`resources/views/pdf/partials/currency-display.blade.php`** — Partial for foreign currency display:
  - Shows original currency amount (e.g., "USD 3,500.00")
  - Shows PHP equivalent (e.g., "PHP 196,000.00")
  - Shows exchange rate used (e.g., "Rate: 1 USD = 56.00 PHP")

### Actions
- **`app/Payroll/Actions/GeneratePayslipPdfAction.php`** — Encapsulates PDF generation logic:
  - Determines which template to use based on `has_external_salary` flag
  - Prepares view data including currency conversion details
  - Calls `Pdf::loadView()` from barryvdh/laravel-dompdf
  - Sets PDF options: paper size A4, portrait orientation
  - Returns the generated PDF content

### Models
- **`app/Payroll/Models/Payslip.php`** — Ensure the following fields exist:
  - `pdf_path` (nullable string) in `$fillable`
  - `generated_at` (nullable datetime) in `$fillable` and `$casts`

### Migrations
- If `pdf_path` and `generated_at` do not already exist on the `payslips` table:
  - **`database/migrations/xxxx_add_pdf_fields_to_payslips_table.php`**
    - Add `pdf_path` column (string, nullable)
    - Add `generated_at` column (datetime, nullable)

### Routes & Controllers
- **`GET /payslips/{payslip}/download`** — Route name: `payslips.download`
- **`app/Http/Controllers/Web/PayslipDownloadController@show`**
  - Middleware: `auth`
  - Authorizes via `PayslipPolicy@download`
  - Returns the stored PDF file as a download response
  - Returns 404 if PDF has not been generated yet

### Policies & Authorization
- **`app/Payroll/Policies/PayslipPolicy.php`** — Add `download()` method:
  - Employees can download only their own finalized payslips
  - Owner, payroll_officer, and finance can download any finalized payslip

### Tests
- **`tests/Feature/Payroll/GeneratePayslipPdfTest.php`**
  - PDF is generated for each payslip upon run finalization
  - PDF content includes correct employee name, period, earnings, deductions, and net pay
  - External salary employee PDF uses simplified template with no deductions section
  - External salary PDF shows "External Salary -- No Deductions Applied" label
  - Foreign currency employee PDF shows original currency amount and PHP equivalent
  - Foreign currency PDF shows exchange rate used
  - `pdf_path` is saved correctly on the payslip record
  - `generated_at` is set after PDF generation
  - Re-downloading the same payslip returns identical content
  - Unauthorized users cannot download payslips they do not own
  - Job retries up to 3 times on failure

### Edge Cases & Notes
- PDF generation is asynchronous; the employee notification (PAY-062) should account for a brief delay before the PDF is downloadable
- Use decimal formatting for all monetary values (2 decimal places for PHP, appropriate precision for foreign currencies)
- Ensure consistent number formatting with thousands separator
- PDF file size should be reasonable; avoid embedding large images or fonts
- Store PDFs in a non-public disk; serve through a controller that checks authorization
- If a payslip PDF needs to be regenerated (e.g., due to a bug fix), a separate admin action should handle it — this ticket covers only initial generation on finalization

## Acceptance Criteria

- [ ] A PDF is generated for each payslip in the run upon finalization
- [ ] PDF content accurately reflects the payslip data: earnings, deductions, bonuses, and net pay
- [ ] External salary employees receive a simplified PDF with no deduction section and the label "External Salary -- No Deductions Applied"
- [ ] Foreign currency employees see the original currency amount, PHP equivalent, and exchange rate used

- [ ] `generated_at` timestamp is set upon successful PDF generation
- [ ] Re-downloading a payslip produces identical output
