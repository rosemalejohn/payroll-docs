# [GW] Payroll - Payroll Run Audit History Tab

## Description

Add an "Audit History" tab to the payroll run detail page that displays all audit entries related to a specific payroll run. This includes state transitions, bonus changes, employee additions/removals, and payslip recomputations. The tab presents entries in a timeline-style layout with the newest events first, giving authorized users a clear chronological record of everything that happened to a run.

## Technical Implementation

### Queries

- Payroll run audit query: builds a union query across multiple auditable types — PayrollRun audits for the run ID, Payslip audits for payslips belonging to the run, and PayrollRunBonus audits for bonuses linked to the run. Combines and sorts by created timestamp descending. Returns paginated results (20 per page).

### What Actions

- Get Payroll Run Audit History action: fetches all audit entries related to a specific payroll run across multiple auditable types, merges and sorts them, eager loads the actor (user) relationship.

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}/audit-history` — audit history data for the tab.

### API Resources

- Payroll Run Audit Entry resource: includes ID, event type, human-readable model type, generated summary description (e.g., "Status changed from draft to submitted"), old/new values, actor name and role, formatted timestamp with relative time, and a state transition flag for prominent styling.

### Policies & Authorization

- Owner, Payroll Officer, and Finance roles can view the audit history tab.
- Employee role cannot access.

### Tests

- Assert Owner, Payroll Officer, and Finance can view the audit history tab.
- Assert Employee role receives 403.
- Assert state transitions appear in the audit history.
- Assert payslip changes related to the run appear.
- Assert bonus additions/removals appear.
- Assert entries are sorted newest first.
- Assert results are paginated.
- Assert entries include correct actor information.

### Edge Cases & Notes

- The union query across multiple auditable types needs proper indexing on the audits table.
- State transition entries should be visually distinct from regular field changes.
- For large runs with many payslip audits, pagination is critical for performance.

## Acceptance Criteria

### Client-side

#### Behavior

- An "Audit History" tab is visible on the payroll run detail page for Owner, Payroll Officer, and Finance roles
- The tab shows all audit events related to the specific payroll run in a timeline-style layout
- State transitions are displayed prominently with colored badges (e.g., draft to submitted in blue, finalized in green)
- Each entry shows the actor (user name and role) and timestamp (relative time with full timestamp on hover)
- Entries are expandable to show detailed old/new value changes
- Results are paginated with "Load more" functionality
- Employee role cannot access the audit history tab

---

### Server-side

#### Side-effects

- The audit history includes all events related to the run: state transitions, payslip changes, bonus changes, and employee additions/removals
- Entries are sorted by newest first
- Results are paginated for performance
- Only Owner, Payroll Officer, and Finance roles can access the audit history
