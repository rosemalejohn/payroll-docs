# [GW] Payroll - Per-Run Bonus Configuration UI

## Description

Build the Bonuses Tab on the Payroll Run Detail page including the payroll run bonuses database table and model. This allows Payroll Officers and Owners to manage which bonuses are active for a specific payroll run. While the run is in draft state, users can add bonuses from the catalog, edit per-run amount/rate overrides, and toggle bonuses active or inactive. Any modification triggers a full recomputation of all payslips. The tab becomes read-only once the run leaves draft state. Per-run overrides affect only the current payroll run and do not modify the bonus types catalog defaults.

## Technical Implementation

### What Migrations

- Create the payroll run bonuses table with ULID primary key, payroll run reference, bonus type reference, denormalized bonus code and name, amount override (nullable decimal), rate override (nullable decimal with 4 decimal places), percentage basis (gross/basic, nullable), attendance threshold days (nullable integer), apply-to-all flag (default true), active flag (default true), added by user reference (nullable), and standard timestamps. Unique constraint on payroll run and bonus type. Index on payroll run.

### What Models

- Create PayrollRunBonus model in the Bonus domain with ULID support.
- Relationships: belongs to PayrollRun, belongs to BonusType, has many PayrollRunBonusExclusions, belongs to added-by User.
- Scopes: active bonuses.
- Casts: amount to decimal:2, rate to decimal:4, apply-to-all and is-active to boolean.

### What Actions

- Attach Bonus to Run action: validates the run is in draft state, validates the bonus type is not already attached to the run, creates a payroll run bonus record with defaults from the bonus type or user-provided overrides, and triggers recomputation.
- Update Run Bonus action: validates the run is in draft state, updates per-run amount, rate, percentage basis, attendance threshold, and apply-to-all on the payroll run bonus record without modifying the bonus types catalog. Triggers recomputation.
- Toggle Run Bonus Active action: validates the run is in draft state, flips the is-active flag, and triggers recomputation.

### Validation & DTOs

- Store Payroll Run Bonus request: validates bonus type ID (required, must exist, unique per run) and optional overrides for amount, rate, percentage basis, and attendance threshold.
- Update Payroll Run Bonus request: validates optional amount, rate, percentage basis, attendance threshold, and apply-to-all fields.

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}/bonuses` — list attached bonuses with available (unattached) bonus types.
- `POST /payroll-runs/{payrollRun}/bonuses` — attach a bonus to the run.
- `PUT /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}` — update per-run bonus config.
- `PATCH /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/toggle` — toggle active/inactive.

### Policies & Authorization

- Owner and Payroll Officer can manage per-run bonuses.
- All mutating actions are blocked when the run is not in draft state.

### Tests

- Assert a bonus can be added to a draft run.
- Assert a bonus cannot be added to a submitted/finalized run (422).
- Assert bonus amount can be overridden per run without changing bonus types catalog defaults.
- Assert toggle active/inactive works in draft state.
- Assert toggle is blocked in non-draft states.
- Assert duplicate bonus type attachment is rejected.
- Assert recomputation is triggered after add/edit/toggle.
- Assert available bonus list excludes already-attached types.

## Acceptance Criteria

### Client-side

#### Behavior

- The Bonuses tab on the payroll run detail page displays all attached bonuses with: Name, Code, Type, Amount/Rate, Apply to All (badge), Active (toggle switch), and Actions
- An "Add Bonus" button opens a modal showing only bonus types not already attached to the run, each with name, code, type, and default amount/rate
- Clicking a bonus in the modal attaches it to the run with default values
- An edit action opens a form to override per-run amount (for fixed), rate and percentage basis (for percentage), and attendance threshold (for attendance-type bonuses)
- A toggle switch allows activating/deactivating a bonus for the run
- All mutating actions show a loading state and refresh the page data on completion
- When the run is not in draft state, all buttons, toggles, and edit actions are disabled with a locked banner

---

### Server-side

#### Side-effects

- Adding a bonus copies default values from the bonus types catalog and creates a payroll run bonus record
- Per-run amount/rate overrides are saved to the payroll run bonus record and do not affect the catalog defaults
- Toggling a bonus inactive sets its active flag to false; toggling back sets it to true
- Every modification (add, edit, toggle) triggers recomputation and payslip totals are updated
- All mutating actions are blocked when the payroll run is not in draft state
- One bonus type per run is enforced by the unique constraint
- Only Owner and Payroll Officer roles can manage per-run bonuses
