# PAY-007: Create Users Migration, Model, and Factory

## Description

Customize the default Laravel `users` table migration to match the PayHRIS schema (ULID primary keys, company_id, employee_id, is_active, last_login_at) and relocate the User model to the Company domain. The User model integrates with Spatie roles (from PAY-005) and establishes relationships to Company and Employee models. This ticket also updates authentication configuration to work with the relocated model.

## Technical Implementation

### What Migrations
- Modify the default `create_users_table` migration (or create a new one if the default has already run):
  ```php
  Schema::create('users', function (Blueprint $table) {
      $table->char('id', 26)->primary();              // ULID
      $table->char('company_id', 26);                  // ref: companies.id (app-layer)
      $table->char('employee_id', 26)->nullable();     // ref: employees.id (nullable)
      $table->string('name', 255);
      $table->string('email', 255)->unique();
      $table->timestamp('email_verified_at')->nullable();
      $table->string('password', 255);
      $table->rememberToken();
      $table->boolean('is_active')->default(true);
      $table->timestamp('last_login_at')->nullable();
      $table->timestamps();

      $table->index('company_id', 'idx_users_company');
  });
  ```
- No database-level FK constraints on `company_id` or `employee_id`.

### What Models
- Move `app/Models/User.php` to `app/Company/Models/User.php` with namespace `App\Company\Models`.
- Update the model:
  ```php
  namespace App\Company\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Foundation\Auth\User as Authenticatable;
  use Illuminate\Notifications\Notifiable;
  use Spatie\Permission\Traits\HasRoles;

  class User extends Authenticatable
  {
      use HasUlids, HasFactory, HasRoles, Notifiable;

      protected $fillable = [
          'company_id', 'employee_id', 'name', 'email',
          'password', 'is_active', 'last_login_at',
      ];

      protected $hidden = ['password', 'remember_token'];

      protected $casts = [
          'email_verified_at' => 'datetime',
          'last_login_at' => 'datetime',
          'is_active' => 'boolean',
          'password' => 'hashed',
      ];

      // Relationships
      public function company() { return $this->belongsTo(Company::class); }
      public function employee() { return $this->belongsTo(\App\Employee\Models\Employee::class); }

      // Scopes
      public function scopeActive($query) { return $query->where('is_active', true); }
  }
  ```
- Update `config/auth.php` providers to point to the new model location:
  ```php
  'providers' => [
      'users' => [
          'driver' => 'eloquent',
          'model' => App\Company\Models\User::class,
      ],
  ],
  ```
- Update any Breeze auth controllers that reference `App\Models\User` to use `App\Company\Models\User`.

### Factories
- Create or update `database/factories/UserFactory.php`:
  ```php
  namespace Database\Factories;

  class UserFactory extends Factory
  {
      protected $model = \App\Company\Models\User::class;

      public function definition(): array
      {
          return [
              'company_id' => \App\Company\Models\Company::factory(),
              'name' => fake()->name(),
              'email' => fake()->unique()->safeEmail(),
              'email_verified_at' => now(),
              'password' => Hash::make('password'),
              'remember_token' => Str::random(10),
              'is_active' => true,
          ];
      }

      // Helper methods for role assignment
      public function owner(): static { return $this->afterCreating(fn ($user) => $user->assignRole('owner')); }
      public function payrollOfficer(): static { return $this->afterCreating(fn ($user) => $user->assignRole('payroll_officer')); }
      public function finance(): static { return $this->afterCreating(fn ($user) => $user->assignRole('finance')); }
      public function employee(): static { return $this->afterCreating(fn ($user) => $user->assignRole('employee')); }
  }
  ```
- Update `AdminUserSeeder` (from PAY-005) to assign `company_id` from the seeded company.

### Tests
- `tests/Feature/Company/UserModelTest.php` (Pest):
  - Asserts a user can be created with ULID primary key.
  - Asserts `belongsTo` relationship to Company works.
  - Asserts `is_active` and `last_login_at` casts work correctly.
  - Asserts factory role helpers work: `User::factory()->owner()->create()` produces a user with the `owner` role.
  - Asserts `active` scope filters correctly.
- `tests/Feature/Auth/AuthWithCustomUserTest.php` (Pest):
  - Asserts authentication still works after model relocation (login, logout, session persistence).
  - Asserts registration creates a user at the new model path.

### Edge Cases & Notes
- Moving the User model is a significant change. All imports across the project must be updated (`App\Models\User` to `App\Company\Models\User`). Use a project-wide search-and-replace.
- The `employee_id` is nullable because admin-only users may not have a corresponding employee record.
- `last_login_at` should be updated on successful login (can be handled via a listener on the `Login` event in a future ticket).
- Ensure Spatie's `HasRoles` trait guard matches the auth guard (`web`).
