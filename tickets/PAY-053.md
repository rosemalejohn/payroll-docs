# [GW] Payroll - Per-Run Bonus Configuration UI

## Description

Implement the Bonuses Tab on the Payroll Run Detail page, allowing payroll officers and owners to manage which bonuses are active for a specific payroll run. While the run is in `draft` state, users can add bonuses from the catalog, edit per-run amount/rate overrides, and deactivate bonuses. Any modification triggers a full recomputation of all payslips in the run. The tab becomes read-only once the run leaves draft state.

Per-run overrides affect only the current payroll run and do not modify the `bonus_types` catalog defaults.

## Technical Implementation

### Controller

**`app/Http/Controllers/Web/PayrollRunBonusController.php`**

- `index(PayrollRun $payrollRun)` — Returns bonuses tab data as part of payroll run detail Inertia props. Lists all `payroll_run_bonuses` for the run with bonus type info. Also returns available (unattached) bonus types for the "Add Bonus" modal.
- `store(StorePayrollRunBonusRequest $request, PayrollRun $payrollRun)` — Attaches a new bonus type to the run via `AttachBonusToRunAction`. Triggers recomputation.
- `update(UpdatePayrollRunBonusRequest $request, PayrollRun $payrollRun, PayrollRunBonus $payrollRunBonus)` — Updates per-run bonus config via `UpdateRunBonusAction`. Triggers recomputation.
- `toggleActive(PayrollRun $payrollRun, PayrollRunBonus $payrollRunBonus)` — Toggles `is_active` between 0 and 1. Triggers recomputation.

### Routes

```
GET    /payroll-runs/{payrollRun}/bonuses                          -> PayrollRunBonusController@index
POST   /payroll-runs/{payrollRun}/bonuses                          -> PayrollRunBonusController@store
PUT    /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}         -> PayrollRunBonusController@update
PATCH  /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/toggle  -> PayrollRunBonusController@toggleActive
```

Middleware: `auth`, `role:owner|payroll_officer`.

### Actions

**`app/Bonus/Actions/AttachBonusToRunAction.php`**

- Validates the run is in `draft` state.
- Validates the bonus type is not already attached to the run.
- Creates a `payroll_run_bonuses` record with defaults from `bonus_types` or user-provided overrides.
- Dispatches `RecomputePayrollRunAction`.

**`app/Bonus/Actions/UpdateRunBonusAction.php`**

- Validates the run is in `draft` state.
- Updates `amount`, `rate`, `percentage_basis`, `attendance_threshold_days`, `apply_to_all` on the `payroll_run_bonuses` record.
- Does not modify `bonus_types` defaults.
- Dispatches `RecomputePayrollRunAction`.

**`app/Bonus/Actions/ToggleRunBonusActiveAction.php`**

- Validates the run is in `draft` state.
- Flips `is_active` on the `payroll_run_bonuses` record.
- Dispatches `RecomputePayrollRunAction`.

### Form Requests

**`app/Http/Requests/Bonus/StorePayrollRunBonusRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`, and run is in `draft` state.
- Rules: `bonus_type_id` required|exists, unique per run; optional overrides for `amount`, `rate`, `percentage_basis`, `attendance_threshold_days`.

**`app/Http/Requests/Bonus/UpdatePayrollRunBonusRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`, and run is in `draft` state.
- Rules: optional `amount` (decimal), `rate` (decimal 0-1), `percentage_basis` (enum), `attendance_threshold_days` (int), `apply_to_all` (boolean).

### Frontend

**`resources/js/pages/payroll-runs/show.tsx` — Bonuses Tab**

- Tab within the payroll run detail page, shown alongside Employees, Summary, etc.
- Renders a table of attached bonuses with columns: Name, Code, Type, Amount/Rate, Apply to All (badge), Active (toggle switch), Actions.
- "Add Bonus" button opens a modal listing available bonus types not yet attached to the run. Each item shows name, code, computation_type, and default amount/rate. Click to attach with defaults.
- Edit action opens a modal or inline form to override: `amount` (for fixed), `rate` and `percentage_basis` (for percentage), `attendance_threshold_days` (for attendance-type bonuses).
- Toggle switch for active/inactive sends PATCH request.
- All mutating actions show a loading state and refresh the page data on completion.

**Read-only mode:**

- When `payrollRun.state !== 'draft'`, all buttons, toggles, and edit actions are disabled.
- A banner displays: "Bonuses are locked. The payroll run is no longer in draft state."

### Recomputation

Every bonus modification (add, edit, toggle) dispatches `RecomputePayrollRunAction` which:

1. Iterates all payslips in the run.
2. Re-runs `ComputePayslipAction` for each employee.
3. Updates payroll run totals (`total_gross`, `total_deductions`, `total_net`).

### Tests

- `tests/Feature/Bonus/PayrollRunBonusCrudTest.php`
  - Bonus can be added to a draft run.
  - Bonus cannot be added to a submitted/finalized run (422).
  - Bonus amount can be overridden per run without changing bonus_types defaults.
  - Toggle active/inactive works in draft state.
  - Toggle blocked in non-draft states.
  - Duplicate bonus type attachment is rejected.
  - Recomputation is triggered after add/edit/toggle.
  - Available bonus list excludes already-attached types.
  - Read-only mode renders correctly for non-draft runs.

## Acceptance Criteria

- [ ] Bonuses tab displays all attached bonuses for the current payroll run with correct details.
- [ ] "Add Bonus" modal shows only bonus types not already attached to the run.
- [ ] Adding a bonus copies default values from `bonus_types` and creates a `payroll_run_bonuses` record.
- [ ] Per-run amount/rate overrides are saved to `payroll_run_bonuses` and do not affect `bonus_types` catalog defaults.
- [ ] Toggling a bonus inactive sets `is_active = 0` (soft toggle, not hard delete); toggling back sets `is_active = 1`.
- [ ] Every modification (add, edit, toggle) triggers `RecomputePayrollRunAction` and payslip totals are updated.
- [ ] All mutating actions are blocked when the payroll run is not in `draft` state, returning 422.
- [ ] The bonuses tab becomes fully read-only with a locked banner when the run leaves draft state.
- [ ] Only owner and payroll_officer roles can manage per-run bonuses.
