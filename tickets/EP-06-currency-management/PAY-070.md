# [GW] Payroll - Currency Conversion Details in Run View

## Description

Add a "Currency Rates" tab to the payroll run detail page that displays all foreign currency exchange rates snapshotted for the run, along with a list of employees whose salaries are in foreign currencies. For finalized runs, rates are shown as historical and immutable. For draft runs where the current rate differs from the snapshot, both rates are displayed with a discrepancy indicator.

## Technical Implementation

### Queries

- Dedicated query class that loads currency snapshots for the run, joins with currencies for display info, and loads employees with foreign-currency salaries showing their original amount, exchange rate used, and PHP equivalent.
- For non-finalized runs, also loads current exchange rates for comparison.

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}/currency-rates` — currency rates tab content.
- Accessible to Owner, Payroll Officer, and Finance roles.

### API Resources

- Currency snapshot resource: currency code, name, snapshot rate, snapshot date, current rate (for non-finalized runs only), and rate changed indicator.
- Foreign currency employee resource: employee name and number, salary currency, original amount, exchange rate, and PHP equivalent.

### Policies & Authorization

- Uses the existing payroll run view policy. Employee role receives 403.

### Tests

- Assert Owner, Payroll Officer, and Finance can view the currency rates tab.
- Assert Employee role receives 403.
- Assert snapshot rates are displayed for each currency used in the run.
- Assert foreign currency employees are listed with correct conversion details.
- Assert PHP equivalent is correctly calculated.
- Assert finalized runs show rates as historical.
- Assert draft runs show current rates alongside snapshot rates when they differ.
- Assert the tab is hidden or shows empty state when no foreign currencies are used.

### Edge Cases & Notes

- PHP (base currency) should not appear in the snapshot list — it is always rate 1.0.
- Exchange rates display with up to 6 decimal places.
- For finalized runs, the current rate comparison is informational only.

## Acceptance Criteria

### Client-side

#### Layout

- A "Currency Rates" tab is available on the payroll run detail page when the run involves foreign currencies

#### Behavior

- The tab shows a table of snapshot exchange rates: Currency Code, Name, Snapshot Rate, and Snapshot Date
- Below the rates, a list of foreign currency employees shows: Employee Name, Employee Number, Salary Currency, Original Amount, Exchange Rate Used, and PHP Equivalent
- For finalized runs, a note states "These rates are historical and were locked at finalization"
- For draft/submitted runs, if the current rate differs from the snapshot rate, both rates are shown with a visual indicator highlighting the difference
- The tab is hidden or shows "All employees in this run are paid in PHP" when no foreign currencies are involved

---

### Server-side

#### Side-effects

- Only Owner, Payroll Officer, and Finance roles can view the currency rates tab
- Employee role is denied access
- Snapshot rates are read-only and never modified through this view
