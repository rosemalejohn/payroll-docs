# [GW] Payroll - ComputePayslipAction - Gross Pay Calculation (Step 1)

## Description

Implement Step 1 of the 10-step gross-to-net payroll computation pipeline within ComputePayslipAction. This step calculates the employee's gross pay from their salary basis (monthly or daily), handles foreign currency conversion using snapshot rates, records a BASIC earning line item, and snapshots salary fields onto the payslip for historical integrity.

## Technical Implementation

### What Actions

- ComputePayslipAction (Step 1 segment): loads employee compensation data, checks the external salary flag, loads the currency snapshot rate for the employee's salary currency, calculates gross pay based on salary basis (monthly salary directly or daily rate multiplied by working days), converts to PHP using the snapshot rate, snapshots salary fields on the payslip, and records a BASIC earning line item.

### Tests

- Assert monthly-basis employee receives their full monthly salary (converted to PHP) as gross pay.
- Assert daily-basis employee receives daily rate multiplied by working days (converted to PHP) as gross pay.
- Assert PHP-currency employees use a snapshot rate of 1.0.
- Assert foreign-currency employees have their salary converted using the snapshot rate.
- Assert a BASIC earning line item is created with the computed gross pay amount.
- Assert salary fields are snapshotted on the payslip (monthly salary, daily rate, currency ID, currency code, exchange rate).
- Assert missing compensation record throws an appropriate exception.
- Assert missing currency snapshot for a foreign-currency employee throws an appropriate exception.
- Assert all monetary values use decimal types with no floating point arithmetic.

### Edge Cases & Notes

- If the employee has no compensation record, the action must fail with a clear error.
- If the currency snapshot is not found for a foreign currency employee, the action must fail with a clear error.
- Zero monthly salary or daily rate results in gross pay of 0, but the BASIC line item is still created.
- All values must be stored using DECIMAL types with appropriate precision — no float arithmetic.

## Acceptance Criteria

### Server-side

#### Side-effects

- Monthly-basis employees receive their full monthly salary (converted to PHP) as gross pay
- Daily-basis employees receive their daily rate multiplied by working days (converted to PHP) as gross pay
- PHP-currency employees use a snapshot rate of 1.0 (no conversion effect)
- Foreign-currency employees have their salary converted using the snapshot rate from the payroll run currency snapshots
- A BASIC earning line item is created with the computed gross pay amount
- Salary fields are snapshotted on the payslip: monthly salary, daily rate, salary currency ID, salary currency code, and salary exchange rate
- Missing compensation records or missing currency snapshots result in a clear error
