# [GW] Payroll - RecomputePayrollRunAction - Batch Recompute All Payslips

## Description

Implement a batch recomputation action that recalculates all payslips in a payroll run from scratch. We will be using two recomputation strategies to avoid performance bottlenecks:

1. **Full Recomputation (`RecomputePayrollRunAction`)** — Rebuilds every payslip in the run. Safe but slow. Used only for critical steps such as pre-submission.
2. **Granular Recomputation (`RecomputeAffectedPayslipsAction`)** — Recomputes only affected payslips and updates totals via adjustments. Default for most draft-run updates.

## Technical Implementation

### 1. Full Recomputation Strategy (The "Safety Net")

#### Action: `app/Payroll/Actions/RecomputePayrollRunAction.php`

- Only allowed for **draft** runs.
- Deletes all line items in the run.
- Recomputes each payslip using `ComputePayslipAction`.
- Recalculates run totals using `SUM()` aggregation.

```php
class RecomputePayrollRunAction
{
    public function handle(PayrollRun $payrollRun): void
    {
        if (! $payrollRun->state->equals(DraftState::class)) {
            throw new InvalidRunStateException("Only draft runs can be recomputed.");
        }

        DB::transaction(function () use ($payrollRun) {
            PayslipLineItem::whereIn(
                'payslip_id',
                $payrollRun->payslips()->pluck('id')
            )->delete();

            $compute = app(ComputePayslipAction::class);
            $payrollRun->payslips->each(fn (Payslip $p) => $compute->handle($p));

            $payrollRun->update([
                'total_gross'      => $payrollRun->payslips()->sum('gross_pay'),
                'total_deductions' => $payrollRun->payslips()->sum('total_deductions'),
                'total_net'        => $payrollRun->payslips()->sum('net_pay'),
            ]);
        });
    }
}
```

---

### 2. Granular Recomputation Strategy (The "Fast Path")

#### Action: `app/Payroll/Actions/RecomputeAffectedPayslipsAction.php`

- Accepts a list of payslip IDs.
- Recomputes only those payslips.
- Calculates the difference between old and new totals.
- Dispatches `UpdatePayrollRunTotalsAction` with adjustment values.

#### Action: `app/Payroll/Actions/UpdatePayrollRunTotalsAction.php`

- Accepts adjustment deltas (e.g., `['total_gross' => +150.75]`).
- Updates totals using atomic `increment` / `decrement` operations.
- Avoids costly full-table `SUM()` queries.

### Triggers and Usage

- **Full Recompute**  
  - Triggered before submitting a payroll run for review.

- **Granular Recompute**  
  - Triggered when changes affect only specific employees, such as:  
    - Absent days  
    - Bonuses  
    - Currency rate updates  
    - Salary changes

### Performance Considerations

- Granular recomputation is significantly faster for large runs.  
- Incremental total updates prevent repeated aggregate scans.  
- Full recompute remains the final consistency checkpoint.

## Acceptance Criteria

- [ ] Full recomputation rebuilds all payslips and totals.  
- [ ] Granular recomputation updates only specified payslips.  
- [ ] Totals adjust correctly using deltas.  
- [ ] Isolated employee updates trigger granular recomputation.  
- [ ] Run submission triggers a full recomputation.  
- [ ] Both actions enforce draft-only restrictions.  
- [ ] Performance improves for partial updates on large runs.
