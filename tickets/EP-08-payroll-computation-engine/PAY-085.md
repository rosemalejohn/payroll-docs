# [GW] Payroll - BIR Tax Override Per Employee

## Description

Allow Payroll Officers and Owners to manually override the computed BIR withholding tax for a specific employee within a draft payroll run. Overrides require a reason and are tracked in the audit trail with both the original computed value and the new override value. Overridden BIR amounts are marked with an asterisk on the payslip to indicate manual adjustment. Recomputation of the payroll run preserves manual overrides and does not reset them to the computed value.

## Technical Implementation

### What Migrations

- Add BIR override fields to the payslips table: original computed BIR value (decimal, nullable), override flag (boolean, default false), and override reason (text, nullable). Index on the override flag for filtering.

### What Actions

- Override BIR Tax action: validates the payslip's payroll run is in draft state. Stores the original computed BIR value if not already stored. Sets the new BIR amount, the override flag to true, and the override reason. Triggers an audit log entry.
- Update the compute payslip action (Step 9): before computing BIR, check if the override flag is set. If overridden, skip BIR computation and retain the manually set value. Recompute net pay using the overridden BIR amount.
- Update the recompute payroll run action: when iterating payslips, pass the override flag so overridden values are not recalculated.

### Validation & DTOs

- Override BIR Tax request: validates override amount (numeric, must be 0 or greater) and reason (required, minimum 10 characters). Payslip must exist and its payroll run must be in draft state.
- Override BIR Tax DTO: payslip ID, override amount, and reason.

### Routes & Controllers

- `PATCH /payroll-runs/{payrollRun}/payslips/{payslip}/bir-override` — override BIR tax.

### Policies & Authorization

- Owner and Payroll Officer can override BIR tax.
- Payslip's payroll run must be in draft state.

### Tests

- Assert Payroll Officer can override BIR tax for an employee in a draft run.
- Assert override requires a reason (422 if missing or too short).
- Assert override amount must be 0 or greater (422 for negative values).
- Assert override is not allowed on non-draft runs (422).
- Assert audit trail records old (computed) and new (override) BIR values.
- Assert recomputation of the payroll run preserves the overridden BIR value.
- Assert overridden BIR is reflected in net pay calculation.
- Assert the original computed BIR value is preserved for reference.
- Assert non-authorized roles receive 403.

### Edge Cases & Notes

- The original computed BIR field preserves the system-calculated value so it can be displayed alongside the override for transparency.
- Multiple overrides on the same payslip update the existing override, not create duplicates.
- Each override is captured as a separate audit entry with full before/after values.

## Acceptance Criteria

### Client-side

#### Behavior

- Each employee row in the payroll run detail page shows the BIR amount with an "Override" button (visible in draft state for authorized users)
- Clicking the button opens a modal showing the current computed BIR amount (read-only), an override amount input field, and a reason text area (required, minimum 10 characters)
- Overridden BIR values are displayed with an asterisk and a tooltip showing the original computed value and reason
- On the payslip detail/PDF, the BIR line item shows an asterisk if overridden, with a footnote explaining the adjustment

#### Validation

- An error is shown if the reason is empty or shorter than 10 characters
- An error is shown if the override amount is negative

---

### Server-side

#### Side-effects

- The override amount replaces the computed BIR tax on the payslip
- A reason is required for every override and is recorded in the audit trail
- The original system-computed BIR value is preserved for reference
- Overrides persist through payroll run recomputation — they are not automatically reset
- Net pay is recalculated using the overridden BIR amount
- Only Owner and Payroll Officer roles can override BIR tax, and only on draft runs
- All overrides are captured in the audit trail with before/after values
