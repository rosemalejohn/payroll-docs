# PAY-036: Deactivate Currency with Safety Checks

## Description

As an owner, I want to deactivate a currency that is no longer needed, with safety checks preventing deactivation when active employees still have compensation in that currency.

## Technical Implementation

### Action
- **`app/Currency/Actions/DeactivateCurrencyAction.php`**
  - Prevents deactivating PHP base currency (`is_base = 1`)
  - Checks for `employee_compensation` records referencing this currency where the linked employee is active
  - If found: returns error with count "X employees still have compensation in [CURRENCY]. Please reassign before deactivating."
  - If safe: sets `is_active = 0`

### Controller
- **`CurrencyController@toggleActive`** — `PATCH /settings/currencies/{currency}/toggle-active`

### Business Rules
- Deactivated currencies no longer appear in currency selector for employee compensation
- Deactivated currencies are NOT included in future payroll run currency snapshots
- Existing payslips referencing this currency are unaffected (they use snapshot data)
- PHP base currency can never be deactivated

### Frontend
- Toggle button on currency row (owner only)
- If blocked: error toast with employee count message
- If safe: confirmation dialog "Deactivate [CODE]?"

### Tests
- Deactivation blocked when active employees use the currency
- PHP cannot be deactivated
- Deactivated currency hidden from compensation currency selector
- Existing payslips unaffected
- Reactivation works (toggle back to active)

## Acceptance Criteria

- [ ] Deactivation blocked with message if active employees use the currency
- [ ] PHP base currency cannot be deactivated
- [ ] Deactivated currency excluded from currency selector
- [ ] Deactivated currency not included in future payroll run snapshots
- [ ] Existing payslips and historical data unaffected by deactivation
- [ ] Currency can be reactivated (toggle back to `is_active = 1`)
