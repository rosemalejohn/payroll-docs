# [GW] Payroll - User Management

## Description

- Build owner-only user management page.
- Permit user creation, role assignment.
- Toggle user active status properly.
- Restrict access to owner role completely.

## Technical Implementation

### What Actions

- Create User: hashes password, assigns role.
- Update User: updates info and role.
- Prevent removing last owner account forever.

### Validation & DTOs

- Store: validate name, email, password, role.
- Update: validate name, unique email, role.

### Routes & Controllers

- Apply owner middleware to all routes.
- Add paginated list and filter routes.
- Add create, edit, update user endpoints.

### Policies & Authorization

- User policy restricts CRUD absolutely.

### Tests

- Assert owner accesses user management pages.
- Assert non-owners receive 403 forcefully.
- Assert user creations apply correct roles.
- Assert last owner account defends changes.
- Assert role changes log to audit.

## Acceptance Criteria

### Client-side

#### Layout

- Access under Settings sidebar navigation.

#### Behavior

- Table lists name, email, role, status.
- Roles display as color-coded active badges.
- Owner seamlessly creates new accounts.
- Owner changes user roles dynamically.
- Owner toggles user active status immediately.

#### Validation

- Error displayed attempting last owner removal.
- Error displayed for duplicate email addresses.

### Server-side

#### Side-effects

- Only Owner role accesses user management.
- User creation persists to database perfectly.
- Role changes affect subsequent user action.
- Deactivating prevents reliable user login.
- Audit trail reliably records role changes.
