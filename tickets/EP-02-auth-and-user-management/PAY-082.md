# [GW] Payroll - Employee Draft Payslip Access Restriction

## Description

Restrict employees to viewing only their own payslips from finalized payroll runs. Employees must not be able to access payslips that are still in draft, submitted, or under review status through any means — the payslip list, direct URL navigation, or API endpoints. When a payroll run is finalized, employees receive a notification that their payslip is available.

## Technical Implementation

### Policies & Authorization

- Payslip policy enforces that employees can only view payslips that belong to them AND are from finalized payroll runs.
- Other roles (owner, payroll_officer, finance) can view payslips in any state.

### Queries

- Dedicated employee payslip query class that filters to finalized runs only and matches the authenticated employee. Results ordered by most recent period first, paginated.

### Routes & Controllers

- `GET /my-payslips` — employee payslip list (finalized only)
- `GET /my-payslips/{payslip}` — employee payslip detail
- Both routes restricted to employee role.
- The list controller uses the dedicated query class; the detail controller authorizes via the payslip policy.

### API Resources

- Employee payslip resource shapes payslip data for employee view: period, gross pay, deductions breakdown, bonuses, net pay. Excludes internal flags and draft notes.

### Events & Listeners

- Listen for payroll run finalization event and send notifications to each employee with a payslip in the finalized run. Uses queue to avoid blocking finalization.

### Mail & Notifications

- In-app notification to employees containing the payroll period name, net pay amount, and a link to view the payslip.

### Tests

- Assert employee can list their own finalized payslips.
- Assert employee cannot see payslips from non-finalized runs in the list.
- Assert employee accessing a non-finalized payslip via direct URL receives 403.
- Assert employee cannot access another employee's payslip.
- Assert employee receives notification when a new finalized payslip is available.
- Assert other roles can view payslips in any state.

### Edge Cases & Notes

- Both the query-level filter and the policy must enforce the finalized-only rule — defense in depth.
- Ensure no shared props or global queries leak draft payslip counts or summaries to the employee role.

## Acceptance Criteria

### Client-side

#### Layout

- Employees have a "My Payslips" page accessible from the sidebar navigation

#### Behavior

- The payslip list shows only payslips from finalized payroll runs, sorted by most recent first
- Each payslip entry shows the period, gross pay, total deductions, and net pay
- Clicking a payslip opens the detail view with a full breakdown of earnings, deductions, and bonuses
- Employees receive an in-app notification when a new payslip is available after finalization
- A PDF download button is available if the payslip PDF has been generated

---

### Server-side

#### Side-effects

- Employees can only see their own payslips — accessing another employee's payslip is denied
- Payslips from draft, submitted, or under review payroll runs are completely hidden from employees
- Directly navigating to a non-finalized payslip URL returns a 403 response
- No partial or preview payslip data is exposed to the employee role through any endpoint
- Other roles (Owner, Payroll Officer, Finance) can still view payslips in any state
