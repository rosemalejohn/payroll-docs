# [GW] Payroll - Bonus Types CRUD in Settings

## Description

Implement the full CRUD interface for managing bonus types under Settings. Bonus types define the catalog of bonuses available for attachment to payroll runs. There are two categories: system bonuses (`is_system = 1`) which are seeded and cannot be deleted or have their code changed, and custom bonuses (`is_system = 0`) which support full CRUD operations.

The owner and payroll_officer roles can create custom bonus types. Only the owner role can delete custom bonus types, and deletion is only permitted if the bonus type has never been used in any payroll run (no `payroll_run_bonuses` records reference it).

System bonuses (Meal Allowance, Attendance Bonus, 13th Month Pay, June Special Bonus) are always visible and allow editing of `default_amount` and `default_rate`, but their `code`, `is_system`, and `computation_type` fields are locked.

## Technical Implementation

### Controller

**`app/Http/Controllers/Web/BonusTypeController.php`**

- `index()` — Returns Inertia page `settings/bonuses/index` with all bonus types for the company, sorted by `sort_order`. Props include bonus types list with usage counts (whether referenced in `payroll_run_bonuses`).
- `store(StoreBonusTypeRequest $request)` — Creates a new custom bonus type via `CreateBonusTypeAction`. Returns redirect with flash message.
- `update(UpdateBonusTypeRequest $request, BonusType $bonusType)` — Updates a bonus type via `UpdateBonusTypeAction`. For system bonuses, only `default_amount`, `default_rate`, `description`, `is_active`, and attendance config fields are mutable.
- `destroy(BonusType $bonusType)` — Deletes a custom bonus type via `DeleteBonusTypeAction`. Returns redirect with flash message.

### Routes

```
GET    /settings/bonuses              -> BonusTypeController@index
POST   /settings/bonuses              -> BonusTypeController@store
PUT    /settings/bonuses/{bonusType}  -> BonusTypeController@update
DELETE /settings/bonuses/{bonusType}  -> BonusTypeController@destroy
```

Middleware: `auth`, `role:owner|payroll_officer` for index/store/update; `role:owner` for destroy.

### Actions

**`app/Bonus/Actions/CreateBonusTypeAction.php`**

- Accepts `BonusTypeData` DTO.
- Auto-generates `code` from `name` (uppercase, underscores, e.g., "Transport Allowance" -> "TRANSPORT_ALLOWANCE"). Code is editable before save.
- Sets `is_system = 0`.
- Validates uniqueness of `code` within the company.

**`app/Bonus/Actions/UpdateBonusTypeAction.php`**

- Accepts `BonusTypeData` DTO and existing `BonusType` model.
- If `is_system = 1`: only allows updating `default_amount`, `default_rate`, `percentage_basis`, `attendance_threshold_days`, `attendance_threshold_type`, `description`, `is_active`, `sort_order`.
- If `is_system = 0`: allows updating all mutable fields except `id` and `company_id`.

**`app/Bonus/Actions/DeleteBonusTypeAction.php`**

- Validates `is_system = 0` (system bonuses cannot be deleted).
- Validates no `payroll_run_bonuses` records reference this bonus type.
- Throws `BonusTypeInUseException` if referenced.
- Hard deletes the record.

### Models

**`app/Bonus/Models/BonusType.php`**

- Implements `Auditable` (owen-it/laravel-auditing).
- Fillable: `company_id`, `code`, `name`, `is_system`, `computation_type`, `default_amount`, `default_rate`, `percentage_basis`, `attendance_threshold_days`, `attendance_threshold_type`, `auto_trigger`, `auto_trigger_month`, `is_active`, `description`, `sort_order`.
- Relationships: `hasMany(PayrollRunBonus::class)`, `belongsTo(Company::class)`.
- Scope: `scopeActive($query)`, `scopeSystem($query)`, `scopeCustom($query)`.
- Accessor: `is_deletable` — returns `true` if `is_system = 0` and has no associated `payroll_run_bonuses`.

### DTOs

**`app/Bonus/DTOs/BonusTypeData.php`** (spatie/laravel-data)

- `name`: string, required
- `code`: string, required, unique per company
- `computation_type`: enum (fixed, percentage, formula)
- `default_amount`: nullable decimal
- `default_rate`: nullable decimal
- `percentage_basis`: nullable enum (gross, basic)
- `attendance_threshold_days`: nullable int
- `attendance_threshold_type`: nullable enum (days, percentage)
- `auto_trigger`: boolean
- `auto_trigger_month`: nullable int (1-12)
- `description`: nullable string
- `is_active`: boolean
- `sort_order`: int

### Form Requests

**`app/Http/Requests/Bonus/StoreBonusTypeRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`.
- Rules: `name` required|max:150, `code` required|max:50|unique within company, `computation_type` required|in:fixed,percentage,formula, conditional rules for amount/rate based on computation_type.

**`app/Http/Requests/Bonus/UpdateBonusTypeRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`.
- Additional validation: if bonus type `is_system = 1`, rejects changes to `code`, `name`, `computation_type`, `is_system`.

### Policy

**`app/Bonus/Policies/BonusTypePolicy.php`**

- `viewAny()`: owner, payroll_officer
- `create()`: owner, payroll_officer
- `update()`: owner, payroll_officer
- `delete()`: owner only; returns `false` if `is_system = 1` or bonus type is referenced by `payroll_run_bonuses`

### Frontend

**`resources/js/pages/settings/bonuses/index.tsx`**

- Table displaying all bonus types with columns: Name, Code, Type (computation_type), Auto Trigger (badge), Active (toggle), System (badge), Actions.
- System bonuses have a "System" badge and no delete button.
- "Create Bonus Type" button opens a modal/dialog.
- Edit button opens a modal/dialog pre-filled with current values.
- Delete button (custom bonuses only) shows confirmation dialog; disabled with tooltip if bonus is used in a run.

**Create/Edit Modal:**

- Fields: `name`, `code` (auto-generated from name, editable), `computation_type` (select), `default_amount` (shown when computation_type=fixed), `default_rate` (shown when computation_type=percentage), `percentage_basis` (shown when computation_type=percentage), `attendance_threshold_days` (shown when computation_type=percentage), `auto_trigger` (checkbox), `auto_trigger_month` (shown when auto_trigger=true, month select), `description`, `is_active`.
- For system bonuses in edit mode: `name`, `code`, `computation_type` fields are disabled.
- Zod schema mirrors server validation for UX feedback.

### Tests

- `tests/Feature/Bonus/BonusTypeCrudTest.php`
  - Owner can create a custom bonus type.
  - Payroll officer can create a custom bonus type.
  - Finance role cannot create bonus types (403).
  - Employee role cannot access bonus settings (403).
  - System bonus `code` cannot be changed via update.
  - System bonus cannot be deleted (403/422).
  - Custom bonus can be deleted if not used in any run.
  - Custom bonus cannot be deleted if used in a run (422).
  - Duplicate `code` within company is rejected (422).
  - Computation type controls required fields validation.

## Acceptance Criteria

- [ ] All four system bonus types are visible on the settings page with a "System" badge.
- [ ] System bonuses allow editing `default_amount`, `default_rate`, attendance config, and `is_active` but not `code` or `computation_type`.
- [ ] System bonuses cannot be deleted; the delete button is hidden or disabled.
- [ ] Custom bonus types can be created by owner and payroll_officer roles.
- [ ] Custom bonus types can be fully edited (name, code, computation_type, amount/rate, auto_trigger settings).
- [ ] Custom bonus types can only be deleted by the owner role.
- [ ] Deletion is blocked with an error message if the bonus type is referenced by any `payroll_run_bonuses` record.
- [ ] The `code` field is auto-generated from `name` but remains editable before save.
- [ ] The `computation_type` selection controls which fields are visible and required (fixed shows amount, percentage shows rate and basis, formula hides both).
- [ ] Auto-trigger month select is only shown when auto_trigger is enabled.
- [ ] All changes are captured in the audit trail via `owen-it/laravel-auditing`.
- [ ] Finance and employee roles cannot access the bonus type settings page (403).
