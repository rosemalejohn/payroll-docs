# [GW] Payroll - Approve Payroll Run for Finalization (Finance)

## Description

Implement the finance approval step for payroll runs. Finance users can approve a payroll run that is currently in the under review state, marking it as ready for the Owner to finalize. This is an intermediate approval step — Finance cannot finalize the run directly. Upon approval, the Owner is notified that the run is ready for finalization.

## Technical Implementation

### What Migrations

- Add approved_by (nullable ULID) and approved_at (nullable timestamp) to the payroll runs table.

### What Actions

- Approve Payroll Run action: validates the run is in under review state and not already approved, sets approved_by and approved_at, and dispatches PayrollRunApproved event.

### Validation & DTOs

- Approve Payroll Run request: validates the run exists and is in the correct state.
- Approve Payroll Run DTO using spatie/laravel-data.

### Routes & Controllers

- POST /payroll-runs/{payrollRun}/approve — approve the run for finalization.

### Policies & Authorization

- Only Finance role can approve.
- Run must be in under review state and not already approved.

### Events & Listeners

- PayrollRunApproved event fired after successful approval.
- Listener notifies all users with the Owner role.

### Mail & Notifications

- PayrollRunApproved notification via database and mail channels with run details and a finalization link.

### Tests

- Assert Finance user can approve an under review payroll run.
- Assert non-Finance users (Payroll Officer, Employee, Owner) receive 403.
- Assert approving a run not in under review state returns 422.
- Assert approving an already-approved run returns 422.
- Assert approved_by and approved_at are correctly set.
- Assert Owner receives a notification after approval.
- Assert approval is recorded in the audit trail.

### Edge Cases & Notes

- A payroll run can only be approved once; re-approval returns a validation error.
- If the run is returned to a previous state, the approval fields are cleared.

## Acceptance Criteria

### Client-side

#### Behavior

- An "Approve for Finalization" button is visible to Finance users on under review runs that have not yet been approved
- After approval, a confirmation flash message is shown
- The button is hidden for non-Finance roles

---

### Server-side

#### Side-effects

- Finance users can approve payroll runs that are in the under review state
- Finance cannot finalize a payroll run directly — only approve it for the Owner
- The Owner is notified (in-app and email) when a payroll run is approved
- Non-Finance roles are denied approval access
- Payroll runs not in under review state cannot be approved
- Already-approved runs cannot be re-approved
- The approved by and approved at fields are correctly recorded
- Approval is logged in the audit trail
