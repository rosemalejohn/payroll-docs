# PAY-084: Bulk Apply Deductions to Payroll Run

## Description

Implement a bulk deduction application action that computes SSS, PhilHealth, Pag-IBIG, and HMO for all eligible employees in a draft payroll run in a single operation. This replaces the need to compute deductions employee-by-employee when setting up a new run. The action uses each employee's gross pay and configuration for bracket/rate lookups, automatically skips employees with `has_external_salary = true`, and presents a summary modal showing the results before confirming.

Individual overrides remain possible after the bulk application, and the action triggers a full recomputation of the payroll run totals.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/BulkApplyDeductionsAction.php`** — Single-responsibility action that:
  - Accepts a `PayrollRun` model (must be in `draft` state)
  - Iterates over all payslips in the run
  - For each payslip where the employee does NOT have `has_external_salary = true`:
    - Computes SSS using bracket-based lookup on gross_pay (Step 1)
    - Computes PhilHealth at 2.5% employee share, subject to salary floor/ceiling
    - Computes Pag-IBIG at fixed ₱100
    - Computes HMO from the employee's HMO configuration
    - Updates the payslip record with computed deduction amounts
  - Skips employees with `has_external_salary = true` and tracks the skip count
  - Returns a `BulkDeductionSummaryData` DTO with results
  - Dispatches `RecomputePayrollRunAction` after all payslips are updated
  - Wraps all mutations in a database transaction
- **`app/Payroll/Actions/RecomputePayrollRunAction.php`** — Existing action; triggered after bulk apply to recalculate run totals (net pay, total deductions, etc.)

### DTOs
- **`app/Payroll/DTOs/BulkDeductionSummaryData.php`** — spatie/laravel-data DTO:
  - `employees_affected`: int (count of employees with deductions applied)
  - `employees_skipped`: int (count of external salary employees skipped)
  - `total_sss`: string (decimal, sum of all SSS deductions)
  - `total_philhealth`: string (decimal, sum of all PhilHealth deductions)
  - `total_pagibig`: string (decimal, sum of all Pag-IBIG deductions)
  - `total_hmo`: string (decimal, sum of all HMO deductions)
  - `total_deductions`: string (decimal, grand total)

### Validation & DTOs
- **`app/Http/Requests/Payroll/BulkApplyDeductionsRequest.php`** — Validates:
  - Payroll run exists
  - Payroll run is in `draft` state (rejects if submitted/under_review/finalized)

### Routes & Controllers
- **`POST /payroll-runs/{payrollRun}/bulk-apply-deductions`** — Route name: `payroll-runs.bulk-apply-deductions`
- **`app/Http/Controllers/Web/PayrollRunBulkDeductionController@store`**
  - Middleware: `auth`, `role:owner|payroll_officer`
  - Authorizes via `PayrollRunPolicy@bulkApplyDeductions`
  - Delegates to `BulkApplyDeductionsAction`
  - Returns redirect with flash containing summary data

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Add `bulkApplyDeductions()` method:
  - `owner` and `payroll_officer` roles can bulk apply
  - Run must be in `draft` state

### Frontend (Inertia/React)
- **`resources/js/pages/payroll-runs/show.tsx`** — Add "Bulk Apply Deductions" button:
  - Visible only when run is in `draft` state and user has permission
  - Opens confirmation dialog before submitting
- **`resources/js/components/modals/bulk-deduction-summary-modal.tsx`** — Summary modal displaying:
  - Number of employees affected
  - Number of external salary employees skipped
  - Total per deduction type (SSS, PhilHealth, Pag-IBIG, HMO)
  - Grand total of all deductions
  - "Close" button to dismiss
- Post-submission flow: action executes, page reloads with flash data, summary modal auto-opens with results

### Tests
- **`tests/Feature/Payroll/BulkApplyDeductionsTest.php`**
  - Bulk apply computes correct SSS for each employee based on their gross pay bracket
  - Bulk apply computes correct PhilHealth (2.5%, capped)
  - Bulk apply sets Pag-IBIG to ₱100 for all eligible employees
  - Bulk apply sets HMO based on each employee's configuration
  - Employees with `has_external_salary = true` are skipped
  - Skip count is accurately reported in summary
  - Bulk apply only works on draft runs (422 for other states)
  - Individual override after bulk apply is preserved and functional
  - `RecomputePayrollRunAction` is triggered after bulk apply
  - Payroll run totals are accurate after bulk apply and recomputation
  - Non-authorized roles receive 403
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- Bulk apply overwrites any previously computed deductions for the affected payslips — this is expected behavior, not a bug
- Individual overrides made AFTER bulk apply are preserved on subsequent recomputations (handled by override flags on payslip fields)
- If all employees in a run have `has_external_salary = true`, the action completes successfully with zero employees affected
- Large payroll runs should be processed within acceptable time limits; consider chunking if runs exceed 500 employees
- The action must be idempotent — running it multiple times on the same draft run produces the same results
