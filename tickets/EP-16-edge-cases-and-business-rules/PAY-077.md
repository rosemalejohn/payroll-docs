# [GW] Payroll - Mid-Month Employee Termination Handling

## Description

Handle cases where an employee is terminated during an active payroll period. The payroll officer sets the employee's last working day, and the system prorates gross pay based on days worked, recalculates deductions using the prorated amount, and adds the termination details to the payslip. This ensures the employee is paid correctly for the portion of the period they actually worked.

## Technical Implementation

### What Migrations

- Add termination fields to payslips: status, dates, proration basis.

### What Models

- Add termination fields to Payslip fillable and casts.

### What Actions

- Prorate terminated employee payslips within the payroll period.
- Recalculate deductions using prorated gross pay.
- Store termination date, last day, and proration basis.

### Validation & DTOs

- DTO includes run, employee, termination, and last-day fields.
- Validate dates and employee membership in payroll run.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/payslips/{payslip}/termination` sets termination.

### Policies & Authorization

- Only Payroll Officer and Owner can set termination.
- Payroll run must be draft.
- Payslip must not already be terminated.

### Tests

- Assert monthly gross pay is prorated correctly.
- Assert daily-rate gross pay is prorated correctly.
- Assert deductions recalculate from prorated gross.
- Assert withholding tax recalculates from prorated gross.
- Assert payslip notes include termination details.
- Assert last working day stays within run period.
- Assert out-of-period last working day fails validation.
- Assert payslip terminated flag is set.
- Assert only draft runs allow termination.
- Assert unauthorized roles receive 403.
- Assert external-salary employees skip deductions after proration.
- Assert net pay is correct after proration.

### Edge Cases & Notes

- External-salary employees skip deductions after gross proration.
- Pag-IBIG proration should remain configurable.
- Handle zero-day terminations with zero-gross payslips.
- Working days must exclude weekends and holidays.
- Allow recomputation after termination corrections.

## Acceptance Criteria

### Client-side

#### Behavior

- Show "Mark as Terminated" on draft payslips.
- Open dialog for termination and last working day.
- Limit last working day to payroll period.
- Show days worked and gross preview.
- Display a "Terminated" badge on payslips.
- Show termination details on the payslip.
- Restrict termination actions to Payroll Officer and Owner.

---

### Server-side

#### Side-effects

- Prorate monthly gross using worked versus total days.
- Calculate daily-rate gross from days worked.
- Recalculate deductions using prorated gross.
- Record termination date and proration basis on payslip.
- Allow termination only in draft runs.
- Restrict termination to Payroll Officer and Owner.
- Prorate external-salary gross, then skip deductions.
