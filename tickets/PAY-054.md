# PAY-054: Per-Employee Bonus Exclusions

## Description

Implement the ability to exclude specific employees from receiving a particular bonus within a payroll run. When a bonus has `apply_to_all` toggled off, the payroll officer can select which employees should be excluded from that bonus. Excluded employees are recorded in the `payroll_run_bonus_exclusions` table with an optional reason. Exclusions are checked during Step 2 of the payroll computation pipeline, and any change to exclusions triggers recomputation for the affected employees.

## Technical Implementation

### Controller

**`app/Http/Controllers/Web/BonusExclusionController.php`**

- `index(PayrollRun $payrollRun, PayrollRunBonus $payrollRunBonus)` — Returns the list of all employees in the run with their exclusion status for this bonus. Props include: employee list (id, name, department, position), current exclusions (employee_id, reason), and the bonus detail.
- `store(ManageBonusExclusionsRequest $request, PayrollRun $payrollRun, PayrollRunBonus $payrollRunBonus)` — Replaces the full set of exclusions for this bonus via `ManageBonusExclusionsAction`. Accepts an array of `{ employee_id, reason }` objects. Triggers recomputation.

### Routes

```
GET  /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/exclusions  -> BonusExclusionController@index
POST /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/exclusions  -> BonusExclusionController@store
```

Middleware: `auth`, `role:owner|payroll_officer`.

### Action

**`app/Bonus/Actions/ManageBonusExclusionsAction.php`**

- Accepts: `PayrollRunBonus $runBonus`, `array $exclusions` (each with `employee_id` and optional `reason`)
- Validates the payroll run is in `draft` state.
- Sets `apply_to_all = 0` on the `payroll_run_bonuses` record if exclusions are provided.
- Sets `apply_to_all = 1` if the exclusion list is empty (all employees included).
- Performs a sync operation: deletes existing exclusions for this bonus, inserts the new set.
- Identifies affected employees (newly excluded or newly included) and dispatches recomputation only for those employees.
- Returns the updated exclusion list.

**Logic:**

```php
public function handle(PayrollRunBonus $runBonus, array $exclusions): void
{
    $payrollRun = $runBonus->payrollRun;

    // Validate draft state
    throw_unless($payrollRun->state === 'draft', PayrollRunNotEditableException::class);

    // Get previous exclusions for diff
    $previousExcludedIds = $runBonus->exclusions()->pluck('employee_id')->toArray();

    // Sync exclusions
    $runBonus->exclusions()->delete();

    $newExcludedIds = [];
    foreach ($exclusions as $exclusion) {
        PayrollRunBonusExclusion::create([
            'payroll_run_bonus_id' => $runBonus->id,
            'employee_id'          => $exclusion['employee_id'],
            'reason'               => $exclusion['reason'] ?? null,
        ]);
        $newExcludedIds[] = $exclusion['employee_id'];
    }

    // Update apply_to_all flag
    $runBonus->update(['apply_to_all' => empty($newExcludedIds)]);

    // Recompute affected employees (those whose exclusion status changed)
    $affectedIds = array_unique(array_merge(
        array_diff($previousExcludedIds, $newExcludedIds), // newly included
        array_diff($newExcludedIds, $previousExcludedIds), // newly excluded
    ));

    foreach ($affectedIds as $employeeId) {
        app(ComputePayslipAction::class)->handle($payrollRun, $employeeId);
    }

    // Update run totals
    app(UpdatePayrollRunTotalsAction::class)->handle($payrollRun);
}
```

### Models

**`app/Bonus/Models/PayrollRunBonusExclusion.php`**

- Fillable: `payroll_run_bonus_id`, `employee_id`, `reason`.

### Form Request

**`app/Http/Requests/Bonus/ManageBonusExclusionsRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`, and run is in `draft` state.
- Rules: `exclusions` array, `exclusions.*.employee_id` required|exists in employees table and must be an employee in the current payroll run, `exclusions.*.reason` nullable|string|max:500.

### Frontend

**Exclusion UI (within bonus detail on payroll run):**

- When the user clicks "Manage Exclusions" on a bonus row (or toggles "Apply to All" off), a modal or slide-over panel opens.
- Displays a checkbox list of all employees in the payroll run, grouped or sorted by department.
- Search/filter input to find employees by name, employee number, or department.
- Checked employees receive the bonus; unchecked employees are excluded.
- Each unchecked (excluded) employee row shows an optional reason text input.
- "Save" button submits the full exclusion list and triggers recomputation.
- Loading state while recomputation processes.
- Current exclusion count is displayed as a badge on the bonus row in the bonuses tab (e.g., "3 excluded").

**Read-only mode:**

- When run is not in `draft`, the exclusion list is viewable but not editable.

### Computation Integration

In Step 2 of `ComputePayslipAction`, before applying each bonus:

```php
foreach ($activeBonuses as $runBonus) {
    if (!$runBonus->apply_to_all) {
        $excluded = $runBonus->exclusions()
            ->where('employee_id', $employee->id)
            ->exists();
        if ($excluded) {
            continue; // Skip this bonus for this employee
        }
    }
    // Apply bonus...
}
```

### Tests

- `tests/Feature/Bonus/BonusExclusionTest.php`
  - Excluding an employee prevents them from receiving the bonus in computed payslip.
  - Excluding and then re-including an employee restores the bonus in their payslip.
  - Exclusion reason is saved correctly.
  - `apply_to_all` is set to 0 when exclusions exist, 1 when none.
  - Exclusions cannot be modified when run is not in draft (422).
  - Only employees in the current payroll run can be excluded (validation error for others).
  - Recomputation is triggered only for affected employees (those whose status changed).
  - Exclusion count is returned in bonus list props.

## Acceptance Criteria

- [ ] Toggling "Apply to All" off on a bonus reveals the employee exclusion interface.
- [ ] The employee list shows all employees in the current payroll run with checkboxes.
- [ ] Unchecked employees are added to `payroll_run_bonus_exclusions` with an optional reason.
- [ ] Excluded employees do not receive the bonus in their computed payslip (Step 2 skips them).
- [ ] Exclusion reasons are saved and visible on the bonus detail view.
- [ ] Saving exclusion changes triggers recomputation for affected employees only.
- [ ] The `apply_to_all` flag on `payroll_run_bonuses` auto-updates based on whether exclusions exist.
- [ ] Exclusion management is blocked when the payroll run is not in `draft` state.
- [ ] Exclusion counts are displayed as badges on the bonuses tab for each bonus.
- [ ] Search/filter allows finding employees quickly in the exclusion list.
