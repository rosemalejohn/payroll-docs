# PAY-060: Finance Review Page for Submitted Runs

## Description

As a finance user, I want to review a submitted payroll run in detail so that I can verify all figures, flag discrepancies, and either approve or return the run for corrections. The review page shows all payslip details in read-only mode with the ability to add flags and comments.

## Technical Implementation

### Controller
- **`FinanceReviewController@show`** — `GET /finance/payroll-runs/{payrollRun}/review`
- Middleware: `auth`, `role:finance|owner`
- Returns Inertia page with:
  - Run summary (name, period, state, totals)
  - All payslips with line-item detail (eager loaded)
  - Existing flags and their statuses
  - Available actions based on current state

### Policy
- **`PayrollRunPolicy@review`** — only `finance` and `owner` roles can access
- Run must be in `submitted` or `under_review` state

### Actions
- **`MoveToUnderReviewAction`** — transitions `submitted → under_review`, sets `reviewed_by`, `reviewed_at`
- **`FlagPayslipLineAction`** — creates a flag on a payslip line item
  - Model: `PayslipFlag` (payslip_id, payslip_line_item_id nullable, flagged_by, note, status enum open/resolved, resolved_by, resolved_at, resolved_note, timestamps)
  - Migration for `payslip_flags` table

### Frontend
- **`resources/js/pages/finance/review.tsx`**
- Layout:
  - Run summary header with state badge and totals
  - Employee-by-employee expandable payslip breakdown
  - Each payslip shows: earnings, bonuses, deductions (all line items), net pay
  - Flag icon on each line item — click to add flag with text note
  - Flagged items highlighted in orange/red
  - Existing flags shown inline with status (open/resolved)
- Action buttons:
  - "Place Under Review" (visible when state is `submitted`)
  - "Return to Draft" with reason dialog
  - "Approve for Finalization" (see PAY-061)
- All figure fields are read-only — finance cannot edit payslip values directly

### Tests
- `tests/Feature/Finance/ReviewPayrollRunTest.php`
  - Finance user can access review page for submitted run
  - Employee role cannot access review page (403)
  - Payroll officer cannot access review page (403)
  - Finance can place run under review
  - Finance can flag a line item
  - Finance can return run to draft
  - Finance cannot edit payslip figures

## Acceptance Criteria

- [ ] Finance and owner can access the review page
- [ ] Payroll officer and employee roles get 403
- [ ] All payslip details displayed in read-only mode
- [ ] Finance can place a submitted run under review (state transition to `under_review`)
- [ ] Finance can add flag with text note on any payslip line item
- [ ] Flagged items are highlighted in the UI for all authorized users
- [ ] Finance can return run to payroll_officer (back to draft with reason)
- [ ] Finance cannot edit payslip figures directly
- [ ] Existing flags and their resolution status are visible
- [ ] `reviewed_by` and `reviewed_at` set when placed under review
