# [GW] Payroll - Employer Contributions in Payroll Run Summary

## Description

Add employer-side contribution calculations and display to the payroll run detail and summary report. For each employee, the system computes the employer share of SSS, PhilHealth, and Pag-IBIG using the same bracket tables and rates as the employee deductions. These amounts are displayed alongside employee deductions in the run summary, included in the total employer cost calculation, and exported in the CSV report. Employer contributions are informational and do not affect the employee's net pay.

## Technical Implementation

### What Migrations

- Add employer_sss, employer_philhealth, employer_pagibig, and employer_contributions_total (all decimal, default 0) to the payslips table.

### What Actions

- Compute Employer Contributions action: accepts a payslip and its gross pay. Looks up employer SSS from the bracket table, computes employer PhilHealth at 2.5% of basic salary (same floor/ceiling), and sets employer Pag-IBIG to PHP 100. Returns all three amounts and their sum. Returns zeroes for external salary employees.
- Update the compute payslip action to invoke the employer contributions action and store results on the payslip.

### Queries

- Update the payroll run summary query to aggregate employer contributions (SSS, PhilHealth, Pag-IBIG) and compute total employer cost as net pay plus total contributions.

### API Resources

- Update the payslip resource to include employer contribution fields.
- Update the payroll run summary resource to include aggregate employer contributions and total employer cost.

### Tests

- Assert employer SSS is correctly looked up from the bracket table.
- Assert employer PhilHealth is 2.5% of basic salary, capped at ceiling.
- Assert employer Pag-IBIG is fixed at PHP 100.
- Assert external salary employees have zero employer contributions.
- Assert employer contributions are stored on payslip records.
- Assert payroll run summary aggregates employer contributions correctly.
- Assert total employer cost equals total net pay plus total employer contributions.
- Assert CSV export includes employer contribution columns.

### Edge Cases & Notes

- Employer contributions use the same gross pay (Step 1 basic salary) as the basis, not adjusted gross.
- SSS employer share is a separate column in the bracket table — not the same as employee share.
- Employer contributions do not affect the employee's net pay calculation.

## Acceptance Criteria

### Client-side

#### Behavior

- The payroll run detail page shows employer SSS, PhilHealth, and Pag-IBIG contribution columns alongside employee share columns
- The run summary section displays aggregate employer contributions and total employer cost
- Employer and employee shares are clearly distinguished with separate column headers

---

### Server-side

#### Side-effects

- Employer SSS, PhilHealth, and Pag-IBIG contributions are computed using the same bracket tables and rates as employee deductions
- Employer contributions are based on the same gross pay (Step 1 basic salary) as employee deductions
- Total employer cost is calculated as the sum of net pay plus all employer contributions
- External salary employees have zero employer contributions
- Employer contributions are stored per payslip for historical accuracy
- CSV export includes employer contribution columns
- Employer contributions do not affect the employee's net pay
