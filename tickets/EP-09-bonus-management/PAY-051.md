# [GW] Payroll - Bonus Types CRUD in Settings

## Description

Build the bonus type management settings page including the bonus types database table, model, and seeder for the 4 system bonus types. Bonus types define the catalog of bonuses available for attachment to payroll runs. There are two categories: system bonuses (seeded, cannot be deleted or have their code changed) and custom bonuses (support full CRUD). The Owner and Payroll Officer can create custom bonus types. Only the Owner can delete custom bonus types, and deletion is only permitted if the bonus type has never been used in any payroll run.

System bonuses are: Meal Allowance (fixed, auto-trigger every month), Attendance Bonus (percentage, manual trigger), 13th Month Pay (formula, auto-trigger December), and June Special Bonus (formula, auto-trigger June).

## Technical Implementation

### What Migrations

- Create the bonus types table with ULID primary key, company reference, code (unique per company), name, system flag (default false), computation type (fixed/percentage/formula), default amount (nullable decimal), default rate (nullable decimal with 4 decimal places), percentage basis (gross/basic, nullable), attendance threshold days (nullable integer), attendance threshold type (days/percentage, nullable), auto trigger flag (default false), auto trigger month (nullable integer 1-12), active flag (default true), description (nullable text), sort order (integer), and standard timestamps. Unique constraint on company and code.

### What Models

- Create BonusType model in the Bonus domain with ULID support, factory, and audit trail integration.
- Relationships: belongs to Company, has many PayrollRunBonuses.
- Scopes: active bonus types, system bonus types, custom bonus types, auto-triggerable for a given month.
- Helpers: check if deletable (not system and not referenced by any payroll run bonus).
- Factory with states for system, custom, fixed, percentage, formula, and auto-trigger.

### What Actions

- Create Bonus Type action: creates a new custom bonus type with auto-generated code from name (uppercase, underscores), system flag set to false, and validated uniqueness of code within the company.
- Update Bonus Type action: for system bonuses, only allows updating default amount, default rate, percentage basis, attendance config, description, active status, and sort order. For custom bonuses, allows updating all mutable fields.
- Delete Bonus Type action: validates the bonus type is not a system bonus and has no payroll run bonus references. Hard deletes the record.

### Validation & DTOs

- Store Bonus Type request: validates name (required), code (required, unique within company), computation type (required, must be fixed/percentage/formula), and conditional fields based on computation type.
- Update Bonus Type request: for system bonuses, rejects changes to code, name, computation type, and system flag.
- Bonus Type DTO using spatie/laravel-data.

### Routes & Controllers

- `GET /settings/bonuses` — paginated list with usage counts.
- `POST /settings/bonuses` — create new custom bonus type.
- `PUT /settings/bonuses/{bonusType}` — update bonus type.
- `DELETE /settings/bonuses/{bonusType}` — delete custom bonus type (Owner only).

### Policies & Authorization

- Owner and Payroll Officer can view and manage bonus types.
- Only Owner can delete custom bonus types.
- System bonuses cannot be deleted.
- Finance and Employee roles cannot access bonus settings.

### Commands

- Bonus Type seeder: seeds 4 system bonus types (Meal Allowance, Attendance Bonus, 13th Month Pay, June Special Bonus) with correct computation types, auto-trigger settings, and trigger months.

### Tests

- Assert Owner can create a custom bonus type.
- Assert Payroll Officer can create a custom bonus type.
- Assert Finance and Employee roles cannot create bonus types (403).
- Assert system bonus code cannot be changed via update.

## Acceptance Criteria

### Client-side

#### Layout

- The Bonus Types page is accessible from the Settings section in the sidebar for Owner and Payroll Officer roles

#### Behavior

- The bonus type list displays: Name, Code, Type (computation type), Auto Trigger (badge), Active (toggle), System (badge), and Actions
- All four system bonus types are visible with a "System" badge
- System bonuses allow editing default amount, default rate, attendance config, and active status, but not code or computation type
- System bonuses have no delete button
- Custom bonus types can be fully edited (name, code, computation type, amount/rate, auto-trigger settings)
- A "Create Bonus Type" button opens a form with: Name, Code (auto-generated from name, editable), Computation Type (dropdown), Default Amount (shown for fixed type), Default Rate (shown for percentage type), Percentage Basis (shown for percentage type), Auto Trigger (checkbox), Auto Trigger Month (shown when auto-trigger is enabled), Description, and Active status
- Custom bonuses can only be deleted by the Owner role
- The delete button is disabled with a tooltip if the bonus type has been used in any payroll run

#### Validation

- An error is shown if the code is duplicated within the company
- An error is shown if required fields are missing based on the computation type
- An error is shown when attempting to delete a system bonus or a bonus type that has been used

---

### Server-side

#### Side-effects

- Four system bonus types are seeded: Meal Allowance (fixed, every month), Attendance Bonus (percentage, manual), 13th Month Pay (formula, December), June Special Bonus (formula, June)
- System bonuses cannot be deleted and their code and computation type cannot be changed
- Custom bonus types support full CRUD operations
- Deletion is only permitted if the bonus type has never been used in any payroll run
- The code field is auto-generated from the name but remains editable before save
- Only Owner and Payroll Officer roles can manage bonus types; only Owner can delete
- All changes are recorded in the audit trail
