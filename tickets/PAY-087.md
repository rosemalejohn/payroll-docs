# [GW] Payroll - Dashboard Page with Payroll Overview

## Description

As a user, I want to see a role-specific dashboard upon login that gives me an overview of relevant payroll information and quick actions so that I can efficiently navigate to my most important tasks.

## Technical Implementation

### Controller
- **`DashboardController@index`** — `GET /dashboard`
- Middleware: `auth`
- Returns role-specific data based on authenticated user's role:

### Role-Specific Data

**Owner:**
- Recent payroll runs (last 5) with state badges
- Total payroll cost for current month (from latest finalized run)
- Pending actions count: runs awaiting finalization (`under_review` state)
- Total active employee count
- Quick actions: "Create New Run", "Review Pending", "Manage Employees"

**Payroll Officer:**
- Current draft runs (editable)
- Submitted runs awaiting review
- Recent activity (last 5 audit entries for payroll runs)
- Quick actions: "Create New Run", "View Draft Runs"

**Finance:**
- Runs awaiting review (`submitted` and `under_review` states) with count
- Recently finalized runs (last 5)
- Quick actions: "Review Queue", "Export Reports"

**Employee:**
- Latest payslip summary: period, gross pay, total deductions, net pay
- Year-to-date earnings total (sum of net_pay from all finalized payslips in current year)
- Quick actions: "View All Payslips", "Download Latest"

### Frontend
- **`resources/js/pages/dashboard/index.tsx`**
- Uses role from shared props to conditionally render sections
- Card-based layout using Shadcn Card components
- Stat cards for key numbers (employee count, pending reviews, etc.)
- Recent runs table (compact, max 5 rows)
- Quick action buttons linking to relevant pages

### Queries
- **`app/Payroll/Queries/DashboardStatsQuery.php`** — aggregates dashboard data efficiently
- Use `DB::raw()` for aggregate queries, eager load only needed relationships
- Cache expensive aggregates for 5 minutes (Redis)

### Tests
- `tests/Feature/DashboardTest.php`
  - Owner sees pending finalization count
  - Payroll officer sees draft runs
  - Finance sees review queue
  - Employee sees latest payslip and YTD
  - Employee does not see admin-level data
  - Dashboard loads within acceptable time

## Acceptance Criteria

- [ ] Dashboard renders role-specific content based on authenticated user
- [ ] Owner sees pending actions, recent runs, employee count
- [ ] Payroll officer sees draft and submitted runs
- [ ] Finance sees runs awaiting review
- [ ] Employee sees latest payslip summary and YTD earnings
- [ ] Quick action links navigate to correct pages
- [ ] Data displays correctly and matches actual database state
- [ ] Page loads efficiently (no N+1 queries, aggregates cached)
- [ ] Employee cannot see admin-level data (run management, employee list)
