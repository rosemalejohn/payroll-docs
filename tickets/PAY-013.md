# PAY-013: Create Payroll Runs Migration and Model with State Machine

## Description

Create the `payroll_runs` table, Eloquent model, factory, and state machine integration using `spatie/laravel-model-states`. A payroll run represents a single payroll processing cycle tied to a payroll period. It tracks aggregate totals (gross, deductions, net) and employee count, and progresses through an irreversible state machine: `draft` -> `submitted` -> `under_review` -> `finalized`. Each state transition records the actor and timestamp. All transitions are audited via `owen-it/laravel-auditing`.

- Configure state machine in `PayrollRunState`:
  ```php
  namespace App\Payroll\States;

  use Spatie\ModelStates\State;
  use Spatie\ModelStates\StateConfig;

  abstract class PayrollRunState extends State
  {
      public static function config(): StateConfig
      {
          return parent::config()
              ->default(Draft::class)
              ->allowTransition(Draft::class, Submitted::class)
              ->allowTransition(Submitted::class, UnderReview::class)
              ->allowTransition(UnderReview::class, Finalized::class);
      }
  }
  ```

- Create `app/Payroll/Models/PayrollRun.php`:
  ```php
  namespace App\Payroll\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;
  use OwenIt\Auditing\Contracts\Auditable;
  use OwenIt\Auditing\Auditable as AuditableTrait;
  use Spatie\ModelStates\HasStates;
  use App\Payroll\States\PayrollRunState;

  class PayrollRun extends Model implements Auditable
  {
      use HasUlids, HasFactory, HasStates, AuditableTrait;

      protected $fillable = [
          'company_id', 'payroll_period_id', 'name', 'state',
          'base_currency_id', 'rates_snapshotted_at',
          'total_gross', 'total_deductions', 'total_net', 'employee_count',
          'submitted_by', 'submitted_at',
          'reviewed_by', 'reviewed_at',
          'finalized_by', 'finalized_at',
          'notes', 'created_by',
      ];

      protected $casts = [
          'state' => PayrollRunState::class,
          'rates_snapshotted_at' => 'datetime',
          'total_gross' => 'decimal:2',
          'total_deductions' => 'decimal:2',
          'total_net' => 'decimal:2',
          'employee_count' => 'integer',
          'submitted_at' => 'datetime',
          'reviewed_at' => 'datetime',
          'finalized_at' => 'datetime',
      ];

      // Relationships
      public function company()
      {
          return $this->belongsTo(\App\Company\Models\Company::class);
      }

      public function payrollPeriod()
      {
          return $this->belongsTo(\App\Payroll\Models\PayrollPeriod::class);
      }

      public function baseCurrency()
      {
          return $this->belongsTo(\App\Currency\Models\Currency::class, 'base_currency_id');
      }

      public function payslips()
      {
          return $this->hasMany(\App\Payroll\Models\Payslip::class);
      }

      public function bonuses()
      {
          return $this->hasMany(\App\Bonus\Models\PayrollRunBonus::class);
      }

      public function currencySnapshots()
      {
          return $this->hasMany(\App\Currency\Models\PayrollRunCurrencySnapshot::class);
      }

      public function submitter()
      {
          return $this->belongsTo(\App\Models\User::class, 'submitted_by');
      }

      public function reviewer()
      {
          return $this->belongsTo(\App\Models\User::class, 'reviewed_by');
      }

      public function finalizer()
      {
          return $this->belongsTo(\App\Models\User::class, 'finalized_by');
      }

      public function creator()
      {
          return $this->belongsTo(\App\Models\User::class, 'created_by');
      }

      // Scopes
      public function scopeByCompany($query, string $companyId)
      {
          return $query->where('company_id', $companyId);
      }

      public function scopeDraft($query)
      {
          return $query->whereState('state', \App\Payroll\States\Draft::class);
      }

      public function scopeFinalized($query)
      {
          return $query->whereState('state', \App\Payroll\States\Finalized::class);
      }

      // Helpers
      public function isDraft(): bool
      {
          return $this->state->equals(\App\Payroll\States\Draft::class);
      }

      public function isFinalized(): bool
      {
          return $this->state->equals(\App\Payroll\States\Finalized::class);
      }
  }
  ```

### Factories
- Create `database/factories/PayrollRunFactory.php`:
  ```php
  namespace Database\Factories;

  class PayrollRunFactory extends Factory
  {
      protected $model = \App\Payroll\Models\PayrollRun::class;

      public function definition(): array
      {
          return [
              'company_id' => \App\Company\Models\Company::factory(),
              'payroll_period_id' => \App\Payroll\Models\PayrollPeriod::factory(),
              'name' => 'Payroll Run ' . fake()->monthName() . ' ' . fake()->year(),
              'state' => 'draft',
              'total_gross' => 0,
              'total_deductions' => 0,
              'total_net' => 0,
              'employee_count' => 0,
          ];
      }

      public function submitted(): static
      {
          return $this->state([
              'state' => 'submitted',
              'submitted_at' => now(),
          ]);
      }

      public function underReview(): static
      {
          return $this->state([
              'state' => 'under_review',
              'submitted_at' => now()->subDay(),
              'reviewed_at' => now(),
          ]);
      }

      public function finalized(): static
      {
          return $this->state([
              'state' => 'finalized',
              'submitted_at' => now()->subDays(3),
              'reviewed_at' => now()->subDay(),
              'finalized_at' => now(),
          ]);
      }
  }
  ```

### Edge Cases & Notes
- The state machine is **irreversible** by design. Once a run reaches `finalized`, it cannot be modified. This is critical for payroll compliance.
- Transition classes should record the actor (`submitted_by`, `reviewed_by`, `finalized_by`) and timestamp during each transition.
- The `rates_snapshotted_at` timestamp records when currency rates were captured for this run. It is set during run creation/computation, not during state transitions.
- Aggregate fields (`total_gross`, `total_deductions`, `total_net`, `employee_count`) are updated after payslip computation, not during creation.
