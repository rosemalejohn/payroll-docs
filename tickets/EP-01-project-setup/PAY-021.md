# [GW] Payroll - System Data Bootstrapping

## Description

- Create salary components table and initial seeders.
- Bootstrap application with essential system data.
- Include company, roles, base PHP currency.
- Add 11 common system salary components.
- Add 4 built-in system bonus types.
- Master seeder orchestrates all data correctly.

## Technical Implementation

### What Migrations

- Create salary components table with ULID.
- Add type, taxability, and system flags.
- Add composite unique constraint on codes.

### What Models

- Create SalaryComponent model in Compensation domain.
- Add filtration scopes for active types.
- Provide helper checking if component deletable.
- Provide factory state for diverse types.

### Tests

- Assert seeder runs purely without errors.
- Assert exactly one seeded company exists.
- Assert 4 core system roles exist.
- Assert base PHP currency is accurate.
- Assert 11 salary components are seeded.
- Assert 4 system bonus types exist.
- Assert admin user has owner role.
- Assert seeder is idempotent and safe.
- Assert system components cannot be deleted.
- Assert duplicate component codes are prevented.

### Edge Cases & Notes

- Seeders must be inherently idempotent.
- ULIDs are dynamically generated at runtime.
- System components logically appear first everywhere.

## Acceptance Criteria

### Server-side

#### Side-effects

- Master seeder securely populates system data.
- Initial system has 1 company, 4 roles.
- System seeds 11 salary components perfectly.
- Admin user successfully logs in initially.
- Salary components have proper taxability logic.
- System-provided components deny manual user deletion.
- Running seeder again creates zero duplicates.
