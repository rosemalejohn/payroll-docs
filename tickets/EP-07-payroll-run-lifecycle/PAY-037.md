# [GW] Payroll - Payroll Period CRUD

## Description

As a payroll officer, I want to create and manage payroll periods (e.g., "March 2026") that define the time window and working days for payroll runs.

## Technical Implementation

### Controller
- **`PayrollPeriodController`** — `app/Http/Controllers/Web/PayrollPeriodController.php`
  - `index`: paginated list with filters
  - `create`: create form
  - `store`: create period
  - `edit`: edit form (only unlocked periods)
  - `update`: update period

### Routes
- Standard resource routes: `GET/POST /payroll-periods`, `GET/PUT /payroll-periods/{period}`, `GET /payroll-periods/create`, `GET /payroll-periods/{period}/edit`

### Actions
- **`app/Payroll/Actions/CreatePayrollPeriodAction.php`**
- **`app/Payroll/Actions/UpdatePayrollPeriodAction.php`** — blocked if `is_locked = 1`

### Form Request
- **`StorePayrollPeriodRequest`**
  - `name`: required, string, max:150
  - `period_type`: required, in:monthly,semi_monthly,weekly
  - `start_date`: required, date
  - `end_date`: required, date, after:start_date
  - `working_days`: required, integer, min:1, max:31

### Frontend
- **`resources/js/pages/payroll-periods/index.tsx`**: list with name, type, date range, working days, locked badge
- **`resources/js/pages/payroll-periods/create.tsx`**: form with date pickers
- **`resources/js/pages/payroll-periods/edit.tsx`**: edit form (disabled fields if locked)
- Locked periods show lock icon and edit button is disabled

### Tests
- Period created with valid data
- Validation prevents start_date >= end_date
- Locked periods cannot be edited
- Working days defaults to 22

## Acceptance Criteria

- [ ] Payroll period can be created with name, type, date range, and working days
- [ ] Validation prevents `start_date >= end_date`
- [ ] `working_days` defaults to 22, editable before locking
- [ ] Locked periods (`is_locked = 1`) cannot be edited
- [ ] Period list shows locked status with visual indicator
- [ ] Payroll officer and owner can manage periods
