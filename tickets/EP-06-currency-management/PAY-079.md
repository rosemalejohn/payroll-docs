# [GW] Payroll - Missing Currency Rate Detection

## Description

During payroll computation, detect when an employee's salary currency does not have a snapshot rate in the current payroll run. Affected payslips are flagged with a "Missing Rate" indicator. The system blocks the transition from submitted to under review until all currency rates are configured. Once the missing rate is added, affected payslips are automatically recomputed.

## Technical Implementation

### What Actions

- Detect Missing Currency Rates action: cross-references employees in the run with the currency snapshot table to find currencies without rates.
- Enhanced Compute Payslip action: if the employee's salary currency has no snapshot rate, the payslip is flagged as having a missing rate with null monetary values.
- Transition to Under Review action: checks for missing rates and blocks the transition if any are found.
- Resolve Missing Currency Rate action: creates or updates the snapshot for the missing currency, recomputes all affected payslips, and clears the missing rate flag.

### What Migrations

- Add a missing rate flag (boolean) and missing rate currency reference (nullable) to the payslips table.

### Routes & Controllers

- Enhanced payroll run show route: includes missing rate summary (affected currencies and employee count) in page props.
- `POST /payroll-runs/{payrollRun}/resolve-missing-rate` — resolve a missing rate by adding the snapshot (Owner only).

### Tests

- Assert payslips for employees with missing currency rates are flagged with null monetary values.
- Assert transition from submitted to under review is blocked when missing rates exist.
- Assert validation error lists affected currencies and employee count.
- Assert adding the missing rate resolves the flag and recomputes the payslip with correct values.
- Assert PHP currency employees are never flagged.
- Assert multiple missing currencies can be handled simultaneously.
- Assert all missing rates must be resolved before the transition is allowed.

### Edge Cases & Notes

- PHP (base currency) always has rate 1.0 and should never be flagged as missing.
- Multiple currencies may be missing simultaneously — all must be resolved.
- When resolving, only payslips for the specific resolved currency are recomputed, not all payslips.

## Acceptance Criteria

### Client-side

#### Behavior

- Warning banner lists missing currencies and employee count
- Banner links to Currency Settings for Owner
- "Missing Rate" badge on affected employees, amounts show "—"
- "Under Review" button disabled until rates resolved

#### Validation

- Transition error lists affected currencies and employee counts

---

### Server-side

#### Side-effects

- Missing currency payslips flagged with null values
- Submitted to under review blocked until resolved
- Adding missing rate auto-recomputes affected payslips
- PHP employees never affected
