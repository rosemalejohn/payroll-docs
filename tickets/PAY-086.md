# [GW] Payroll - Dual Currency Payslip Display for Foreign Salary Employees

## Description

For employees whose salary is denominated in a foreign currency (non-PHP), display dual currency amounts on the payslip detail page and generated PDF. The payslip shows the original salary in the employee's currency alongside the PHP equivalent, the exchange rate used with its snapshot date, and all amounts with PHP as the primary display currency. Deductions are shown in PHP with a note that they are computed on the PHP-converted gross. Employees paid in PHP see the standard single-currency payslip with no conversion section.

## Technical Implementation

### Queries
- **`app/Payroll/Queries/PayslipWithCurrencyQuery.php`** — Query class that:
  - Loads the payslip with its payroll run's currency snapshot
  - Joins `payroll_run_currency_snapshots` to get the exchange rate for the employee's salary currency
  - Joins `employees` to get the `salary_currency` field
  - Returns the payslip data enriched with currency conversion context

### Routes & Controllers
- **`GET /payroll-runs/{payrollRun}/payslips/{payslip}`** — Existing route
- **`app/Http/Controllers/Web/PayslipController@show`** — Update to:
  - Detect if the employee has a non-PHP salary currency
  - If foreign currency: include currency snapshot data in Inertia page props
    - `original_currency`: currency code (e.g., "USD")
    - `original_salary_amount`: salary in original currency
    - `exchange_rate`: rate used for conversion
    - `snapshot_date`: date the rate was captured
    - `php_equivalent`: converted amount in PHP
  - If PHP-only: omit currency conversion props entirely

### API Resources
- **`app/Http/Resources/Payroll/PayslipDetailResource.php`** — Update to conditionally include:
  - `currency_conversion` object (only for foreign currency employees):
    - `original_currency`: string (e.g., "USD")
    - `original_salary`: string (decimal in original currency)
    - `exchange_rate`: string (decimal, e.g., "60.41")
    - `snapshot_date`: string (date, e.g., "2026-03-01")
    - `rate_display`: string (formatted, e.g., "1 USD = ₱60.41 (as of March 1, 2026)")
  - For PHP employees: `currency_conversion` is `null`

### Frontend (Inertia/React)
- **`resources/js/pages/payslips/show.tsx`** — Update payslip detail page:
  - If `currency_conversion` is present:
    - Display salary line: "₱XXX,XXX.XX (USD X,XXX.XX)" with PHP amount primary, foreign amount secondary
    - Exchange rate banner: "1 USD = ₱60.41 (as of March 1, 2026)"
    - Net pay: primary in PHP, secondary in original currency below
    - Deductions section: all amounts in PHP with note "Computed on PHP-converted gross"
  - If `currency_conversion` is null: render standard single-currency payslip
- **`resources/js/components/payslip/currency-conversion-banner.tsx`** — Reusable component:
  - Displays exchange rate and snapshot date
  - Styled as an informational banner within the payslip layout
- **`resources/js/components/payslip/dual-amount.tsx`** — Reusable component:
  - Displays primary PHP amount and secondary foreign currency amount
  - Accepts `phpAmount`, `foreignAmount`, `currencyCode` props

### PDF Generation
- **`app/Payroll/Services/PayslipPdfGenerator.php`** — Update PDF template to:
  - Include currency conversion section for foreign salary employees
  - Show exchange rate and snapshot date
  - Display dual amounts for salary and net pay
  - Add footnote for deductions: "All deductions computed on PHP-converted gross salary"
  - PHP-only employees: standard PDF layout unchanged

### Tests
- **`tests/Feature/Payroll/DualCurrencyPayslipTest.php`**
  - Foreign currency employee payslip shows original currency and PHP equivalent
  - Exchange rate and snapshot date are displayed correctly
  - Net pay is shown primarily in PHP, secondarily in original currency
  - Deductions are shown in PHP with "Computed on PHP-converted gross" note
  - PHP-only employee payslip does not show any conversion section
  - Currency conversion uses the snapshot rate from the payroll run (not live rate)
  - PDF includes dual currency information for foreign salary employees
  - PDF for PHP-only employees has standard layout without conversion section
  - Conversion math is accurate: `original_amount * exchange_rate = php_amount`

### Edge Cases & Notes
- Exchange rates are always sourced from `payroll_run_currency_snapshots`, never from live rates — this ensures payslip amounts match the rates locked at run creation
- The rate format is "1 [FOREIGN] = X PHP" (e.g., "1 USD = ₱60.41"), consistent with the currency system defined in `docs/04-currency.md`
- Reverse conversion for display (PHP to foreign) uses the inverse of the snapshot rate
- If a currency snapshot is missing for an employee's salary currency (data integrity issue), the controller should log a warning and display PHP amounts only with a note indicating the rate was unavailable

## Acceptance Criteria

- [ ] Foreign currency employees see dual amounts on payslip: original currency and PHP equivalent
- [ ] Exchange rate and snapshot date are displayed (e.g., "1 USD = ₱60.41 (as of March 1, 2026)")
- [ ] PHP is the primary display currency for all amounts
- [ ] Deductions are shown in PHP with a note: "Computed on PHP-converted gross"
- [ ] PHP-only employees see the standard payslip view with no conversion section
- [ ] Conversion uses the snapshot rate from the payroll run, not live rates
- [ ] PDF output includes dual currency information for foreign salary employees
