# [GW] Payroll - Authenticated Layout

## Description

- Create primary authenticated layout shell.
- Wrap all post-login application pages seamlessly.
- Include collapsible sidebar with role navigation.
- Include top header with user information.
- Provide flash message display toast system.
- Ensure complete visual mobile responsiveness.
- Create Inertia middleware sharing cross-cutting data.
- Pass authenticated user, roles, and permissions.
- Share unread system notification count reliably.

## Technical Implementation

### Routes & Controllers

- Apply middleware globally to web group.

### Edge Cases & Notes

- Sidebar filtering remains strictly UX-only.
- Route-level middleware enforces actual access control.
- Flash messages integrate with Inertia middleware.
- Layout avoids slow full page reloads.
- Sidebar state persists using local storage.
- Keep shared props data exceptionally lightweight.

## Acceptance Criteria

### Client-side

#### Layout

- Layout features sidebar, header, main area.

#### Positioning

- Sidebar groups and displays navigation links.
- Header displays name, user info, logout.
- Flash messages appear top-right as toasts.

#### Screen-size Friendliness

- Desktop sidebar collapses via toggle button.
- Mobile sidebar collapses into slide-out drawer.

#### Behavior

- Navigation filters via assigned user role.
- Owner sees all navigation links available.
- Officer sees dashboard, employees, runs, bonuses.
- Active page link displays prominent highlight.
- Navigation avoids full DOM page reloads.
- Sidebar collapsed state persists between routes.
- Logout signs out user to login.
- Pages display logged-in user name successfully.
- Flash messages notify after server actions.
- Notification bell shows unread message count.

### Server-side

#### Side-effects

- Authed requests attach basic user roles.
- Unauthenticated requests deny private user payloads.
- Passwords are completely restricted from frontend.
