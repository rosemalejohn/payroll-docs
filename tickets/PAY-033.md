# PAY-033: Currency Management Page (Settings)

## Description

As an owner, I want to view and manage currencies in the Settings section so that I can configure exchange rates for employees paid in foreign currencies. Non-owner roles can view currencies but not modify them.

## Technical Implementation

### Controller
- **`CurrencyController@index`** — `GET /settings/currencies`
- Middleware: `auth`
- Returns all currencies for the company with rate, last updated, active status

### Policy
- **`CurrencyPolicy`**
  - `viewAny`: all authenticated users
  - `create`, `update`, `toggleActive`: owner only

### Frontend
- **`resources/js/pages/settings/currencies/index.tsx`**
- Table columns: code, name, symbol, rate (formatted with 6 decimal places), last updated (relative time), active status badge
- Owner actions: Edit rate button, Add currency button, Toggle active button
- Non-owner view: read-only table (no action buttons)
- PHP base currency row: no edit/toggle buttons (immutable)

### Inertia Props
- `currencies`: paginated list of currencies
- `canManage`: boolean (true if owner)

### Tests
- All roles can view currency list
- Owner sees action buttons
- Non-owner sees read-only view
- PHP currency has no action buttons

## Acceptance Criteria

- [ ] All active currencies listed with rates and last updated timestamp
- [ ] Owner sees edit, add, and toggle buttons
- [ ] Non-owner roles see read-only currency list
- [ ] PHP base currency row has no edit or toggle buttons
- [ ] Rate displayed with 6 decimal places
- [ ] Last updated shows relative time (e.g., "2 days ago")
