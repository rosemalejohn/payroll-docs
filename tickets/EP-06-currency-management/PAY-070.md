# [GW] Payroll - Currency Conversion Details in Run View

## Description

Add a "Currency Rates" tab to the payroll run detail page that displays all foreign currency exchange rates snapshotted for the run, along with a list of employees whose salaries are in foreign currencies. The tab shows each currency's snapshot rate and date, and for each foreign currency employee: their name, salary currency, original salary amount, exchange rate used, and PHP equivalent.

For finalized runs, the snapshot rates are historical and immutable. For draft runs where the current rate differs from the snapshot rate, both rates are displayed with a note highlighting the discrepancy.

## Technical Implementation

### Routes & Controllers
- **`GET /payroll-runs/{payrollRun}/currency-rates`** — Route name: `payroll-runs.currency-rates`
- **`app/Http/Controllers/Web/PayrollRunCurrencyController@index`**
  - Middleware: `auth`, `role:owner,payroll_officer,finance`
  - Authorizes via `PayrollRunPolicy@view`
  - Loads currency snapshots for the run and employees with foreign currency salaries
  - Also loads current exchange rates for comparison (draft/submitted runs only)
  - Returns Inertia response (can be a partial reload for tab switching)

### Queries
- **`app/Payroll/Queries/PayrollRunCurrencyQuery.php`** — Query class:
  - Loads all records from `payroll_run_currency_snapshots` for the given run
  - Joins with the currencies table to get currency names and codes
  - Loads employees in the run with `salary_currency != 'PHP'`
  - For each foreign currency employee, retrieves:
    - Employee name and number
    - Salary currency code
    - Original salary amount (in foreign currency)
    - Snapshot exchange rate
    - PHP equivalent (original_amount * exchange_rate)
  - For non-finalized runs: also loads current exchange rates from the `currencies` table for comparison

### API Resources
- **`app/Http/Resources/Payroll/CurrencySnapshotResource.php`** — Shapes currency snapshot data:
  - `currency_code`: string (e.g., "USD")
  - `currency_name`: string (e.g., "US Dollar")
  - `snapshot_rate`: decimal (e.g., 56.000000)
  - `snapshot_date`: formatted date
  - `current_rate`: decimal or null (only for non-finalized runs)
  - `rate_changed`: boolean (true if current_rate differs from snapshot_rate)
- **`app/Http/Resources/Payroll/ForeignCurrencyEmployeeResource.php`** — Shapes employee data:
  - `employee_name`: string
  - `employee_number`: string
  - `salary_currency`: string
  - `original_amount`: decimal
  - `exchange_rate`: decimal (snapshot rate)
  - `php_equivalent`: decimal

### Policies & Authorization
- Uses existing `PayrollRunPolicy@view` — accessible to `owner`, `payroll_officer`, and `finance` roles

### Frontend Components
- **`resources/js/pages/payroll-runs/show.tsx`** — Add "Currency Rates" tab to the existing payroll run detail page:
  - Tab is visible only if the run has foreign currency employees or currency snapshots
  - Uses partial Inertia reload when switching to this tab
- **`resources/js/components/payroll/currency-rates-tab.tsx`** — Tab content component:
  - **Snapshot Rates Section**:
    - Table columns: Currency Code, Currency Name, Snapshot Rate, Snapshot Date
    - For non-finalized runs with rate changes: additional "Current Rate" column and a warning badge/note on rows where `rate_changed` is true (e.g., "Current rate differs: 57.50")
    - For finalized runs: note stating "These rates are historical and were locked at finalization"
  - **Foreign Currency Employees Section**:
    - Table columns: Employee Name, Employee Number, Salary Currency, Original Amount, Exchange Rate, PHP Equivalent
    - Sorted by currency, then by employee name
- **`resources/js/components/payroll/rate-change-badge.tsx`** — Small badge/tooltip component:
  - Shows when snapshot rate differs from current rate
  - Displays both rates with a visual indicator (e.g., arrow up/down)

### Tests
- **`tests/Feature/Payroll/CurrencyConversionDetailsTest.php`**
  - Owner, payroll_officer, and finance can view currency rates tab
  - Employee role receives 403
  - Snapshot rates are displayed for each currency used in the run
  - Foreign currency employees are listed with correct conversion details
  - PHP equivalent is correctly calculated (original_amount * exchange_rate)
  - For finalized runs, rates are marked as historical and immutable
  - For draft runs, current rates are shown alongside snapshot rates
  - Rate change indicator appears when current rate differs from snapshot rate
  - Runs with only PHP employees show an empty state or note
  - Currency rates tab is not visible for runs with no foreign currency data
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- The `payroll_run_currency_snapshots` table stores rates at run creation time; these never change for the run
- PHP (Philippine Peso) is always rate 1.0 and should not appear in the snapshot list (it is the base currency)
- If a currency rate was updated multiple times between run creation and finalization, only the snapshot rate and the current rate matter — intermediate changes are irrelevant
- For finalized runs, the "current rate" comparison is informational only and does not affect any calculations
- Ensure exchange rates display with appropriate precision (6 decimal places as per the schema)
- Empty state: if no foreign currencies are used in the run, the tab can either be hidden or show "All employees in this run are paid in PHP"
