# PAY-043: Absent Days Entry Per Employee Per Run

## Description

Provide an inline-editable absent days field on each employee row within a draft payroll run's detail view. Updating absent days triggers payslip recomputation so that absent deductions and net pay reflect the change immediately. An optional note field captures the reason for the absence.

## Technical Implementation

### Action

**`app/Payroll/Actions/UpdatePayslipAbsentDaysAction.php`**

- Accepts `Payslip` model, `absent_days` (decimal, 0-2 decimal places), and optional `absent_note` (string|null).
- Validates the parent `PayrollRun` is in `draft` state; throws `InvalidRunStateException` otherwise.
- Validates `absent_days >= 0`.
- Validates `absent_days <= payroll_run.payroll_period.working_days`.
- Updates `payslips.absent_days` and `payslips.absent_note`.
- Calls `ComputePayslipAction` to recompute the individual payslip.
- Updates the parent run totals via `RecomputePayrollRunAction` (or a lightweight total-recalculation helper).

### Controller

Add method on `PayrollRunController` or dedicated `PayslipController`:

- `PATCH /payroll-runs/{payrollRun}/payslips/{payslip}/absent-days`
  - Request body: `{ absent_days: number, absent_note?: string }`

### Form Request: `UpdateAbsentDaysRequest`

```php
public function rules(): array
{
    return [
        'absent_days' => ['required', 'numeric', 'min:0', 'max:' . $this->payslip->payrollRun->payrollPeriod->working_days],
        'absent_note' => ['nullable', 'string', 'max:500'],
    ];
}
```

### Frontend (Inertia/React)

- Each employee row in the payroll run detail table has an inline-editable `absent_days` input field (number input, step=0.5 to support half-days).
- On blur or Enter key, the value is submitted via `router.patch()`.
- An optional expandable note field (textarea) is shown on click/icon toggle.
- Validation feedback shown inline: red border and error text if value exceeds working days or is negative.
- After successful update, the row's computed values (absent deduction, net pay) update to reflect recomputation.
- Fields are read-only (non-editable) when the run is not in `draft` state.

### DTO

```php
// app/Payroll/DTOs/UpdateAbsentDaysData.php
class UpdateAbsentDaysData extends Data {
    public float $absent_days;
    public ?string $absent_note;
}
```

## Acceptance Criteria

- [ ] Absent days are editable inline on each employee row when the run is in `draft` state.
- [ ] Absent days field is read-only on non-draft runs.
- [ ] Validation prevents negative values (returns error).
- [ ] Validation prevents values exceeding the payroll period's `working_days` (returns error).
- [ ] Half-day increments (e.g., 0.5, 1.5) are supported.
- [ ] Changing absent days triggers automatic payslip recomputation.
- [ ] Recomputation updates the absent deduction and net pay on the payslip record.
- [ ] The optional absent note is saved on the payslip record.
- [ ] Updated values are persisted and reflected on page reload.
- [ ] Parent run totals (`total_deductions`, `total_net`) are recalculated after the update.
