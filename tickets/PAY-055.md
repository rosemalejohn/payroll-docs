# [GW] Payroll - Cash Advance CRUD

## Description

Implement the CRUD interface for managing employee cash advances. Cash advances track money issued to employees that is later recovered through payroll deductions. The payroll officer and owner roles can create, view, and update cash advances. Each cash advance has an amount, a tracked recovery balance, and a status that transitions from `active` to `fully_recovered` (automatically when balance reaches zero) or `cancelled` (manually by authorized users).

Cash advances can be viewed per employee or as a global list filterable by employee and status.

## Technical Implementation

### Controller

**`app/Http/Controllers/Web/CashAdvanceController.php`**

- `index(Request $request)` — Returns Inertia page `cash-advances/index` with paginated list of all cash advances. Supports filters: `employee_id`, `status` (active, fully_recovered, cancelled), `date_from`, `date_to`. Props include employee list for the filter dropdown.
- `store(StoreCashAdvanceRequest $request)` — Creates a cash advance via `CreateCashAdvanceAction`. Redirects with flash message.
- `update(UpdateCashAdvanceRequest $request, CashAdvance $cashAdvance)` — Updates editable fields (notes, status for cancellation) via `UpdateCashAdvanceAction`. Only `active` advances can be updated.
- `show(CashAdvance $cashAdvance)` — Returns Inertia page with cash advance detail including recovery history (from audit trail).

### Routes

```
GET    /cash-advances                  -> CashAdvanceController@index
POST   /cash-advances                  -> CashAdvanceController@store
GET    /cash-advances/{cashAdvance}     -> CashAdvanceController@show
PUT    /cash-advances/{cashAdvance}     -> CashAdvanceController@update
```

Middleware: `auth`, `role:owner|payroll_officer`.

### Actions

**`app/CashAdvance/Actions/CreateCashAdvanceAction.php`**

- Accepts `CashAdvanceData` DTO.
- Sets `balance = amount` (no recovery yet).
- Sets `amount_recovered = 0.00`.
- Sets `status = 'active'`.
- Sets `created_by` to the authenticated user.
- Creates the `cash_advances` record.

**`app/CashAdvance/Actions/UpdateCashAdvanceAction.php`**

- Validates the cash advance is in `active` status (fully_recovered and cancelled cannot be edited).
- Allows updating: `notes`, `status` (only to `cancelled`), `amount` (only if `amount_recovered = 0`, i.e., no recovery has started).
- If status is changed to `cancelled`, sets balance to 0.
- If amount is changed, recalculates `balance = amount - amount_recovered`.

### Models

**`app/CashAdvance/Models/CashAdvance.php`**

- Implements `Auditable` (owen-it/laravel-auditing).
- Fillable: `company_id`, `employee_id`, `amount`, `amount_recovered`, `balance`, `issued_date`, `status`, `notes`, `created_by`.
- Casts: `amount` as `decimal:2`, `amount_recovered` as `decimal:2`, `balance` as `decimal:2`, `issued_date` as `date`, `status` as enum or string.
- Relationships: `belongsTo(Company::class)`, `belongsTo(Employee::class)`, `belongsTo(User::class, 'created_by')`.
- Scopes: `scopeActive($query)`, `scopeForEmployee($query, $employeeId)`.
- Accessor: `is_fully_recovered` — returns `balance <= 0`.

### DTOs

**`app/CashAdvance/DTOs/CashAdvanceData.php`** (spatie/laravel-data)

- `employee_id`: string, required
- `amount`: decimal, required, min:0.01
- `issued_date`: date, required
- `notes`: nullable string

### Form Requests

**`app/Http/Requests/CashAdvance/StoreCashAdvanceRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`.
- Rules: `employee_id` required|exists in employees, `amount` required|decimal|min:0.01, `issued_date` required|date, `notes` nullable|string|max:1000.

**`app/Http/Requests/CashAdvance/UpdateCashAdvanceRequest.php`**

- Authorization: user has role `owner` or `payroll_officer`.
- Rules: `notes` nullable|string|max:1000, `status` nullable|in:cancelled, `amount` nullable|decimal|min:0.01 (only if amount_recovered is 0).

### Policy

**`app/CashAdvance/Policies/CashAdvancePolicy.php`**

- `viewAny()`: owner, payroll_officer
- `view()`: owner, payroll_officer, or employee (own cash advances only)
- `create()`: owner, payroll_officer
- `update()`: owner, payroll_officer (only if status is `active`)
- `cancel()`: owner, payroll_officer (only if status is `active`)

### Queries

**`app/CashAdvance/Queries/CashAdvanceIndexQuery.php`**

- Handles filtering by `employee_id`, `status`, date range (`issued_date` between `date_from` and `date_to`).
- Eager loads `employee` (name, employee_number, department).
- Orders by `issued_date` descending.
- Paginates (default 20 per page).

### Frontend

**`resources/js/pages/cash-advances/index.tsx`**

- Table with columns: Employee Name, Employee Number, Amount, Recovered, Balance, Status (badge), Issued Date, Actions.
- Status badges: `active` (blue), `fully_recovered` (green), `cancelled` (gray).
- Filters: Employee select (searchable dropdown), Status select, Date range picker.
- "Create Cash Advance" button opens a modal with fields: Employee (searchable select), Amount (currency input), Issued Date (date picker), Notes (textarea).
- Edit action (only for active advances): opens modal to update notes or cancel.
- Balance column is auto-calculated: `amount - amount_recovered`.

**`resources/js/pages/cash-advances/show.tsx`**

- Detail view showing full cash advance info and audit history (recovery timeline from auditing records).

## Acceptance Criteria

- [ ] Cash advances can be created by owner and payroll_officer roles with employee, amount, issued date, and optional notes.
- [ ] Balance is automatically calculated as `amount - amount_recovered` and displayed correctly.
- [ ] New cash advances have `status = 'active'`, `amount_recovered = 0`, and `balance = amount`.
- [ ] The cash advance list supports filtering by employee, status, and date range.
- [ ] The list is paginated and shows employee name, amount, recovered, balance, status, and issued date.
- [ ] Active cash advances can have their notes updated and can be cancelled.
- [ ] Cancelled and fully recovered cash advances cannot be modified.
- [ ] Amount can only be changed on active advances where no recovery has occurred (`amount_recovered = 0`).
- [ ] Finance and employee roles cannot create or manage cash advances (403).
- [ ] All changes to cash advances are captured in the audit trail via `owen-it/laravel-auditing`.
