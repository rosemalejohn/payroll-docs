# [GW] Payroll - Create Salary Components Migration, Model, and Seeder

## Description

Create the `salary_components` table, Eloquent model, factory, and seeder. Salary components define reusable component types (earnings, deductions, benefits) used across the payroll system. Eleven system components are seeded by default and cannot be deleted. These system components map directly to the steps in the 10-step gross-to-net payroll computation pipeline.

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_salary_components_table.php`:
  ```php
  Schema::create('salary_components', function (Blueprint $table) {
      $table->char('id', 26)->primary();                    // ULID
      $table->char('company_id', 26);                        // ref: companies.id
      $table->string('name', 150);
      $table->string('code', 50);                            // e.g. BASIC_MONTHLY, SSS, BIR_TAX
      $table->enum('type', ['earning', 'deduction', 'benefit']);
      $table->boolean('is_taxable')->default(true);
      $table->boolean('is_system')->default(false);          // system components cannot be deleted
      $table->text('description')->nullable();
      $table->integer('sort_order')->default(0);
      $table->boolean('is_active')->default(true);
      $table->timestamps();

      // Composite unique constraint
      $table->unique(['company_id', 'code'], 'uq_salary_components_company_code');
      $table->index('company_id', 'idx_salary_components_company');
  });
  ```

### What Models
- Create `app/Compensation/Models/SalaryComponent.php`:
  ```php
  namespace App\Compensation\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;

  class SalaryComponent extends Model
  {
      use HasUlids, HasFactory;

      protected $fillable = [
          'company_id', 'name', 'code', 'type',
          'is_taxable', 'is_system', 'description',
          'sort_order', 'is_active',
      ];

      protected $casts = [
          'is_taxable' => 'boolean',
          'is_system' => 'boolean',
          'is_active' => 'boolean',
          'sort_order' => 'integer',
      ];

      // Relationships
      public function company()
      {
          return $this->belongsTo(\App\Company\Models\Company::class);
      }

      // Scopes
      public function scopeActive($query)
      {
          return $query->where('is_active', true);
      }

      public function scopeSystem($query)
      {
          return $query->where('is_system', true);
      }

      public function scopeCustom($query)
      {
          return $query->where('is_system', false);
      }

      public function scopeEarnings($query)
      {
          return $query->where('type', 'earning');
      }

      public function scopeDeductions($query)
      {
          return $query->where('type', 'deduction');
      }

      public function scopeBenefits($query)
      {
          return $query->where('type', 'benefit');
      }

      public function scopeByCompany($query, string $companyId)
      {
          return $query->where('company_id', $companyId);
      }

      public function scopeOrdered($query)
      {
          return $query->orderBy('sort_order')->orderBy('name');
      }

      // Helpers
      public function isSystem(): bool
      {
          return $this->is_system;
      }

      public function isDeletable(): bool
      {
          return !$this->is_system;
      }
  }
  ```

### Edge Cases & Notes
- System components (`is_system = true`) must not be deletable through the application. The `isDeletable()` method provides this check, and delete actions/controllers should verify this before proceeding.
- The `type` enum has three values: `earning`, `deduction`, `benefit`. All 11 system components are either `earning` or `deduction`. The `benefit` type exists for future custom components.
- `is_taxable` is `false` for all deduction-type system components because they are deducted before or alongside tax computation. The two earning components (`BASIC_MONTHLY`, `DAILY_RATE`) are taxable.
- `sort_order` determines display order in payslips and UI. System components use low numbers (1-21) to appear first; custom components should use higher numbers (100+).
- The unique constraint allows the same code to exist for different companies but not duplicated within one company.
- Running the seeder on an already-seeded database should be idempotent or handled gracefully (use `firstOrCreate` if preferred over catching unique constraint violations).
