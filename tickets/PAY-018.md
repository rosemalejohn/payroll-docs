# [GW] Payroll - Create Payroll Run Bonus Exclusions Migration and Model

## Description

Create the `payroll_run_bonus_exclusions` table and model to track which employees are excluded from receiving a specific bonus in a specific payroll run. Used when `apply_to_all = 0` on the parent `payroll_run_bonuses` record.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_payroll_run_bonus_exclusions_table.php`**
  ```sql
  id                    CHAR(26) PRIMARY KEY  -- ULID
  payroll_run_bonus_id  CHAR(26) NOT NULL     -- ref: payroll_run_bonuses.id
  employee_id           CHAR(26) NOT NULL     -- ref: employees.id
  reason                TEXT NULL
  timestamps
  UNIQUE KEY uq_exclusions_bonus_employee (payroll_run_bonus_id, employee_id)
  ```

### Model
- **`app/Bonus/Models/PayrollRunBonusExclusion.php`**
  - Uses `HasUlids` trait
  - Relationships: `belongsTo(PayrollRunBonus::class)`, `belongsTo(Employee::class)`
  - Fillable: `payroll_run_bonus_id`, `employee_id`, `reason`

### Usage in Computation
- During Step 2 (bonus application), check if employee has an exclusion record:
  ```php
  $isExcluded = PayrollRunBonusExclusion::where('payroll_run_bonus_id', $runBonus->id)
      ->where('employee_id', $employee->id)
      ->exists();
  ```

### Tests
- Unique constraint prevents duplicate exclusion for same bonus + employee
- Excluded employee does not receive the bonus during computation
- Reason is optional

## Acceptance Criteria

- [ ] Migration runs without errors
- [ ] One exclusion per bonus per employee enforced (unique constraint)
- [ ] Exclusion record prevents bonus application during payslip computation
- [ ] Reason field is optional (nullable)
- [ ] Relationships to `PayrollRunBonus` and `Employee` work correctly
