# [GW] Payroll - Create Employees Migration and Model

## Description

Create the `employees` table, Eloquent model, factory, and auditing configuration. The employees table is the local mirror of employee data synced from the HRIS application. It stores both HRIS-synced fields (overwritten on every sync) and payroll-specific fields (managed locally, never overwritten). This is a critical model as nearly every payroll computation depends on employee data.

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_employees_table.php`:
  ```php
  Schema::create('employees', function (Blueprint $table) {
      $table->char('id', 26)->primary();                    // ULID (local)
      $table->char('company_id', 26);                        // ref: companies.id
      $table->char('hris_employee_id', 26)->unique();        // ULID from HRIS (sync key)

      // Synced from HRIS (overwritten on every sync)
      $table->string('employee_number', 50);
      $table->string('first_name', 150);
      $table->string('last_name', 150);
      $table->string('middle_name', 150)->nullable();
      $table->string('email', 255)->nullable();
      $table->string('department', 150)->nullable();
      $table->string('position', 150)->nullable();
      $table->enum('employment_type', ['regular', 'contractual', 'probationary'])->default('regular');
      $table->enum('employment_status', ['active', 'inactive', 'on_leave', 'terminated'])->default('active');
      $table->date('date_hired')->nullable();

      // Payroll-specific (managed locally, never overwritten by sync)
      $table->enum('pay_basis', ['monthly', 'daily'])->default('monthly');
      $table->boolean('has_external_salary')->default(false);
      $table->boolean('has_mp2')->default(false);
      $table->boolean('has_sss_wisp')->default(false);
      $table->decimal('hmo_amount', 12, 2)->default(0.00);
      $table->string('sss_number', 50)->nullable();
      $table->string('philhealth_number', 50)->nullable();
      $table->string('pagibig_number', 50)->nullable();
      $table->string('bir_tin', 50)->nullable();

      $table->timestamp('last_synced_at')->nullable();
      $table->timestamps();

      // Indexes
      $table->index('company_id', 'idx_employees_company');
      $table->index('hris_employee_id', 'idx_employees_hris_id');
      $table->index('employment_status', 'idx_employees_status');
  });
  ```

### What Models
- Create `app/Employee/Models/Employee.php`:
  ```php
  namespace App\Employee\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;
  use OwenIt\Auditing\Contracts\Auditable;
  use OwenIt\Auditing\Auditable as AuditableTrait;

  class Employee extends Model implements Auditable
  {
      use HasUlids, HasFactory, AuditableTrait;

      protected $fillable = [
          'company_id', 'hris_employee_id', 'employee_number',
          'first_name', 'last_name', 'middle_name', 'email',
          'department', 'position', 'employment_type', 'employment_status',
          'date_hired', 'pay_basis', 'has_external_salary', 'has_mp2',
          'has_sss_wisp', 'hmo_amount', 'sss_number', 'philhealth_number',
          'pagibig_number', 'bir_tin', 'last_synced_at',
      ];

      protected $casts = [
          'date_hired' => 'date',
          'has_external_salary' => 'boolean',
          'has_mp2' => 'boolean',
          'has_sss_wisp' => 'boolean',
          'hmo_amount' => 'decimal:2',
          'last_synced_at' => 'datetime',
      ];

      // Audit only payroll-specific fields (not HRIS-synced fields)
      protected $auditInclude = [
          'has_external_salary', 'has_mp2', 'has_sss_wisp',
          'hmo_amount', 'pay_basis',
      ];

      // Accessors
      public function getFullNameAttribute(): string
      {
          return trim("{$this->first_name} {$this->middle_name} {$this->last_name}");
      }

      // Relationships
      public function company() { return $this->belongsTo(\App\Company\Models\Company::class); }
      public function user() { return $this->hasOne(\App\Company\Models\User::class); }
      public function compensation() { return $this->hasOne(\App\Compensation\Models\EmployeeCompensation::class); }

      // Scopes
      public function scopeActive($query) { return $query->where('employment_status', 'active'); }
      public function scopeByDepartment($query, string $department) { return $query->where('department', $department); }
      public function scopeByCompany($query, string $companyId) { return $query->where('company_id', $companyId); }
  }
  ```

### Edge Cases & Notes
- `hris_employee_id` is the sync anchor from the HRIS system. It must never be regenerated or changed after initial creation.
- `has_external_salary = true` means the employee's net pay equals gross pay and all deduction/tax steps (3-9) are skipped during payroll computation.
- The `auditInclude` array limits auditing to payroll-specific fields only, since HRIS-synced fields change frequently via the sync job and would create noise in the audit log.
- `hmo_amount` is stored on the employee rather than in `employee_compensation` because it is a fixed personal deduction, not a salary component amount.
