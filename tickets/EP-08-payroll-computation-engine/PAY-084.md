# [GW] Payroll - Bulk Apply Deductions to Payroll Run

## Description

Implement a bulk deduction application action that computes SSS, PhilHealth, Pag-IBIG, and HMO for all eligible employees in a draft payroll run in a single operation. The action uses each employee's gross pay and configuration for bracket/rate lookups, automatically skips employees with external salary enabled, and presents a summary showing the results. Individual overrides remain possible after the bulk application.

## Technical Implementation

### What Actions

- Bulk Apply Deductions action: accepts a payroll run in draft state. Iterates over all payslips in the run. For each payslip where the employee does not have external salary enabled: computes SSS using bracket-based lookup on gross pay, computes PhilHealth at 2.5% employee share subject to salary floor/ceiling, computes Pag-IBIG at fixed PHP 100, and computes HMO from the employee's HMO configuration. Updates each payslip with the computed deduction amounts. Skips external salary employees and tracks the skip count. Returns a summary with employees affected, employees skipped, totals per deduction type, and grand total. Triggers a full recomputation of the payroll run after all payslips are updated. Wraps all mutations in a database transaction.

### Validation & DTOs

- Bulk Apply Deductions request: validates payroll run exists and is in draft state.
- Bulk Deduction Summary DTO: employees affected count, employees skipped count, total SSS, total PhilHealth, total Pag-IBIG, total HMO, and grand total of all deductions.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/bulk-apply-deductions` — bulk apply deductions.

### Policies & Authorization

- Owner and Payroll Officer can perform bulk apply.
- Run must be in draft state.

### Tests

- Assert bulk apply computes correct SSS for each employee based on their gross pay bracket.
- Assert bulk apply computes correct PhilHealth (2.5%, capped).
- Assert bulk apply sets Pag-IBIG to PHP 100 for all eligible employees.
- Assert bulk apply sets HMO based on each employee's configuration.
- Assert employees with external salary enabled are skipped.
- Assert skip count is accurately reported in the summary.
- Assert bulk apply only works on draft runs (422 for other states).
- Assert individual override after bulk apply is preserved and functional.
- Assert recomputation is triggered after bulk apply.
- Assert payroll run totals are accurate after bulk apply and recomputation.
- Assert non-authorized roles receive 403.

### Edge Cases & Notes

- Bulk apply overwrites any previously computed deductions for the affected payslips.
- Individual overrides made after bulk apply are preserved on subsequent recomputations.
- If all employees in a run have external salary enabled, the action completes with zero employees affected.
- The action must be idempotent — running it multiple times produces the same results.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Bulk Apply Deductions" button is visible on the payroll run detail page in draft state for authorized users
- Clicking the button opens a confirmation dialog before proceeding
- After completion, a summary modal displays: number of employees affected, number of employees skipped (external salary), total per deduction type (SSS, PhilHealth, Pag-IBIG, HMO), and grand total

---

### Server-side

#### Side-effects

- Bulk apply computes correct SSS, PhilHealth, Pag-IBIG, and HMO per employee based on their gross pay and configuration
- Employees with external salary enabled are skipped, with skip count reported
- Only draft payroll runs can have bulk deductions applied
- Individual deduction overrides still work after bulk application
- A full recomputation of the payroll run is triggered after bulk apply to update run totals
- Only Owner and Payroll Officer roles can perform bulk apply
