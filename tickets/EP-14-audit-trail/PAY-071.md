# [GW] Payroll - Configure Audit Trail with Laravel Auditing

## Description

Install and configure the `owen-it/laravel-auditing` package to provide a comprehensive audit trail across the PayHRIS Payroll application. The Auditable interface must be implemented on all critical models: PayrollRun, Payslip, EmployeeCompensation, Currency, and Employee (payroll-specific flags). Each model must declare which attributes are audited and which are excluded, ensuring that state transitions, salary changes, currency rate changes, and payroll flag changes are all captured with old/new values and the acting user.

The audit system uses Laravel Auditing's default `audits` table and stores the authenticated user as the actor on every audit record. This forms the foundation for the audit trail UI (PAY-072) and payroll run audit history (PAY-073).

## Technical Implementation

### Package Installation & Configuration
- **`composer require owen-it/laravel-auditing`**
- **`php artisan vendor:publish --provider="OwenIt\Auditing\AuditingServiceProvider" --tag="config"`** — Publishes `config/audit.php`
- **`php artisan vendor:publish --provider="OwenIt\Auditing\AuditingServiceProvider" --tag="migrations"`** — Publishes audits table migration
- **`php artisan migrate`** — Creates the `audits` table

### Configuration (`config/audit.php`)
- Set `'user.morph_prefix'` to `'user'`
- Set `'user.guards'` to `['web']`
- Enable `'console'` auditing to `false` (only track web user actions)
- Set `'driver'` to `'database'`
- Set `'implementation'` to the default `OwenIt\Auditing\Models\Audit::class`

### Models
- **`app/Payroll/Models/PayrollRun.php`** — Implement `OwenIt\Auditing\Contracts\Auditable` interface and use `OwenIt\Auditing\Auditable` trait:
  - `$auditInclude`: `['status', 'name', 'period_start', 'period_end', 'pay_date', 'approved_by', 'approved_at', 'finalized_by', 'finalized_at', 'cancelled_at', 'cancelled_by', 'notes']`
  - `$auditEvents`: `['created', 'updated']`
  - Custom audit transformation for state transitions to capture `old_status → new_status`

- **`app/Payroll/Models/Payslip.php`** — Implement Auditable:
  - `$auditInclude`: `['gross_pay', 'net_pay', 'total_deductions', 'total_bonuses', 'sss_contribution', 'philhealth_contribution', 'pagibig_contribution', 'withholding_tax', 'hmo_deduction', 'status', 'notes']`
  - `$auditEvents`: `['created', 'updated']`

- **`app/Payroll/Models/EmployeeCompensation.php`** — Implement Auditable:
  - `$auditInclude`: `['basic_salary', 'daily_rate', 'salary_currency_id', 'compensation_type', 'effective_date']`
  - `$auditEvents`: `['created', 'updated']`

- **`app/Payroll/Models/Currency.php`** — Implement Auditable:
  - `$auditInclude`: `['code', 'name', 'rate_to_php', 'is_active']`
  - `$auditEvents`: `['created', 'updated']`

- **`app/Employee/Models/Employee.php`** — Implement Auditable:
  - `$auditInclude`: `['has_external_salary', 'has_mp2', 'has_hmo', 'hmo_monthly_amount', 'has_cash_advance', 'cash_advance_monthly_deduction']`
  - `$auditExclude`: all HRIS-synced fields (name, email, department, position, etc.) since those are managed externally
  - `$auditEvents`: `['updated']` (creation is via HRIS sync)

### Migrations
- **`database/migrations/xxxx_create_audits_table.php`** — Published from package:
  - `id` (bigIncrements)
  - `user_type`, `user_id` (nullable, morphs for actor)
  - `event` (string: created, updated, deleted)
  - `auditable_type`, `auditable_id` (morphs for audited model)
  - `old_values`, `new_values` (JSON)
  - `url` (nullable text)
  - `ip_address` (nullable)
  - `user_agent` (nullable)
  - `tags` (nullable string)
  - `created_at` (datetime)
  - Index on `auditable_type`, `auditable_id`
  - Index on `user_id`, `user_type`
  - Index on `created_at`

### Tests
- **`tests/Feature/Audit/AuditConfigurationTest.php`**
  - Updating PayrollRun status creates an audit record with old/new values
  - Updating Payslip monetary fields creates an audit record
  - Updating EmployeeCompensation salary creates an audit record
  - Updating Currency rate creates an audit record
  - Updating Employee payroll flags creates an audit record
  - HRIS-synced fields on Employee are excluded from audit
  - Audit record captures the authenticated user as actor
  - Audit record stores correct `event` type (created/updated)
  - Unauthenticated changes (e.g., queue jobs) use the `resolveUser` fallback or are attributed to system

### Edge Cases & Notes
- Queue jobs that modify auditable models must set the user context via `Audit::withUser()` or a custom user resolver so that automated changes are properly attributed
- The `audits` table can grow large; consider adding a scheduled cleanup job for records older than a configurable retention period (e.g., 2 years)
- HRIS-synced fields on the Employee model must be excluded to avoid noisy audit entries from daily syncs
- Decimal monetary values must be stored accurately in the JSON old/new values columns; verify no floating-point precision loss
