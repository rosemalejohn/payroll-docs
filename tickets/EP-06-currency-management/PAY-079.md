# [GW] Payroll - Missing Currency Rate Detection

## Description

During payroll computation, detect when an employee's salary currency does not have a snapshot rate in the current payroll run. Affected payslips are flagged with a "Missing Rate" indicator on the payroll run detail page. The system blocks the transition from `submitted` to `under_review` until all currency rates are configured. Finance users can see the warning and are directed to notify the owner to configure rates in Settings. Once the rate is set and the snapshot is updated, the warning is dismissed and affected figures are automatically recomputed.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/DetectMissingCurrencyRatesAction.php`** — Checks for missing rates during computation:
  - Accepts: payroll run ID
  - Queries all employees in the run whose `salary_currency_id` is not PHP
  - Cross-references with `payroll_run_currency_snapshots` for the run
  - Returns list of employees/payslips with missing snapshot rates, grouped by currency

- **`app/Payroll/Actions/ComputePayslipAction.php`** — Enhanced to handle missing rates:
  - Before computing: check if the employee's salary currency has a snapshot rate
  - If missing: set payslip status to `missing_rate`, set `gross_pay` and `net_pay` to null, add note "Missing exchange rate for {currency_code}"
  - If present: proceed with normal computation

- **`app/Payroll/Actions/SubmitPayrollRunAction.php`** — Enhanced to check for missing rates:
  - Payroll officers can still submit with missing rates (issue moves to finance review)

- **`app/Payroll/Actions/TransitionToUnderReviewAction.php`** — Blocks transition if missing rates:
  - Calls `DetectMissingCurrencyRatesAction`
  - If missing rates found: throw `ValidationException` with message listing affected currencies and employee count
  - Transition is blocked until all rates are resolved

- **`app/Payroll/Actions/ResolveMissingCurrencyRateAction.php`** — Called when a missing rate is added:
  - Accepts: payroll run ID, currency ID
  - Creates or updates the snapshot for this currency in the run
  - Recomputes all affected payslips that had `missing_rate` status
  - Clears the `missing_rate` flag on resolved payslips

### Models
- **`app/Payroll/Models/Payslip.php`** — Add:
  - `has_missing_rate` (boolean, default false) to `$fillable`
  - `missing_rate_currency_id` (nullable ULID) to `$fillable`
  - Scope: `scopeMissingRate($query)` — filters payslips with `has_missing_rate = true`

### Migrations
- **`database/migrations/xxxx_add_missing_rate_fields_to_payslips_table.php`**
  - Add `has_missing_rate` (boolean, default false)
  - Add `missing_rate_currency_id` (ULID, nullable)
  - Index on `has_missing_rate`

### Routes & Controllers
- **`GET /payroll-runs/{payrollRun}`** — Existing show route, enhanced:
  - Includes `missing_rate_summary`: array of `{ currency_code, currency_name, employee_count }` for currencies with missing rates
  - Includes `has_missing_rates`: boolean flag

- **`POST /payroll-runs/{payrollRun}/resolve-missing-rate`** — Route name: `payroll-runs.resolve-missing-rate`
- **`app/Http/Controllers/Web/PayrollRunMissingRateController@store`**
  - Middleware: `auth`, `role:owner`
  - Accepts currency ID, triggers `ResolveMissingCurrencyRateAction`
  - Redirects back with flash message

### Frontend
- **`resources/js/pages/payroll-runs/show.tsx`** — Enhanced with missing rate warnings:
  - If `has_missing_rates` is true, show a prominent warning banner at the top of the run detail page
  - Banner text: "Missing exchange rates for: {currency_list}. {employee_count} employee(s) affected. Payslips cannot be computed until rates are configured."
  - Link in banner: "Go to Currency Settings" → `/settings/currencies`
  - State transition button to `under_review` is disabled with tooltip explaining missing rates

- **`resources/js/pages/payroll-runs/partials/employee-list.tsx`** — Enhanced employee row:
  - Employees with missing rates show a red "Missing Rate" badge next to their currency indicator
  - Payslip amounts show "—" instead of computed values
  - Tooltip: "Exchange rate for {currency_code} not configured for this run"

- **`resources/js/components/payroll/missing-rate-banner.tsx`** — Reusable warning banner:
  - Props: `missing_currencies` array, `employee_count`
  - Displays warning icon, affected currencies, employee count, and link to settings

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Update `transitionToUnderReview()`:
  - Must additionally check for missing rates before allowing transition

### Tests
- **`tests/Feature/Payroll/MissingCurrencyRateDetectionTest.php`**
  - Payslip for employee with missing currency rate is flagged with `has_missing_rate = true`
  - Flagged payslip has null gross_pay and net_pay
  - Payslip notes indicate missing rate and currency code
  - Transition from submitted to under_review is blocked when missing rates exist
  - Validation error lists affected currencies and employee count
  - Adding the missing rate and resolving clears the flag and recomputes the payslip
  - Recomputed payslip has correct values after rate resolution
  - PHP currency employees are never flagged (base currency)
  - Run detail page shows missing rate summary in props
  - Multiple missing currencies are handled simultaneously
  - All missing rates must be resolved before transition is allowed

### Edge Cases & Notes
- PHP (Philippine Peso) is the base currency and always has rate 1.0; it should never be flagged as missing
- A payroll run might have multiple missing currencies simultaneously; all must be resolved before transition
- When a rate is resolved, only payslips for that specific currency should be recomputed, not all payslips in the run
- If the owner adds a new currency rate while a run is in `submitted` state, the payroll officer should see the missing rate warning disappear on page refresh
- Consider sending a notification to the owner when a run is submitted with missing rates
