# [GW] Payroll - Initialize Laravel 12 Project with Inertia.js, React, and Companies Database

## Description

Set up the foundational Laravel 12 application with Inertia.js v2, React 19, and TypeScript. This also includes creating the companies table — the foundational tenant record that all other domain models reference. The goal is a working dev environment where Laravel serves an Inertia-powered React page through Vite, with a seeded company record.

## Technical Implementation

### What Migrations

- Create the companies table with ULID primary key, company name, code (unique), address, government registration numbers (TIN, SSS, PhilHealth, Pag-IBIG), logo path, and active flag.
- Default Laravel migrations (sessions, cache, jobs) are included as-is.

### What Models

- Create Company model in the Company domain with ULID support, factory, and relationships to users, employees, currencies, and salary components.
- Factory should generate realistic company data.

### Routes & Controllers

- Default welcome route returning an Inertia response.

### Tests

- Assert GET `/` returns 200 and renders the correct Inertia component.
- Assert a company can be created with a valid ULID primary key.
- Assert the company code has a unique constraint.
- Assert the factory generates valid records.

### Environment & Config

- Configure Vite with Laravel and React plugins.
- Create TypeScript configuration with strict mode and path aliases.
- Create Inertia root Blade view with asset directives.
- Create Inertia app entry point with dynamic page resolution.
- Create a basic welcome page component.
- Configure `.env.example` with database, Redis, HRIS API, queue, cache, and mail variables.
- Register Inertia middleware in the web middleware group.

### Edge Cases & Notes

- Ensure TypeScript compiles without errors.
- Vite dev server must proxy correctly to Laravel's dev server.
- No database-level foreign key constraints — referential integrity enforced at the application layer.
- All primary keys use ULIDs (26-character string), not auto-incrementing integers.

## Acceptance Criteria

### Server-side

**Q: List down all known effects to data.**

- Running the project setup commands installs all dependencies and starts the application without errors
- The application can connect to the database and run all migrations successfully
- A company record can be created and persisted in the system
- Each company has a unique code that prevents duplicates

### Client-side

#### Behavior

**Q: How is the element expected to behave?**

- The welcome page loads and is rendered by React through Inertia (not a server-rendered Blade view)
- The Vite development server starts and watches for file changes
- The production build completes without errors
