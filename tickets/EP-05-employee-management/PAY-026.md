# [GW] Payroll - Employee Management

## Description

- Build primary employee management list page.
- Build employee creation and editing forms.
- Create employees table and specialized model.
- Construct list, queries, forms, API resources.
- Mirror HRIS data alongside payroll fields.
- Orchestrate functional employee account deactivation flows.

- Create employees table with ULID keys.

- Track timestamp for latest sync status.

### What Models

- Create Employee model in Employee domain.
- Separate HRIS-synced from static payroll fields.
- Audit purely tracks payroll-specific field changes.
- Add list filtering and active scopes.

### What Actions

- Create: assigns ULID, company, local flags.
- Update: updates strictly allowed record fields.
- Updates avoid overriding unique payroll settings.
- Deactivate: Provide reliable action modifying states.
- Evaluate corresponding incomplete database transaction items.

### Queries

- Support search, department, status, basis filters.

### Validation & DTOs

- Validate provided list query string parameters.
- Store: validate name, unique number, employment.

### Routes & Controllers

- Map employee paginated filtered list route.

### API Resources

## Acceptance Criteria

### Client-side

#### Layout

- Owners and Officers access Employee navigation.
- Create accessible via list add button.
- Edit accessible from actions list column.

#### Behavior

- List details numbers, names, departments, bases.
- Search queries strictly by name, number.
- Users filter by status and departments.
- Columns sort by clicking appropriate headers.
- Forms divide cleanly into logical sections.
- Edit inputs pre-fill accurate existing information.
- Deactivate switch appears purely alongside profiles.
- Submissions dynamically locate affiliated database activity.
- Discovered components trigger complex warning interfaces.
- Standard submissions present simple continuation dialogs.

#### Validation

- Missing required fields block explicit submission.
- Duplicate numbers trigger immediate format validation.
- Future hire dates trigger validation failure.

### Server-side

#### Side-effects

- Finance and Employee roles strictly blocked.
- Combines HRIS-synced and payroll-specific internal fields.
- System updates isolated employee records flawlessly.
- Processing strictly applies passive boolean parameters.
- Processes automatically circumvent subsequent batch calculations.
