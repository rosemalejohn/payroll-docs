# [GW] Payroll - Mid-Month Employee Termination Handling

## Description

Handle the scenario where an employee is marked as terminated with an effective date falling within an active payroll run period. The payroll officer specifies the employee's last working day, and the system prorates the employee's gross pay based on actual days worked, recomputes all deductions on the prorated amount, and annotates the payslip with termination details. This ensures terminated employees are compensated accurately for their partial period.

## Technical Implementation

### What Migrations

- Add termination fields to payslips table: is_terminated (boolean, default false), termination_date (nullable date), last_working_day (nullable date), proration_basis (nullable string).

### What Models

- Payslip model: add termination fields (is_terminated, termination_date, last_working_day, proration_basis) to fillable attributes and casts.

### What Actions

- Handle Terminated Employee Payslip action: orchestrates termination proration — validates last working day falls within the run period, calculates days worked vs total working days, prorates gross pay (monthly: salary / total_working_days × days_worked; daily rate: daily_rate × days_worked), triggers payslip recomputation with prorated gross, and updates payslip notes with termination info.
- Prorate Deductions action: prorates mandatory deductions based on prorated gross — SSS bracket lookup on prorated gross, PhilHealth at 2.5% of prorated gross respecting floor/ceiling, Pag-IBIG fixed at ₱100, HMO prorated by working days ratio, withholding tax annualized from prorated gross.
- Set Employee Termination action: marks the payslip as terminated with termination date, last working day, and proration basis note.

### Validation & DTOs

- Terminated Employee Payslip DTO: payroll run ID, employee ID, last working day, termination date.
- Set Employee Termination request: validates last_working_day is required, is a date, and falls within the payroll run period. Validates termination_date is required and is a date. Validates employee is part of the payroll run.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/payslips/{payslip}/termination` — sets termination and triggers proration.

### Policies & Authorization

- Only Payroll Officer and Owner roles can set termination.
- Payroll run must be in draft state.
- Payslip must not already be marked as terminated.

### Tests

- Assert monthly employee gross pay is prorated: salary / working_days × days_worked.
- Assert daily rate employee gross pay is prorated: daily_rate × days_worked.
- Assert SSS, PhilHealth, and HMO deductions are recalculated based on prorated gross.
- Assert withholding tax is recalculated based on prorated gross.
- Assert payslip notes include termination date and proration basis.
- Assert last working day must be within the run period.
- Assert last working day after period end returns validation error.
- Assert terminated flag is set on the payslip.
- Assert only draft runs allow termination marking.
- Assert non-Payroll Officer/Owner roles receive 403.
- Assert external salary employees have gross prorated but skip deduction steps 3-9.
- Assert net pay is computed correctly after proration.

### Edge Cases & Notes

- External salary employees still have their gross prorated but skip deduction steps 3-9 per computation engine rules.
- Pag-IBIG proration rules may vary by company policy — current implementation keeps the fixed ₱100 but this should be configurable.
- If the termination date is the first day of the period, the employee has 0 working days — handle gracefully with zero gross payslip.
- Working days calculation must account for weekends and potentially company holidays.
- If the employee is un-terminated (correction), the payslip should be recomputable to the full period amount.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Mark as Terminated" option is available on an employee's payslip within a draft payroll run
- Clicking it opens a dialog prompting for termination date and last working day
- The last working day date picker is constrained to dates within the payroll run period
- A preview shows the calculated days worked and prorated gross estimate before confirming
- Terminated payslips display a "Terminated" badge
- Termination details (date, last working day, proration basis) are visible on the payslip
- Only Payroll Officer and Owner roles can see and use the termination option

---

### Server-side

#### Side-effects

- Monthly employee gross pay is prorated based on actual days worked vs total working days in the period
- Daily rate employee gross pay is calculated as daily rate multiplied by days worked
- All mandatory deductions (SSS, PhilHealth, HMO, withholding tax) are recalculated based on the prorated gross
- The payslip is flagged as terminated with the termination date and proration basis recorded
- Only draft payroll runs allow termination marking
- Only Payroll Officer and Owner roles can set termination on a payslip
- External salary employees have gross prorated but skip deduction computation steps
