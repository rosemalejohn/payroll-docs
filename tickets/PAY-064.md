# PAY-064: Payslip Preview with Draft Watermark

## Description

Implement an in-memory payslip PDF preview for payroll runs that are in `draft` or `submitted` states. The preview uses the same layout as the final payslip PDF but overlays a prominent "DRAFT" watermark across the page. The preview is generated on-the-fly and is never saved to disk.

The preview is accessible to `payroll_officer` and `owner` roles and can be triggered from the payroll run detail page without navigating away (via a modal or side panel).

## Technical Implementation

### Routes & Controllers
- **`GET /payroll-runs/{payrollRun}/payslips/{payslip}/preview`** — Route name: `payroll-runs.payslips.preview`
- **`app/Http/Controllers/Web/PayslipPreviewController@show`**
  - Middleware: `auth`, `role:owner,payroll_officer`
  - Authorizes via `PayslipPolicy@preview`
  - Generates an in-memory PDF using the same Blade template as the final payslip (PAY-063)
  - Adds "DRAFT" watermark overlay
  - Returns the PDF as an inline response (Content-Disposition: inline) for browser viewing
  - Does **not** persist the file to disk

### Actions
- **`app/Payroll/Actions/GeneratePayslipPreviewAction.php`** — Generates preview PDF:
  - Reuses the same view data preparation logic as `GeneratePayslipPdfAction`
  - Passes an additional `is_preview: true` flag to the Blade template
  - Generates PDF via `barryvdh/laravel-dompdf` with the watermark applied
  - Returns the raw PDF content (not saved)

### Views (Blade Templates)
- **`resources/views/pdf/payslip.blade.php`** — Update to conditionally render watermark:
  - When `$is_preview` is true, overlay a diagonal "DRAFT" watermark using CSS:
    - Large, semi-transparent text rotated at -45 degrees
    - Positioned center of the page
    - Applied via `position: fixed` with `z-index` above content but readable beneath
  - All other content remains identical to the final payslip layout
- The external salary template (`payslip-external.blade.php`) also supports the watermark

### Policies & Authorization
- **`app/Payroll/Policies/PayslipPolicy.php`** — Add `preview()` method:
  - Only `owner` and `payroll_officer` roles can preview
  - Payroll run must be in `draft` or `submitted` state
  - Returns false for `under_review` or `finalized` runs (use download for finalized)

### Frontend Components
- **`resources/js/components/payslip-preview-modal.tsx`** — Modal or side panel component:
  - Triggered by a "Preview" button on each payslip row in the payroll run detail page
  - Embeds the PDF in an `<iframe>` or uses a PDF viewer library
  - Closes without navigation
  - Shows loading state while PDF is being generated
- **`resources/js/pages/payroll-runs/show.tsx`** — Add preview button to payslip rows:
  - Button visible only for `owner` and `payroll_officer` roles
  - Button visible only when run is in `draft` or `submitted` state

### Tests
- **`tests/Feature/Payroll/PayslipPreviewTest.php`**
  - Payroll officer can preview a payslip for a `draft` run
  - Payroll officer can preview a payslip for a `submitted` run
  - Owner can preview a payslip for a `draft` or `submitted` run
  - Finance and employee roles receive 403
  - Preview of a payslip for an `under_review` run returns 403
  - Preview of a payslip for a `finalized` run returns 403
  - Preview response has correct content type (`application/pdf`)
  - Preview response has inline content disposition
  - Preview content includes "DRAFT" watermark
  - Preview content matches current computation data (not stale data)
  - No file is written to disk during preview

### Edge Cases & Notes
- Preview generates the PDF synchronously (not queued) since it must return immediately
- For large payroll runs, preview is per-payslip (not bulk) to keep response times reasonable
- If computation data changes between previews, the new preview reflects the latest data
- The watermark must be clearly visible but not obscure the underlying data
- Consider caching the preview PDF briefly (e.g., 60 seconds) in memory/Redis to avoid regeneration on repeated views of the same payslip

## Acceptance Criteria

- [ ] Preview is available for payslips in `draft` and `submitted` payroll runs
- [ ] "DRAFT" watermark is clearly visible on the preview PDF
- [ ] Preview content matches the current payslip computation data
- [ ] Preview PDF is not saved to disk
- [ ] Preview is accessible by `payroll_officer` and `owner` roles only
- [ ] Finance and employee roles receive 403 when attempting to preview
- [ ] Preview for `under_review` or `finalized` runs returns 403
- [ ] Preview can be accessed without navigating away from the payroll run page (modal or side panel)
- [ ] Loading state is shown while the preview PDF is being generated
