# [GW] Payroll - Payroll Run State Transitions

## Description

Implement the payroll run state machine using `spatie/laravel-model-states`. Payroll runs follow an irreversible linear workflow: `draft -> submitted -> under_review -> finalized`. Each transition is handled by a dedicated Action class that enforces authorization, validates preconditions, records metadata (actor, timestamp), and logs to the audit trail.

State transitions are strictly one-directional and cannot be reversed. Each transition must validate role-based access and record who performed the action and when.

## Technical Implementation

### State Classes

Define states under `app/Payroll/States/PayrollRunState.php` using `spatie/laravel-model-states`:

- `DraftState`
- `SubmittedState`
- `UnderReviewState`
- `FinalizedState`

Register allowed transitions in `PayrollRunState::config()`:

- `DraftState` -> `SubmittedState`
- `SubmittedState` -> `UnderReviewState`
- `UnderReviewState` -> `FinalizedState`

### Actions

**`app/Payroll/Actions/SubmitPayrollRunAction.php`**

- Validates the run is in `draft` state.
- Validates the run has at least one employee (payslips exist).
- Triggers payslip computation for all employees in the run (calls `RecomputePayrollRunAction`).
- Sets `submitted_by` (authenticated user ULID) and `submitted_at` (now).
- Transitions state from `draft` to `submitted`.
- Logs transition via `owen-it/laravel-auditing`.

**`app/Payroll/Actions/MoveToUnderReviewAction.php`**

- Validates the run is in `submitted` state.
- Authorization: only `finance` or `owner` roles may invoke.
- Sets `reviewed_by` and `reviewed_at`.
- Transitions state from `submitted` to `under_review`.
- Logs transition via audit trail.

**`app/Payroll/Actions/FinalizePayrollRunAction.php`**

- Validates the run is in `under_review` state.
- Authorization: only `owner` role may invoke.
- Locks all payslips associated with the run (`payslips.is_locked = true`).
- Locks the payroll period (`payroll_periods.is_locked = true`).
- Sets `finalized_by` and `finalized_at`.
- Transitions state from `under_review` to `finalized`.
- Logs transition via audit trail.

### Controller

Add transition endpoint methods on `PayrollRunController`:

- `POST /payroll-runs/{payrollRun}/submit` -> `SubmitPayrollRunAction`
- `POST /payroll-runs/{payrollRun}/review` -> `MoveToUnderReviewAction`
- `POST /payroll-runs/{payrollRun}/finalize` -> `FinalizePayrollRunAction`

Each endpoint uses a Form Request for authorization and delegates to the corresponding Action.

### Audit Trail

Each transition dispatches an `PayrollRunTransitioned` event captured by `owen-it/laravel-auditing`, recording:

- `old_state`, `new_state`
- `actor_id`, `actor_name`, `actor_role`
- `timestamp`

## Acceptance Criteria

- [ ] States transition in the correct order only: `draft -> submitted -> under_review -> finalized`.
- [ ] Attempting an invalid transition (e.g., `draft -> finalized`) throws a `TransitionNotAllowed` exception.
- [ ] Attempting a backward transition (e.g., `submitted -> draft`) throws a `TransitionNotAllowed` exception.
- [ ] `SubmitPayrollRunAction` rejects runs with zero employees.
- [ ] `SubmitPayrollRunAction` triggers payslip computation and sets `submitted_by`/`submitted_at`.
- [ ] `MoveToUnderReviewAction` is restricted to `finance` and `owner` roles; other roles receive 403.
- [ ] `FinalizePayrollRunAction` is restricted to `owner` role only; other roles receive 403.
- [ ] Finalization sets `is_locked = true` on all payslips in the run.
- [ ] Finalization sets `is_locked = true` on the associated payroll period.
- [ ] Audit trail records actor ID, actor name, old state, new state, and timestamp for every transition.
- [ ] All transitions are idempotent-safe (re-submitting an already submitted run returns an appropriate error, not a crash).
