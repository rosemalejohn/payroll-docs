# PAY-019: Create Cash Advances Migration and Model

## Description

Create the `cash_advances` table and model to track cash advance issuance and recovery per employee. Active cash advances are summed during payroll computation (Step 8) and deducted from net pay. After payroll finalization, recovered amounts are updated on the cash advance records.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_cash_advances_table.php`**
  ```sql
  id               CHAR(26) PRIMARY KEY      -- ULID
  company_id       CHAR(26) NOT NULL
  employee_id      CHAR(26) NOT NULL          -- ref: employees.id
  amount           DECIMAL(12,2) NOT NULL     -- original amount
  amount_recovered DECIMAL(12,2) NOT NULL DEFAULT 0.00
  balance          DECIMAL(12,2) NOT NULL     -- amount - amount_recovered
  issued_date      DATE NOT NULL
  status           ENUM('active','fully_recovered','cancelled') NOT NULL DEFAULT 'active'
  notes            TEXT NULL
  created_by       CHAR(26) NULL              -- ref: users.id
  timestamps
  ```
- Index: `idx_cash_advances_employee` on `(employee_id, status)`

### Model
- **`app/CashAdvance/Models/CashAdvance.php`**
  - Uses `HasUlids` trait
  - Casts: `amount` → `decimal:2`, `amount_recovered` → `decimal:2`, `balance` → `decimal:2`, `issued_date` → `date`
  - Relationships: `belongsTo(Employee::class)`, `belongsTo(Company::class)`, `belongsTo(User::class, 'created_by')`
  - Scopes: `scopeActive()` (where status = 'active'), `scopeForEmployee($employeeId)`
  - Computed: `balance` should always equal `amount - amount_recovered`
  - Method: `markFullyRecovered()` sets status to `fully_recovered` when balance reaches 0

### Factory
- `CashAdvanceFactory` with states: `active`, `fullyRecovered`, `cancelled`

### Tests
- Balance equals amount minus amount_recovered
- Active scope returns only active advances
- Status transitions from active to fully_recovered when balance = 0

## Acceptance Criteria

- [ ] Migration runs without errors

- [ ] Status transitions: `active` → `fully_recovered` (when balance = 0), `active` → `cancelled`
- [ ] Decimal casts on all monetary fields
- [ ] Factory generates valid test data with different states
