# [GW] Payroll - Create Payroll Run Action

## Description

As a payroll officer, I want to create a new payroll run for a specific period so that I can begin computing employee salaries. The run auto-populates with all active employees, snapshots currency rates, and attaches auto-trigger bonuses.

## Technical Implementation

### Action
- **`app/Payroll/Actions/CreatePayrollRunAction.php`**
  - Validates: no existing non-cancelled run for the same period
  - Creates `PayrollRun` record in `draft` state
  - Calls `SnapshotCurrencyRatesAction` to snapshot all active currency rates
  - Calls `AutoAttachBonusesToRunAction` to attach applicable auto-trigger bonuses
  - Creates a `Payslip` record for each active employee (initial empty values)
  - Sets `employee_count`, `base_currency_id` (PHP), `created_by`

### Controller
- **`PayrollRunController@store`** — `POST /payroll-runs`

### Form Request
- **`StorePayrollRunRequest`**
  - `payroll_period_id`: required, exists
  - `name`: required, string, max:255
  - `notes`: nullable, string

### Validation
- Duplicate check: `PayrollRun::where('payroll_period_id', $periodId)->whereNotIn('state', ['cancelled'])->exists()` → error if true

### Frontend
- "Create Payroll Run" button on runs list page
- Form: select payroll period (dropdown of unlocked periods), run name (auto-suggested from period name), notes
- Shows recent runs for selected period to help avoid duplicates
- On success: redirect to run detail page

### Tests
- Run created in draft state with correct data
- All active employees auto-populated
- Currency rates snapshotted
- Auto-trigger bonuses attached
- Duplicate run for same period blocked
- Inactive employees excluded

## Acceptance Criteria

- [ ] Run created in `draft` state
- [ ] All active employees auto-populated with payslip records
- [ ] Currency rates snapshotted from all active currencies
- [ ] Auto-trigger bonuses attached based on period month
- [ ] Duplicate run for same period (non-cancelled) blocked with error
- [ ] `employee_count` set correctly
- [ ] Redirects to run detail page on success
