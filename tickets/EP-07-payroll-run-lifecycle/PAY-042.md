# [GW] Payroll - Add/Remove Employees from Draft Payroll Run

## Description

Allow Payroll Officers and Owners to add or remove employees from a payroll run while it is in draft state. Adding an employee creates an initial payslip record with zero values and snapshots their salary data. Removing an employee deletes their payslip record and requires a reason, which is logged in the audit trail. These operations are blocked on any non-draft run.

## Technical Implementation

### What Actions

- Add Employee to Run action: validates the run is in draft state, validates the employee does not already have a payslip in this run, creates a payslip record with snapshotted employee data and zero computed fields, and increments the run's employee count.
- Remove Employee from Run action: validates the run is in draft state, requires a reason (minimum 5 characters), deletes the payslip and all associated line items, decrements the run's employee count, and logs removal in the audit trail with employee details, reason, and actor.

### Validation & DTOs

- Add Employee request: validates employee ID (required, must exist in employees table).
- Remove Employee request: validates reason (required, minimum 5 characters).
- DTOs for add and remove employee data using spatie/laravel-data.

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/employees` — add an employee to the run.
- `DELETE /payroll-runs/{payrollRun}/employees/{employee}` — remove an employee from the run.

### Policies & Authorization

- Owner and Payroll Officer can add and remove employees from draft runs.
- Operations are blocked on non-draft runs (returns 422).

### Tests

- Assert an employee can be added to a draft run, creating a payslip with initial zero values.
- Assert adding a duplicate employee to the same run returns a validation error.
- Assert the employee search filters out employees already in the run.
- Assert removing an employee requires a non-empty reason (minimum 5 characters).
- Assert removal deletes the payslip and all associated line items.
- Assert removal is logged in the audit trail with employee details, reason, and actor.
- Assert the run's employee count is updated on both add and remove.
- Assert add and remove operations are blocked on non-draft runs (422).

## Acceptance Criteria

### Client-side

#### Behavior

- An "Add Employee" button on the draft run detail page opens a searchable employee modal
- The employee search filters out employees already in the run and allows search by name, employee ID, or department
- A confirmation button adds the selected employee to the run
- Each employee row in the draft run detail view has a "Remove" button
- Clicking "Remove" opens a dialog requiring a reason text input (minimum 5 characters)
- On confirmation, the employee row is removed from the table with a toast notification
- Add and remove controls are only visible when the run is in draft state

#### Validation

- An error is shown if attempting to add an employee who is already in the run
- An error is shown if the removal reason is empty or too short

---

### Server-side

#### Side-effects

- Adding an employee creates a payslip record with snapshotted employee data and zero computed fields
- Adding a duplicate employee to the same run is prevented by the unique constraint
- Removing an employee deletes the payslip and all associated line items
- Removing an employee requires a reason, which is logged in the audit trail with the actor and employee details
- The run's employee count is updated on both add and remove operations
- Add and remove operations are blocked on non-draft runs
