# PAY-024: Build Reusable Form Components

## Description

Create a library of reusable form field wrapper components built on Shadcn/ui primitives. Each field component handles its own label, validation error display, and optional description text. These components integrate with Inertia's `useForm` hook for simple forms and display Laravel server-side validation errors inline. This library will be used across all create/edit pages in the application.

## Technical Implementation

### What Actions
- No domain actions in this ticket (frontend component only).

### Frontend Components

#### `resources/js/components/forms/text-input.tsx`
- Wraps Shadcn `Input` with `Label`.
- Props: `label`, `name`, `value`, `onChange`, `error` (string, from Laravel validation), `description` (optional helper text), `placeholder`, `type` (text/email/password), `required`, `disabled`.
- Error text renders below the input in red (`text-destructive`).
- Description text renders below the input in muted color.
- `id` auto-generated from `name` for label association.

#### `resources/js/components/forms/number-input.tsx`
- Wraps Shadcn `Input` with `type="number"`.
- Props: same as text-input plus `min`, `max`, `step`.
- Formats display value; stores raw number.
- Prevents non-numeric input.

#### `resources/js/components/forms/select-field.tsx`
- Wraps Shadcn `Select` with `Label`.
- Props: `label`, `name`, `value`, `onChange`, `options` (array of `{ value: string; label: string }`), `error`, `description`, `placeholder`, `required`, `disabled`.
- Renders a `SelectTrigger`, `SelectContent`, and `SelectItem` for each option.

#### `resources/js/components/forms/date-picker.tsx`
- Wraps a date input or Shadcn Popover-based calendar picker.
- Props: `label`, `name`, `value` (ISO date string), `onChange`, `error`, `description`, `minDate`, `maxDate`, `required`, `disabled`.
- Value stored as `YYYY-MM-DD` string for Laravel compatibility.

#### `resources/js/components/forms/toggle-field.tsx`
- Wraps Shadcn `Switch` with `Label`.
- Props: `label`, `name`, `checked` (boolean), `onChange`, `error`, `description`, `disabled`.
- Label renders to the right of the switch.

#### `resources/js/components/forms/textarea-field.tsx`
- Wraps Shadcn `Textarea` with `Label`.
- Props: `label`, `name`, `value`, `onChange`, `error`, `description`, `placeholder`, `rows`, `required`, `disabled`.

#### `resources/js/components/forms/form-section.tsx`
- Groups related form fields with a title and optional description.
- Props: `title` (string), `description` (optional string), `children` (form fields).
- Renders a card-like container with a header and body area using Shadcn `Card` or a styled `div`.
- Used to visually separate form sections (e.g., "Personal Information", "Compensation Details").

#### `resources/js/components/forms/form-actions.tsx`
- Renders a bottom bar with submit and cancel buttons.
- Props: `submitLabel` (default "Save"), `cancelHref` (Inertia link for cancel), `processing` (boolean to disable submit), `isDirty` (boolean for unsaved changes warning).
- Submit button shows loading spinner when `processing` is true.
- Cancel button navigates via Inertia `Link` component.

### Integration Pattern with `useForm`
- Each field component accepts `error` as a direct prop rather than reaching into Inertia context.
- The parent page component uses `useForm` and passes form state to fields:
  ```typescript
  const { data, setData, post, processing, errors } = useForm({ name: '', email: '' });
  // ...
  <TextInput label="Name" name="name" value={data.name}
    onChange={(e) => setData('name', e.target.value)} error={errors.name} />
  ```
- This keeps field components decoupled from Inertia and reusable with other form libraries.

### Tests
- No automated backend tests for this ticket.
- Manual verification: each field type renders correctly, labels associate with inputs, validation errors display inline, useForm integration works end-to-end.

### Edge Cases & Notes
- Field components must handle `undefined` error prop gracefully (no error displayed).
- All inputs must have proper `id` and `htmlFor` attributes for accessibility.
- Number input must handle edge cases: empty string on clear, pasting non-numeric text.
- Date picker value must always be an ISO string to match Laravel's date validation.
- Form section component should support nested form sections without visual conflicts.

## Acceptance Criteria

- [ ] `NumberInput` renders a number input with min/max/step support and blocks non-numeric entry.
- [ ] `SelectField` renders a dropdown with options and displays the selected value.
- [ ] `DatePicker` renders a date picker and stores the value as `YYYY-MM-DD`.
- [ ] `ToggleField` renders a switch with label and handles boolean state.
- [ ] `TextareaField` renders a multi-line text area with configurable rows.
- [ ] `FormSection` groups fields with a title and optional description.
- [ ] `FormActions` renders submit and cancel buttons with processing state support.
- [ ] All field components display Laravel validation errors inline below the field in red text.
- [ ] All field components integrate cleanly with Inertia `useForm` hook.
- [ ] All inputs have proper `id` and `htmlFor` label associations for accessibility.
- [ ] Submit button shows a loading state and is disabled while `processing` is true.
- [ ] All components are fully typed with TypeScript.
