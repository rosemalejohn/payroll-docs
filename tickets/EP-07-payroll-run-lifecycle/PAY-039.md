# [GW] Payroll - Payroll Run List Page

## Description

Build the payroll run list page where the Payroll Officer and Owner can view all payroll runs with their statuses, track payroll history, and quickly navigate to active runs. The list is filterable by state and date range, paginated, and sorted by newest first.

## Technical Implementation

### Queries

- Payroll run list query with filtering by state (multi-select) and date range (created date). Eager loads payroll period data. Default sort by created date descending, with additional sort options for name and state.

### Routes & Controllers

- `GET /payroll-runs` — paginated list of payroll runs with filters.

### Policies & Authorization

- Owner, Payroll Officer, and Finance can view the payroll run list.
- Employee role cannot access the run list.

### Tests

- Assert payroll runs are listed with correct data (period, name, state, date, employee count, total net).
- Assert filters by state and date range work correctly.
- Assert pagination works with configurable page sizes.
- Assert default sort is newest first.
- Assert Employee role cannot access the run list (403).
- Assert finalized runs show the download summary action.

## Acceptance Criteria

### Client-side

#### Layout

- Accessible from sidebar for Owner, Payroll Officer, Finance

#### Behavior

- List shows period, name, state badge, date, count, net
- State colors: Draft gray, Submitted blue, Review yellow, Finalized green
- Filter by state (multi-select) and date range
- Paginated with page size selector (10, 25, 50)
- Default sort newest first
- Row click navigates to run detail page
- "Download Summary" shown on finalized runs only
- "Create New Run" button for Owner/Payroll Officer

---

### Server-side

#### Side-effects

- Paginated results with eager-loaded period data
- Only Owner, Payroll Officer, Finance can access
- Employee role denied access
