# [GW] Payroll - Update Currency Rate Action

## Description

As an owner, I want to update exchange rates for foreign currencies so that payroll computations use accurate conversion rates. The system should warn me if the rate change affects open payroll runs.

## Technical Implementation

### Action
- **`app/Currency/Actions/UpdateCurrencyRateAction.php`**
  - Validates: `$newRate > 0`
  - Prevents updating base currency (`is_base = 1` throws exception)
  - Updates: `rate`, `rate_updated_at = now()`, `updated_by`
  - Checks for open payroll runs (draft/submitted/under_review) with employees in this currency
  - Returns warning payload if affected runs exist

### Controller
- **`CurrencyController@update`** — `PUT /settings/currencies/{currency}`

### Form Request
- **`UpdateCurrencyRateRequest`**
  - `rate`: required, numeric, gt:0

### Frontend
- Edit modal on currency row (owner only):
  - Currency name and code (read-only)
  - Current rate field (editable, 6 decimal precision)
  - Last updated info: "Last updated on [date] at [time]"
  - Live preview: "1 [CODE] = ₱[rate]"
  - Save and Cancel buttons
- If affected runs detected: warning dialog listing run names before saving

### Audit
- Rate changes captured by laravel-auditing with old/new values

### Tests
- Rate updates correctly
- Base currency update blocked
- Rate ≤ 0 rejected
- Audit log captures old/new rate
- Warning returned when open runs affected

## Acceptance Criteria

- [ ] Rate updates immediately on save
- [ ] Base currency (PHP) rate cannot be edited
- [ ] Rate ≤ 0 rejected with validation error
- [ ] Audit log records old and new rate values
- [ ] Warning shown if open payroll runs have employees in this currency
- [ ] `rate_updated_at` and `updated_by` set on rate change
- [ ] Only owner can update rates
