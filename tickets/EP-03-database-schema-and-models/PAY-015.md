# [GW] Payroll - Create Payslip Line Items Migration and Model

## Description

Create the `payslip_line_items` table and Eloquent model to store individual line entries on a payslip (each earning, bonus, or deduction as a separate row). These line items power the itemized payslip PDF and detail views.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_payslip_line_items_table.php`**
  ```sql
  id             CHAR(26) PRIMARY KEY  -- ULID
  payslip_id     CHAR(26) NOT NULL     -- ref: payslips.id
  component_code VARCHAR(50) NOT NULL  -- ref: salary_components.code or bonus type code
  component_name VARCHAR(150) NOT NULL -- display name
  type           ENUM('earning','deduction','bonus') NOT NULL
  amount         DECIMAL(14,2) NOT NULL DEFAULT 0.00  -- in PHP
  sort_order     INT NOT NULL DEFAULT 0
  notes          TEXT NULL
  timestamps
  ```
- Index: `idx_line_items_payslip` on `payslip_id`

### Model
- **`app/Payroll/Models/PayslipLineItem.php`**
  - Uses `HasUlids` trait
  - Casts: `amount` → `decimal:2`
  - Relationship: `belongsTo(Payslip::class)`
  - Fillable: all columns
  - Default ordering scope by `sort_order`

### Factory
- `PayslipLineItemFactory` with states for earning, deduction, bonus types

### Tests
- Line item belongs to payslip
- Sort order determines display order
- Enum type validation works
