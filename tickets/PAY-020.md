# PAY-020: Create Payroll Run Currency Snapshots Migration and Model

## Description

Create the `payroll_run_currency_snapshots` table and model to capture all active currency rates at the moment a payroll run is created. Payslip calculations reference snapshot rates (not live rates) to ensure historical accuracy and payroll integrity.

## Technical Implementation

### Migration
- **`database/migrations/xxxx_create_payroll_run_currency_snapshots_table.php`**
  ```sql
  id             CHAR(26) PRIMARY KEY      -- ULID
  payroll_run_id CHAR(26) NOT NULL         -- ref: payroll_runs.id
  currency_id    CHAR(26) NOT NULL         -- ref: currencies.id
  currency_code  VARCHAR(10) NOT NULL      -- denormalized for readability
  rate           DECIMAL(18,6) NOT NULL    -- rate at time of payroll run creation
  snapshotted_at TIMESTAMP NOT NULL
  timestamps
  ```
- Index: `idx_currency_snapshots_run` on `payroll_run_id`

### Model
- **`app/Currency/Models/PayrollRunCurrencySnapshot.php`**
  - Uses `HasUlids` trait
  - Casts: `rate` → `decimal:6`, `snapshotted_at` → `datetime`
  - Relationships: `belongsTo(PayrollRun::class)`, `belongsTo(Currency::class)`
  - Fillable: all columns

### Usage
- Lookup during payslip computation:
  ```php
  $snapshotRate = PayrollRunCurrencySnapshot::where('payroll_run_id', $run->id)
      ->where('currency_id', $compensation->currency_id)
      ->value('rate');
  ```

### SnapshotCurrencyRatesAction
- **`app/Currency/Actions/SnapshotCurrencyRatesAction.php`**
  - Called when a PayrollRun is created
  - Snapshots all active currencies (`is_active = 1`) for the company
  - Sets `payroll_runs.rates_snapshotted_at = now()`

### Tests
- Snapshot created for each active currency on run creation
- Snapshot rate used in computation (not live rate)
- Rate stored as DECIMAL(18,6)
- Inactive currencies not included in snapshot

## Acceptance Criteria

- [ ] Migration runs without errors
- [ ] Snapshot stores the exchange rate at the moment of payroll run creation
- [ ] Rate uses DECIMAL(18,6) for precision
- [ ] `currency_code` denormalized for readability
- [ ] Lookup by `payroll_run_id + currency_id` returns the correct snapshot rate
- [ ] Only active currencies (`is_active = 1`) are snapshotted
- [ ] `payroll_runs.rates_snapshotted_at` updated after snapshot completes
