# [GW] Payroll - Per-Employee Bonus Exclusions

## Description

Build the ability to exclude specific employees from receiving a particular bonus within a payroll run, including the payroll run bonus exclusions database table and model. When a bonus has apply-to-all toggled off, the Payroll Officer can select which employees should be excluded from that bonus. Exclusions are checked during Step 2 of the payroll computation pipeline, and any change to exclusions triggers recomputation for the affected employees.

## Technical Implementation

### What Migrations

- Create the payroll run bonus exclusions table with ULID primary key, payroll run bonus reference, employee reference, reason (nullable text), and standard timestamps. Unique constraint on payroll run bonus and employee.

### What Models

- Create PayrollRunBonusExclusion model in the Bonus domain with ULID support.
- Relationships: belongs to PayrollRunBonus, belongs to Employee.
- Update PayrollRunBonus model with: has many PayrollRunBonusExclusions relationship and helper method to check if an employee is excluded.

### What Actions

- Manage Bonus Exclusions action: validates the payroll run is in draft state. Sets apply-to-all to false if exclusions are provided, or true if the exclusion list is empty. Performs a sync operation: deletes existing exclusions for this bonus and inserts the new set. Identifies affected employees (newly excluded or newly included) and triggers recomputation only for those employees. Updates payroll run totals.

### Validation & DTOs

- Manage Bonus Exclusions request: validates an array of exclusions where each entry has an employee ID (required, must exist and be in the current payroll run) and an optional reason (maximum 500 characters).

### Routes & Controllers

- `GET /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/exclusions` — list all employees with their exclusion status.
- `POST /payroll-runs/{payrollRun}/bonuses/{payrollRunBonus}/exclusions` — replace the full set of exclusions.

### Policies & Authorization

- Owner and Payroll Officer can manage bonus exclusions.
- Exclusion management is blocked when the run is not in draft state.

### Tests

- Assert excluding an employee prevents them from receiving the bonus in the computed payslip.
- Assert excluding and then re-including an employee restores the bonus in their payslip.
- Assert exclusion reason is saved correctly.
- Assert apply-to-all is set to false when exclusions exist and true when none.
- Assert exclusions cannot be modified when the run is not in draft (422).
- Assert only employees in the current payroll run can be excluded.
- Assert recomputation is triggered only for affected employees.
- Assert exclusion count is returned in the bonus list props.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Manage Exclusions" button on each bonus row (or toggling "Apply to All" off) opens an employee list panel
- The panel shows all employees in the payroll run with checkboxes — checked employees receive the bonus, unchecked employees are excluded
- A search/filter input allows finding employees by name, employee number, or department
- Each excluded employee row has an optional reason text input
- A "Save" button submits the exclusion list and triggers recomputation
- The exclusion count is displayed as a badge on each bonus row in the bonuses tab (e.g., "3 excluded")
- When the run is not in draft state, the exclusion list is viewable but not editable

#### Validation

- An error is shown if an excluded employee is not part of the current payroll run

---

### Server-side

#### Side-effects

- Excluded employees do not receive the bonus during payslip computation (Step 2 skips them)
- The apply-to-all flag on the payroll run bonus auto-updates based on whether exclusions exist
- Exclusion reasons are saved and stored with the exclusion record
- Saving exclusion changes triggers recomputation for affected employees only (those whose exclusion status changed)
- One exclusion per bonus per employee is enforced by the unique constraint
- Exclusion management is blocked when the payroll run is not in draft state
- Only Owner and Payroll Officer roles can manage exclusions
