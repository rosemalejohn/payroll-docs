# [GW] Payroll - CSV and PDF Export for Finalized Runs

## Description

Implement CSV and PDF export functionality for finalized payroll runs. The CSV export uses `maatwebsite/laravel-excel` and includes per-employee payroll data with all earnings, deductions, bonuses, and currency details. The PDF export uses `barryvdh/laravel-dompdf` and generates a company-level summary report matching the payroll summary report page (PAY-067).

Files are named with the payroll period slug for easy identification: `payroll-{period-slug}.csv` and `payroll-summary-{period-slug}.pdf`.

## Technical Implementation

### Routes & Controllers
- **`GET /payroll-runs/{payrollRun}/export/csv`** — Route name: `payroll-runs.export.csv`
- **`GET /payroll-runs/{payrollRun}/export/pdf`** — Route name: `payroll-runs.export.pdf`
- **`app/Http/Controllers/Web/ExportController@csv`**
  - Middleware: `auth`, `role:owner,finance`
  - Authorizes via `PayrollRunPolicy@export`
  - Returns a download response via `maatwebsite/laravel-excel`
- **`app/Http/Controllers/Web/ExportController@pdf`**
  - Middleware: `auth`, `role:owner,finance`
  - Authorizes via `PayrollRunPolicy@export`
  - Generates summary PDF via `barryvdh/laravel-dompdf`
  - Returns download response

### Export Classes
- **`app/Payroll/Exports/PayrollRunExport.php`** — Implements `FromQuery`, `WithHeadings`, `WithMapping`, `ShouldAutoSize`:
  - CSV columns (in order):
    1. Employee Name
    2. Employee Number
    3. Department
    4. Pay Basis
    5. Gross Pay
    6. SSS
    7. PhilHealth
    8. Pag-IBIG
    9. HMO
    10. BIR Withholding Tax
    11. MP2
    12. SSS WISP
    13. Cash Advance
    14. Total Deductions
    15. 13th Month Pay (if applicable)
    16. Other Bonuses (comma-separated labels + amounts)
    17. Total Bonuses
    18. Net Pay
    19. Salary Currency
    20. Exchange Rate
    21. PHP Equivalent (gross)
  - Query: all payslips in the payroll run, eager loading employee, deductions, bonuses
  - Mapping: format decimals to 2 places, handle null currency fields (PHP employees show "PHP", rate "1.0000")
  - Foreign currency employees: `Salary Currency` shows the foreign code, `Exchange Rate` shows the snapshot rate, `PHP Equivalent` shows the converted gross amount

### Views (Blade Templates)
- **`resources/views/pdf/payroll-summary.blade.php`** — Summary report PDF template:
  - Header: company name, report title "Payroll Summary Report", payroll period, finalization date
  - Employee count
  - Earnings summary: total gross pay
  - Deductions summary table: type, total amount per type
  - Employer contributions table: SSS employer, PhilHealth employer, Pag-IBIG employer
  - Net pay total
  - Total employer cost
  - Footer: generation timestamp, generated by user name

### Actions
- **`app/Payroll/Actions/GeneratePayrollSummaryPdfAction.php`** — Generates the summary PDF:
  - Reuses `PayrollSummaryQuery` from PAY-067 for data aggregation
  - Renders the summary Blade template
  - Returns the PDF content via `barryvdh/laravel-dompdf`

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Add `export()` method:
  - Only `owner` and `finance` roles can export
  - Payroll run must be in `finalized` state

### DTOs
- **`app/Payroll/DTOs/PayrollExportRowData.php`** — spatie/laravel-data DTO for type-safe row mapping in the export class

### Tests
- **`tests/Feature/Payroll/PayrollRunExportTest.php`**
  - CSV downloads with correct filename format: `payroll-{period-slug}.csv`
  - CSV contains correct column headers
  - CSV data matches finalized payslip data for each employee
  - CSV includes all deduction types with correct amounts
  - CSV includes bonus data
  - Foreign currency employees show salary currency, exchange rate, and PHP equivalent in CSV
  - PHP currency employees show "PHP" as currency with rate "1.0000"
  - PDF downloads with correct filename format: `payroll-summary-{period-slug}.pdf`
  - PDF summary totals match the payroll summary report page data
  - Only `owner` and `finance` roles can export (others receive 403)
  - Export of non-finalized run returns 403
  - Empty run (no payslips) produces valid CSV with headers only
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- For large payroll runs, CSV export should use chunked query reading to avoid memory issues (`FromQuery` handles this via `maatwebsite/laravel-excel`)
- Period slug should be URL-safe: e.g., "2026-03-01-to-2026-03-15"
- Ensure CSV properly escapes commas in employee names or department names
- Decimal values in CSV should not have thousands separators (raw numbers for spreadsheet compatibility)
- PDF summary should use formatted numbers with thousands separators for readability
- If a deduction type has zero total across all employees, it can be omitted from the PDF summary or shown as zero
