# [GW] Payroll - Configure Roles and Permissions with Spatie

## Description

Install and configure the `spatie/laravel-permission` package to implement the four-role authorization system for PayHRIS Payroll. This ticket creates the role definitions, seeds them into the database, and provides helper utilities for role checking in controllers, policies, and middleware. The roles are: `owner`, `payroll_officer`, `finance`, and `employee`.

## Technical Implementation

### What Migrations
- Run the Spatie migrations: `php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"`, then `php artisan migrate`.
- This creates the standard Spatie tables: `roles`, `permissions`, `model_has_roles`, `model_has_permissions`, `role_has_permissions`.

### What Models
- Update `app/Models/User.php` (or current User model location) to add the `HasRoles` trait from `Spatie\Permission\Traits\HasRoles`.

### Routes & Controllers
- Register Spatie's `RoleMiddleware` and `PermissionMiddleware` in `bootstrap/app.php`:
  ```php
  ->withMiddleware(function (Middleware $middleware) {
      $middleware->alias([
          'role' => \Spatie\Permission\Middleware\RoleMiddleware::class,
          'permission' => \Spatie\Permission\Middleware\PermissionMiddleware::class,
      ]);
  })
  ```
- Routes can now use `->middleware('role:owner')` or `->middleware('role:owner|payroll_officer')`.

### Environment & Config
- Publish Spatie config: `php artisan vendor:publish --tag="permission-config"`.
- Configure `config/permission.php`:
  - Set `teams` to `false` (single-company model, no team-based permissions).
  - Set `cache.expiration_time` to a reasonable value (e.g., 1440 minutes).
- Ensure the User model's guard matches the Spatie config guard.

### Seeders
- Create `database/seeders/RoleSeeder.php`:
  ```php
  Role::create(['name' => 'owner', 'guard_name' => 'web']);
  Role::create(['name' => 'payroll_officer', 'guard_name' => 'web']);
  Role::create(['name' => 'finance', 'guard_name' => 'web']);
  Role::create(['name' => 'employee', 'guard_name' => 'web']);
  ```
- Update `database/seeders/DatabaseSeeder.php` to call `RoleSeeder`.
- Create `database/seeders/AdminUserSeeder.php` that creates a default admin user and assigns the `owner` role:
  ```php
  $user = User::factory()->create([
      'name' => 'Admin',
      'email' => 'admin@payhris.com',
      'password' => Hash::make('password'),
  ]);
  $user->assignRole('owner');
  ```

### Shared Inertia Props
- Update `HandleInertiaRequests` middleware to share the authenticated user's roles:
  ```php
  'auth' => [
      'user' => $request->user() ? [
          'id' => $request->user()->id,
          'name' => $request->user()->name,
          'email' => $request->user()->email,
          'roles' => $request->user()->getRoleNames(),
      ] : null,
  ],
  ```
- Create `resources/js/hooks/use-role.ts` helper hook:
  ```typescript
  export function useRole() {
      const { auth } = usePage().props;
      const hasRole = (role: string) => auth.user?.roles?.includes(role);
      const isOwner = hasRole('owner');
      // ... etc.
      return { hasRole, isOwner, isPayrollOfficer, isFinance, isEmployee };
  }
  ```

### Tests
- `tests/Feature/RoleTest.php` (Pest):
  - Asserts all 4 roles exist after seeding.
  - Asserts a user can be assigned and checked for a role (`$user->hasRole('owner')` returns true).
  - Asserts role middleware blocks unauthorized access (a user with `employee` role cannot access an `owner`-only route).
  - Asserts the admin seeder creates a user with the `owner` role.
- `tests/Feature/SharedPropsTest.php` (Pest):
  - Asserts authenticated Inertia responses include the user's roles in shared props.

### Edge Cases & Notes
- Roles are authorization labels only; granular permissions are not needed at this stage but the infrastructure supports adding them later.
- The `employee` role is the most restrictive: read-only access to own finalized payslips only.
- Role checks on the frontend are UX-only. All enforcement must happen server-side via middleware and policies.

## Acceptance Criteria

- [ ] `spatie/laravel-permission` package is installed and configured.
- [ ] Spatie migrations run successfully, creating roles/permissions tables.
- [ ] `RoleSeeder` seeds all 4 roles: `owner`, `payroll_officer`, `finance`, `employee`.
- [ ] `php artisan db:seed` runs without errors and all roles are present in the `roles` table.
- [ ] `AdminUserSeeder` creates a default user with the `owner` role.
- [ ] `$user->hasRole('owner')` returns `true` for the seeded admin user.
- [ ] `$user->assignRole('finance')` successfully assigns a role to a user.
- [ ] Role middleware (`role:owner`) protects routes and returns 403 for unauthorized roles.
- [ ] `HandleInertiaRequests` shares the authenticated user's roles in page props.
- [ ] `use-role.ts` hook is available for frontend role checking.
- [ ] All Pest tests pass (`php artisan test --filter=Role`).
- [ ] Spatie permission cache can be cleared via `php artisan permission:cache-reset`.
