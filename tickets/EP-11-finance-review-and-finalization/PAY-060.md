# [GW] Payroll - Finance Review Page for Submitted Runs

## Description

Build the finance review page where Finance users can review a submitted payroll run in detail, verify all figures, flag discrepancies, and either approve or return the run for corrections. The review page shows all payslip details in read-only mode with the ability to add flags and comments on individual line items.

## Technical Implementation

### What Migrations

- Create the payslip_flags table with ULID primary key, payslip and line item references (line item nullable), flagged-by and resolved-by user references, note, status (open/resolved), resolved note (nullable), and standard timestamps.

### What Models

- Create PayslipFlag model in the Payroll domain with ULID support.
- Relationships: belongs to Payslip, optionally belongs to PayslipLineItem, belongs to flagged-by User, optionally belongs to resolved-by User.

### What Actions

- Move to Under Review action: transitions the run from submitted to under review and records reviewer and timestamp.
- Flag Payslip Line action: creates a flag on a payslip or line item with a note and open status.

### Routes & Controllers

- GET /finance/payroll-runs/{payrollRun}/review — finance review page showing run summary, payslips with line-item detail, existing flags, and available actions.

### Policies & Authorization

- Finance and Owner roles can access the review page.
- Payroll Officer and Employee roles cannot access (403).
- Run must be in submitted or under review state.

### Tests

- Assert Finance user can access the review page for a submitted run.
- Assert Employee role cannot access the review page (403).
- Assert Payroll Officer cannot access the review page (403).
- Assert Finance can place a run under review (state transition).
- Assert Finance can flag a line item with a note.
- Assert Finance can return a run to draft.
- Assert Finance cannot edit payslip figures directly.
- Assert existing flags and their resolution status are visible.

## Acceptance Criteria

### Client-side

#### Layout

- The finance review page is accessible from the payroll runs list for Finance and Owner roles

#### Behavior

- The page shows the run summary header with state badge and financial totals
- Employee-by-employee payslip breakdowns are expandable, showing all earnings, bonuses, and deductions as line items
- A flag icon on each line item allows Finance users to add a flag with a text note
- Flagged items are highlighted visually (orange/red) for all authorized users
- Existing flags show inline with their status (open/resolved)
- A "Place Under Review" button is visible when the run is in submitted state
- A "Return to Draft" button with reason dialog is available
- All figure fields are read-only — Finance cannot edit payslip values directly

---

### Server-side

#### Side-effects

- Finance and Owner roles can access the review page; other roles are denied
- Placing a run under review transitions the state from submitted to under review and records who reviewed it and when
- Flags can be added to payslip line items with a note and are stored with an open status
- Finance cannot edit payslip figures directly — all fields are read-only
