# [GW] Payroll - Build Authenticated Layout with Sidebar Navigation

## Description

Create the primary authenticated layout shell that wraps all post-login pages. The layout includes a collapsible sidebar with role-based navigation links, a top header with user information and logout, and a flash message display system. This is the visual foundation for every authenticated page and must be mobile-responsive.

## Technical Implementation

### Edge Cases & Notes

- Sidebar navigation filtering is UX-only; route-level middleware enforces actual access control.
- Flash messages from the session must be picked up by the Inertia shared props middleware.
- The layout must not cause full page reloads when navigating between links (Inertia SPA navigation).
- Sidebar collapsed state should persist across page navigations using local storage.

## Acceptance Criteria

### Client-side

#### Layout

- The layout consists of a sidebar on the left, a header across the top, and a main content area filling the remaining space

#### Positioning

- The sidebar displays navigation links grouped by section
- The header displays the application name, user information with role badge, and a logout button
- Flash messages appear as toast notifications at the top-right of the viewport

#### Screen-size Friendliness

- On desktop, the sidebar is visible and collapsible via a toggle button
- On mobile, the sidebar collapses into a slide-out drawer triggered by a hamburger button in the header

#### Behavior

- Navigation links are filtered based on the logged-in user's role:
  - Owner sees all navigation links
  - Payroll Officer sees Dashboard, Employees, Payroll Runs, Cash Advances, and Bonus Types
  - Finance sees Dashboard and Payroll Runs
  - Employee sees only Dashboard and My Payslips
- The active page's navigation link is visually highlighted
- Clicking a navigation link navigates without a full page reload
- The sidebar collapsed/expanded state persists across page navigations
- The logout button signs the user out and redirects to the login page
- Success flash messages appear with a green accent; error messages with a red accent
- Flash messages auto-dismiss after 5 seconds with a manual close button available
