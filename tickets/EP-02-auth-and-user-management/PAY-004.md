# [GW] Payroll - Authentication System

## Description

- Setup session authentication using Laravel Breeze.
- Integrate Inertia and React stack completely.
- Create users table with ULID keys.
- Add company and employee references.
- Add active status tracking firmly.
- Relocate User model to Company domain.
- Provide login and registration access.

## Technical Implementation

### What Migrations

- Customize users table with ULID keys.
- Add company and optional employee references.
- Add active flag and login timestamp.
- Include Breeze reset tokens and sessions.

### What Models

- Move User model to Company domain.
- Extend Authenticatable with ULID support.
- Add Spatie roles and Notifiable trait.
- Define company and employee relationships.
- Add role helpers to user factory.

### Validation & DTOs

- Use Breeze standard Form Requests.
- Frontend uses Inertia inline form validation.

### Routes & Controllers

- Install Laravel Breeze scaffolding thoroughly.
- Add login, register, reset, verification routes.
- Authed users redirect to dashboard.
- Guests redirect to login page.

### Tests

- Assert valid login redirects to dashboard.
- Assert invalid credentials return firm errors.
- Assert guests redirect to login directly.
- Assert valid data creates new user.
- Assert logout completely destroys user session.
- Assert user creation with ULID works.
- Assert factory helpers produce correct roles.
- Assert authentication works with relocated model.

### Environment & Config

- Set session driver to database.
- Configure auth guards for web guard.
- Configure secure remember me functionality.
- Update model location in auth config.

### Edge Cases & Notes

- Restore any overwritten files from Breeze.
- Employee reference remains fully nullable.
- Last login timestamp updates upon success.

## Acceptance Criteria

### Client-side

#### Behavior

- Login accepts valid email and password.
- Registration creates brand new account directly.
- Forgot password requests reset link heavily.
- Users redirect to dashboard after login.
- Logout clears session unconditionally.
- Invalid login shows error on form.
- Remember me keeps session persistently active.

#### Validation

- Registration shows inline validation errors.
- Login shows credential matching error.

### Server-side

#### Side-effects

- Registration persists new user properly.
- Users link to distinct company record.
- Unauthenticated requests bounce to login.
- Inactive users absolutely cannot log in.
