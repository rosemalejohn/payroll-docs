# PAY-062: Finalize Payroll Run Action (Owner Only)

## Description

Implement the finalization step for payroll runs, restricted exclusively to users with the `owner` role. Finalization is the terminal, irreversible state transition (`under_review` -> `finalized`) that permanently locks the payroll run, all associated payslips, and the payroll period. Upon finalization, the system dispatches jobs for payslip PDF generation, cash advance recovery processing, and notifications to payroll officers and employees.

Pre-finalization checks ensure all flags are resolved and the run is in the `under_review` state. Once finalized, no further edits are possible.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/FinalizePayrollRunAction.php`** — Single-responsibility action that:
  - Validates the payroll run is in the `under_review` state
  - Runs pre-finalization checks (all flags resolved, data integrity verified)
  - Transitions the run state from `under_review` to `finalized` via spatie/laravel-model-states
  - Sets `finalized_by` (authenticated user ULID) and `finalized_at` (current timestamp)
  - Locks all associated payslips (`is_locked = true`)
  - Locks the payroll period (`is_locked = 1`)
  - Dispatches `PayrollRunFinalizedEvent`
  - Wraps all mutations in a database transaction

### Models
- **`app/Payroll/Models/PayrollRun.php`** — Add:
  - `finalized_by` (nullable ULID) and `finalized_at` (nullable datetime) to `$fillable`
  - `finalizedBy()` belongsTo relationship to User
  - `isFinalized(): bool` accessor
- **`app/Payroll/Models/Payslip.php`** — Ensure `is_locked` boolean field exists and is cast
- **`app/Payroll/Models/PayrollPeriod.php`** — Ensure `is_locked` boolean field exists and is cast

### Migrations
- **`database/migrations/xxxx_add_finalization_fields_to_payroll_runs_table.php`**
  - Add `finalized_by` column (ULID, nullable)
  - Add `finalized_at` column (datetime, nullable)
  - Index on `finalized_by`

### Validation & DTOs
- **`app/Payroll/DTOs/FinalizePayrollRunData.php`** — spatie/laravel-data DTO containing:
  - `payroll_run_id`: string (ULID)
- **`app/Http/Requests/Payroll/FinalizePayrollRunRequest.php`** — Validates run exists and is in the correct state

### Routes & Controllers
- **`POST /payroll-runs/{payrollRun}/finalize`** — Route name: `payroll-runs.finalize`
- **`app/Http/Controllers/Web/PayrollRunFinalizationController@store`**
  - Middleware: `auth`, `role:owner`
  - Authorizes via `PayrollRunPolicy@finalize`
  - Delegates to `FinalizePayrollRunAction`
  - Redirects back with flash message

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Add `finalize()` method:
  - **Only** `owner` role can finalize
  - Run must be in `under_review` state
  - All flags must be resolved (no unresolved payslip flags)

### Events & Listeners
- **`app/Payroll/Events/PayrollRunFinalizedEvent.php`** — Contains payroll run ID and finalizer ID
- **`app/Payroll/Listeners/DispatchPayslipPdfGenerationListener.php`** — Dispatches `GeneratePayslipPdfJob` for each payslip in the run
- **`app/Payroll/Listeners/ProcessCashAdvanceRecoveryListener.php`** — Dispatches `ProcessCashAdvanceRecoveryJob` to update cash advance balances
- **`app/Payroll/Listeners/NotifyStakeholdersOfFinalizationListener.php`** — Notifies payroll officers and employees

### Jobs
- **`app/Payroll/Jobs/GeneratePayslipPdfJob.php`** — Queued on `payslips` queue (detailed in PAY-063)
- **`app/Payroll/Jobs/ProcessCashAdvanceRecoveryJob.php`** — Queued on `payroll` queue; processes cash advance deductions and updates remaining balances

### Notifications
- **`app/Payroll/Notifications/PayrollRunFinalizedNotification.php`** — Database and mail notification to payroll officers
- **`app/Payroll/Notifications/PayslipAvailableNotification.php`** — Database notification to employees informing them their payslip is available

### Tests
- **`tests/Feature/Payroll/FinalizePayrollRunTest.php`**
  - Owner can finalize an `under_review` run
  - Non-owner roles (`payroll_officer`, `finance`, `employee`) receive 403
  - Finalizing a run not in `under_review` state returns 422
  - Unresolved flags prevent finalization (returns 422)
  - `finalized_by` and `finalized_at` are correctly set
  - All payslips are locked after finalization
  - Payroll period is locked after finalization
  - `PayrollRunFinalizedEvent` is dispatched
  - PDF generation jobs are dispatched for each payslip
  - Cash advance recovery job is dispatched
  - Payroll officers receive finalization notification
  - Employees receive payslip-available notification
  - Finalization is recorded in audit trail
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- Finalization is irreversible — there is no state transition out of `finalized`
- All operations must be wrapped in a database transaction; if any step fails, the entire finalization rolls back
- The `GeneratePayslipPdfJob` is dispatched per payslip to allow parallel processing and individual retry on failure
- Cash advance recovery must correctly decrement outstanding balances and mark fully recovered advances as complete
- Race condition: use database-level locking (`lockForUpdate`) on the payroll run row to prevent concurrent finalization attempts

## Acceptance Criteria

- [ ] Only users with the `owner` role can finalize a payroll run
- [ ] Finalized payroll runs are permanently locked with no further edits possible
- [ ] Payroll period associated with the run is locked (`is_locked = 1`)
- [ ] All payslips in the run are locked (`is_locked = true`)
- [ ] `finalized_by` and `finalized_at` are correctly recorded
- [ ] PDF generation is triggered for each payslip upon finalization
- [ ] Cash advance balances are updated based on deductions in the finalized run
- [ ] Payroll officers are notified of finalization
- [ ] Employees are notified that their payslips are available
- [ ] Finalize button is hidden for non-owner roles (including `payroll_officer` and `finance`)
- [ ] API returns 403 for unauthorized finalization attempts by non-owner roles
- [ ] Runs with unresolved flags cannot be finalized
- [ ] Finalization action is logged in the audit trail via `owen-it/laravel-auditing`
