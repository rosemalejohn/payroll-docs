# [GW] Payroll - Dashboard Page with Payroll Overview

## Description

Build a role-specific dashboard page that displays relevant payroll information and quick actions upon login. Owners see pending actions, recent runs, and employee count. Payroll officers see their draft and submitted runs. Finance users see the review queue. Employees see their latest payslip summary and year-to-date earnings.

## Technical Implementation

### Queries

- Dashboard Stats query: aggregates role-specific dashboard data efficiently. Uses aggregate queries for counts and sums, eager loads only needed relationships. Caches expensive aggregates for 5 minutes via Redis.

### Routes & Controllers

- `GET /dashboard` — role-specific dashboard page. Returns different data sets based on the authenticated user's role.

### Policies & Authorization

- All authenticated users can access the dashboard.
- Data is scoped by role — employees cannot see admin-level data (run management, employee lists, payroll costs).

### Tests

- Assert Owner sees pending finalization count and recent runs.
- Assert Payroll Officer sees draft and submitted runs.
- Assert Finance sees the review queue.
- Assert Employee sees latest payslip summary and year-to-date earnings.
- Assert Employee does not see admin-level data.
- Assert dashboard loads efficiently (no N+1 queries).

### Edge Cases & Notes

- Aggregate queries should be cached to avoid performance issues on every page load.
- Year-to-date earnings for employees should sum net pay from all finalized payslips in the current calendar year.
- If no payroll runs exist yet, dashboard should show appropriate empty states rather than errors.

## Acceptance Criteria

### Client-side

#### Layout

- The dashboard is the landing page after login for all authenticated users
- Content is displayed in a card-based layout

#### Behavior

- **Owner view:** displays recent payroll runs (last 5) with state badges, total payroll cost for the current month, count of runs awaiting finalization, total active employee count, and quick action buttons for "Create New Run", "Review Pending", and "Manage Employees"
- **Payroll Officer view:** displays current draft runs, submitted runs awaiting review, recent activity entries, and quick action buttons for "Create New Run" and "View Draft Runs"
- **Finance view:** displays runs awaiting review with count, recently finalized runs (last 5), and quick action buttons for "Review Queue" and "Export Reports"
- **Employee view:** displays latest payslip summary (period, gross pay, deductions, net pay), year-to-date earnings total, and quick action buttons for "View All Payslips" and "Download Latest"
- Quick action buttons navigate to the correct pages
- Empty states are shown when no data is available (e.g., no payroll runs yet)

---

### Server-side

#### Side-effects

- Dashboard data is scoped by the authenticated user's role
- Employees can only see their own payslip data — no access to run management or employee lists
- Aggregate data (counts, totals) is cached for performance
- Year-to-date earnings are calculated from all finalized payslips in the current calendar year
