# [GW] Payroll - Dashboard Page with Payroll Overview

## Description

Build a role-specific dashboard page that displays relevant payroll information and quick actions upon login. Owners see pending actions, recent runs, and employee count. Payroll officers see their draft and submitted runs. Finance users see the review queue. Employees see their latest payslip summary and year-to-date earnings.

## Technical Implementation

### Queries

- Aggregate role-specific dashboard stats and cache expensive results.

### Routes & Controllers

- `GET /dashboard` returns role-specific dashboard data.

### Policies & Authorization

- All authenticated users can access the dashboard.
- Scope dashboard data by authenticated user role.

### Tests

- Assert Owner sees pending finalization count and recent runs.
- Assert Payroll Officer sees draft and submitted runs.
- Assert Finance sees the review queue.
- Assert Employee sees payslip summary and year-to-date earnings.
- Assert Employee does not see admin-level data.
- Assert dashboard loads efficiently (no N+1 queries).

### Edge Cases & Notes

- Cache aggregate queries to avoid repeated expensive loads.
- Year-to-date earnings sum finalized payslips this calendar year.
- Show empty states when payroll data is unavailable.

## Acceptance Criteria

### Client-side

#### Layout

- Show dashboard after login for authenticated users.
- Display dashboard content in cards.

#### Behavior

- Owner view shows runs, counts, payroll cost, quick actions.
- Officer view shows drafts, submissions, activity, quick actions.
- Finance view shows reviews, finalized runs, quick actions.
- Employee view shows payslip summary, earnings, quick actions.
- Quick actions navigate to the correct pages.
- Show empty states when no data exists.

---

### Server-side

#### Side-effects

- Scope dashboard data by authenticated user role.
- Employees only see their own payslip data.
- Cache aggregate counts and totals for performance.
- Calculate year-to-date earnings from finalized payslips.
