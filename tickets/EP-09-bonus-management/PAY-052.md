# [GW] Payroll - AutoAttachBonusesToRunAction

## Description

Implement the action that automatically attaches applicable bonus types to a payroll run at creation time. When a new payroll run is created, the system iterates over all active bonus types with auto-trigger enabled and attaches those whose auto-trigger month matches the run's payroll period month (or is null, meaning every month). Each attached bonus creates a payroll run bonus record with default values copied from the bonus types catalog. This ensures recurring bonuses like Meal Allowance (every month), 13th Month Pay (December), and June Special Bonus (June) are automatically included.

## Technical Implementation

### What Actions

- Auto Attach Bonuses to Run action: accepts a payroll run, determines the period month from the payroll period's start date. Queries all active bonus types for the company with auto-trigger enabled where the auto-trigger month matches the period month or is null. For each eligible bonus type, creates a payroll run bonus record with defaults copied from the bonus type catalog (default amount, default rate, percentage basis, attendance threshold days). Sets apply-to-all to true and is-active to true. Handles duplicate attachment attempts gracefully (skip if already attached). Called automatically inside the Create Payroll Run action after the run is created and currency snapshots are taken.

### Tests

- Assert December run attaches Meal Allowance and 13th Month Pay, skips June Special.
- Assert June run attaches Meal Allowance and June Special, skips 13th Month.
- Assert March run attaches only Meal Allowance (no month-specific bonuses match).
- Assert Attendance Bonus is never auto-attached (auto-trigger disabled).
- Assert inactive bonus types are not attached even if auto-trigger is enabled.
- Assert custom bonus with auto-trigger enabled and null month is attached every month.
- Assert custom bonus with auto-trigger enabled and specific month is attached only in that month.
- Assert default values are correctly copied from the bonus types catalog.
- Assert created records have apply-to-all and is-active set to true by default.
- Assert duplicate attachment is prevented by the unique constraint.

### Edge Cases & Notes

- If a bonus type has been deactivated since the last run, it will not be auto-attached.
- Custom bonus types with auto-trigger enabled follow the exact same logic as system bonus types.
- The action does not trigger payslip computation — that is handled separately after employee assignment.

## Acceptance Criteria

### Server-side

#### Side-effects

- Meal Allowance (auto-trigger enabled, no specific month) is attached to every payroll run regardless of month
- 13th Month Pay (auto-trigger enabled, December) is attached only to payroll runs whose period starts in December
- June Special Bonus (auto-trigger enabled, June) is attached only to payroll runs whose period starts in June
- Attendance Bonus (auto-trigger disabled) is never auto-attached; it must be manually added
- Custom bonus types with auto-trigger enabled follow the same month-matching logic
- Default values (amount, rate, percentage basis, attendance threshold) are copied from the bonus types catalog
- Created payroll run bonus records have apply-to-all and is-active set to true by default
- Inactive bonus types are not attached even if auto-trigger is enabled
- Duplicate attachment attempts are handled gracefully without errors
- The action is called automatically during payroll run creation
