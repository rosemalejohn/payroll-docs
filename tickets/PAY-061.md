# [GW] Payroll - Approve Payroll Run for Finalization (Finance)

## Description

Implement the finance approval step for payroll runs. Finance users can approve a payroll run that is currently in the `under_review` state, marking it as ready for the owner to finalize. This is an intermediate approval step — finance cannot finalize the run directly. Upon approval, the owner is notified that the run is ready for finalization.

The approval adds `approved_by` and `approved_at` fields to the payroll run record. The run remains in the `under_review` state but is flagged as approved, distinguishing it from runs still awaiting finance review.

## Technical Implementation

### Actions
- **`app/Payroll/Actions/ApprovePayrollRunAction.php`** — Single-responsibility action that:
  - Validates the payroll run is in the `under_review` state
  - Validates the run has not already been approved
  - Sets `approved_by` (authenticated user ULID) and `approved_at` (current timestamp)
  - Dispatches `PayrollRunApprovedEvent`

### Models
- **`app/Payroll/Models/PayrollRun.php`** — Add the following:
  - `approved_by` (nullable ULID) and `approved_at` (nullable datetime) to `$fillable`
  - `approvedBy()` belongsTo relationship to the User model
  - `isApproved(): bool` accessor method
  - Auditable trait from `owen-it/laravel-auditing`

### Migrations
- **`database/migrations/xxxx_add_approval_fields_to_payroll_runs_table.php`**
  - Add `approved_by` column (ULID, nullable)
  - Add `approved_at` column (datetime, nullable)
  - Index on `approved_by`

### Validation & DTOs
- **`app/Payroll/DTOs/ApprovePayrollRunData.php`** — spatie/laravel-data DTO containing:
  - `payroll_run_id`: string (ULID)
- **`app/Http/Requests/Payroll/ApprovePayrollRunRequest.php`** — Validates that the payroll run exists and is in the correct state

### Routes & Controllers
- **`POST /payroll-runs/{payrollRun}/approve`** — Route name: `payroll-runs.approve`
- **`app/Http/Controllers/Web/PayrollRunApprovalController@store`**
  - Middleware: `auth`, `role:finance`
  - Authorizes via `PayrollRunPolicy@approve`
  - Delegates to `ApprovePayrollRunAction`
  - Redirects back with flash message

### Policies & Authorization
- **`app/Payroll/Policies/PayrollRunPolicy.php`** — Add `approve()` method:
  - Only `finance` role can approve
  - Run must be in `under_review` state
  - Run must not already be approved

### Events & Listeners
- **`app/Payroll/Events/PayrollRunApprovedEvent.php`** — Contains payroll run ID and approver ID
- **`app/Payroll/Listeners/NotifyOwnerOfApprovalListener.php`** — Sends notification to all users with `owner` role

### Notifications
- **`app/Payroll/Notifications/PayrollRunApprovedNotification.php`** — Database and mail notification to owner with payroll run details and link to finalize

### Tests
- **`tests/Feature/Payroll/ApprovePayrollRunTest.php`**
  - Finance user can approve an `under_review` payroll run
  - Non-finance users (payroll_officer, employee, owner) receive 403
  - Approving a run not in `under_review` state returns 422
  - Approving an already-approved run returns 422
  - `approved_by` and `approved_at` are correctly set after approval
  - Owner receives notification after approval
  - Approval is recorded in audit trail
  - Unauthenticated user is redirected to login

### Edge Cases & Notes
- A payroll run can only be approved once; re-approval attempts should return a validation error
- If the owner rejects and sends the run back to a previous state, the approval fields should be cleared
- Audit trail must capture the approval action with before/after state via `owen-it/laravel-auditing`

## Acceptance Criteria

- [ ] Finance users can approve payroll runs that are in the `under_review` state
- [ ] Owner is notified (database + email) when a payroll run is approved
- [ ] Finance users cannot finalize a payroll run directly — only approve
- [ ] Non-finance roles receive a 403 response when attempting to approve
- [ ] Payroll runs not in `under_review` state cannot be approved
- [ ] Already-approved runs cannot be re-approved
- [ ] `approved_by` and `approved_at` fields are correctly populated
- [ ] Approval action is logged in the audit trail via `owen-it/laravel-auditing`
- [ ] Approve button is visible only to finance users on `under_review` runs
- [ ] Flash message confirms successful approval
