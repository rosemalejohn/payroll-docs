# [GW] Payroll - Build Shared Props Middleware

## Description

Create and configure the Inertia middleware to share cross-cutting data with every page response. Shared props include the authenticated user (with roles and permissions), flash messages, application name, and unread notification count. Corresponding frontend types must be created for type-safe consumption.

## Technical Implementation

### Routes & Controllers

- No new routes or controllers. The middleware is applied globally to the web middleware group.

### Tests

- Assert unauthenticated requests have no user data shared.
- Assert authenticated requests include user information with roles and permissions.
- Assert flash messages are populated from session data.
- Assert the application name matches the configured value.
- Assert notification count is included for authenticated users.
- Assert no sensitive data (password, tokens) is exposed in shared props.

### Edge Cases & Notes

- Shared props are serialized on every response — keep them lightweight.
- Permission lookups can be expensive; consider caching if performance becomes an issue.
- Flash messages follow standard session flash behavior (appear once, then cleared).
- Roles and permissions must be plain lists for frontend consumption.

## Acceptance Criteria

### Client-side

#### Behavior

- Every page shows the logged-in user's name and role information when authenticated
- Flash messages (success and error) appear when triggered by server actions and disappear after being shown
- The application name is displayed consistently across all pages
- The notification bell shows the correct count of unread notifications

---

### Server-side

#### Side-effects

- Authenticated requests include user information with their assigned roles and permissions
- Guest/unauthenticated requests do not include any user data
- No sensitive information (passwords, tokens) is ever sent to the frontend
