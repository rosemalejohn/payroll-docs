# PAY-012: Create Payroll Periods Migration and Model

## Description

Create the `payroll_periods` table, Eloquent model, and factory. Payroll periods define the date ranges and working day counts for payroll processing cycles. Each period belongs to a company and can be of type monthly, semi-monthly, or weekly. A locked period prevents new payroll runs from being created against it, ensuring finalized periods remain immutable.

## Technical Implementation

### What Migrations
- Create migration `database/migrations/xxxx_xx_xx_create_payroll_periods_table.php`:
  ```php
  Schema::create('payroll_periods', function (Blueprint $table) {
      $table->char('id', 26)->primary();                          // ULID
      $table->char('company_id', 26);                              // ref: companies.id
      $table->string('name', 150);                                 // e.g. "March 2026", "Mar 1-15, 2026"
      $table->enum('period_type', ['monthly', 'semi_monthly', 'weekly']);
      $table->date('start_date');
      $table->date('end_date');
      $table->integer('working_days')->default(22);
      $table->boolean('is_locked')->default(false);
      $table->char('created_by', 26)->nullable();                  // ref: users.id
      $table->timestamps();

      $table->index('company_id', 'idx_payroll_periods_company');
  });
  ```

### What Models
- Create `app/Payroll/Models/PayrollPeriod.php`:
  ```php
  namespace App\Payroll\Models;

  use Illuminate\Database\Eloquent\Concerns\HasUlids;
  use Illuminate\Database\Eloquent\Factories\HasFactory;
  use Illuminate\Database\Eloquent\Model;

  class PayrollPeriod extends Model
  {
      use HasUlids, HasFactory;

      protected $fillable = [
          'company_id', 'name', 'period_type',
          'start_date', 'end_date', 'working_days',
          'is_locked', 'created_by',
      ];

      protected $casts = [
          'start_date' => 'date',
          'end_date' => 'date',
          'working_days' => 'integer',
          'is_locked' => 'boolean',
      ];

      // Relationships
      public function company()
      {
          return $this->belongsTo(\App\Company\Models\Company::class);
      }

      public function payrollRuns()
      {
          return $this->hasMany(\App\Payroll\Models\PayrollRun::class);
      }

      public function creator()
      {
          return $this->belongsTo(\App\Models\User::class, 'created_by');
      }

      // Scopes
      public function scopeUnlocked($query)
      {
          return $query->where('is_locked', false);
      }

      public function scopeLocked($query)
      {
          return $query->where('is_locked', true);
      }

      public function scopeByCompany($query, string $companyId)
      {
          return $query->where('company_id', $companyId);
      }

      public function scopeByType($query, string $periodType)
      {
          return $query->where('period_type', $periodType);
      }

      // Helpers
      public function isLocked(): bool
      {
          return $this->is_locked;
      }

      public function canCreateRun(): bool
      {
          return !$this->is_locked;
      }

      public function lock(): void
      {
          $this->update(['is_locked' => true]);
      }
  }
  ```

### Factories
- Create `database/factories/PayrollPeriodFactory.php`:
  ```php
  namespace Database\Factories;

  class PayrollPeriodFactory extends Factory
  {
      protected $model = \App\Payroll\Models\PayrollPeriod::class;

      public function definition(): array
      {
          $startDate = fake()->dateTimeBetween('-6 months', 'now');
          $endDate = (clone $startDate)->modify('+1 month -1 day');

          return [
              'company_id' => \App\Company\Models\Company::factory(),
              'name' => 'Payroll ' . $startDate->format('F Y'),
              'period_type' => 'monthly',
              'start_date' => $startDate,
              'end_date' => $endDate,
              'working_days' => 22,
              'is_locked' => false,
          ];
      }

      public function monthly(): static
      {
          return $this->state(['period_type' => 'monthly', 'working_days' => 22]);
      }

      public function semiMonthly(): static
      {
          return $this->state(['period_type' => 'semi_monthly', 'working_days' => 11]);
      }

      public function weekly(): static
      {
          return $this->state(['period_type' => 'weekly', 'working_days' => 5]);
      }

      public function locked(): static
      {
          return $this->state(['is_locked' => true]);
      }
  }
  ```

### Edge Cases & Notes
- The `is_locked` flag is the primary mechanism to prevent new payroll runs from being created against a period. Controllers/actions should check `canCreateRun()` before allowing run creation.
- `working_days` defaults to 22 (standard Philippine monthly working days) but can be adjusted for periods with holidays or partial months.
- Period date ranges are not validated for overlap at the database level; this should be enforced at the application/validation layer.
