# PAY-040: Payroll Run Detail Page

## Description

As a payroll officer, I want to view a detailed payroll run page with employee payslips, bonuses, currency rates, and audit history organized in tabs so that I can manage and review the run effectively.

## Technical Implementation

### Controller
- **`PayrollRunController@show`** — `GET /payroll-runs/{payrollRun}`
- Returns run with relationships based on active tab
- Eager loads based on context to avoid N+1

### Frontend
- **`resources/js/pages/payroll-runs/show.tsx`**

### Layout
- **Header**: run name, period, state badge, totals (gross, deductions, net), employee count
- **Action bar** (role and state dependent):
  - Draft: "Submit for Review" button (payroll_officer/owner)
  - Submitted: "Place Under Review" (finance/owner)
  - Under Review: "Approve" (finance), "Finalize" (owner only)
  - Any non-finalized: "Return to Draft" with reason
- **Tabs**:
  1. **Employees**: payslip table with employee name, gross, total bonuses, total deductions, net pay, absent days (editable in draft)
  2. **Bonuses**: attached bonuses with amount/rate, apply_to_all status, add/remove/edit (draft only)
  3. **Currency Rates**: snapshotted rates with code, rate, snapshot date
  4. **Audit History**: timeline of state transitions and changes

### State-Based Behavior
- **Draft**: all fields editable, absent days inline edit, bonus management active
- **Submitted/Under Review**: read-only, review actions visible to finance
- **Finalized**: fully read-only, download actions (PDF, CSV)

### Tests
- Detail page loads with correct data
- Tabs switch correctly
- Draft mode allows edits
- Finalized mode is read-only
- Role-based actions visible appropriately

## Acceptance Criteria

- [ ] Run detail page shows name, period, state, and financial totals
- [ ] Employees tab shows payslip summary per employee
- [ ] Bonuses tab shows attached bonuses with configuration
- [ ] Currency Rates tab shows snapshotted exchange rates
- [ ] Audit History tab shows timeline of changes
- [ ] Draft runs are editable (absent days, bonuses)
- [ ] Submitted/under_review runs are read-only with review actions
- [ ] Finalized runs are fully read-only with download options
- [ ] Role-based action buttons: finance sees review, owner sees finalize
