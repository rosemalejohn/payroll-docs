# [GW] Payroll - Build Authenticated Layout with Sidebar Navigation

## Description

Create the primary authenticated layout shell that wraps all post-login pages in the application. The layout includes a collapsible sidebar with role-based navigation links, a top header with user information and logout, and a flash message display system. This layout is the visual foundation for every authenticated page and must be mobile-responsive.

## Technical Implementation

### What Actions
- No domain actions in this ticket (frontend layout only).

### Routes & Controllers
- No new routes or controllers. The layout is consumed by Inertia pages via the `layout` property.

### Frontend Components

#### `resources/js/layouts/authenticated-layout.tsx`
- Accept `children` as the main content slot.
- Compose the layout from `Sidebar`, `Header`, and `FlashMessages` components.
- Read auth user and flash messages from Inertia shared props (`usePage().props`).
- Apply a CSS grid or flex layout: sidebar on the left, header on top, main content area fills remaining space.
- Sidebar collapses on mobile (hidden by default, toggled via hamburger button in header).
- Use Zustand or React state for sidebar open/closed state, persisting preference in localStorage.

#### `resources/js/components/layout/sidebar.tsx`
- Render navigation links grouped by section.
- Navigation items defined in a config array, each with: `label`, `href`, `icon` (Lucide icon), `requiredRoles` (array of role strings).
- Filter visible links based on the authenticated user's roles from shared props.
- Navigation structure:
  - **Dashboard** (`/dashboard`) - all roles
  - **Employees** (`/employees`) - owner, payroll_officer
  - **Payroll Runs** (`/payroll-runs`) - owner, payroll_officer, finance
  - **Cash Advances** (`/cash-advances`) - owner, payroll_officer
  - **Currencies** (`/currencies`) - owner
  - **Bonus Types** (`/bonus-types`) - owner, payroll_officer
  - **My Payslips** (`/my-payslips`) - employee
  - **Settings** (`/settings`) - owner
- Highlight active link using `usePage().url` matching.
- Use Shadcn `Button` (variant `ghost`) and `Tooltip` for collapsed sidebar state.
- Collapsible toggle button (chevron icon) at the bottom of the sidebar.
- On mobile (below `lg` breakpoint): sidebar renders as a Shadcn `Sheet` (slide-out drawer) triggered by a hamburger button in the header.

#### `resources/js/components/layout/header.tsx`
- Display the application name (from shared props `appName`).
- Show the authenticated user's name and role badge (using Shadcn `Badge`).
- Mobile hamburger button (visible below `lg` breakpoint) to toggle the sidebar sheet.
- Logout button using Inertia `router.post('/logout')`.
- Optional: notification bell icon with unread count badge (from shared props).

#### `resources/js/components/layout/flash-messages.tsx`
- Read `flash.success` and `flash.error` from Inertia shared props.
- Render flash messages as toast-style notifications at the top-right of the viewport.
- Auto-dismiss after 5 seconds with a close button for manual dismissal.
- Style success messages with green accent, error messages with red accent.
- Use Shadcn `Toast` or a custom implementation with Tailwind animations.
- Handle multiple flash messages in a stack.

#### `resources/js/components/layout/role-badge.tsx`
- Small utility component rendering a Shadcn `Badge` with role-specific colors:
  - `owner`: purple
  - `payroll_officer`: blue
  - `finance`: green
  - `employee`: gray

### TypeScript Types

#### `resources/js/types/index.d.ts` (or `types/global.d.ts`)
- Define `AuthUser` type: `{ id: string; name: string; email: string; roles: string[]; permissions: string[] }`.
- Define `FlashMessages` type: `{ success?: string; error?: string }`.
- Define `SharedProps` type extending Inertia's page props.

### Tests
- No automated backend tests for this ticket.
- Manual verification: sidebar renders, links filter by role, flash messages appear, mobile sidebar toggles.

### Edge Cases & Notes
- Sidebar navigation filtering is UX-only; route-level middleware enforces actual access control.
- Flash messages from Laravel's `session()->flash()` must be picked up by the Inertia shared props middleware (PAY-025).
- The layout must not cause a full page reload when navigating between sidebar links (Inertia SPA navigation).
- Sidebar collapsed state should persist across page navigations.

## Acceptance Criteria

- [ ] `authenticated-layout.tsx` renders a sidebar, header, and main content area.
- [ ] Sidebar displays navigation links filtered by the authenticated user's roles.
- [ ] Owner role sees all navigation links; employee role sees only Dashboard and My Payslips.
- [ ] Active navigation link is visually highlighted based on the current URL.
- [ ] Header displays the user's name, role badge, and a working logout button.
- [ ] Logout button sends a POST to `/logout` and redirects to the login page.
- [ ] Flash messages (success and error) display as toast notifications and auto-dismiss after 5 seconds.
- [ ] Sidebar is collapsible on desktop; collapsed state persists in localStorage.
- [ ] On mobile viewports (below `lg`), sidebar renders as a slide-out sheet triggered by a hamburger button.
- [ ] Layout does not cause full page reloads when navigating between links.
- [ ] All components use Shadcn/ui primitives and Tailwind CSS for styling.
