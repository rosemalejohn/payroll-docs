# [GW] Payroll - Update Currency Rate Action

## Description

Allow the Owner to update exchange rates for foreign currencies. The system warns if the rate change affects open payroll runs. Rate changes are tracked in the audit trail.

## Technical Implementation

### What Actions

- Update Currency Rate action: validates the new rate is positive, prevents updating the base currency (PHP), updates the rate and rate update timestamp, and checks for open payroll runs with employees in this currency to return a warning if affected runs exist.

### Validation & DTOs

- Update Currency Rate request: validates rate (required, must be greater than zero).

### Routes & Controllers

- `PUT /settings/currencies/{currency}` — update currency rate (Owner only).

### Tests

- Assert rate updates correctly.
- Assert base currency (PHP) update is blocked.
- Assert rate of zero or negative is rejected.
- Assert audit log captures old and new rate values.
- Assert warning is returned when open payroll runs have employees in this currency.
- Assert only Owner can update rates.

## Acceptance Criteria

### Client-side

#### Behavior

- Clicking the Edit button on a currency row opens a modal showing the currency name and code (read-only) with an editable rate field
- The "Last updated" date and time is displayed for reference
- A live preview shows "1 [CODE] = ₱[rate]" as the user types
- If the rate change affects open payroll runs, a warning dialog lists the affected runs before saving

#### Validation

- An error is shown if the rate is zero or negative
- The PHP (Philippine Peso) rate cannot be edited

---

### Server-side

#### Side-effects

- The exchange rate is updated immediately on save
- The rate update timestamp is recorded
- Rate changes are captured in the audit trail with old and new values
- A warning is returned if open payroll runs contain employees paid in this currency
- Only the Owner role can update currency rates
