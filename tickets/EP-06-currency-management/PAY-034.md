# [GW] Payroll - Add New Currency

## Description

Allow the Owner to add a new currency to the system so that employees can be compensated in foreign currencies with proper exchange rate tracking.

## Technical Implementation

### What Actions

- Create Currency action: validates the code is unique per company, rate is positive, sets the currency as non-base and active, records the rate update timestamp.

### Validation & DTOs

- Store Currency request: validates code (required, uppercase, unique per company), name (required), symbol (required), and rate (required, must be greater than zero).

### Routes & Controllers

- `POST /settings/currencies` — create new currency (Owner only).

### Policies & Authorization

- Only the Owner role can create currencies.

### Tests

- Assert currency is created with correct data.
- Assert duplicate code per company is rejected.
- Assert rate of zero or negative is rejected.
- Assert non-owner roles receive 403.
- Assert new currency is immediately available in employee compensation currency selector.

## Acceptance Criteria

### Client-side

#### Behavior

- An "Add Currency" button is available on the currency settings page (Owner only)
- Clicking it opens a modal form with fields: Currency Code, Name, Symbol, and Exchange Rate
- A live preview shows the rate formula as the user types: "1 [CODE] = ₱[rate]"

#### Validation

- An error is shown if the currency code already exists in the system
- An error is shown if the rate is zero or negative

---

### Server-side

#### Side-effects

- A new currency is created as active and non-base
- The new currency is immediately available in the employee compensation currency selector
- Only the Owner role can add currencies
