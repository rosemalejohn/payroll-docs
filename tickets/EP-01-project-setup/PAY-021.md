# [GW] Payroll - Create Database Seeders for Initial Data

## Description

Create all database seeders required to bootstrap the PayHRIS Payroll application with essential system data. This includes the company record, roles and permissions, the PHP base currency, all 11 system salary components, the 4 built-in bonus types, and an admin (owner) user. A master `DatabaseSeeder` must orchestrate all seeders in the correct dependency order so that `php artisan db:seed` produces a fully functional starting state from a fresh database.

## Technical Implementation

### What Actions
- No domain actions in this ticket (seeder-only).

### What Migrations
- No new migrations. Seeders depend on tables created by prior migration tickets.

### Seeders

#### `database/seeders/CompanySeeder.php`
- Create a single default company record in the `companies` table.
- Fields: `id` (generated ULID), `name` ("PayHRIS Demo Company"), `code` ("PAYHRIS"), `is_active` (true).
- Use `Str::ulid()` for the ULID primary key.
- Store the company ID in a config or return it for dependent seeders.

#### `database/seeders/RoleSeeder.php`
- Uses `spatie/laravel-permission` to create 4 roles: `owner`, `payroll_officer`, `finance`, `employee`.
- Assign permissions to each role as defined in the project spec:
  - `owner`: full access (all permissions).
  - `payroll_officer`: create/manage payroll runs (draft through under_review), manage employees, manage cash advances.
  - `finance`: review payroll runs, transition under_review to finalized.
  - `employee`: read-only access to own finalized payslips.
- Guard name: `web`.

#### `database/seeders/CurrencySeeder.php`
- Seed the PHP base currency into the `currencies` table.
- Fields: `code` ("PHP"), `name` ("Philippine Peso"), `symbol` ("\u20b1"), `rate` (1.000000), `is_base` (1), `is_active` (1).
- `company_id` references the company created by `CompanySeeder`.

#### `database/seeders/SalaryComponentSeeder.php`
- Seed all 11 system salary components into `salary_components` with `is_system = 1`:

| code | name | type | is_taxable | sort_order |
|---|---|---|---|---|
| `BASIC_MONTHLY` | Basic Monthly Salary | earning | 1 | 10 |
| `DAILY_RATE` | Daily Rate | earning | 1 | 20 |
| `SSS` | SSS Contribution | deduction | 0 | 110 |
| `PHILHEALTH` | PhilHealth Contribution | deduction | 0 | 120 |
| `PAGIBIG` | Pag-IBIG Contribution | deduction | 0 | 130 |
| `HMO` | HMO Deduction | deduction | 0 | 140 |
| `MP2` | MP2 Voluntary Contribution | deduction | 0 | 150 |
| `SSS_WISP` | SSS WISP Voluntary | deduction | 0 | 160 |
| `CASH_ADVANCE` | Cash Advance Recovery | deduction | 0 | 170 |
| `BIR_TAX` | BIR Withholding Tax | deduction | 0 | 180 |
| `ABSENT_DEDUCTION` | Absent Deduction | deduction | 0 | 100 |

- All records use the company ID from `CompanySeeder` and `is_active = 1`.

#### `database/seeders/BonusTypeSeeder.php`
- Seed the 4 built-in bonus types into `bonus_types` with `is_system = 1`:

| code | name | computation_type | auto_trigger | auto_trigger_month | default_amount |
|---|---|---|---|---|---|
| `MEAL_ALLOWANCE` | Meal Allowance | fixed | 1 | NULL | configurable |
| `ATTENDANCE_BONUS` | Attendance Bonus | percentage | 0 | NULL | NULL |
| `13TH_MONTH` | 13th Month Pay | formula | 1 | 12 | NULL |
| `JUNE_SPECIAL` | June Special Bonus | formula | 1 | 6 | NULL |

- All records use the company ID from `CompanySeeder` and `is_active = 1`.

#### `database/seeders/AdminUserSeeder.php`
- Create a default owner user in the `users` table.
- Fields: `name` ("Admin User"), `email` ("admin@payhris.com"), `password` (bcrypt of "password"), `company_id` (from `CompanySeeder`), `is_active` (1), `email_verified_at` (now).
- Assign the `owner` role via `$user->assignRole('owner')`.

#### `database/seeders/DatabaseSeeder.php`
- Call seeders in dependency order:
  1. `CompanySeeder`
  2. `RoleSeeder`
  3. `CurrencySeeder`
  4. `SalaryComponentSeeder`
  5. `BonusTypeSeeder`
  6. `AdminUserSeeder`
- Use `$this->call([...])` for clean orchestration.

### Tests
- `tests/Feature/Seeders/DatabaseSeederTest.php` (Pest):
  - Assert `php artisan db:seed` runs without exception.
  - Assert `companies` table has 1 record.
  - Assert 4 roles exist (`owner`, `payroll_officer`, `finance`, `employee`).
  - Assert `currencies` table has 1 record with `code = 'PHP'` and `is_base = 1`.
  - Assert `salary_components` table has 11 records with `is_system = 1`.
  - Assert `bonus_types` table has 4 records with `is_system = 1`.
  - Assert admin user exists with email `admin@payhris.com` and has the `owner` role.
  - Assert admin user can authenticate with seeded credentials.

### Edge Cases & Notes
- Seeders must be idempotent: use `firstOrCreate` or `updateOrCreate` so re-running `db:seed` does not create duplicates.
- ULIDs are generated at seeder runtime, not hardcoded, to avoid collisions across environments.
- The admin password ("password") is for development only; production deployments must change it.

## Acceptance Criteria

- [ ] `currencies` table contains PHP with `is_base = 1` and `rate = 1.000000`.
