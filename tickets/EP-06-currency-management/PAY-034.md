# [GW] Payroll - Add New Currency Action

## Description

As an owner, I want to add a new currency to the system so that employees can be compensated in foreign currencies with proper exchange rate tracking.

## Technical Implementation

### Action
- **`app/Currency/Actions/CreateCurrencyAction.php`**
  - Validates code is unique per company
  - Validates rate > 0
  - Sets `is_base = 0`, `is_active = 1`, `rate_updated_at = now()`
  - Creates currency record

### Controller
- **`CurrencyController@store`** — `POST /settings/currencies`

### Form Request
- **`StoreCurrencyRequest`**
  - `code`: required, string, max:10, uppercase, unique per company
  - `name`: required, string, max:100
  - `symbol`: required, string, max:10
  - `rate`: required, numeric, gt:0

### Frontend
- "Add Currency" button on currency list page (owner only)
- Modal form with fields: code, name, symbol, rate
- Live preview as user types rate: "1 [CODE] = ₱[rate]"
- Validation errors displayed inline

### Tests
- Currency created with correct data
- Duplicate code per company rejected
- Rate ≤ 0 rejected
- Non-owner cannot create currency (403)
- New currency immediately available for employee compensation

## Acceptance Criteria

- [ ] New currency created with `is_base = 0`, `is_active = 1`
- [ ] Rate stored as DECIMAL(18,6)
- [ ] Unique constraint on `company_id + code` enforced
- [ ] Validation rejects rate ≤ 0
- [ ] Live formula preview shows "1 [CODE] = ₱[rate]" as user types
- [ ] New currency immediately available in compensation currency selector
- [ ] Only owner can add currencies
