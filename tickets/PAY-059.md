# PAY-059: Return Payroll Run to Draft

## Description

As a payroll officer or owner, I want to return a submitted or under-review payroll run back to draft status so that errors identified during review can be corrected. This action requires a reason comment and notifies relevant stakeholders.

## Technical Implementation

### Action
- **`app/Payroll/Actions/ReturnRunToDraftAction.php`** (lorisleiva/laravel-actions)
  - Accepts: `PayrollRun $payrollRun`, `User $returnedBy`, `string $reason`
  - Validates run is in `submitted` or `under_review` state (NOT `finalized`)
  - Transitions state back to `draft` via spatie/laravel-model-states
  - Stores reason in audit trail
  - Dispatches notifications:
    - If returning from `under_review`: notify the finance user who placed it under review
    - If returning from `submitted`: notify the payroll_officer who submitted it

### Controller
- **`PayrollRunController@returnToDraft`** — `POST /payroll-runs/{payrollRun}/return-to-draft`
- Middleware: `auth`, `role:payroll_officer|owner`

### Form Request
- **`ReturnToDraftRequest`**
  - `reason`: required, string, min:10, max:1000

### Notifications
- **`PayrollRunReturnedToDraftNotification`** — database channel
  - Content: "Payroll run '{run_name}' has been returned to draft. Reason: {reason}"
  - Recipients determined by the state it was returned from

### Frontend
- "Return to Draft" button visible on run detail page when state is `submitted` or `under_review`
- Modal dialog with:
  - Reason textarea (required, minimum 10 characters)
  - Submit and Cancel buttons
- Button not visible on finalized runs

### Audit
- Audit record captures: old state, new state (draft), actor, reason comment

### Tests
- `tests/Feature/PayrollRun/ReturnToDraftTest.php`
  - Returns submitted run to draft with reason
  - Returns under_review run to draft with reason
  - Rejects return of finalized run (422)
  - Rejects return without reason (validation error)
  - Verifies notification sent to appropriate users
  - Verifies run is editable again after return

## Acceptance Criteria

- [ ] Run can be returned to draft from `submitted` or `under_review` state
- [ ] Reason comment is required (minimum 10 characters)
- [ ] Cannot return a finalized run to draft
- [ ] Audit trail captures return reason and actor
- [ ] Run becomes editable again after returning to draft
- [ ] Finance user notified when run returned from `under_review`
- [ ] Payroll officer notified when run returned from `submitted`
- [ ] All edits made after returning to draft are logged in audit trail
