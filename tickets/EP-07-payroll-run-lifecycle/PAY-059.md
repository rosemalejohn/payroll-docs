# [GW] Payroll - Return Payroll Run to Draft

## Description

Allow Payroll Officers and Owners to return a submitted or under-review payroll run back to draft state so that errors identified during review can be corrected. This action requires a reason comment and notifies relevant stakeholders. Finalized runs cannot be returned to draft.

## Technical Implementation

### What Actions

- Return Run to Draft action: validates the run is in submitted or under review state (not finalized), requires a reason (minimum 10 characters), transitions state back to draft, stores the reason in the audit trail, and dispatches notifications to relevant stakeholders based on the previous state.

### Validation & DTOs

- Return to Draft request: validates reason (required, minimum 10 characters, maximum 1000 characters).

### Routes & Controllers

- `POST /payroll-runs/{payrollRun}/return-to-draft` — return to draft with reason.

### Policies & Authorization

- Owner and Payroll Officer can return runs to draft.
- Finalized runs cannot be returned to draft.

### Mail & Notifications

- PayrollRunReturnedToDraft notification via database channel: includes the run name and the reason for return.
- If returning from under review: notify the finance user who placed it under review.
- If returning from submitted: notify the payroll officer who submitted it.

### Tests

- Assert a submitted run can be returned to draft with a reason.
- Assert an under review run can be returned to draft with a reason.
- Assert a finalized run cannot be returned to draft (422).
- Assert return without a reason is rejected (validation error).
- Assert return with a reason shorter than 10 characters is rejected.
- Assert the appropriate notification is sent based on the previous state.
- Assert the run is editable again after returning to draft.
- Assert the audit trail captures the return reason and actor.

## Acceptance Criteria

### Client-side

#### Behavior

- A "Return to Draft" button is visible on the payroll run detail page when the state is submitted or under review
- The button is not visible on finalized runs
- Clicking the button opens a modal dialog with a reason text area (required, minimum 10 characters) and Submit/Cancel buttons
- After confirmation, the run returns to draft state

#### Validation

- An error is shown if the reason is empty or shorter than 10 characters

---

### Server-side

#### Side-effects

- A payroll run can be returned to draft from submitted or under review state
- A reason comment is required (minimum 10 characters) and is recorded in the audit trail
- Finalized runs cannot be returned to draft
- The run becomes editable again after returning to draft (bonuses, employee data, and absent days are unlocked)
- The finance user is notified when a run is returned from under review
- The payroll officer is notified when a run is returned from submitted
- All changes after returning to draft are logged in the audit trail
