# [GW] Payroll - Configure Tailwind CSS v4 and Shadcn/ui Component Library

## Description

Install and configure Tailwind CSS v4 with project-specific design tokens, then initialize the Shadcn/ui component library with a default theme. Install all base UI primitives needed across the application to ensure the design system is in place before any feature pages are built.

## Technical Implementation

### Environment & Config

- Install Tailwind CSS v4 with its Vite plugin.
- Define project design tokens (colors, border radius, font family) using Tailwind v4's CSS-first configuration.
- Initialize Shadcn/ui with appropriate style configuration and path aliases.
- Create a class-name utility helper using clsx and tailwind-merge.
- Install supporting dependencies: clsx, tailwind-merge, class-variance-authority, lucide-react.
- Install 14 base Shadcn/ui primitives: Button, Input, Label, Card, Dialog, Sheet, Select, Table, Badge, Dropdown Menu, Tabs, Tooltip, Separator, Skeleton.

### Edge Cases & Notes

- Tailwind v4 uses CSS-first configuration — no JavaScript config file should be created.
- Shadcn/ui components are copied into the project (not installed as a dependency), so they can be freely customized.

## Acceptance Criteria

### Client-side

#### Behavior

- Tailwind utility classes apply correctly in the browser.
- All 14 base UI components are available for import and render correctly.
- Design tokens (colors, spacing, fonts) are consistent across the application.
- The production build completes without CSS or component-related errors.
