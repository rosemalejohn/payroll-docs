# [GW] Payroll - Dual Currency Payslip Display for Foreign Salary Employees

## Description

For employees whose salary is denominated in a foreign currency (non-PHP), display dual currency amounts on the payslip detail page and generated PDF. The payslip shows the original salary in the employee's currency alongside the PHP equivalent, the exchange rate used with its snapshot date, and all amounts with PHP as the primary display currency. Deductions are shown in PHP with a note that they are computed on the PHP-converted gross. Employees paid in PHP see the standard single-currency payslip with no conversion section.

## Technical Implementation

### Queries

- Payslip with currency query: loads the payslip with the payroll run's currency snapshot, joins the currency snapshots table to get the exchange rate for the employee's salary currency, and returns the payslip data enriched with currency conversion context.

### Routes & Controllers

- Update the payslip detail routes to detect if the employee has a non-PHP salary currency. If foreign currency, include currency snapshot data in the page props (original currency code, original salary amount, exchange rate, snapshot date, PHP equivalent). If PHP-only, omit the currency conversion props.

### API Resources

- Update the Payslip Detail resource to conditionally include a currency conversion object (only for foreign currency employees): original currency code, original salary in foreign currency, exchange rate, snapshot date, and formatted rate display string.

### Tests

- Assert foreign currency employee payslip shows original currency and PHP equivalent.
- Assert exchange rate and snapshot date are displayed correctly.
- Assert net pay is shown primarily in PHP, secondarily in original currency.
- Assert deductions are shown in PHP with "Computed on PHP-converted gross" note.
- Assert PHP-only employee payslip does not show any conversion section.
- Assert currency conversion uses the snapshot rate from the payroll run (not live rate).
- Assert PDF includes dual currency information for foreign salary employees.
- Assert conversion math is accurate.

### Edge Cases & Notes

- Exchange rates are always sourced from payroll run currency snapshots, never from live rates.
- Rate format is "1 [FOREIGN] = X PHP", consistent with the currency system.
- If a currency snapshot is missing (data integrity issue), display PHP amounts only with a note that the rate was unavailable.

## Acceptance Criteria

### Client-side

#### Behavior

- Foreign currency employees see dual amounts on the payslip: original currency and PHP equivalent
- An exchange rate banner displays the rate and snapshot date (e.g., "1 USD = PHP 60.41 (as of March 1, 2026)")
- PHP is the primary display currency for all amounts
- Deductions are shown in PHP with a note: "Computed on PHP-converted gross"
- PHP-only employees see the standard payslip view with no conversion section

---

### Server-side

#### Side-effects

- Currency conversion uses the snapshot rate from the payroll run, not live rates
- Foreign salary employees have their original currency amount, exchange rate, and PHP equivalent included in the payslip data
- PHP-only employees have no currency conversion data included
- PDF output includes dual currency information for foreign salary employees
- If a snapshot rate is missing, PHP amounts are shown with a note about the unavailable rate
